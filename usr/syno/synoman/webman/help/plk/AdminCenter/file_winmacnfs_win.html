SMB

Protokół SMB (Server Message Block) umożliwia klientom z systemami Windows, Mac OS i Linux z obsługą protokołów SMB/CIFS dostęp do danych przechowywanych na serwerze Synology NAS. Aby przeglądać pliki na serwerze Synology NAS za pośrednictwem programu Eksplorator Windows lub montować foldery współdzielone jako dyski sieciowe, włącz tę opcję na stronie Panel sterowania > Usługi plików > SMB/AFP/NFS.

Aby włączyć protokół SMB:

  1. Zaznacz opcję Włącz usługę SMB.
  2. Kliknij Zastosuj.

Uwaga:

  • Po włączeniu protokołu SMB można zamontować w pełni indeksowane foldery w komputerze Mac za pośrednictwem SMB i wyszukiwać zawartość plików w zamontowanych folderach przy użyciu aplikacji Finder (nie dotyczy NVR216 oraz VS360HD).

Grupa robocza

ta opcja umożliwia przypisanie systemu jako członka określonej grupy roboczej do celów zarządzania siecią.

Uwaga:

  • Nazwa grupy roboczej może zawierać od 1 do 15 znaków. Nie może zawierać następujących znaków: [ ] ; : .< >* + = \ / | ? ,
  • Jeżeli używany jest komputer Mac z systemem OS X 10.9, a Synology NAS dołączył do domeny Windows, można się zalogować, dodając localhost\ przed nazwą użytkownika lokalnego Synology NAS, aby uzyskać dostęp do plików przechowywanych w urządzeniu Synology NAS za pośrednictwem protokołu SMB.

Nie zezwalaj na dostęp do poprzednich wersji

W programie Eksplorator Windows użytkownicy mogą uzyskiwać dostęp do migawek folderów lub plików, klikając we właściwościach folderu współdzielonego kartę Poprzednie wersje, co pozwala na przywracanie lub kopiowanie danych bez udziału administratora. Jeżeli opcja Nie zezwalaj na dostęp do poprzednich wersji zostanie zaznaczona, na karcie Poprzednie wersje nie będzie żadnych informacji.

Uwaga:

  • Ta funkcja jest obsługiwana wyłącznie przez VDSM i modele o następujących architekturach pakietów: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview oraz Grantley. Aby sprawdzić, jaką architekturę pakietów obsługuje dany model, zapoznaj się z tym artykułem.
  • Użytkownicy mogą uzyskać dostęp do wcześniejszej wersji tylko wtedy, gdy w momencie tworzenia migawki odpowiadającej wcześniejszej wersji ich konta użytkowników były utworzone i miały uprawnienia do odczytu folderu współdzielonego, którego migawka była tworzona.

Włączanie rejestrowania transferu w logu

Zaznacz pole wyboru Włącz log transferu, aby włączyć rejestrowanie operacji na plikach wykonywanych za pośrednictwem protokołu CIFS/SMB. Log transferu domyślnie monitoruje tylko usunięcie pliku lub folderu, jednak możesz również przejść do Ustawienia logu, aby wybrać inne zdarzenia operacji na plikach – tworzenie, przenoszenie, zmianę nazwy, odczyt, zapis – które system ma rejestrować. Należy zwrócić uwagę, że im więcej wybranych zdarzeń operacji na plikach, tym większe prawdopodobieństwo wpływu na wydajność systemu.

Ustawienia zaawansowane

Kliknij przycisk Ustawienia zaawansowane, aby wyświetlić poniższe opcje.

  • Serwer WINS: Microsoft Windows Internet Name Service (WINS) to nazwa usługi odwzorowywania nazw NetBIOS na adresy IP. Użytkownicy systemu Windows znajdą system w sieciach TCP/IP znacznie łatwiej, jeżeli zostanie on skonfigurowany tak, aby rejestrować się na serwerze WINS.
  • Maksymalny protokół SMB: w zależności od konfiguracji sieci określ najnowszy protokół SMB, który będzie obsługiwany przez urządzenie Synology NAS.
    • SMB1: Protokół SMB1 (zwany także CIFS) jest obsługiwany od wersji Windows NT.
    • SMB2: Protokół SMB2 jest obsługiwany od wersji Windows Vista i jest udoskonaloną wersją protokołu SMB. SMB2 umożliwia łączenie wielu działań SMB w jedno żądanie w celu zmniejszenia liczby pakietów sieciowych i poprawienia wydajności.
    • SMB2 i Large MTU: Maksymalna jednostka transmisji (MTU, Maximum Transmission Unit) to największa jednostka danych, którą można przesłać przy użyciu protokołu telekomunikacyjnego. Aby umożliwić wykorzystanie szybszych interfejsów sieciowych, takich jak Ethernet 1 Gb lub 10 Gb, w funkcji Large MTU zwiększono MTU do 1 megabajta (MB). Włączenie funkcji Large MTU zwiększa szybkość i wydajność transferów dużych plików, zmniejszając jednocześnie liczbę pakietów do przetworzenia.
    • SMB3: protokół SMB3 jest obsługiwany od wersji Windows 8 i Windows Server 2012. Jest to rozszerzona wersja protokołu SMB2. Protokół SMB3 obsługuje transmisję plików szyfrowanych algorytmem AES, co pozwala poprawić bezpieczeństwo przesyłania plików między komputerami.

    Uwaga:

    • Protokół SMB2 jest obsługiwany w systemach Windows Vista, Windows Server 2008 R2, Windows 7 i nowszych.
    • Funkcja Large MTU jest obsługiwana w systemach Windows Server 2008 R2, Windows 7 i nowszych.
    • W celu uzyskania efektywnej obsługi dużych MTU na komputerze z systemem Windows należy zmodyfikować ustawienia rejestru komputera. Więcej informacji można znaleźć w materiałach pomocy technicznej firmy Microsoft (takich jak ten artykuł).
  • Minimalny protokół SMB: w zależności od konfiguracji sieci określ najstarszy protokół SMB, który będzie obsługiwany przez urządzenie Synology NAS.
  • Tryb szyfrowania transferu: po włączeniu protokołu SMB3 protokół SMB doda szyfrowanie transferu, aby zwiększyć bezpieczeństwo transmisji plików.
    • Wyłącz: wyłączenie szyfrowania transferu.
    • Auto: włączenie szyfrowania transferu na klientach obsługujących tę funkcję.
    • Wymuś: Wymuszenie szyfrowania transferu. Po wybraniu tej opcji klienci nieobsługujący szyfrowania transferu nie będą mogli korzystać z protokołu SMB.

    • Uwaga:

    • Funkcja szyfrowania transferu jest obsługiwana przez system Windows 8, Windows Server 2012 i nowsze wersje.
    • W przypadku włączenia szyfrowania transferu logowanie anonimowe w protokole SMB nie będzie obsługiwane.
    • Po włączeniu szyfrowania należy pozostawić opcję Włącz Opportunistic Locking zaznaczoną, aby uniknąć przekraczania limitów czasu oczekiwania aplikacji.
  • Włącz Opportunistic Locking: zaznacz tę opcję, aby włączyć funkcję Opportunistic Locking CIFS, dzięki której klient będzie mógł sprawniej dodawać do pamięci podręcznej zawartość plików i zapobiegać niewielkiemu wyprzedzeniu odczytu/zapisu. Jeżeli opcja ta nie zostanie zaznaczona, aplikacje bazy danych lepiej sprawdzą się w uzyskiwaniu dostępu do plików bazy danych przechowywanych na Synology NAS za pośrednictwem sieci.
    • Włącz dzierżawę SMB2: włączenie tej opcji sprawi, że dostępnych będzie więcej ustawień powiązanych z SMB2 umożliwiających dostosowanie zapewnianego przez Opportunistic Locking mechanizmu zapisu danych w pamięci podręcznej klienta.
  • Włącz Local Master Browser: kiedy w tej samej podsieci istnieje wiele komputerów z systemem Windows, w wyniku negocjacji jeden z nich zostanie wybrany jako Local Master Browser (główna przeglądarka lokalna). Komputer pełniący funkcję Local Master Browser przechowuje listy pozostałych komputerów z podsieci oraz ich udostępnionych zasobów i udostępnia im te listy. Ta opcja umożliwia urządzeniu Synology NAS objęcie roli Local Master Browser (głównej przeglądarki lokalnej).

    • Uwaga:

    • Włączenie tej funkcji spowoduje wyłączenie funkcji hibernacji dysku twardego i aktywowanie konta użytkownika guest (gość) niezabezpieczonego hasłem.
  • Włącz moduł DirSort VFS: Sortowanie nazw plików i wyświetlanie ich na liście w kolejności alfabetycznej. Włączenie tej opcji obciąża system.
  • Odrzuć pliki: Ukrywanie plików według określonych kryteriów. Kryteria mogą obejmować symbole wieloznaczne (*), a większą liczbę kryteriów należy rozdzielić ukośnikami (/). Na przykład: /abc*/*.txt/*.conf/. Włączenie tej opcji obciąża system.
  • Zezwalaj na łącza symboliczne w folderach współdzielonych: zaznacz tę opcję w celu zezwolenia na wskazywanie plików lub folderów docelowych przez łącza symboliczne w tym samym folderze współdzielonym. Łącza symboliczne można tworzyć przy użyciu terminala systemu DSM.
    • Zezwalaj na łącza symboliczne między folderami współdzielonymi: zaznacz tę opcję w celu zezwolenia na wskazywanie plików lub folderów przez łącza symboliczne w różnych folderach współdzielonych. Łącza symboliczne można tworzyć przy użyciu terminala systemu DSM. Użytkownicy nieposiadający odpowiednich uprawnień będą mogli uzyskiwać dostęp do docelowych folderów współdzielonych łączy symbolicznych.
  • Włącz moduł VFS w celu konwersji znaków specjalnych Mac: zaznacz, aby podczas tworzenia pliku znaki zarezerwowane dla SMB (np. < > " * ?) były konwertowane na podstawowy alfabet łaciński. Pliki takie będą rozpoznawalne przez AFP.
  • Włącz moduł MSDFS VFS: Zezwolenie na wskazywanie przez łącza symboliczne plików lub folderów znajdujących się na innych serwerach SMB. W przypadku starszych wersji systemu Windows, takich jak Windows XP, przed włączeniem tej opcji należy dołączyć klienta i Synology NAS do domeny Windows. Więcej informacji o tworzeniu łączy MSDFS można uzyskać tutaj. Pamiętaj, że po wyłączeniu tej opcji trzeba wyczyścić pamięć podręczną urządzenia klienckiego (ponowne uruchomienie urządzenia klienckiego jest również zalecane). W przeciwnym razie wydajność może ulec pogorszeniu.
  • Wyłącz wiele połączeń z tego samego adresu IP: Kiedy ta opcja jest włączona, urządzenie Synology NAS zezwala na tylko jedno połączenie z każdego adresu IP w przypadku klientów używających protokołu SMB. Jeżeli wiele klientów z tego samego adresu IP używa protokołu SMB, Synology NAS utrzyma najnowsze połączenie i odrzuci wszystkie starsze. W środowisku NAT klient może używać jednego adresu IP. W takim przypadku włączenie tej opcji może wpływać negatywnie na normalne działanie.
  • Gromadź logi debugowania: Przechowywanie szczegółowych dzienników do celów debugowania. Włączenie tej opcji obciąża system.
  • Zastosuj domyślne uprawnienia UNIX: Włączenie tej opcji powoduje zastosowanie domyślnych uprawnień systemu UNIX podczas przesyłania lub tworzenia plików i folderów. Uprawnienia systemu UNIX mają wartość 744 w przypadku plików i 755 w przypadku folderów. Po wyłączeniu tej opcji uprawnienia systemu UNIX mają wartość 777 w przypadku plików i folderów.

    • Uwaga:

    • W przypadku folderów współdzielonych z obsługą listy kontroli dostępu systemu Windows (ACL) (wszystkie udostępnienia z wyłączeniem udostępnień „photo” i udostępnień na zewnętrznych dyskach) należy uruchomić polecenie chmod w kliencie Linux lub FTP, aby zmienić rodzaje uprawnień dotyczące folderów i plików z listy kontroli dostępu systemu Windows (ACL) na UNIX (nie dotyczy EDS14).
    • Włączenie tej opcji może spowodować problemy z niespójnością uprawnień między różnymi protokołami. Aby uniknąć niespójności, nie zaleca się włączania tej opcji.
  • Nie rezerwuj przestrzeni na dysku podczas tworzenia plików: Włącz tę opcję, aby zapobiec rezerwowaniu przestrzeni na dysku podczas tworzenia plików na serwerze Synology NAS.
  • Włącz pamięć podręczną wyszukiwania z symbolami wieloznacznymi: włącz tę opcję, aby poprawić wydajność wyszukiwania, jeżeli klienci wysyłają wielokrotne żądania wyszukiwania z użyciem symboli maski.
  • Włącz trwałą obsługę protokołu SMB (międzyprotokołowe blokowanie plików zostanie wyłączone): Włączenie tej opcji umożliwi użytkownikom przywrócenie utraconego połączenia SMB2 z tym serwerem Synology NAS po tymczasowym rozłączeniu. Otwarte pliki pozostaną dostępne podczas przerwania połączenia. Międzyprotokołowe blokowanie plików między protokołami zostanie wyłączone, zatem użytkownicy będą mogli uzyskiwać dostęp do zablokowanego pliku lub modyfikować plik przy użyciu protokołów innych niż SMB2 (np. AFP).
  • Wyczyść pamięć podręczną SMB: kliknięcie tego przycisku pozwala usunąć następujące pliki pamięci podręcznej SMB w ramach rozwiązywania problemów:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
