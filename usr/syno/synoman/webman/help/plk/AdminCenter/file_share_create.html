Tworzenie folderu współdzielonego

Foldery współdzielone to podstawowe katalogi, w których można przechowywać pliki i zarządzać nimi na serwerze Synology NAS. Przed zapisaniem plików należy utworzyć w systemie DSM co najmniej jeden folder współdzielony.

Aby utworzyć folder współdzielony:

  1. Wybierz pozycje Panel sterowania > Folder współdzielony, a następnie kliknij opcję Utwórz.
  2. Określ nazwę i opis folderu współdzielonego.
  3. Wybierz pamięć masową, w której ma być utworzony folder współdzielony (pomiń ten krok, jeżeli istnieje tylko jeden wolumen).
  4. W razie potrzeby zmodyfikuj następujące opcje:
    • Ukryj ten folder współdzielony w „Moje miejsca sieciowe”: zapobiega wyświetlaniu folderu współdzielonego w grupie „Sieć” w programie Eksplorator plików Windows. Ta opcja nie wpływa na uprawnienia dostępu do folderu współdzielonego.

      Uwaga:

      • Opcja Ukrywanie folderu współdzielonego w „Moje miejsca sieciowe” nie wpływa na prawa dostępu. Prawa dostępu do ukrytego folderu współdzielonego nie ulegną zmianie. Użytkownicy, którzy mają odpowiednie uprawnienia dostępu do folderu współdzielonego, mogą nadal uzyskać do niego dostęp za pośrednictwem ścieżki „\nazwa serwera\nazwa folderu współdzielonego”.
    • Ukryj podfoldery i pliki przed użytkownikami bez uprawnień: po włączeniu ustawienia Ukryj podfoldery i pliki przed użytkownikami bez uprawnień, użytkownicy niemający uprawnień do odczytu, którzy spróbują uzyskać dostęp do folderu współdzielonego przy użyciu protokołu Windows File Sharing, nie będą mogli wyświetlić podfolderów ani plików w tym folderze współdzielonym. Zapobiega to dezorientacji użytkowników łączących się z urządzeniami Synology, którzy widzą wiele plików i podfolderów, ale nie mogą ich otworzyć.

      Uwaga:

      • Ta funkcja jest obsługiwana wyłącznie za pośrednictwem protokołu Windows File Sharing.
      • Ustawienia uprawnień dotyczące każdego folderu i pliku można zmodyfikować w aplikacji File Station, klikając go prawym przyciskiem myszy. Z menu rozwijanego wybierz polecenie Właściwości, a następnie kliknij kartę Uprawnienie.
      • Sam folder współdzielony nie zostanie ukryty przed użytkownikami bez uprawnień po włączeniu ustawienia Ukryj podfoldery i pliki przed użytkownikami bez uprawnień. Ukryte zostaną tylko podfoldery i pliki w tym folderze współdzielonym.
  5. Włącz kosz: pliki usunięte z folderu współdzielonego są przenoszone do folderu o nazwie #recycle. Dostęp do kosza można ograniczyć do użytkowników należących do grupy administrators.
  6. Włącz limit folderu współdzielonego: aby określić maksymalną pojemność folderów współdzielonych, zaznacz pole wyboru Włącz limit folderu współdzielonego, a w polu wprowadź maksymalną pojemność (np. 10 GB).

    Uwaga:

    • Opcja folderów współdzielonych Włącz limit folderu współdzielonego będzie dostępna tylko wtedy, gdy w pamięci masowej używany jest system plików Btrfs.
    • Łączne użycie pamięci masowej katalogów home wszystkich użytkowników nie może przekraczać limitu katalogu homes.
  7. Włączanie sumy kontrolnej danych w celu uzyskania zaawansowanej integralności danych: Aby zapewnić ochronę folderu współdzielonego za pomocą strategii Suma kontrolna CRC32 i Kopiowanie przy zapisie, można włączyć sumę kontrolną danych dla zaawansowanej ochrony integralności danych podczas tworzenia folderu współdzielonego. Suma kontrolna CRC32 stosowana jest do sprawdzania zgodności danych ze stanem z momentu ich oryginalnego zapisu. System automatycznie wykorzysta funkcję nadmiarowości RAID do ich naprawy. Kopiowanie przy zapisie pozwala poprawić spójność danych w przypadku nieprawidłowego zamknięcia systemu.

    Uwaga:

    • Opcja ta jest dostępna tylko w przypadku wolumenów sformatowanych w systemie plików Btrfs.
    • Można włączyć/wyłączyć tę opcję dla folderu współdzielonego podczas jego tworzenia. Nie można jej zmodyfikować po utworzeniu folderu współdzielonego.
    • Włączenie tej opcji pozwala systemowi na przeprowadzenie czyszczenia danych w pełnej skali. Dowiedz się więcej o czyszczeniu danych
    • W celu zapewnienia wysokiej jakości usług zalecamy niewłączanie sumy kontrolnej danych, jeżeli folder współdzielony będzie używany dla następujących usług:
      • Hosting baz danych lub maszyn wirtualnych
      • Przechowywanie nagrań wideo aplikacji Surveillance Station
      • Uruchamianie usług wymagających niewielkiego i losowego wzorca zapisu
  8. Włącz kompresję plików: Dane w utworzonym folderze współdzielonym będą automatycznie kompresowane przez system plików Btrfs w celu oszczędzania miejsca w pamięci. Podczas odzyskiwania danych do użytku będą one automatycznie dekompresowane.

    Uwaga:

    • Opcja ta jest dostępna tylko wtedy, gdy folder współdzielony został utworzony w wolumenie Btrfs lub gdy włączono opcję Zaawansowana ochrona integralności danych.
    • Nie można jej zmodyfikować po utworzeniu folderu współdzielonego.
    • Stopień kompresji zależy od typu danych: Pliki tekstowe mają najwyższy możliwy stopień kompresji, podczas gdy pliki multimedialne (np. obrazy, muzyka lub filmy) nie mogą zostać mocno skompresowane, ponieważ są one już skompresowane do celów wyświetlania.
    • Tej opcji nie można włączyć, jeśli włączone jest szyfrowanie folderu współdzielonego.
  9. Zaszyfruj ten folder współdzielony: aby zaszyfrować folder współdzielony, wykonaj następujące czynności:
    1. Określ i potwierdź nowy klucz szyfrowania. Klucz szyfrowania nie może zawierać przecinków (,) ani znaków równości (=).
    2. W razie potrzeby zaznacz pole wyboru Dodaj klucz szyfrowania do Menedżera kluczy, aby szyfrowany folder był montowany automatycznie podczas uruchamiania. Jeżeli opcja jest wyłączona, zaszyfrowany folder współdzielony pozostanie niezamontowany po kolejnym uruchomieniu serwera Synology NAS. Aby zobaczyć niezamontowany folder współdzielony i uzyskać do niego dostęp, trzeba będzie ręcznie go zamontować, importując lub wprowadzając klucz szyfrowania.
  10. Kliknij OK, aby zakończyć.

    11. Uwaga:

    • Niektóre funkcje i opcje folderu współdzielonego są obsługiwane wyłącznie przez VDSM i modele o następujących architekturach pakietów: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview oraz Grantley. Aby sprawdzić, jaką architekturę pakietów obsługuje dany model, zapoznaj się z tym artykułem.

Reguły nazewnictwa folderu współdzielonego:

  • W nazwach folderów współdzielonych nie jest istotna wielkość liter i mogą one mieć od 1 do 32 znaków. Nazwy nie mogą zawierać następujących znaków: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Następujące nazwy są zarezerwowane dla systemu: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Pierwszym znakiem nazwy folderu współdzielonego nie może być znak minus ani spacja, a ostatni znak nie może być spacją.
  • W opisie folderu współdzielonego istotna jest wielkość liter i powinien on zawierać do 64 wyświetlanych znaków Unicode.

Szyfrowanie folderów współdzielonych:

Szyfrowanie przy użyciu 256-bitowego algorytmu AES chroni dane, blokując wszystkie nieautoryzowane próby dostępu. Po zaszyfrowaniu folderu współdzielonego inni użytkownicy nie będą mogli uzyskać dostępu do zaszyfrowanych danych ani ich zmodyfikować bez klucza szyfrowania nawet po usunięciu dysków twardych z serwera Synology NAS i zamontowaniu ich w innym urządzeniu.

Uwaga:

  • Nazwa pliku lub folderu w zaszyfrowanym folderze współdzielonym nie może zawierać więcej niż 143 znaki angielskie lub 47 znaków azjatyckich (CJK). Jeżeli dostęp do folderów współdzielonych odbywa się na urządzeniu z systemem Mac OS, nazwa pliku lub folderu w zaszyfrowanym folderze współdzielonym nie może przekraczać 130 znaków angielskich lub 43 znaków azjatyckich (CJK).
  • Zaszyfrowane foldery współdzielone nie są dostępne za pośrednictwem NFS.
  • Jeżeli na serwerze Synology NAS przy użyciu przycisku Reset zostaną przywrócone domyślne hasła, zaszyfrowane foldery współdzielone zostaną odmontowane, a opcja automatycznego montowania zaszyfrowanych folderów współdzielonych zostanie wyłączona.
  • Szyfrowane foldery współdzielone są obsługiwane wyłącznie w trybie wysokich uprawnień (dotyczy tylko Dockerx64). Aby użyć tej funkcji, przejdź do strony Docker w systemie DSM hosta, wybierz system Docker DSM, dla którego chcesz włączyć szyfrowane foldery współdzielone, kliknij Edytuj, a następnie zaznacz pole obok Uruchom kontener przy użyciu wysokich uprawnień.
