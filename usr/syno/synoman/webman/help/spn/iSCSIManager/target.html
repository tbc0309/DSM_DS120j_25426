Destino

Dependiendo del modelo, Synology NAS permite crear múltiples destinos a los que se pueden asignar LUN.

Observación: El número de destinos admitido varía en función de los modelos de Synology NAS.

Para crear un destino:

  1. Haga clic en el botón Crear.
  2. Introduzca la información del destino, incluidos Nombre, IQN y si se habilita o no Autenticación CHAP.
  3. Haga clic en Siguiente.
  4. Configurar la asignación de iSCSI LUN. Seleccione Crear un nuevo iSCSI LUN o Asignar LUN existentes.
  5. Haga clic en Siguiente.
  6. Haga clic en Atrás para modificar o en Aplicar para confirmar la configuración.

Terminología

  • IQN: IQN (iSCSI Qualified Name) es un conjunto de nombre exclusivo para un destino. No debe entrar en conflicto con otros IQN de destino en otros servidores. IQN debe tener menos de 128 caracteres y puede incluir letras (sin tener en cuenta mayúsculas y minúsculas), números, dos puntos, guiones y puntos. El formato de nomenclatura sugerido es iqn.[fecha].[dominio]:[identificador del dispositivo], por ejemplo: iqn.2000-04.com.synology:storage.diskarray.1.
  • Autenticación CHAP mutua: En CHAP mutua, tanto el cliente como el servidor se autenticarán entre sí. Cualquier contraseña CHAP empleada para una autenticación de cliente no deberá configurarse para la autenticación de cualquier servidor, y cualquier contraseña CHAP empleada para la autenticación del servidor no deberá configurarse para la configuración de un cliente.

Complete la Detección en primer lugar y después habilite la autenticación CHAP antes de realizar la conexión a un destino. Si lo prefiere, puede realizar la conexión a un destino con la IP y el IQN.

Para eliminar un destino:

Puede eliminar un destino y cualquier LUN asignado. Al quitar un iSCSI LUN, se eliminarán todos los datos del iSCSI LUN.

  1. Seleccione el destino que desea eliminar.
  2. Haga clic en Quitar.
  3. Si desea eliminar los LUN asignados, pulse Eliminar también los LUN asignados y seleccione los que desee eliminar.
  4. Haga clic en OK.

Para editar un destino:

  1. Seleccione el destino que desea editar.
  2. Seleccione Editar en el menú desplegable Acción.
  3. Edite la configuración según sus preferencias. Haga clic en OK para guardar los cambios.

Terminología

  • Suma de comprobación CRC: habilite Suma de comprobación CRC para comprobar si se ha producido algún error durante la transmisión de datos.
  • Varias sesiones: permita que varias sesiones de más de un iniciador se conecten al mismo destino al mismo tiempo. Para proteger sus archivos de daños o pérdida inesperada de datos, se sugiere encarecidamente que use un sistema de archivos compartido o un sistema de clúster tal como VMware Virtual Machine File System (VMFS) y Oracle Cluster File System (OCFS).
  • Enmascaramiento: el enmascaramiento permite que a destino accedan exclusivamente determinados IQN del iniciador iSCSI.

Para habilitar o deshabilitar un destino:

  1. Seleccione el destino que desea habilitar o deshabilitar.
  2. Seleccione Habilitar o Deshabilitar en el menú desplegable Acción.
