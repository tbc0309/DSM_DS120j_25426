SMB

El protocolo SMB (Server Message Block) permite a los clientes de Windows, Mac y Linux compatibles con SMB/CIFS acceder a los datos almacenados en Synology NAS. Si desea explorar archivos en Synology NAS a través de Windows Explorer o montar carpetas compartidas como unidades de red, habilite esta opción en Panel de control > Servicios de archivos > SMB/AFP/NFS.

Para activar el protocolo SMB:

  1. Marque Habilitar servicio SMB.
  2. Haga clic en Apply.

Observación:

  • Tras habilitar el protocolo SMB, puede montar carpetas indexadas en su ordenador Mac a través del protocolo SMB y utilizar Finder para buscar en el contenido de los archivos en las carpetas montadas (no aplicable a NVR216 ni VS360HD).

Grupo de trabajo

esta opción le permite designar el sistema como miembro de un grupo de trabajo específico para fines de administración de redes.

Observación:

  • El nombre de un grupo de trabajo puede contener entre 1 y 15 caracteres. El nombre no puede incluir los siguientes caracteres: [ ] ; : "< >* + = \ / | ? ,
  • Si utiliza un Mac con OS X 10.9 y su Synology NAS se ha unido a un dominio Windows, puede iniciar sesión añadiendo localhost\ delante del nombre de usuario local de Synology NAS para acceder a los archivos almacenados en Synology NAS a través del protocolo SMB.

No permitir el acceso a versiones anteriores

En Windows Explorer, los usuarios pueden acceder a las instantáneas de carpetas o archivos haciendo clic en la pestaña Versiones anteriores dentro de una carpeta compartida, permitiendo restaurar o copiar datos sin ayuda de un administrador. Si elige No permitir el acceso a versiones anteriores, la página “Versiones anteriores” no mostrará ninguna información.

Nota:

  • Esta función solo es compatible con VDSM y los modelos con las siguientes arquitecturas de paquete: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview y Grantley. Para ver qué arquitectura de paquete es compatible con su modelo, consulte este artículo.
  • Se permite que los usuarios accedan a una versión anterior siempre y cuando, en el momento de tomar la instantánea de la versión anterior, sus cuentas de usuario se hayan creado y se les haya otorgado el permiso necesario para leer la carpeta compartida correspondiente a la instantánea tomada.

Habilitar registro de transferencia

Marque la casilla de verificación Habilitar registro de transferencia para habilitar el registro de operaciones de archivos realizadas a través del protocolo CIFS/SMB. De forma predeterminada, el registro de transferencia solo supervisa la eliminación de un archivo o carpeta; sin embargo, también puede ir a Configuración del registro para seleccionar otros eventos de operaciones de archivo (crear, mover, renombrar, leer o escribir) que debe registrar el sistema. Tenga en cuenta que cuantos más eventos de operaciones de archivo seleccione, más probable será que el rendimiento del sistema se vea afectado.

Configuración avanzada

Haga clic en el botón Configuración avanzada para ver las siguientes opciones.

  • Servidor WINS: el Servicio de nombres Internet de Windows (WINS) de Microsoft es un servicio de asignación de nombre NetBIOS a dirección IP. Los usuarios de Windows localizarán el sistema con mayor facilidad en redes TCP/IP si el sistema se ha configurado para registrarse con un servidor WINS.
  • Protocolo SMB máximo: según la configuración de red, establezca el protocolo SMB más reciente compatible con Synology NAS.
    • SMB1: SMB1 (también llamado CIFS) es compatible desde Windows NT.
    • SMB2: SMB2 es compatible desde Windows Vista y es la versión mejorada de SMB. SMB2 añade la posibilidad de componer varias acciones de SMB en una sola solicitud para reducir el número de paquetes de red y mejorar así el rendimiento.
    • SMB2 y Large MTU: la unidad de transmisión máxima (MTU) es la unidad de datos más grande que se puede pasar a través de un protocolo de comunicaciones. Para aprovechar interfaces de red más rápidas, como Ethernet de 1 gigabit o de 10 gigabits, Large MTU aumenta la unidad de transmisión máxima a 1 megabyte (MB). Al habilitar Large MTU se mejora la velocidad y la eficacia de las transferencias de archivos grandes, a la vez que se reduce el número de paquetes que se tienen que procesar.
    • SMB3: SMB3 es compatible desde Windows 8 y Windows Server 2012. Es la versión mejorada de SMB2. SMB3 es compatible con la transmisión cifrada de archivos basados en AES y por ello mejora la seguridad de las transferencias de archivos entre iguales.

    Observación:

    • SMB2 es compatible con Windows Vista, Windows Server 2008 R2, Windows 7 y versiones posteriores.
    • Large MTU es compatible con Windows Server 2008 R2, Windows 7 y versiones posteriores.
    • Para hacer que Large MTU funcione con su ordenador Windows, tendrá que modificar la configuración del registro. Para más detalles, consulte el soporte técnico de Microsoft (como este artículo).
  • Protocolo SMB mínimo: según la configuración de red, utilice el nivel de protocolo SMB más antiguo compatible con Synology NAS.
  • Modo de cifrado de transporte: cuando se habilita SMB3, el protocolo SMB agregará un cifrado de transporte para reforzar la seguridad de la transmisión de archivos.
    • Deshabilitar: no se aplicará ningún cifrado de transporte.
    • Automático: el cifrado de transporte solo se aplicará a clientes compatibles con esta característica.
    • Forzar: el cifrado de transporte se aplicará siempre. Esto hará que los clientes no compatibles con el cifrado de transporte no puedan utilizar el protocolo SMB.

    • Observación:

    • El cifrado de transporte es compatible con Windows 8, Windows Server 2012 y versiones posteriores.
    • El inicio de sesión anónimo del protocolo SMB no es compatible cuando se habilita el cifrado de transporte.
    • Cuando el cifrado de transporte esté activado, deje marcada la opción Habilitar Opportunistic Locking para evitar errores de tiempo de espera de la aplicación.
  • Habilitar Opportunistic Locking: seleccione esta opción para activar el 'Opportunistic locking' de CIFS, de modo que el cliente pueda almacenar mejor el contenido de los archivos en la memoria caché y evitar pequeñas sobrecargas de lectura/escritura. Cuando esta opción está desactivada, mejorará el rendimiento de las aplicaciones de bases de datos a la hora de acceder a los archivos de base de datos almacenados en Synology NAS a través de la red.
    • Habilitar concesión SMB2: la activación de esta opción permite aplicar más ajustes relacionados con SMB2 para afinar el mecanismo de caché del cliente proporcionado por Opportunistic Locking.
  • Habilitar Local Master Browser: cuando existen múltiples ordenadores Windows en la misma subred, estos negociarán y elegirán un ordenador como 'Local Master Browser'. El Local Master Browser mantiene las listas de los otros ordenadores de la subred y sus recursos compartidos, y comparte estas listas con los demás ordenadores. Esta opción permite que Synology NAS adopte el papel de Local Master Browser.

    • Observación:

    • Al habilitar esta función, se deshabilita la hibernación del disco duro y se activa la cuenta guest sin contraseña.
  • Habilitar el módulo VFS DirSort: ordena los nombres de las carpetas y los enumera por orden alfabético. La habilitación de esta opción afecta el rendimiento del sistema.
  • Vetar archivos: oculta archivos de acuerdo con unos criterios especificados. Entre los criterios se pueden incluir comodines (*) y las entradas múltiples se deben separar con una barra inclinada (/). Por ejemplo: /abc*/*.txt/*.conf/. La habilitación de esta opción afecta el rendimiento del sistema.
  • Permitir enlaces simbólicos dentro de carpetas compartidas: seleccione esta opción para permitir que los enlaces simbólicos apunten a archivos o carpetas dentro de la misma carpeta compartida. Los enlaces simbólicos se pueden crear utilizando el terminal DSM.
    • Permitir enlaces simbólicos entre carpetas compartidas: seleccione esta opción para permitir que los enlaces simbólicos apunten a archivos o carpetas en carpetas compartidas diferentes. Los enlaces simbólicos se pueden crear utilizando el terminal DSM. Sin embargo, los usuarios sin privilegios podrían acceder a las carpetas compartidas de destino de enlaces simbólicos.
  • Habilitar el módulo VFS para convertir caracteres especiales Mac: seleccione esta opción para convertir los caracteres reservados para SMB (p. ej. < > " * ?) en los nombres de archivo a Latín básico durante la creación del archivo. Dichos archivos se podrán reconocer a través de AFP.
  • Habilitar el módulo VFS MSDFS: permite a los enlaces simbólicos apuntar a archivos o carpetas situados en otros servidores SMB. Con versiones anteriores de Windows, como Windows XP, el cliente y Synology NAS deben unirse a un dominio Windows antes de habilitar esta opción. Para obtener más información acerca de la creación de enlaces MSDFS, haga clic aquí. Tenga en cuenta que la caché del dispositivo cliente se debe borrar después de deshabilitar esta opción (también se recomienda reiniciar el dispositivo cliente); de lo contrario, el rendimiento y comportamiento normales se podrían ver afectados.
  • Desactivar las conexiones múltiples desde la misma dirección IP: cuando se habilita esta opción, Synology NAS solo permite una conexión por dirección IP de clientes que utilizan el protocolo SMB. Cuando varios clientes con la misma dirección IP utilizan el protocolo SMB, Synology NAS conservará el último de ellos e interrumpirá las conexiones anteriores. En un entorno NAT, es posible que el cliente utilice una dirección IP individual, en cuyo caso habilitar esta opción puede afectar al rendimiento normal.
  • Recoger registros de depuración: cuando se habilita esta opción, se almacenarán registros detallados para realizar depuraciones. La habilitación de esta opción afecta el rendimiento del sistema.
  • Aplicar permisos UNIX predeterminados: habilite esta opción para aplicar los permisos UNIX predeterminados cuando se cargan o crean archivos y carpetas. Los permisos UNIX serán 744 para archivos y 755 para carpetas. Cuando se desactiva esta opción, el permiso UNIX es 777 para archivos y carpetas.

    • Observación:

    • Para las carpetas compartidas con Windows ACL (todos los recursos compartidos excepto 'photo' y los recursos compartidos en discos externos), ejecute el comando chmod en el cliente Linux o FTP para cambiar los tipos de permisos de carpeta y de archivo de Windows ACL a UNIX (no aplicable a EDS14).
    • Si se habilita esta opción se pueden provocar problemas de permisos que no mantienen la consistencia entre protocolos diferentes. Para evitar inconsistencias, sugerimos no habilitar esta opción.
  • No reserve espacio de disco al crear archivos: active esta opción para no permitir la reserva de espacio en disco al crear archivos en su Synology NAS.
  • Habilitar búsqueda con comodines en caché: habilite esta opción para mejorar el rendimiento de la búsqueda si los clientes envían solicitudes de búsqueda repetidas con comodines.
  • Habilitar identificadores duraderos SMB (se deshabilitará el bloqueo de archivos entre protocolos): la activación de esta opción permite a los usuarios restaurar la conexión SMB2 perdida a este Synology NAS después de una desconexión temporal; los archivos abiertos permanecerán disponibles durante la interrupción de la conexión a la red. El bloqueo de archivos entre protocolos se deshabilitará para que los usuarios puedan acceder o modificar un archivo bloqueado a través de protocolos distintos a SMB2 (p. Ej. AFP).
  • Borrar caché SMB: haga clic en este botón para borrar los siguientes archivos de la caché SMB para resolver problemas:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
