Cliente de SSO

Cliente de SSO de dominio/LDAP

Si su Synology NAS se ha unido al mismo dominio o servicio LDAP que el servidor SSO, puede configurar su Synology NAS como cliente de SSO. Los usuarios pueden acceder a los servicios proporcionados por su Synology NAS una vez inician sesión en el servidor SSO con sus credenciales.

Antes de comenzar, asegúrese de que su Synology NAS se ha unido al mismo dominio o servicio LDAP que el servidor SSO.

Para configurar su Synology NAS como cliente de SSO:

  1. Acceda a Panel de control > Dominio/LDAP > Cliente de SSO.
  2. Active la casilla de verificación Habilitar servicio SSO.
  3. Especifique la URL del servidor SSO y el ID de aplicación en los siguientes campos debajo de la casilla de verificación.

    Nota:

    Para obtener el ID de aplicación, escoja uno de los siguientes métodos:

    • Método 1:
      1. Inicie sesión en el DSM que actúa como el servidor SSO.
      2. Añada su Synology NAS, que hará las veces de aplicación cliente en la lista de aplicaciones de SSO Server. Para obtener información detallada, consulte el artículo de ayuda para SSO Server > Lista de aplicaciones.
      3. Copie y pegue el ID de aplicación asignado por SSO Server una vez su Synology NAS se haya añadido al servidor SSO.
    • Método 2:
      1. Tras introducir la URL del servidor SSO*, haga clic en Registro rápido junto al campo del ID de la aplicación.
      2. En el cuadro de registro, introduzca las credenciales de una cuenta de administrador del servidor SSO en los campos Administrador del servidor y Contraseña.
      3. Especifique el nombre de la aplicación cliente en el campo Nombre de la aplicación.
      4. Asegúrese de que Redirigir URI hace referencia exactamente a la página de inicio de sesión de su Synology NAS.
      5. Haga clic en OK y se añadirá su Synology NAS a SSO Server, mientras que el ID de la aplicación se rellenará automáticamente.

    ______

    * Se le pedirá que introduzca la URL idéntica del servidor SSO (nombre de host o IP) al configurar cada cliente SSO.

  4. Haga clic en Aplicar para guardar la configuración.

Nota:

  • Si el servidor SSO utiliza el protocolo HTTPS, puede que los exploradores que incumplen un certificado de confianza bloqueen el inicio de sesión SSO.
  • Al acceder al servidor SSO mediante el protocolo HTTP, si su Synology NAS (establecido como cliente de SSO) utiliza conexión HTTPS, los usuarios necesitan ajustar la configuración de sus exploradores para permitir el uso de una conexión no segura.
  • La dirección IP del servidor SSO en formato IPv6 no será compatible si ha marcado la casilla de verificación Mejorar la seguridad con el encabezado HTTP de la Directiva de seguridad de contenido (CSP) en Panel de control > Seguridad > Seguridad en su DSM.

Cliente de SSO de Azure AD

Si su Synology NAS se ha unido al dominio de Azure AD con Site-to-Site VPN o al dominio de Windows que está sincronizado con un dominio de Azure AD, puede configurar su Synology NAS como cliente de SSO de Azure. Los usuarios pueden acceder a los servicios proporcionados por su Synology NAS una vez inician sesión en el servidor SSO de Azure con sus credenciales.

Antes de comenzar, asegúrese de que su Synology NAS se ha unido al dominio de Azure AD a través de VPN o al dominio de Windows que ya está sincronizado con el dominio de Azure AD.

Para configurar su Synology NAS como cliente de SSO de Azure:

  1. Acceda a Panel de control > Dominio/LDAP > Cliente de SSO.
  2. Active la casilla de verificación Habilitar servicio Azure SSO.
  3. Especifique el ID de cliente, el Secreto del cliente, el ID de inquilino y la URI de redirección en los campos correspondientes.
  4. Haga clic en Aplicar para guardar la configuración.

Nota:

  • Para conocer los pasos detallados para obtener el ID/Secreto del cliente y el ID de inquilino, consulte las secciones que aparecen a continuación.
  • Por motivos de seguridad, los usuarios deben iniciar sesión en DSM con Azure SSO a través de HTTPS.

Para obtener el ID de cliente y el Secreto del cliente para una aplicación:

  1. Inicie sesión en el portal clásico de Azure con una cuenta de administrador.
  2. En ACTIVE DIRECTORY, seleccione el directorio en el que se incluirán las cuentas de usuario.
  3. Vaya a APLICACIONES y haga clic en AGREGAR en la parte inferior para agregar una nueva aplicación.
  4. Haga clic en Agregar una aplicación que mi organización está desarrollando.
  5. Rellene el Nombre de su aplicación y seleccione la casilla de verificación APLICACIÓN WEB Y/O API WEB.
  6. Rellene la URL DE INICIO DE SESIÓN. Por ejemplo, https://yourdomain:port/webman/login.cgi
  7. Rellene URI DE ID. DE APLICACIÓN. Por ejemplo, https://yourdomain:port/
  8. Después de completar la configuración, seleccione su aplicación y encontrará el ID DE CLIENTE.
  9. Agregue una clave y haga clic en AGREGAR. En ese momento obtendrás el SECRETO DEL CLIENTE en texto sin formato.

Para obtener el ID de inquilino para una aplicación:

  1. Repita los pasos del 1 al 3 de la sección anterior.
  2. En la pestaña APLICACIONES, seleccione una aplicación y haga clic en VER EXTREMOS en la parte inferior.
  3. Se mostrará una secuencia de ID en cada campo de texto. Copie y pegue el ID en el campo ID de inquilino.

Nota:

  • La interfaz de usuario de Microsoft Azure™ está sujeta a cambios sin previo aviso.
  • Para obtener más información sobre el mecanismo de Azure AD, consulte las siguientes páginas web:
    • Documentación de Azure Directory
    • Documentación de Active Directory Domain Services
