Crear una carpeta compartida

Las carpetas compartidas son los directorios básicos en los que puede almacenar y administrar archivos y carpetas de su Synology NAS. Antes de almacenar archivos, debe crear al menos una carpeta compartida en DSM.

Para crear una carpeta compartida:

  1. Vaya a Panel de control > Carpeta compartida y haga clic en Crear.
  2. Especifique un nombre y una descripción para la carpeta compartida.
  3. Seleccione un almacenamiento en el que desea crear la carpeta compartida (omita este paso si solo existe un almacenamiento).
  4. Si fuera necesario, modifique las siguientes opciones:
    • Ocultar esta carpeta compartida en 'Mis sitios de red': evita que la carpeta compartida aparezca en 'Red' en el explorador de archivos de Windows. Esta opción no afecta los privilegios de acceso de la carpeta compartida.

      Observación:

      • Ocultar una carpeta compartida en 'Mis sitios de red' no afecta a sus derechos de acceso. Los derechos de acceso de las carpetas compartidas ocultas seguirán siendo los mismos. Los usuarios que tengan los derechos de acceso adecuados a la carpeta compartida seguirán pudiendo acceder a ella escribiendo '\\nombre del servidor\nombre de la carpeta compartida'.
    • Ocultar subcarpetas y archivos a usuarios sin permisos: cuando la opción Ocultar subcarpetas y archivos a usuarios sin permisos esté habilitada, si un usuario sin privilegios de lectura intenta acceder a una carpeta compartida a través del protocolo para compartir archivos de Windows, el usuario no podrá ver subcarpetas ni archivos dentro de la carpeta compartida. Esto evita que los usuarios puedan confundirse al conectarse a sus productos Synology y encuentren un gran número de archivos y subcarpetas que no pueden abrir.

      Observación:

      • Esta función solo es compatible con el protocolo para compartir de archivos de Windows.
      • Puede modificar los ajustes de privilegios de cada carpeta y archivo de File Station haciendo clic en ellos con el botón derecho. Elija Propiedades en el menú desplegable y haga clic en la pestaña Permiso.
      • La carpeta compartida no se ocultará a los usuarios sin permiso al habilitar Ocultar subcarpetas y archivos a usuarios sin permisos. Solo se ocultarán las subcarpetas y los archivos dentro de esta carpeta compartida.
  5. Habilitar papelera de reciclaje: cuando se eliminan archivos de la carpeta compartida, se moverán a una carpeta llamada #recycle. El acceso a la papelera de reciclaje estará limitado a usuarios que pertenecen al grupo administrators.
  6. Habilitar la cuota de carpetas compartidas: para especificar la capacidad máxima de cada carpeta compartida marque la casilla de verificación Habilitar la cuota de carpetas compartidas y escriba la capacidad máxima (por ejemplo, 10 GB) en el siguiente campo.

    Observación:

    • Solo cuando el almacenamiento utilice el sistema de archivos Btrfs, la opción Habilitar la cuota de carpetas compartidas estará disponible en las carpetas compartidas.
    • El uso del almacenamiento combinado de todos los directorios home de los usuarios no puede exceder la cuota de 'homes'.
  7. Habilitar la suma de comprobación para la integridad de datos avanzada: para proteger una carpeta compartida con estrategias de suma de comprobación CRC32 y copia en escritura, puede habilitar la suma de comprobación de datos para la integración de datos avanzada durante la creación de la carpeta compartida. Se aplica una suma de comprobación de CRC32 para verificar si los datos siguen siendo idénticos a los grabados originalmente y si el sistema utiliza automáticamente redundancia RAID para corregir los datos dañados. La copia en escritura ayuda a mejorar la consistencia de datos cuando se produce un cierre anómalo del sistema.

    Observación:

    • Esta opción está disponible solo para volúmenes que utilizan el sistema de archivos Btrfs.
    • Puede habilitar/deshabilitar esta opción para cada carpeta compartida durante su creación. Esta opción no se puede modificar después de crear la carpeta compartida.
    • Al activar esta opción se permite al sistema llevar a cabo una depuración de datos a gran escala. Más información sobre la depuración de datos
    • Para garantizar la calidad del servicio, le recomendamos que no habilite la suma de comprobación de datos cuando la carpeta compartida se utilice para lo siguiente:
      • Alojar bases de datos o máquinas virtuales
      • Almacenar grabaciones de vídeo de Surveillance Station
      • Ejecutar servicios que requieren un patrón de escritura pequeño y aleatorio
  8. Habilitar compresión de archivos: el sistema de archivos Btrfs comprime automáticamente los datos de la carpeta compartida creada para ahorrar espacio de almacenamiento. Cuando se recuperan para su uso, los datos se descomprimen automáticamente.

    Observación:

    • Esta opción solo está disponible cuando se crea una carpeta compartida en un volumen Btrfs o cuando se habilita Protección avanzada de la integridad de datos.
    • Esta opción no se puede modificar después de crear la carpeta compartida.
    • La tasa de compresión difiere de los tipos de datos: Los archivos de texto tienen la mayor tasa de compresión, mientras que los archivos multimedia (p. ej. imágenes, música o vídeos) pueden no comprimirse adecuadamente, porque ya están comprimidos para la salida.
    • Esta opción no puede activarse si está habilitado el cifrado de las carpetas compartidas.
  9. Cifrar esta carpeta compartida: si desea cifrar la carpeta compartida, haga lo siguiente:
    1. Especifique y confirme una nueva clave de cifrado. La clave de cifrado no puede incluir comas (,) ni signos de igual (=).
    2. Si fuera necesario, marque la casilla Agregar clave de cifrado a Administrador de claves para montar la carpeta cifrada automáticamente cuando arranque el sistema. Si esta opción está deshabilitada, la carpeta compartida cifrada se desmontará la próxima vez que se reinicie Synology NAS. Para ver una carpeta compartida desmontada o acceder a ella, deberá montar la carpeta manualmente importando o escribiendo la clave de cifrado.
  10. Haga clic en OK para terminar.

    11. Observación:

    • Algunas funciones y opciones de carpeta compartida solo son compatibles con VDSM y los modelos con las siguientes arquitecturas de paquete: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview y Grantley. Para ver qué arquitectura de paquete es compatible con su modelo, consulte este artículo.

Reglas de nomenclatura de carpetas compartidas:

  • Los nombres de las carpetas compartidas no distinguen entre mayúsculas y minúsculas y pueden incluir de 1 a 32 caracteres. Los nombres no pueden incluir los siguientes caracteres: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Los siguientes nombres están reservados para el uso del sistema: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • El primer carácter de un nombre de carpeta compartida no puede ser un signo menos o un espacio y el último no puede ser un espacio.
  • La descripción de la carpeta compartida distingue entre mayúsculas y minúsculas, y está limitada a 64 caracteres Unicode que se pueden mostrar.

Cifrado de carpetas compartidas:

El cifrado AES de 256 bits protege los datos bloqueando todos los intentos de acceso no autorizados. Una vez que se haya cifrado una carpeta compartida, ninguna otra persona sin clave de cifrado podrá acceder ni modificar los datos cifrados, aunque retiren los discos duros de su Synology NAS y los monten en otro dispositivo.

Observación:

  • El nombre de un archivo o carpeta dentro de una carpeta compartida cifrada no puede superar los 143 caracteres ingleses o 47 caracteres asiáticos (CJK); si accede a las carpetas compartidas desde Mac OS, el nombre de un archivo o carpeta dentro de una carpeta compartida cifrada no puede superar los 130 caracteres ingleses o 43 caracteres asiáticos (CJK).
  • No se puede acceder a las carpetas compartidas a través de NFS.
  • Si restaura las contraseñas predeterminadas utilizando el botón de reinicio situado en Synology NAS, las carpetas compartidas cifradas se desmontarán y la opción de montar automáticamente las carpetas compartidas codificadas se inhabilitará.
  • Las carpetas compartidas cifradas solo son compatibles en el modo con privilegios altos (solo aplicable a Dockerx64). Para usar esta función vaya al paquete Docker de su DSM host, seleccione el Docker DSM para el que desea habilitar las carpetas compartidas cifradas, haga clic en Editar y marque la casilla junto a Ejecutar contenedor usando privilegios altos.
