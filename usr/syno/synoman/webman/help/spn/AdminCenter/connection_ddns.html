DDNS

DDNS (Servicio dinámico de nombres de dominio) simplifica la conexión a su Synology NAS a través de Internet asignando un nombre de host a su dirección IP. Por ejemplo, DDNS le permite acceder a su Synology NAS con un nombre de dominio (p. ej., www.john.synology.me), de forma que no sea necesario recordar una dirección IP (p. ej., 172.16.254.1).

Configurar DDNS

Puede asignar un nombre de host existente a la dirección IP de su Synology NAS o bien registrar uno nuevo suministrado por Synology o por otros servidores DDNS. Consulte a cada proveedor para obtener más información sobre el registro de nombres de host.

Para configurar un nombre de host de DDNS:

  1. Haga clic en el botón Agregar.
  2. Aparece un cuadro de diálogo pidiéndole que edite las siguientes opciones:
    • Proveedor de servicios: seleccione un proveedor de servicios. Para registrar un nombre de host gratuito de Synology, elija Synology en el menú desplegable.
    • Nombre de host: escriba un nombre de host DDNS registrado, como john.synology.me.
    • Nombre de usuario/Correo electrónico: Introduzca el nombre de usuario/correo electrónico de su proveedor de DDNS.
    • Contraseña/Clave: introduzca la contraseña/clave de su proveedor de DDNS.
    • Dirección externa: introduzca la dirección IP externa del Synology NAS que utilizará el nombre de host.
    • Heartbeat: habilite esta opción para recibir alertas sobre el estado del nombre de host asignado (solo para nombres de host proporcionados por Synology).
  3. Haga clic en Probar conexión para comprobar que la configuración es correcta.
  4. Haga clic en OK para guardar y finalizar.
  5. Si ha seleccionado Synology como proveedor de servicios, aparecerá un cuadro de mensaje si no hay un certificado SSL que coincida con su nombre de host de DDNS para recomendarle que firme un certificado de Let's Encrypt para el nombre de host y lo establezca como certificado predeterminado. Haga clic en Sí.

    Observación: Si un certificado SSL ya coincide con su nombre de host de DDNS, aparecerá un cuadro de mensaje y se le preguntará si desea establecerlo como el certificado predeterminado. Este mensaje aparecerá cada vez que se edite el nombre de host de DDNS. Para obtener más información acerca del certificado SSL, consulte aquí.

Una vez completada la configuración, puede acceder a su Synology NAS a través de Internet introduciendo el nombre de host de DDNS en un explorador web.

Requisitos y limitaciones

  • Los servidores del proveedor de servicios DDNS están funcionando.
  • Synology NAS puede conectarse a Internet.
  • DDNS solo funciona en un entorno de red en el que no sea necesario ningún servidor proxy.
  • Solo puede introducir un nombre de host para cada proveedor DDNS.

Administrar nombres de host

Utilice los siguientes botones para administrar entradas de nombres de host existentes:

  • Eliminar: permite eliminar la entrada de nombre de host seleccionada de la lista.
  • Editar: permite cambiar la configuración de la entrada de nombre de host.
  • Actualizar ahora: permite actualizar el estado de todas las entradas de nombre de host que tienen habilitado el servicio DDNS.

Configurar un proveedor de DDNS personalizado

Puede añadir un proveedor de DDNS personalizado si su proveedor todavía no aparece en la lista de proveedores de servicios.

Para configurar un proveedor de DDNS:

  1. Haga clic en Personalizar.
  2. Aparece un cuadro de diálogo pidiéndole que introduzca las siguientes configuraciones:
    • Proveedor de servicios: Introduzca el nombre de su proveedor de servicios personalizado.
    • URL de consulta: Introduzca la URL de su proveedor de servicios personalizado.
  3. Haga clic en Guardar para añadir el proveedor de servicios personalizado a la lista de proveedores de servicios de DDNS.

Requisitos y limitaciones

  • La URL de consulta se debe adherir al protocolo especificado por su proveedor de servicios. Esta URL se utilizará para actualizar la información de IP en el servidor DDNS de su proveedor.
  • Una URL de consulta normalmente contiene los cuatro parámetros siguientes. Al introducir una URL de consulta, sustituya los siguientes parámetros por la variable correspondiente.
    • Nombre de usuario: __USERNAME__
    • Contraseña: __PASSWORD__
    • Nombre de host: __HOSTNAME__
    • Dirección externa: __MYIP__
  • La URL de consulta no puede contener espacios ni puntos y coma.
