Correo electrónico

Habilitar las notificaciones por correo electrónico permite a Synology NAS enviarle mensajes a su correo electrónico cuando se produzcan determinados eventos, como errores del sistema, pérdidas de conexión a la red, fallos en el comportamiento del ventilador, cortes de alimentación o no se puedan completar las tareas de copias de seguridad.

Observación:

  • El evento de comportamiento del ventilador no está disponible en un modelo sin ventilador, como EDS14. Para obtener más información, visite el sitio web de Synology.

Para habilitar las notificaciones por correo electrónico:

  1. Vaya al Panel de control > Notificación > Correo electrónico.
  2. Marque Habilitar las notificaciones de correo electrónico.
  3. Introduzca una o varias direcciones de correo electrónico en el campo Dirección de correo electrónico del destinatario. Las notificaciones del sistema se enviarán a las direcciones introducidas aquí. Se permite introducir dos direcciones. Puede separar varias direcciones con un punto y coma (;).
  4. Si fuera necesario, agregue un prefijo de asunto en el campo Prefijo del asunto. Este prefijo se agregará al asunto de cada mensaje enviado por Synology NAS para que pueda identificar y filtrar los mensajes con facilidad.
  5. Si desea utilizar un servidor de correo electrónico de terceros (por ejemplo, Gmail o Outlook) para transmitir mensajes, consulte la sección sobre "correos electrónicos de terceros" a continuación. Si desea utilizar un servidor personalizado, consulte la sección sobre "servidor SMTP personalizado".

Para utilizar un servicio de correo electrónico de terceros:

  1. En el menú de Proveedor de servicios, elija un servicio de correo electrónico de terceros, como Gmail o Outlook.
  2. Introduzca su nombre de usuario y contraseña para el servicio de correo electrónico seleccionado.
  3. Haga clic en Aplicar para guardar la configuración.
  4. Puede hacer clic en Enviar un correo electrónico de prueba para ver si la configuración es correcta.

Para utilizar un servidor SMTP personalizado:

  1. En el menú Proveedor de servicios, seleccione Servidor SMTP personalizado.
  2. Introduzca el nombre del servidor SMTP o la dirección IP en el campo Servidor SMTP.
  3. Escriba el número de puerto SMTP en el campo Puerto SMTP (si ha especificado un puerto SMTP diferente al configurar las reglas de reenvío de puerto, introduzca ese número).
  4. Si su servidor SMTP requiere autenticación, marque Se requiere autenticación y escriba su Nombre de usuario y Contraseña para el servidor de correo electrónico.
  5. Si su servidor SMTP requiere una conexión SSL/TLS, marque Es necesaria una conexión segura (SSL/TLS).
  6. Si desea personalizar el nombre del remitente que aparece en los mensajes de correo electrónico enviados por Synology NAS, introduzca el nombre en el campo Nombre del remitente. Si desea personalizar la dirección de correo electrónico del remitente, introduzca la dirección de correo electrónico en el campo Correo electrónico del remitente. Si el campo está en blanco, se utilizará la primera dirección de correo electrónico en el campo Dirección de correo electrónico del destinatario.
  7. Haga clic en Aplicar para guardar la configuración.
  8. Puede hacer clic en Enviar un correo electrónico de prueba para ver si la configuración es correcta.

Observación:

  • Si desea enviar un mensaje de bienvenida a cada nueva cuenta de usuario creada, marque Enviar mensajes de bienvenida a usuarios nuevos y cada usuario recibirá un mensaje de bienvenida cuando se cree su cuenta.
  • El idioma de las notificaciones de correo electrónico se puede configurar en Idioma de notificación. Para cambiar el idioma de notificación por correo electrónico, vaya a Menú principal > Panel de control > Opciones regionales, haga clic en la pestaña Idioma y, a continuación, seleccione el idioma que prefiera en el menú desplegable de la sección Idioma de notificación.
  • Si utiliza el servidor SMTP de Gmail y ha habilitado la verificación en dos pasos, recuerde que debe utilizar su "contraseña específica de la aplicación" proporcionada por Google para la autenticación SMTP.

Nombre del dominio del servidor SMTP:

El servidor SMTP puede ser una dirección IP o un nombre de dominio. En el segundo caso, asegúrese de haber introducido una dirección IP de DNS server válida en Menú principal > Panel de control > Red y que el servidor se encuentre en Internet.

Observación:

Debido a un cambio en la política de API de Google Gmail, los usuarios deben elegir la Configuración personalizada del servidor SMTP en el menú desplegable Proveedor de servicios para enviar mensajes de correo electrónico con estas tres finalidades. Para obtener más información sobre su configuración, consulte esta página.

  • Usar ssmtp binario para enviar mensajes de correo electrónico, por ejemplo, ejecutar scripts definidos por usuario en el Programador de tareas.
  • Crear un sitio web con Web Station y enviar mensajes de correo electrónico con PHP. Esto también incluye enviar mensajes de correo electrónico desde un paquete de un sitio web de terceros como Wordpress, MantisBT, etc.
  • Enviar notificaciones de correo electrónico cuando alguien deje un comentario en una foto o artículo de Photo Station.
