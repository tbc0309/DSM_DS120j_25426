效能

按一下效能頁籤來檢視資源使用情形。當滑鼠移上圖表時，會顯示詳細的日期、時間及資訊。

於總覽頁籤中，CPU、GPU、系統記憶體、GPU 記憶體、網路、磁碟、儲存空間及 iSCSI 的圖表將顯示於此。

CPU

您可查看使用者、系統及 I/O 等待的整體 CPU 使用率。在剛開啟此畫面時，CPU 負載可能會很高，因為系統除了正在收集系統資源的統計資料外，還需負載顯示頁面。

  • 使用者代表 CPU 用於處理使用者應用程式的比例。
  • 系統代表 CPU 用於系統核心、軟 / 硬體中斷 (IRQ) 的比例。
  • I/O 等待則代表用於 I/O 等待的比例。
  • 使用率為使用者及系統 CPU 使用率的總和。
  • 您可看到 1、5、15 分鐘的平均負載，代表運行進程佇列 (執行中的程序數加上等待執行的程序數之總和)。

名詞解釋

  • I/O 等待：I/O 等待代表 CPU 處於閒置狀態且沒有被任何程式使用。

    • I/O 等待是正常的現象。CPU 速度通常遠大於 I/O，因此 CPU 經常需要等候磁碟或網路傳輸完畢。然而，如果 I/O 等待的比例過高，代表 CPU 耗費大部份的時間在 I/O 等待上，而沒有被妥善利用。您可以使用較有效率的磁碟群組配置 (例如 SHR 或 RAID 5) 或網路設定 (例如 Bonding) 來取得更佳的系統效能。

GPU

整體 GPU 使用率將顯示於此。您可以從下拉式選單中選擇切換至 GPU 或 GPU 記憶體。

  • 使用率為使用者及系統 GPU 使用率的總和。
  • 於 GPU 記憶體中，您可以分別檢視已用容量及可用空間的記憶體配置。

注意:

  • 僅適用於支援 GPU 的機種。

系統記憶體

畫面上會顯示整體的實體記憶體使用率及置換空間的記憶體使用率。記憶體使用率維持很高是因為系統將常用資料儲存於快取中，因此能快速地取得資料，不需存取硬碟。當整體記憶體不足時，系統會釋放快取記憶體。

如果置換空間使用率極高，代表系統記憶體不足，將會影響到系統效能。您可以從上方的下拉式選單中選擇置換空間來檢視移入 / 移出率。

網路

網路流量圖會以每秒多少 KB 的方式顯示資料傳送與接收的流量。若有 PPPoE 連線，亦會顯示其網路流量。

磁碟

您可以按一下圖表上方的下拉式選單，來選擇檢視整體的磁碟使用率、傳輸速率或讀寫次數。按一下下方的檢視全部，勾選更多磁碟並將其新增至圖表。

儲存空間

此頁面顯示儲存空間的整體傳輸狀態。按一下圖表上方的下拉式選單來檢視傳輸用途、傳輸速度或傳輸效能 (IOPS)。按一下檢視全部按鈕來選擇個別的儲存空間，並將其新增至圖表。

iSCSI

此頁面顯示當前運作中的 iSCSI LUN 整體狀態。按一下圖表上方的下拉式選單來選取一個 iSCSI LUN 並檢視其狀態。預設會顯示三張圖表：傳輸量、讀寫次數、延遲率。若要取得更多詳細資訊，按一下選擇圖表按鈕來選取其他想要檢視的圖表。
