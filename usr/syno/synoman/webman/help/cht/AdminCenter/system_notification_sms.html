簡訊

您可以啟動簡訊通知，讓 Synology NAS 在系統狀態改變或是錯誤出現的時候寄送訊息通知您，例如連線中斷、風扇狀態改變、斷電、儲存空間不足、備份或還原任務失敗等狀況 。

注意：

  • 風扇狀態改變之事件不適用於無風扇機種，例如 EDS14。請前往 Synology 網站以了解更多資訊。

若要啟動簡訊通知：

勾選啟動簡訊通知並完成下列設定：

  1. 從簡訊服務供應商下拉式選單選擇您欲使用的簡訊服務供應商。預設的服務供應商為「clickatell」、「clickatell-2017」和「SendinBlue-v3」。
  2. 按一下新增簡訊服務供應商來建立新的簡訊供應商設定檔，或是按一下刪除來刪除所選的簡訊供應商設定檔。
  3. 若簡訊供應商要求以使用者帳號、密碼作為驗證方式，需在簡訊服務供應商設定中將供應商指定之參數選擇為對應的使用者帳號、密碼型別，並在使用者帳號欄位，輸入您的簡訊服務帳號，以及在密碼及確認密碼欄位中，輸入您向簡訊供應商申請的使用者密碼，並再輸入一次進行確認。
  4. 若簡訊供應商要求以 API 金鑰作為驗證方式，需在簡訊服務供應商設定中將供應商指定之參數選擇為 API 金鑰，並在API 金鑰欄位輸入簡訊供應商提供之 API 金鑰。
  5. 若簡訊供應商提供更改寄送人名稱的參數，需在簡訊服務供應商設定中將供應商指定之參數選擇為寄送人，並在寄送人欄位輸入寄送人名稱，其格式須符合簡訊供應商的要求。
  6. 輸入主要電話號碼或次要電話號碼。如果您已輸入兩組號碼，則通知簡訊亦會同時發送到這兩組號碼。
  7. 如果您不想接收太多訊息，請勾選設定簡訊間隔限制，並將通知訊息的數量限制為在指定的時間間隔內僅收到一封訊息。
  8. 按一下寄送測試簡訊來寄送測試簡訊至指定的電話號碼。如果未收到訊息，則代表設定不正確。
  9. 按一下套用來儲存設定，或是按一下重置來刪除設定。

注意：

  • 您可以在系統通知語言中設定簡訊通知的語言。若要變更簡訊通知的語言，請前往主選單 > 控制台 > 區域選項，按一下語言頁籤，從系統通知語言區塊中的下拉式選單來選擇您偏好的語言。
  • 除了預設的服務供應商「clickatell」及「clickatell-2017」以外，皆以英文作為簡訊通知語言。

限制：

  • 使用者帳號：有大小寫之分，可設定 1 到 32 個 Unicode 字元。
  • 密碼：區分大小寫，可設定 0 到 128 個可顯示字元，包含字母、數字、符號，和空格。
  • API 金鑰：有大小寫之分，可設定 1 到 256 個 Unicode 字元。
  • 寄件人：有大小寫之分，可設定 1 到 64 個 Unicode 字元。
  • 最小發送簡訊間隔可輸入 1 至 10000 分鐘。

若要新增簡訊服務供應商：

  1. 按一下新增簡訊服務供應商。
  2. 在簡訊服務供應商名稱欄位中，輸入簡訊供應商名稱。
  3. 在測試網址輸入簡訊網址。您可以從簡訊供應者處取得此網址。
  4. 在HTTP 請求方法選擇 GET 或是 POST。您可以從簡訊供應商的 API 文件中取得資訊。
  5. 按一下下一步。
  6. 在編輯 HTTP 請求標頭步驟中，依照簡訊供應商提供的 API 文件，將需要設定在 HTTP 請求標頭的參數加入列表中。
  7. 按一下下一步。
  8. 若選擇的 HTTP 請求方法為 POST，此時會進入編輯 HTTP 請求主體的步驟。請依照簡訊供應商提供的 API 文件，將需要設定在 HTTP 請求主體的參數加入列表中。
  9. 按一下下一步。
  10. 系統會從測試網址、 HTTP 請求標頭，以及 HTTP 請求主體中分析出數個參數。您需要為每個網址的參數設定一個相對應的類別。除了其他參數以外，各參數僅能使用一次。

限制：

  1. 供應商名稱：有大小寫之分，可設定 1 到 32 個 Unicode 字元。
  2. 測試網址：有大小寫之分，可設定 1 到 500 個 Unicode 字元。測試網址僅供設定之用。設定完成之後並不會傳送任何簡訊。網址的格式是由簡訊服務供應商提供。以 Clickatell 為例，網址可能如下：
    https://api.clickatell.com/http/sendmsg?user=TestUser&password=TestPassword&api_id=3148203&to=886123456789&text=Hello+world
  3. 參數名稱：有大小寫之分，可設定 1 到 32 個 Unicode 字元。
  4. 參數值：有大小寫之分，可設定 1 到 256 個 Unicode 字元。
  5. 參數至多可以設定 10 個。
  6. 「簡訊內容」參數必須輸入「Hello」與「world」，「Hello」與「world」之間的字元必須是服務供應商定義的空格字元。以 Clickatell 為例，「簡訊內容」參數必須輸入 Hello+world，因為供應商以「+」來取代空格字元。
