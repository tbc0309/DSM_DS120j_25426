DSM 設定

在控制台 > 網路 > DSM 設定，可以變更用戶端存取 Synology NAS 時使用的連接埠編號。

若要更改預設連接埠編號：

在 DSM 中，HTTP 及 HTTPS 連線永遠維持啟動。預設的 HTTP 連接埠為 5000，預設的 HTTPS 連接埠為 5001。

  1. 在 HTTP 或 HTTPS 欄位中輸入連接埠編號。
  2. 按一下套用。

若要讓 HTTP 連線重新導向至 HTTPS：

當用戶端嘗試透過 HTTP 連接埠存取您的 Synology NAS 時，可將用戶端重新導向至 HTTPS 連接埠。

  1. 選取將 HTTP 連線重新導向至 HTTPS (Web Station 及 Photo Station 除外)。
  2. 按一下套用。

HTTPS 安全連線使用的連接埠：

  • DSM：預設的連接埠編號為 5001，因此您可以輸入以下位址來安全存取 DSM：https://伺服器主機名稱:5001/
  • Web Station 及 Photo Station：預設的連接埠編號為 443，因此您可以輸入以下位址來安全存取 Web Station 及 Photo Station：https://伺服器主機名稱/

若要啟動 HTTP/2：

您可以啟動 HTTP/2 來加速 HTTPS 連線。

  1. 勾選啟動 HTTP/2。
  2. 按一下套用。

若要自訂最大同時 HTTP 連線數：

您可以啟動自訂最大同時 HTTP 連線數來提高 Synology NAS 可負擔的連線數。

注意：

  • 僅適用於 Bromolow、Cedarview、Avoton、Grantley、Broadwell、Dockerx64、Braswell、Broadwellnk、Apollolake 平台，若要了解您的機種是否支援此功能，請參閱 此篇文章。
  • 設定越高的 HTTP 同時連線數，將會需要更多的系統記憶體。

若要自訂 HTTP 回應的「Server」標頭：

  1. 勾選在 HTTP 回應中啟動「Server」標頭。
  2. 指定「Server」標頭的內容 (例如 nginx)。
  3. 按一下套用。

您可以取消勾選核取方塊來停用「Server」標頭。

網域

在網域下方，可以自訂用來存取 Synology NAS 的網域名稱。

若要啟動自訂網域：

  1. 勾選啟動自訂網域。
  2. 為您的 Synology NAS 指定已申請的網域名稱。
  3. 亦可勾選啟動 HSTS，強制此網域使用 HTTPS 加密連線。
  4. 按一下套用。

注意：

  • 每個網域名稱僅可搭配 Synology NAS 上的不同應用程式使用。
  • 如果網域名稱長度未超過 15 個字元並符合NetBIOS 命名慣例，系統便會自動註冊並廣播該網域名稱 (例如 MyDSM)。在相同區域網路內，只須透過與下列相似的網址，Windows 使用者便可存取 DSM 桌面：http://MyDSM 或 https://MyDSM
  • 如果透過使用網域名稱的網址存取應用程式，便會透過標準 HTTP 連接埠 (80) 或標準 HTTPS 連接埠 (443) 連線。
