SMB

O protocolo SMB (Server Message Block) permite que clientes Windows, Mac e Linux com suporte a SMB/CIFS acessem dados armazenados no Synology NAS. Se você desejar procurar arquivos no Synology NAS através do Windows Explorer ou montar pastas compartilhadas como unidades de rede, ative essa opção em Painel de controle > Serviços de arquivos > SMB/AFP/NFS.

Para habilitar o protocolo SMB:

  1. Marque Habilitar serviço SMB.
  2. Clique em Aplicar.

Nota:

Depois de habilitar o protocolo SMB, você pode montar as pastas totalmente indexadas no seu computador Mac via SMB e usar o Finder para pesquisar o conteúdo dos arquivos nas pastas montadas.

Grupo de trabalho

Esta opção permite que você designe um sistema como membro de um grupo de trabalho específico para fins de gerenciamento de rede.

Nota:

  • O nome de um grupo de trabalho pode conter de 1 a 15 caracteres. O nome não pode ter os seguintes caracteres: [ ] ; : " < > * + = \ / | ? ,
  • Se você estiver usando um Mac que executa o OS X 10.9 e seu Synology NAS estiver ingressado em um domínio do Windows, poderá entrar adicionando localhost\ antes do seu nome de usuário local do Synology NAS para acessar arquivos armazenados no Synology NAS através do protocolo SMB.

Desabilitar acesso a versões anteriores

No Windows Explorer, os usuários podem acessar instantâneos de pastas ou arquivos clicando na guia Versões anteriores dentro de uma pasta compartilhada, o que permite a restauração ou cópia de dados sem a ajuda do administrador. Se você escolher Desabilitar acesso a versões anteriores, a página “Versões anteriores” não exibirá nenhuma informação.

Nota:

  • Os usuários têm permissão de acessar apenas uma versão anterior se, no momento em que o instantâneo correspondente à versão anterior foi tirado, suas contas de usuários foram criadas ou receberam permissão de leitura da pasta compartilhada à qual corresponde o instantâneo tirado.

Ativação do registro de transferência

Marcar essa opção ativará o registro das operações de acesso a arquivos, incluindo criação, exclusão, leitura, gravação, transferência e renomeação de arquivos ou pastas através do protocolo CIFS/SMB, também conhecido como protocolo SMB.

Configurações avançadas

Clique no botão Configurações avançadas para ver as opções abaixo.

  • Servidor WINS: O Serviço de cadastramento na Internet do Microsoft Windows (WINS) é um serviço de mapeamento de nome NetBIOS para endereço de IP. Os usuários do Windows localizarão o sistema em redes TCP/IP com mais facilidade se o sistema tiver sido configurado para ser registrado em um servidor WINS.
  • Protocolo SMB máximo: Dependendo da sua configuração de rede, defina o protocolo SMB mais recente que será suportado pelo seu Synology NAS.
    • SMB1: O SMB1 (também chamado de CIFS) é suportado desde o Windows NT.
    • SMB2: O SMB2 é suportado desde o Windows Vista e é uma versão aprimorada do SMB. O SMB2 adiciona a capacidade de incorporar várias ações SMB em uma única solicitação a fim de reduzir o número de pacotes de rede e melhorar o desempenho.
    • SMB2 e MTU grande: A unidade máxima de transmissão (MTU) refere-se à maior unidade de dados que pode ser transmitida através de um protocolo de comunicação. Para aproveitar interfaces de rede mais rápidas, como Ethernet de 1 ou 10 gigabits, a Large MTU aumenta a unidade de transmissão máxima para 1 MB. A ativação da Large MTU aumenta a velocidade e a eficiência da transferência de arquivos grandes e, ao mesmo tempo, reduz o número de pacotes que precisam ser processados.
    • SMB3: O SMB3 é suportado desde o Windows 8 e do Windows Server 2012. Ele é uma versão aprimorada do SMB2. O SMB3 suporta a transmissão com criptografia de arquivo AES, aumentando assim a segurança de transferências de arquivos ponto a ponto.

    Nota:

    • O SMB2 tem suporte no Windows Vista, no Windows Server 2008 R2, no Windows 7 e em versões mais recentes.
    • A Large MTU tem suporte no Windows Server 2008 R2, no Windows 7 e em versões mais recentes.
    • Para tornar eficiente o Large MTU em seu computador com Windows, será necessário modificar as configurações de registro do computador. Para obter mais detalhes, consulte o suporte técnico da Microsoft (por exemplo, este artigo).
  • Protocolo SMB mínimo: Dependendo da sua configuração de rede, defina o protocolo SMB mais antigo que será suportado pelo seu Synology NAS.
  • Modo de criptografia de transporte: Quando o SMB3 for habilitado, o protocolo SMB adicionará criptografia de transporte para fortalecer a segurança da transmissão de arquivos.
    • Desabilitar: Nenhuma criptografia de transporte será aplicada.
    • Automático: A criptografia de transporte será aplicada apenas a clientes que suportam esse recurso.
    • Forçar: A criptografia de transporte será aplicada sempre. Isso fará com que os clientes que não suportam a criptografia de transporte não consigam usar o protocolo SMB.

    • Nota:

    • A criptografia de transporte é suportada no Windows 8, Windows Server 2012 e em versões mais recentes.
    • O login anônimo no protocolo SMB não é suportado quando a criptografia de transporte está habilitada.
    • Quando a criptografia de transporte for habilitada, deixe a opção Desabilitar Opportunistic Locking desmarcada para evitar atingir os tempos limite do aplicativo.
  • Habilitar Opportunistic Locking: Selecione para ativar o "Opportunistic Locking" de CIFS, para que o cliente possa melhor armazenar em cache o conteúdo do arquivo e evitar a sobrecarga de pequenas leituras/gravações. Quando essa opção estiver desmarcada, aplicativos de banco de dados terão melhor desempenho ao acessar arquivos de banco de dados armazenados na Synology NAS pela rede.
    • Habilitar concessão SMB2: Habilitar essa opção permite mais configurações relacionadas a SMB2 para ajustar o mecanismo de cache de cliente oferecido pelo Opportunistic Locking.
  • Ativar Local Master Browser: Quando existem vários computadores com Windows na mesma sub-rede, eles negociarão e elegerão um computador como o "local master browser." O local master browser mantém listas de outros computadores dentro da sub-rede e seus recursos compartilhados e compartilha essas listas com outros computadores. Essa opção permite que o Synology NAS assuma o papel de local master browser.

    • Nota:

    A habilitação desta função desabilita a hibernação do disco rígido e ativa a conta do convidado sem uma senha.

  • Habilitar o módulo VFS DirSort: Classifica os nomes de pasta e lista-os em ordem alfabética. Ativar esta opção afeta o desempenho do sistema.
  • Arquivos de veto: Oculta os arquivos de acordo com os critérios especificados. Os critérios podem incluir curingas (*) e entradas múltiplas devem ser separadas por barras (/). Por exemplo: /abc*/*.txt/*.conf/. Ativar esta opção afeta o desempenho do sistema.
  • Permitir links simbólicos nas pastas compartilhadas: Selecione para permitir que links simbólicos apontem para arquivos ou pastas na mesma pasta compartilhada. Eles podem ser criados usando o terminal de DSM.
    • Permitir links simbólicos entre pastas compartilhadas: Selecione para permitir que os links simbólicos apontem para arquivos ou pastas em diferentes pastas compartilhadas. Eles podem ser criados usando o terminal de DSM. No entanto, usuários sem privilégios talvez possam acessar as pastas compartilhadas de destino dos links simbólicos.
  • Habilitar módulo VFS para converter caracteres especiais do Mac: Selecione esta opção para converter caracteres reservados do SMB (por exemplo, < > " * ?) nos nomes de arquivos para latim básico durante a criação de arquivo. Esses arquivos serão reconhecíveis através de AFP.
  • Habilitar o módulo VFS MSDFS: Permite que links simbólicos apontem para arquivos ou pastas localizadas em outros servidores SMB. Com versões mais antigas do Windows, como o Windows XP, o cliente e o Synology NAS precisam participar de um domínio do Windows antes de ativar essa opção. Para saber mais sobre como criar os links MSDFS, consulte aqui. Observe que o cache do dispositivo cliente deve ser limpo após desabilitar esta opção, ou o desempenho normal poderá ser afetado.
  • Desabilitar múltiplas conexões do mesmo endereço IP: Quando essa opção estiver ativada, o Synology NAS só permite uma conexão por endereço IP de clientes usando o protocolo SMB. Quando vários clientes com o mesmo endereço IP usarem o protocolo SMB, o Synology NAS irá manter o mais recente e derrubar as conexões mais antigas. Em ambientes NAT, o cliente poderá usar um único endereço de IP, caso em que se ativar essa opção isso poderá afetar o desempenho normal.
  • Coletar registros de depuração: Quando essa opção for ativada, registros detalhados serão armazenados para fins de depuração. Ativar esta opção afeta o desempenho do sistema.
  • Aplique as permissões UNIX padrão: Habilite essa opção para aplicar as permissões UNIX padrão ao fazer upload ou criar arquivos e pastas. A permissão UNIX será 744 para arquivos e 755 para pastas. Quando essa opção é desativada, a permissão UNIX é 777 para arquivos e pastas.

    • Nota:

    • Ativar essa opção pode causar problemas de permissão inconsistente entre os diferentes protocolos. Para evitar inconsistências, sugerimos deixar essa opção desabilitada.

    Nota:

    • Para pastas compartilhadas com a ACL do Windows habilitada (todos os compartilhamentos, exceto "foto" e compartilhamentos em unidades externas), execute o comando chmod no cliente Linux ou FTP para alterar os tipos de permissão de arquivos e pastas da ACL do Windows para UNIX.
    • Ativar essa opção pode causar problemas de permissão inconsistente entre os diferentes protocolos. Para evitar inconsistências, sugerimos deixar essa opção desabilitada.
  • Não reservar espaço em disco ao criar arquivos: Habilite essa opção para não permitir que espaço em disco seja reservado ao se criar arquivos no seu Synology NAS.
  • Habilitar identificadores duráveis SMB (o bloqueio de arquivos entre protocolos será desabilitado): Habilitar essa opção permite aos usuários restaurar a conexão SMB2 perdida com esse Synology NAS após a desconexão temporária; os arquivos abertos permanecerão disponíveis durante a paralisação da rede. No entanto, o bloqueio de arquivos entre protocolos será desabilitado para que os usuários possam acessar ou modificar um arquivo bloqueado através de protocolos não SMB2 (por exemplo, AFP).
  • Ativar clone rápido Btrfs: Habilitar essa opção permite que os usuários copiem arquivos em velocidades aceleradas quando a origem e o destino estão no mesmo volume Btrfs.
  • Apagar cache SMB: Clique neste botão para excluir os seguintes arquivos do cache SMB, para fins de solução de problemas:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
