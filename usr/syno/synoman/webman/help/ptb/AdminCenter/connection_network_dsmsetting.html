Configurações do DSM

Em Painel de Controle > Rede > Configurações do DSM, você pode alterar o número da porta que os clientes usam para acessar o Synology NAS.

Para mudar o número de porta padrão:

Conexões HTTP e HTTPS são sempre habilitadas no DSM. A porta HTTP padrão é 5000 e a porta HTTPS padrão é 5001.

  1. Digite o número da porta no campo HTTP ou HTTPS.
  2. Clique em Aplicar.

Para redirecionar conexões HTTP para HTTPS:

Você pode redirecionar clientes para a porta HTTPS quando eles tentarem acessar seu Synology NAS através da porta HTTP.

  1. Selecione Redirecionar automaticamente conexões HTTP para HTTPS (exceto Web Station e Photo Station).
  2. Clique em Aplicar.

Portas usadas para a conexão HTTPS:

  • DSM: O número da porta padrão é 5001, portanto é possível acessar o DSM de forma segura digitando o seguinte endereço: https://server-hostname:5001/
  • Web Station e Photo Station: O número da porta padrão é 443, portanto é possível acessar o Web Station e o Photo Station de forma segura digitando o seguinte endereço: https://server-hostname/

Para habilitar HTTP/2:

Você pode acelerar conexões HTTPS habilitando o HTTP/2.

  1. Marque Habilitar HTTP/2.
  2. Clique em Aplicar.

Para personalizar o máximo de conexões HTTP simultâneas:

Você pode habilitar a opção Personalizar máximo de conexões HTTP simultâneas para aumentar o número de conexões HTTP no seu Synology NAS.

Nota:

  • Quanto maior o número máximo de conexões HTTP simultâneas, mais memória do sistema será consumida.

Para personalizar o cabeçalho "Server" em respostas HTTP:

  1. Marque Habilitar o cabeçalho “Server” em respostas HTTP.
  2. Especifique um cabeçalho para seu servidor Web (por exemplo, nginx).
  3. Clique em Aplicar.

Você pode desabilitar o cabeçalho “Server” desmarcando a caixa de seleção.

Domínio

Em Domínio, você pode especificar um nome de domínio personalizado para acessar o Synology NAS.

Para habilitar o domínio personalizado:

  1. Marque Habilitar domínio personalizado.
  2. Especifique um nome de domínio aplicado para seu Synology NAS.
  3. Você também pode marcar Habilitar HSTS para forçar conexões HTTPS criptografadas nesse domínio.
  4. Clique em Aplicar.

Nota:

  • Cada nome de domínio pode ser usado somente para um aplicativo específico no seu Synology NAS.
  • Se o nome de domínio não exceder 15 caracteres e respeitar as convenções de nomenclatura do NetBIOS (por exemplo, MyDSM), ele será registrado e transmitido pelo sistema automaticamente. Usuários do Windows podem acessar a área de trabalho do DSM na mesma rede local com uma URL semelhante: http://MyDSM ou https://MyDSM
  • Ao acessar aplicativos com URLs que usam nomes de domínio, você passará pela porta HTTP padrão (80) ou pela porta HTTPS padrão (443).
