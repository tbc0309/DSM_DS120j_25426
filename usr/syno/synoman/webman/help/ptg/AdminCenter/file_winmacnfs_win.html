SMB

O protocolo SMB (Server Message Block) permite aos clientes de Windows, Mac e Linux com suporte SMB/CIFS acederem aos dados armazenados em Synology NAS.Se pretende pesquisar ficheiros em Synology NAS através do Explorador do Windows ou montar pastas partilhadas como unidades de rede, ative esta opção em Painel de controlo > Serviços de ficheiros > SMB/AFP/NFS.

Para ativar o protocolo SMB:

  1. Assinale Ativar serviço SMB.
  2. Clique em Aplicar.

Nota:

Depois de ativar o protocolo SMB, pode montar as pastas totalmente indexadas no seu computador Mac via SMB e usar o Finder para procurar os conteúdos de ficheiros nas pastas montadas.

Grupo de Trabalho

Esta opção permite-lhe designar o sistema como membro de um grupo de trabalho específico para fins de gestão de rede.

Nota:

  • O nome de um grupo de trabalho pode incluir 1 a 15 carateres.O nome não pode incluir os seguintes carateres:[ ] ; :" < > * + = \ / | ?,
  • Se estiver a utilizar um Mac com o OS X 10.9 e o seu Synology NAS estiver associado a um domínio Windows, pode iniciar a sessão adicionando localhost\ antes do nome de utilizador local Synology NAS para aceder aos ficheiros armazenados em Synology NAS através do protocolo SMB.

Cancelar acesso a Versões Anteriores

No Explorador do Windows, os utilizadores podem aceder aos instantâneos de pastas ou ficheiros clicando no separador Versões anteriores dentro de uma pasta partilhada, permitindo o restauro ou cópia de dados sem a ajuda de um administrador.Se escolher Cancelar o acesso a versões anteriores, a página "Versões anteriores" não apresentará qualquer informação.

Nota：

  • Os utilizadores têm acesso a uma versão anterior apenas se, no momento em que o instantâneo correspondeu à versão anterior, as suas contas de utilizador tiverem sido criadas e tendo permissão de leitura da pasta partilhada à qual corresponde o instantâneo tirado.

Ativação de Registo de Transferência

Marcar esta opção ativará operações de acesso a registo do ficheiro, incluindo criar, eliminar, ler, escrever, mover e renomear ficheiros ou pastas através do protocolo CIFS/SMB, também conhecido como protocolo SMB.

Definições avançadas

Clique no botão Definições avançadas para ver as opções abaixo.

  • Servidor WINS:O Microsoft Windows Internet Name Service (WINS) é um serviço para mapear o nome NetBIOS para endereços IP.Os utilizadores Windows localizarão o sistema mais facilmente nas redes TCP/IP se o sistema tiver sido configurado para se registar com um servidor WINS.
  • Protocolo SMB máximo:Dependendo da configuração da sua rede, pode definir o protocolo SMB mais recente que será suportado pelo seu Synology NAS.
    • SMB1:SMB1 (também conhecido por CIFS) é suportado desde do Windows NT.
    • SMB2:O SMB2 é suportado desde o Windows Vista e é a versão melhorada do SMB.O SMB2 adiciona a capacidade de compor múltiplas ações SMB num só pedido, para reduzir o número de pacotes de rede e melhorar o desempenho.
    • SMB2 e Large MTU:A unidade máxima de transmissão (MTU) refere-se à unidade máxima de dados que pode ser transmitida num protocolo de comunicação.Para tirar partido de interfaces de rede mais rápidas, como Ethernet de 1 ou 10 gigabits, Large MTU aumenta a unidade máxima de transmissão para 1 megabyte (MB).Ativar Large MTU melhora a velocidade e eficiência de transferências de ficheiros grandes, enquanto reduz o número de pacotes que necessitam de ser processados.
    • SMB3:SMB3 é suportado desde o Windows 8 e Windows Server 2012.É uma versão melhorada de SMB2.SMB3 suporta transmissão de encriptação de ficheiros baseada em AES, melhorando assim a segurança da transferência de ficheiros ponto a ponto.

    Nota:

    • O SMB2 é suportado no Windows Vista, Windows Server 2008 R2, Windows 7 e posterior.
    • Large MTU é suportado no Windows Server 2008 R2, Windows 7 e superior.
    • Para que Large MTU seja eficaz no computador Windows, terá de modificar as definições de registo do computador.Para mais detalhes, consulte o suporte técnico da Microsoft (tal como este artigo).
  • Protocolo SMB mínimo:Dependendo da configuração da sua rede, pode definir o protocolo SMB mais antigo que será suportado pelo seu Synology NAS.
  • Modo de encriptação de transporte:Quando SMB3 está ativado, o protocolo SMB adicionará encriptação de transporte para reforçar a segurança na transmissão de ficheiros.
    • Desativar:Não será aplicada qualquer encriptação de transporte.
    • Auto:A encriptação de transporte será apenas aplicada aos clientes que suportem esta funcionalidade.
    • Forçar:A encriptação de transporte será sempre aplicada.Isto levará a que os clientes que não suportem a encriptação de transporte não consigam usar o protocolo SMB.

    • Nota:

    • A encriptação de transporte é suportada no Windows 8, Windows Server 2012 e superior.
    • Não é suportado o início de sessão anónimo para o protocolo SMB quando ativada a encriptação de transporte.
    • Quando ativada a encriptação de transporte, deixe desmarcado Desativar Opportunistic Locking para evitar inatividade da aplicação.
  • Ativar Opportunistic Locking:Selecione para ativar "Opportunistic Locking" de CIFS, para que o cliente consiga melhor conteúdos cache menores tempos de leitura/escrita.Quando esta opção está desmarcada, as aplicações da base de dados têm melhor desempenho ao aceder a ficheiros de base de dados armazenados no Synology NAS através da rede.
    • Ativar autorização SMB2:Ativar esta opção permite mais definições relacionadas com SMB2 para ajustar o mecanismo de cache do cliente fornecido por Opportunistic Locking.
  • Ativar Local Master Browser:Quando existem vários computadores Windows na mesma sub-rede, negociam e elegem um computador "local master browser"." O local master browser mantém listas dos outros computadores dentro da sub-rede e respetivos recursos partilhados, e partilha estas listas com os outros computadores.Esta opção permite a Synology NAS assumir a função de local master browser.

    • Nota:

    Ativar esta função desativa a hibernação de disco rígido e ativa a conta guest sem uma palavra-passe.

  • Activar módulo DirSort VFS:Ordena os nomes das pastas e lista-as por ordem alfabética.A ativação desta opção afeta o desempenho do sistema.
  • Vetar ficheiros:Oculta ficheiros de acordo com os critérios especificados.Os critérios podem incluir carateres universais (*) e as várias entradas devem estar separadas por uma barra (/).Por exemplo:/abc*/*.txt/*.conf/.A ativação desta opção afeta o desempenho do sistema.
  • Permitir ligações simbólicas dentro de pastas partilhadas:Seleccione para permitir que as ligações simbólicas apontem para ficheiros ou pastas na mesma pasta partilhada.As ligações simbólicas podem ser criadas utilizando o terminal DSM.
    • Permitir ligações simbólicas em pastas partilhadas:Selecione para permitir que as ligações simbólicas apontem para ficheiros ou pastas em diferentes pastas partilhadas.As ligações simbólicas podem ser criadas utilizando o terminal DSM.No entanto, utilizadores sem privilégios podem aceder às pastas partilhadas de destino de ligações simbólicas.
  • Ativar módulo VFS para converter caracteres especiais do Mac:Selecione para converter caracteres reservados-SMB (por ex., < > " * ?) em nomes de ficheiros para Latim Básico durante a criação do ficheiro.Tais ficheiros serão reconhecíveis por AFP.
  • Activar módulo MSDFS VFS:Permite que as ligações simbólicas apontem para ficheiros ou pastas localizados noutros servidores SMB.Em versões antigas do Windows, tais como o Windows XP, o cliente e o Synology NAS têm de aderir a um domínio Windows antes de ativar esta opção.Para mais informações sobre como criar ligações MSDFS, consulte aqui.Tenha em conta que a cache do dispositivo cliente deve ser limpa após desativar esta opção, caso contrário, o desempenho normal pode ser afetado.
  • Desactivar múltiplas ligações a partir do mesmo endereço IP:Quando esta opção está ativada, o Synology NAS apenas permite uma ligação por endereço IP de clientes que utilizem o protocolo SMB.Quando múltiplos clientes com o mesmo endereço IP utilizam o protocolo SMB, o Synology NAS mantém o último e elimina ligações antigas.Num ambiente NAT, o cliente pode estar a utilizar um endereço IP único. Neste caso, ao ativar esta opção, o desempenho normal pode ser afetado.
  • Reunir registos de depuração:Quando esta opção está ativada, são armazenados registos detalhados para fins de depuração.A ativação desta opção afeta o desempenho do sistema.
  • Aplicar permissões de UNIX predefinidas:Ative esta opção para aplicar as permissões de UNIX predefinidas quando carregar ou criar ficheiros e pastas.A permissão de UNIX será 744 para ficheiros e 755 para pastas.Quando esta opção estiver desativada, a permissão de UNIX é 777 para ficheiros e pastas.

    • Nota:

    • Ativar esta opção pode provocar questões de inconsistência de permissões entre diferentes protocolos.Para evitar inconsistências, sugerimos que deixe esta opção desativada.

    Nota:

    • Para pastas partilhadas com ativação ACL do Windows (todas as partilhas, excluindo "photo" e partilhas em discos externos) execute o comando chmod no seu Linux ou cliente FTP para alterar os tipos de permissão de pasta e ficheiro do ACL do Windows para UNIX.
    • Ativar esta opção pode provocar questões de inconsistência de permissões entre diferentes protocolos.Para evitar inconsistências, sugerimos que deixe esta opção desativada.
  • Não reservar espaço de disco quando cria ficheiros:Ative esta opção para proibir que mais espaço de disco seja reservado quando cria ficheiros no seu Synology NAS .
  • Ative alcunhas duráveis SMB (Será desativado o bloqueio de ficheiros entre múltiplas plataformas):Ativar esta opção permite aos utilizadores restaurar ligações SMB2 perdidas para este Synology NAS após desconexão temporária. Os ficheiros abertos permanecerão disponíveis durante a falha de rede.Porém, o bloqueio de ficheiros entre múltiplas plataformas será desativado para que os utilizadores possam aceder ou modificar um ficheiro bloqueado através de protocolos não SMB2 (por ex. AFP).
  • Ativar clonagem rápida Btrfs:Ativar esta opção permite aos utilizadores copiar ficheiros a velocidades aceleradas quando tanto a origem como o destino estão dentro do mesmo volume Btrfs.
  • Limpar cache SMB:Clique neste botão para eliminar os seguintes ficheiros da cache SMB para fins de resolução de problemas:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
