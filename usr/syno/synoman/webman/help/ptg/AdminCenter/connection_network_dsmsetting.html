Definições do DSM

Em Painel de Controlo > Rede > Definições DSM, pode alterar o número da porta que os clientes utilizam para aceder ao seu Synology NAS.

Para alterar o número da porta predefinida:

As ligações HTTP e HTTPS estão sempre ativadas no DSM. A porta HTTP predefinida é 5000 e a porta HTTPS predefinida é 5001.

  1. Introduza o número da porta no campo HTTP ou HTTPS.
  2. Clique em Aplicar.

Para redirecionar as ligações HTTP para HTTPS:

Pode redirecionar clientes para a porta HTTPS quando tentam aceder ao seu Synology NAS através da porta HTTP.

  1. Selecione Redireccionar automaticamente as ligações HTTP para HTTPS (excluindo a Web Station e a Photo Station).
  2. Clique em Aplicar.

Portas utilizadas para a ligação HTTPS:

  • DSM: O número da porta predefinido é 5001, por isso, pode aceder de forma segura ao DSM introduzindo o seguinte endereço: https://server-hostname:5001/
  • Web Station e Photo Station: O número da porta predefinido é 443, por isso, pode aceder de forma segura à Web Station e Photo Station introduzindo o seguinte endereço: https://server-hostname/

Para ativar HTTP/2:

Pode acelerar as ligações HTTPS ativando HTTP/2.

  1. Assinale Ativar HTTP/2.
  2. Clique em Aplicar.

Para personalizar o número máximo de ligações HTTP em simultâneo:

Poderá ativar Personalizar o número máximo de ligações HTTP em simultâneo para aumentar as ligações HTTP no seu Synology NAS.

Nota:

  • Quanto mais alto for o número máximo de ligações HTTP em simultâneo, mais será exigido da memória do sistema.

Para personalizar o cabeçalho “Servidor” nas respostas HTTP:

  1. Assinale Ativar cabeçalho “Server” nas respostas HTTP.
  2. Especifique um cabeçalho para o seu servidor web (por ex., nginx).
  3. Clique em Aplicar.

Pode desativar o cabeçalho “Servidor” desmarcando a caixa de verificação.

Domínio

Em Domínio, pode especificar um nome de domínio personalizado para aceder ao Synology NAS.

Para ativar o domínio personalizado:

  1. Marque Ativar domínio personalizado.
  2. Especifique um nome de domínio aplicado para o seu Synology NAS.
  3. Também pode marcar Ativar HSTS para forçar as ligações encriptadas HTTPS neste domínio.
  4. Clique em Aplicar.

Nota:

  • Cada nome de domínio pode ser usado apenas para uma aplicação em separado no seu Synology NAS.
  • Se o nome de domínio não exceder os 15 caracteres de comprimento, e cumprir as convenções de nome NetBIOS, o sistema irá registar e transmitir o nome de domínio automaticamente (por ex., MyDSM). Os utilizadores Windows pode aceder ao ambiente de trabalho DSM na mesma rede local com um URL parecido: http://MyDSM ou https://MyDSM
  • Quando acede a aplicações através de URLs que utiliza nomes de domínio, irá percorrer a porta HTTP padrão (80) ou a porta HTTPS padrão (443).
