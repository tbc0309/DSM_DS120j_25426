Skapa en Delad mapp

Delade mappar är de grundläggande mapparna där du kan lagra och hantera filer och mappar på din Synology NAS. Innan du lagrar några filer behöver du skapa minst en delad mapp i DSM.

Så här gör du för att skapa en delad mapp:

  1. Gå till Kontrollpanel > Delad mapp och klicka på Skapa.
  2. Specificera ett namn och en beskrivning av den delade mappen.
  3. Välj ett lagringsutrymme på vilken den delade mappen ska skapas (hoppa över detta alternativ om det bara finns ett lagringsutrymme).
  4. Ändra, om nödvändigt, följande alternativ:
    • Dölj denna delade mapp i ”Mina nätverksplatser”: Förhindrar att den delade mappen visas under ”Nätverk” i Windows filutforskare. Detta alternativ påverkar inte åtkomstprivilegierna för den delade mappen.

      Obs:

      • Att dölja en delad mapp i ”Mina nätverksplatser” påverkar inte dess åtkomsträttigheter. Åtkomsträttigheterna för en dold delad mapp ändras inte. Användare som har rätt åtkomsträttigheter till den delade mappen kan fortfarande få åtkomst till den genom att ange ”\\servernamn\namn på delad mapp”.
    • Dölj undermappar och filer från användare som saknar behörighet: När Dölj undermappar och filer från användare som saknar behörighet är aktiverat får inte en användare, utan läsrättigheter, som försöker få åtkomst till en delad mapp via protokollet Fildelning i Windows möjlighet att se undermappar eller filer i den delade mappen. Detta förhindrar att användare blir förvirrade när de ansluter till sin Synology-produkt och ser ett stort antal filer och undermappar som de inte kan öppna.

      Obs:

      • Denna funktion stöds bara av protokollet Fildelning i Windows.
      • Du kan ändra behörighetsinställningarna för varje mapp och fil i File Station genom att högerklicka på den. Välj Egenskaper i rullgardinsmenyn och klicka på fliken Behörighet.
      • Den delade mappen i sig självt kommer inte att vara dold för användare utan behörighet genom aktivering av Dölj undermappar och filer från användare som saknar behörighet. Endast undermappar och filer inom den här delade mappen kommer att vara dolda.
  5. Aktivera papperskorgen: När filer i den delade mappen raderas kommer de att flyttas till en mapp som heter Papperskorg. Åtkomst till papperskorgen kan ändras för användare som hör till gruppen administrators.
  6. Aktivera kvot för delade mappar: För att specificera den maximala kapaciteten för varje delad mapp, markera Aktivera kvot för delade mappar och ange den maximala kapaciteten (dvs. 10 GB) i följande fält.

    Obs:

    • Endast när lagringen använder filsystemet Btrfs kommer alternativet Aktivera kvot för delade mappar vara tillgängligt för delade mappar.
    • Den kombinerade användningen av lagringen för alla användares mappar home kan inte överskrida kvoten för ”homes”.
  7. Aktivera avancerat dataintegritetsskydd: För att skydda en delad mapp med strategierna CRC32 checksum och kopiera vid skrivning kan du aktivera avancerat dataintegritetsskydd när den delade mappen skapas. CRC32 checksum tillämpas för att kontrollera att data förblir identiskt med hur de ursprungligen skrevs och meddelar användaren om någon korruption upptäcks. Kopiera vid skrivning hjälper till att förbättra dataöverensstämmelse vid onormal avstängning av systemet.

    Obs:

    • Det här alternativet är bara tillgängligt för de volymer som använder filsystemet Btrfs.
    • Du kan aktivera/avaktivera det här alternativet för varje delad mapp när den skapas. Det här alternativet kan inte ändras när den delade mappen väl har skapats.
    • Om filer såsom databaser konstant redigeras och uppdateras kan mängden fragmenterade filer öka beroende på kopiera vid skrivning. Det är att rekommendera att du avaktiverar det här alternativet för att förbättra prestandan.
  8. Aktivera filkomprimering: Data i den skapade delade mappen komprimeras automatiskt av Btrf-filsystemet för att spara på lagringsutrymmet. När det hämtas för användning blir data automatiskt dekomprimerat.

    Obs:

    • Det här alternativet är endast tillgängligt när du skapar en delad mapp i en Btrfs-volym eller när Avancerat skydd av dataintegritet är aktiverat.
    • Det här alternativet kan ändras när den delade mappen har skapats.
    • Komprimeringsnivån skiljer sig åt mellan datatyper: Textfiler har den högsta nivån av komprimering medan multimediafiler (t.ex. bilder, musik eller videor) inte komprimeras lika bra eftersom de redan har komprimerats för utmatning.
    • Det här alternativet kan inte aktiveras om kryptering för delad mapp är aktiverat.
  9. Kryptera denna delade mapp: Om du vill kryptera den delade mappen, gör följande:
    1. Specificera och bekräfta en ny krypteringsnyckel. Krypteringsnyckeln kan inte inkludera komman (,) eller likhetstecken (=).
    2. Om nödvändigt, markera rutan Lägg till krypteringsnyckel till Nyckelhanterare för att montera den krypterade mappen automatiskt när systemet startas. Om detta alternativ är avaktiverat, kommer den krypterade mappen att vara omonterad nästa gång Synology NAS startar om. För att se eller få åtkomst till en omonterad delad mapp måste du montera mappen manuellt genom att importera eller ange krypteringsnyckeln.
  10. Klicka på OK för att slutföra.

Regler för namngivning av delade mappar:

  • Delade mappars namn är skiftlägeskänsligt och kan innehålla 1-32 tecken. Namn kan inte innehålla följande tecken: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Följande namn är reserverade för systemanvändning: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Det första tecknet i en delad mapps namn får inte vara ett minustecken eller ett mellanslag och det sista tecknet får inte heller vara ett mellanslagstecken.
  • Den delade mappens beskrivning är skiftlägeskänslig och kan innehålla upp till 64 Unicode-tecken.

Kryptera delade mappar:

AES 256-bit-krypteringen skyddar data genom att blockera alla oauktoriserade åtkomstförsök. När en delad mapp är krypterad kan inte de som inte har krypteringsnyckeln få åtkomst till eller ändra krypterad data, detta gäller även om de tar bort hårddiskarna från din Synology Synology NAS och monterar den i annan enhet.

Obs:

  • Namnet på en fil eller en mapp i en krypterad delad mapp kan inte överskrida 143 engelska tecken eller 47 asiatiska (CJK) tecken; om du använder dina delade mappar med Mac OS kan namnet på en fil eller mapp i en krypterad delad mapp inte överskrida 130 engelska tecken eller 43 (CJK) tecken.
  • Det går inte att få åtkomst till krypterade delade mappar via NFS.
  • Om lösenord återställs till standard genom att använda knappen reset som finns på Synology NAS kommer krypterade delade mappar att avmonteras och alternativet att automatisk montera krypterade delade mappar att avaktiveras.
  • Krypterade delade mappar stöds endast i läget högt privilegie. För att använda den här funktionen, gå till filpaketet Docker på din DSM som är värd, välj den Docker DSM som du vill aktivera krypterade delade mappar för, klicka på Redigera och markera rutan bredvid Kör container med högt privilegie.
