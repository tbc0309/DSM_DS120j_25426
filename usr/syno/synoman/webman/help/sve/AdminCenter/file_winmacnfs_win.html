SMB

SMB (Server Message Block)-protokollet gör att Windows-, Mac- och Linux-klienter som har stöd för SMB/CIFS får åtkomst till data lagrad på Synology NAS. Om du vill bläddra bland filer på Synology NAS via Utforskaren eller montera delade mapparna som nätverksenheter, vänligen aktivera detta alternativ i Kontrollpanel > Filtjänster > SMB/AFP/NFS.

Att aktivera SMB-protokoll:

  1. Bocka för Aktivera SMB-tjänst.
  2. Klicka på Tillämpa.

Obs:

Efter aktivering av SMB-protokollet kan du montera fullständigt indexerade mappar på din Mac-dator via SMB och använda Finder för att söka efter filernas innehåll i de monterade mapparna.

Arbetsgrupp

Detta alternativ låter dig designera systemet som en medlem av en specifik arbetsgrupp med syfte nätverkshantering.

Obs:

  • Arbetsgruppens namn kan innehålla 1 till 15 tecken. Namnet kan inte innehålla följande tecken: [ ] ; : " < > * + = \ / | ? ,
  • Om du använder en Mac med OS X 10.9 och din Synology NAS har anslutit till en Windowsdomän, kan du logga in genom att lägga till localhost\ före det lokala användarnamnet på din Synology NAS för att få åtkomst till filer som lagras på Synology NAS via SMB-protokollet.

Tillåt inte åtkomst till tidigare versioner

I Utforskaren kan användare få åtkomst till ögonblicksbilder för mappar och filer genom att klicka på fliken Tidigare versioner i en delad mapp vilket möjliggör återställning eller kopiering av data utan hjälp från en administratör. Om du väljer Tillåt inte åtkomst till tidigare versioner visar sidan ”Tidigare versioner” inte någon information.

Obs:

  • Användare har bara läsbehörighet för kopian av snapshotet om, det vid den tiden snapshotet togs, de redan hade sina konton skapade och de hade tilldelats läsbehörighet för den delade mappen för vilken det motsvarande snapshotet tagits.

Aktivera loggning av överföring

Att markera detta alternativ aktiverar loggning av filåtkomståtgärder, inklusive skapande, borttagning, läsning, skrivning, flyttning och namnbyte för filer eller mappar via protokollen CIFS/SMB, även känt som SMB-protokollet.

Avancerade inställningar

Klicka på knappen Avancerade inställningar för att se nedanstående alternativ.

  • WINS Server: Microsoft Windows Internet Name Service (WINS) är en NetBIOS-tjänst för mappning av namn-till-IP-adress. Windows-användare kommer att hitta systemet lättare på TCP/IP-nätverk om systemet har konfigurerats för att registrera sig med en WINS-server.
  • Max SMB-protokoll: Beroende på din nätverkskonfiguration, ställ in det senaste SMB-protokoll som stöds av din Synology NAS.
    • SMB1: SMB1 (även kallad CIFS) har haft stöd sedan Windows NT.
    • SMB2: SMB2 har haft stöd sedan Windows Vista och är den förbättrade versionen av SMB. SMB2 lägger till möjligheten att sammanställa flera SMB-åtgärder till en enstaka förfrågan för att reducera antalet nätverkspaket och förbättra prestandan.
    • SMB2 och Large MTU: Maximum transmission unit (MTU) (översatt till svenska maximal överföringsenhet) avser den största dataenhet som kan överföras över ett kommunikationsprotokoll. För att kunna dra fördel av snabbare nätverksgränssnitt, såsom 1- eller 10-gigabit Ethernet, ökar Large MTU den maximala överföringsenheten (MTU) till 1 megabyte (MB). Att aktivera Large MTU förbättrar hastigheten och effektiviteten för stora filöverföringar, medan det minskar antalet paket som behöver bearbetas.
    • SMB3: SMB3 har haft stöd sedan Windows 8 och Windows Server 2012. Det är den utökade versionen av SMB2. SMB3 stöder överföring med AES-baserad filkryptering och förbättrar på så sätt säkerheten för överföringar med peer-to-peer.

    Obs:

    • SMB2 stöds av Windows Vista, Windows Server 2008 R2, Windows 7 och senare.
    • Large MTU stöds av Windows Server 2008 R2, Windows 7 och senare.
    • För att göra MTU mer effektivt på din Windowsdator behöver du ändra datorns registerinställningar. För mer information, vänligen se Microsoft tekniska support (såsom denna artikel).
  • Min SMB-protokoll: Beroende på din nätverkskonfiguration, ställ in det tidigaste SMB-protokoll som stöds av din Synology NAS.
  • Transportkrypteringsläge: När SMB3 är aktiverat lägger SMB-protokollet till transportkryptering för att stärka säkerheten för filöverföring.
    • Avaktivera: Ingen transportkryptering tillämpas.
    • Auto: Transportkryptering tillämpas bara på klienter som har stöd för den här funktionen.
    • Framtvinga: Transportkryptering tillämpas alltid. Det gör att klienter som saknar stöd för transportkryptering inte kan använda SMB-protokollet.

    • Obs:

    • Transportkryptering stöds av Windows 8, Windows Server 2012 och senare.
    • Anonym inloggning för SMB-protokollet stöds inte när transportkryptering är aktiverat.
    • När transportkryptering är aktiverat, vänligen lämna Avaktivera Opportunistic Locking omarkerad för att undvika timeouts i applikationen.
  • Avaktivera Opportunistic Locking: Välj för att aktivera ”Opportunistic Locking” för CIFS så att klienten bättre kan cacha filinnehåll och undvika mindre overhead för läsa/skriva. När det här alternativet är avaktiverat blir prestandan för databasapplikationer bättre vid åtkomst till databasfiler som lagras på Synology NAS via nätverk.
    • Aktivera SMB2-lån: Aktivering av det här alternativet tillåter fler SMB2-relaterade inställningar för att finjustera cachemekanism, som tillhandahålls av Opportunistic Locking, för klient.
  • Aktivera Local Master Browser: När flera Windows-datorer finns i samma subnät kommer de att förhandla och välja en dator som blir ”local master browser”. Local master browser underhåller listan över andra datorer inom subnätet och delas delade resurser och delar denna lista med de andra datorerna. Detta alternativ låter Synology NAS anta rollen som local master browser.

    • Obs:

    Att aktivera denna funktion avaktiverar hårddiskens vilolägesmekanism och aktiverar även guest-kontot utan lösenord.

  • Aktivera modulen DirSort VFS: Sorterar mappnamnen och listar dem alfabetiskt. Aktivering av detta alternativ påverkar systemets prestanda.
  • Vetofiler: Döljer filer i enlighet med specifika kriteria. Kriteria kan inkludera jokertecken (*) och flera poster ska separeras med ett snedstreck (/). Till exempel: /abc*/*.txt/*.conf/. Aktivering av detta alternativ påverkar systemets prestanda.
  • Tillåt symboliska länkar i delade mappar: Välj för att tillåta att symboliska länkar pekar direkt på filer eller mappar inom samma delade mapp. Symboliska länkar kan skapas genom att använda DSM terminal.
    • Tillåt symboliska länkar mellan delade mappar: Markera för att tillåta symboliska länkar att peka på filer och mappar i olika delade mappar. Symboliska länkar kan skapas genom att använda DSM terminal. Dock kan användare utan privilegier få åtkomst till målets delade mappar för symboliska länkar.
  • Aktivera modulen VFS för att konvertera specialtecken för Mac: Välj för att konvertera tecken, som reserverats för SMB (t.ex. < > " * ?) i filnamn, till standard Latin när filen skapas. Sådana filer känns igen via AFP.
  • Aktivera modulen MSDFS VFS: Låter symboliska länkar att peka till filer och mappar som finns på andra SMB-servrar. Med äldre versioner av Windows, såsom Windows XP, behöver klienten och Synology NAS ansluta till en Windowsdomän innan detta alternativ aktiveras. För att få veta mer om att skapa MSDFS-länkar, vänligen se här. Vänligen observera att cache på klientenheten ska rensas efter att detta alternativ avaktiverats, i annat fall kan normala funktioner påverkas.
  • Avaktivera flera anslutningar från samma IP-adress: När detta alternativ är aktiverat tillåter Synology NAS endast en anslutning per IP-adress från klienter som använder SMB-protokollet. När flera klienter med samma IP-adress använder SMB-protokollet kommer Synology NAS behålla den senaste och släppa alla äldre anslutningar. I en NAT-miljö kan klienten använda en enskild IP-adress, i sådant fall kan aktivering av detta alternativt påverka normala funktioner.
  • Samla loggfiler över felsökning: När detta alternativ är aktiverat, kan detaljerade loggfiler lagras för felsökningssyften. Aktivering av detta alternativ påverkar systemets prestanda.
  • Tillämpa standardrättigheter för UNIX: Aktivera de här alternativet för att applicera standardrättigheterna för UNIX för att ladda upp eller skapa filer och mappar. Behörigheter för UNIX kommer att vara 744 för filer och 755 för mappar. När detta alternativ är avaktiverat är behörighet för UNIX 777 för filer och mappar.

    • Obs:

    • Aktivering av detta alternativ kan orsaka problem med inkonsekventa behörigheter mellan olika protokoll. För att undvika inkonsekvens rekommenderar vi att detta alternativ lämnas avaktiverat.

    Obs:

    • För delade mappar där Windows ACL aktiverats (alla utdelningar förutom ”photo” och utdelningar på externa enheter), vänligen kör kommandot chmod på din Linux- eller FTP-klient för att ändra typ av filbehörighet från Windows ACL till UNIX.
    • Aktivering av detta alternativ kan orsaka problem med inkonsekventa behörigheter mellan olika protokoll. För att undvika inkonsekvens rekommenderar vi att detta alternativ lämnas avaktiverat.
  • Reservera inte diskutrymme när filer skapas: Aktivera det här alternativet för att inte tillåta att diskutrymme reserveras när filer skapas på din Synology NAS.
  • Aktivera SMB-varaktiga referenser (fillåsning mellan protokoll avaktiveras): Aktivering av det här alternativet låter användare återställa förlorad SMB2-anslutning till denna Synology NAS efter tillfällig frånkoppling; öppnade filer förblir tillgängliga under tiden nätverket är nere. Dock avaktiveras fillåsning mellan protokoll så att användare kan få åtkomst till eller ändra en låst fil via andra protokoll än SMB2 (t.ex. AFP).
  • Aktivera snabb klon för Btrfs: Aktivering av det här alternativet gör att användaren kan kopiera filer med snabbare hastigheter när både källan och målet finns på samma Btrfs volym.
  • Rensa SMB-cache: Klicka på denna knapp för att radera följande SMB-cachefiler för felsökningssyften:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
