Général

Les options génériques de réseau telles que la passerelle par défaut, le serveur DNS, le proxy et le tunneling IPv6, sont disponibles dans Panneau de configuration > Réseau > Général.

Renommage du périphérique Synology NAS

Pour renommer votre périphérique Synology NAS, saisissez le nouveau nom du serveur et cliquez sur Appliquer.

Le nom du serveur n'est pas sensible à la casse et peut inclure de 1 à 15 caractères contenant des lettres, des nombres, des traits de soulignement et des signes moins (-). Le premier caractère du nom doit être une lettre.

Spécification de la passerelle par défaut

Saisissez simplement l'adresse IP de la passerelle par défaut et cliquez sur Appliquer pour enregistrer le paramètre.

Spécification de l'adresse IP du DNS Server

Vous pouvez choisir de configurer manuellement l'adresse IP de votre serveur DNS. Dans ce cas, l'adresse IP de votre serveur DNS préféré doit être saisie. Une autre adresse IP de serveur DNS peut également être saisie, fournissant ainsi une sécurité si le serveur DNS préféré rencontre des problèmes. Une fois les adresses IP de serveur DNS configurées comme vous le souhaitez, cliquez sur Appliquer pour enregistrer vos paramètres.

Paramètres avancés :

  • Répondre à la demande ARP si l'adresse IP cible est identique à une adresse locale configurée sur l'interface entrante : dans le protocole réseau, le protocole ARP ne répond pas à la demande issue de la même adresse IP. Activez cette option pour répondre à la demande ARP et assurez-vous que la table ARP est complète.
  • Activer les passerelles multiples : activez cette option pour prendre en charge plusieurs passerelles dans l'interface.
  • Activer la détection des conflits IP : activez cette option pour détecter automatiquement une adresse IP identique sur le réseau.

Remarque :

Les champs Serveur DNS et Passerelle par défaut seront définis automatiquement et ne peuvent pas être modifiés si toutes vos interfaces réseau sont configurées automatiquement ou lorsque vous utilisez une connexion PPPoE.

Connexion via un serveur proxy

Pour permettre à votre périphérique Synology NAS de se connecter via un serveur proxy, cochez la case Se connecter via un serveur proxy, saisissez l'Adresse et le Port, puis cliquez sur Appliquer.

Remarque :

  • Cliquez sur Paramètres avancés pour configurer le proxy HTTP, le proxy HTTPS et les informations d'authentification.
  • Par défaut, les paramétrages du proxy HTTPS sont les mêmes que ceux du proxy HTTP.
  • Si la case Se connecter via un serveur proxy est cochée, Mise à jour de DSM et Centre de paquets, ainsi que d'autres paquets, se connecteront toujours à Internet via un proxy.
  • Si la case Ignorer le serveur proxy pour les adresses locales est cochée, assurez-vous que le serveur DNS utilisé par le Synology NAS peut résoudre les demandes des périphériques qui se trouvent dans le réseau local.

Activer le tunneling IPv6

Jusqu'à ce que IPv6 supplante complètement IPv4, le tunneling IPv6 permet aux hôtes fonctionnant en IPv6 d'accéder aux services IPv4. Cette technique fournit une manière de transporter un trafic IPv6 dans des réseaux IPv4 existants.
Si vous n'êtes pas sûr de vous quant à la manière de configurer l'IPv6 ou que vous rencontrez des problèmes de connexion, contactez votre fournisseur d'accès Internet.

Pour activer le tunnel IPv6 :

  1. Cochez la case Activer le tunnel pour activer la prise en charge du tunnel.
  2. Dans Adresse du serveur, saisissez le nom d'hôte ou l'adresse IP du fournisseur de service tunnel, par exemple : anonymous.freenet6.net.
  3. Choisissez entre Se connecter de manière anonyme ou Se connecter avec un compte existant selon vos préférences.

Remarque :

Votre adresse IPv6 externe pourra être mise à jour automatiquement par le fournisseur de service. Toutefois, le périphérique Synology NAS ne supprimera pas immédiatement l'ancienne adresse IPv6.
