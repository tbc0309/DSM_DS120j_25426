E-mail

L'activation des notifications par email permet au Synology NAS d'envoyer des messages à votre adresse email lorsque des événements spécifiques se produisent, tels que des erreurs système, une perte de connexion réseau, le changement de comportement du ventilateur, une panne d'alimentation ou l'échec d'une tâche de fond.

Remarque :

  • L'événement correspondant au changement de comportement du ventilateur n'est pas disponible sur un modèle sans ventilateur, tel que l'EDS14. Pour plus d'informations, visitez le site Web Synology.

Pour activer la notification par email :

  1. Accédez à Panneau de configuration > Notification > Email.
  2. Activez la case à cocher Activer les notifications par email.
  3. Saisissez une ou plusieurs adresses email dans le champ Adresse email du destinataire. Les notifications système sont envoyées aux adresses saisies ici. Vous pouvez saisir deux adresses. Les comptes multiples peuvent être séparés par un point-virgule (;).
  4. Si nécessaire, ajoutez un préfixe d'objet dans le champ Préfixe d'objet. Ce préfixe sera ajouté à l'objet de chaque message envoyé par le Synology NAS, ce qui permet d'identifier et de filtrer les messages.
  5. Si vous souhaitez utiliser un serveur de messagerie tiers (Gmail ou Outlook, par exemple) pour relayer les messages, consultez la section « Service de messagerie tiers » ci-dessous. Si vous souhaitez utiliser un serveur personnalisé, consultez la section « Personnaliser un serveur SMTP ».

Pour utiliser un service de messagerie tiers :

  1. Dans le menu Fournisseur de service, choisissez un service de messagerie tiers, tel que Gmail ou Outlook.
  2. Saisissez votre nom d'utilisateur et votre mot de passe pour le service de messagerie sélectionné.
  3. Cliquez sur Appliquer pour enregistrer les paramètres.
  4. Vous pouvez cliquer sur Envoyer un email de test pour vérifier si vos paramètres sont corrects.

Pour utiliser un serveur SMTP personnalisé :

  1. Dans le menu Fournisseur de service, sélectionnez Personnaliser un serveur SMTP.
  2. Saisissez le nom du serveur SMTP ou l'adresse IP dans le champ Serveur SMTP
  3. Entrez le numéro de port SMTP dans le champ Port SMTP. (Si vous avez défini un autre port SMTP lors de la configuration des règles de transmission de port, renseignez le numéro correspondant.)
  4. Si votre serveur SMTP requiert une authentification SMTP, activez la case à cocher Authentification requise, puis saisissez votre Nom d'utilisateur et votre Mot de passe associés au serveur de messagerie.
  5. Si votre serveur SMTP requiert une connexion SSL/TLS, activez la case à cocher Une connexion sécurisée (SSL/TLS) est nécessaire.
  6. Si vous souhaitez personnaliser le nom de l'expéditeur qui apparaît sur les messages électroniques envoyés par le Synology NAS, tapez ce nom dans le champ Nom de l'expéditeur. Si vous souhaitez personnaliser l'adresse email de l'expéditeur, saisissez l'adresse email dans le champ Expéditeur de l'email. Si le champ est vide, c'est la première adresse email renseignée dans le champ Destinataires qui sera utilisée.
  7. Cliquez sur Appliquer pour enregistrer les paramètres.
  8. Vous pouvez cliquer sur Envoyer un email de test pour vérifier si vos paramètres sont corrects.

Remarque :

  • Si vous souhaitez envoyer un message de bienvenue à chaque compte nouvellement créé, activez la case à cocher Envoyer un compte de bienvenue aux nouveaux utilisateurs, et chaque utilisateur recevra un message de bienvenue après la création de son compte.
  • La langue de notification par email est définie dans le champ Langue de notification. Pour changer la langue de notification par email, allez dans Menu principal > Panneau de configuration > Options régionales, cliquez sur l'onglet Langue, puis sélectionnez la langue de votre choix dans le menu déroulant de la section Langue de notification.
  • Si vous utilisez le serveur SMTP de GMail et que vous avez activé la vérification en deux étapes, n'oubliez pas d'utiliser votre « Mot de passe spécifique à l'application » fourni par Google pour l'authentification SMTP.

Nom de domaine du serveur SMTP :

Le serveur SMTP peut être une adresse IP ou un nom de domaine. Si c'est un nom de domaine, assurez-vous qu'une IP DNS server valide a bien été renseignée dans Menu principal > Panneau de configuration > Réseau et que le serveur est bien connecté à Internet.

Remarque :

En raison d'une modification de la stratégie d'API de Google Gmail, les utilisateurs doivent sélectionner l'option Paramètres du serveur SMTP personnalisé dans le menu déroulant Fournisseur de services s'ils souhaitent envoyer un e-mail aux trois fins suivantes. Pour plus d'informations sur cette configuration, veuillez consulter ce tutoriel.

  • Utiliser le binaire ssmtp pour envoyer des e-mails, comme l'exécution de scripts définis par l'utilisateur dans le Planificateur de tâches.
  • Créer un site Web avec Web Station et envoyer des e-mails en utilisant PHP. Cela inclut également l'envoi d'e-mails à partir d'un paquet de site Web tiers, tel que Wordpress, MantisBT, etc.
  • Envoyer des notifications par e-mail lorsque quelqu'un laisse un commentaire sur une photo ou un article dans Photo Station.
