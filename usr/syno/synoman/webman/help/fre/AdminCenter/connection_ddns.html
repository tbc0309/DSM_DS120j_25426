DDNS

DDNS (Dynamic Domain Name Service) simplifie la connexion à votre Synology NAS via Internet en mappant un nom d'hôte à son adresse IP. Par exemple, DDNS vous permet d'accéder à votre Synology NAS en utilisant un nom de domaine (par exemple : www.john.synology.me), sans avoir à vous rappeler d'une adresse IP (par exemple : 172.16.254.1).

Configurer DDNS

Vous pouvez pointer un nom d'hôte existant à l'adresse IP de votre Synology NAS ou en obtenir un nouveau fourni par Synology ou plusieurs autres fournisseurs de DDNS. Veuillez consulter chaque fournisseur pour plus de détails concernant l'enregistrement d'un nom d'hôte.

Pour configurer un nom d'hôte DDNS :

  1. Cliquez sur le bouton Ajouter.
  2. Une boîte de dialogue s’affiche vous invitant à modifier les paramètres suivants :
    • Fournisseur de service : sélectionnez un fournisseur de service. Pour obtenir un nom d'hôte gratuit fourni par Synology, choisissez Synology dans le menu déroulant.
    • Nom d'hôte : entrez un nom d'hôte DDNS enregistré, comme john.synology.me.
    • Nom d'utilisateur / e-mail : entrez le nom d'utilisateur pour votre fournisseur DDNS.
    • Mot de passe / clé : entrez le mot de passe / la clé pour votre fournisseur DDNS.
    • Adresse externe : entrez l'adresse IP externe du Synology NAS qui utilisera le nom d'hôte.
    • Heartbeat : activez cette option pour recevoir des alertes sur l'état du nom d'hôte mappé (uniquement pour les noms d'hôtes fournis par Synology).
  3. Cliquez sur Test de connexion pour voir si les paramètres sont corrects.
  4. Cliquez sur OK pour enregistrer et terminer.
  5. Si vous avez sélectionné Synology comme fournisseur de services, un message s'affiche s'il n'y a pas de certificat SSL correspondant à votre nom d'hôte DDNS. Il est recommandé de signer un certificat Let's Encrypt pour le nom d'hôte et de le définir comme certificat par défaut. Cliquez sur Oui.

    Remarque : si un certificat SSL correspond déjà à votre nom d'hôte DDNS, un message s'affiche et vous demande si vous souhaitez le définir comme certificat par défaut. Ce message s'affiche chaque fois que le nom d'hôte DDNS est modifié. Pour plus d'informations sur le certificat SSL, voir ici.

Une fois la configuration terminée, vous pouvez accéder à votre Synology NAS sur Internet en entrant le nom d'hôte DDNS dans un navigateur Web.

Conditions et limites

  • Les serveurs du fournisseur de service DDNS fonctionnent.
  • Le Synology NAS peut se connecter à Internet.
  • DDNS ne fonctionne que dans un environnement réseau où aucun serveur proxy n'est nécessaire.
  • Vous ne pouvez entrer qu'un nom d'hôte pour chaque fournisseur de service DDNS.

Gestion des noms d'hôtes

Utilisez les boutons suivants pour gérer les entrées des noms d'hôtes existants :

  • Supprimer : supprime le nom d'hôte sélectionné de la liste.
  • Modifier : changez les paramètres de l'entrée du nom d'hôte sélectionné.
  • Mettre à jour maintenant : met à jour le statut de toutes les entrées des noms d'hôtes dont le service DDNS est activé.

Configuration d'un fournisseur DDNS personnalisé

Vous pouvez ajouter un fournisseur DDNS personnalisé si votre fournisseur ne s'affiche pas déjà dans la liste des fournisseurs de services.

Pour configurer un fournisseur DDNS personnalisé :

  1. Cliquez sur Personnaliser.
  2. Une boîte de dialogue s’affiche vous invitant à saisir les paramètres suivants :
    • Fournisseur de service : saisissez le nom de votre fournisseur de services personnalisé.
    • URL de la requête : saisissez l'URL de votre fournisseur de services personnalisé.
  3. Cliquez sur Enregistrer pour ajouter le fournisseur de services personnalisé à la liste des fournisseurs de services DDNS.

Conditions et limites

  • L'URL de la requête doit être conforme au protocole spécifié par votre fournisseur de services. Cette URL est utilisée pour mettre à jour les informations d'IP sur le serveur DDNS de votre fournisseur.
  • Une URL de requête comprend généralement les quatre paramètres ci-dessous. Lorsque vous saisissez l'URL de requête, remplacez les paramètres suivants par la variable correspondante.
    • Nom d'utilisateur : __USERNAME__
    • Mot de passe : __PASSWORD__
    • Nom d'hôte : __HOSTNAME__
    • Adresse externe : __MYIP__
  • L'URL de requête ne peut pas comporter d'espace ni de point-virgule.
