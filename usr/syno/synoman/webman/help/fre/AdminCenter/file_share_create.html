Création d'un dossier partagé

Les dossiers partagés sont les répertoires de base où vous pouvez stocker et gérer les fichiers et dossiers sur votre Synology NAS. Avant de stocker des fichiers, vous devez créer au moins un dossier partagé dans DSM.

Pour créer un dossier partagé :

  1. Accédez à Panneau de configuration > Dossier partagé et cliquez sur Créer.
  2. Spécifiez un nom et une description pour le dossier partagé.
  3. Sélectionnez un stockage sur lequel créer le dossier partagé (passez cette étape s'il n'y a aucun volume existant).
  4. Modifiez les options suivantes si nécessaire :
    • Cacher ce dossier partagé dans « Mes emplacements réseau » : Empêche le dossier partagé d'apparaître sous « Réseau » dans Windows File Explorer. Cette option n'affecte pas les privilèges d'accès du dossier partagé.

      Remarque :

      • Cacher un dossier dans « Favoris réseau » n'affecte pas les droits d'accès. Les droits d'accès du dossier partagé caché demeurent inchangés. Les utilisateurs disposant des droits suffisants peuvent toujours accéder à ce dossier partagé en saisissant « \\nom serveur\nom dossier partagé ».
    • Masquer les sous-dossiers et les fichiers des utilisateurs sans autorisations : Lorsque Masquer les sous-dossiers et les fichiers des utilisateurs sans autorisations est activé, si un utilisateur sans privilèges de lecture tente d'accéder à un dossier partagé via le protocole de partage de fichiers de Windows, l'utilisateur ne pourra pas voir les sous-dossiers ou les fichiers dans le dossier partagé. Cela empêche les utilisateurs de s'y perdre quand ils se connectent à leurs produits Synology et trouvent un grand nombre de fichiers et de sous-dossiers qu'ils ne peuvent pas ouvrir.

      Remarque :

      • Cette fonctionnalité est uniquement prise en charge avec le protocole de partage de fichiers de Windows.
      • Vous pouvez modifier les paramètres de privilège de chaque dossier dans File Station en cliquant-droit dessus. Choisissez Propriétés dans le menu déroulant et cliquez sur l'onglet Permission.
      • Le dossier partagé lui-même ne sera pas caché aux utilisateurs sans autorisation en activant Masquer les sous-dossiers aux utilisateurs sans autorisations. Seuls les sous-dossiers et les fichiers qui se trouvent dans ce dossier partagé seront masqués.
  5. Activer la corbeille :Quand les fichiers dans le dossier partagé sont supprimés, ils seront déplacés vers un dossier nommé #recycle L'accès à la corbeille de recyclage peut être limité aux utilisateurs appartenant au groupe administrators.
  6. Activer le quota du dossier partagé : Pour indiquer la capacité maximale de chaque dossier partagé, cochez la case Activer le quota du dossier partagé et saisissez la capacité maximale (par ex., 10 Go) dans le champ suivant.

    Remarque :

    • Les options de Activer le quota du dossier partagé ne sont disponibles pour les dossiers partagés que lorsque le stockage utilise le système de fichiers Btrfs.
    • L'utilisation du stockage combiné des répertoires home de tous les utilisateurs ne peut pas dépasser le quota de « homes ».
  7. Activation de la somme de contrôle des données pour l'intégrité avancée des données : Pour protéger un dossier partagé à l'aide de la somme de contrôle CRC32 et de stratégies copie lors de l'écriture, vous pouvez activer la somme de contrôle des données pour l'intégrité avancée des données pendant la création du dossier partagé. Une somme de contrôle CRC32 est appliquée afin de vérifier si les données demeurent telles qu'elles ont été écrites à l'origine et le système utilise automatiquement la redondance du RAID pour corriger les données altérées. La copie lors de l'écriture permet d'améliorer la cohérence des données en cas d'arrêt anormal du système.

    Remarque :

    • Cette option est uniquement disponible pour les volumes utilisant le système de fichiers Btrfs.
    • Vous pouvez activer ou désactiver cette option pour chaque dossier partagé lors de sa création. Il est impossible de modifier cette option une fois le dossier partagé créé.
    • L'activation de cette option permet au système d'exécuter un nettoyage des données à grande échelle. En savoir plus sur le nettoyage des données
    • Pour assurer la qualité du service, nous recommandons de ne pas activer la somme de contrôle des données lorsque le dossier partagé est utilisé pour les services suivants :
      • Hébergement de bases de données ou de machines virtuelles
      • Stockage des enregistrements vidéo de Surveillance Station
      • Exécution de services qui nécessitent un petit modèle d'écriture aléatoire
  8. Activer la compression de fichier : Les données dans le dossier partagé créé sont automatiquement compressés par le système de fichiers Btrfs afin d'économiser l'espace de stockage. Lorsqu'elles sont récupérées en vue de leur utilisation, les données sont automatiquement décompressées.

    Remarque :

    • Cette option n'est disponible que si vous créez un dossier partagé sur un volume Btrfs ou si l'option Protection avancée de l'intégrité des données est activée.
    • Il est impossible de modifier cette option une fois le dossier partagé créé.
    • Le taux de compression diffère en fonction du type de données : Les fichiers de texte bénéficient du taux de compression le plus élevé, alors que les fichiers multimédias (par ex., images, musique, vidéos) peuvent ne pas être correctement compressés car ils le sont déjà pour la sortie.
    • Il est impossible d'activer cette option si le chiffrement des dossiers partagés est activé.
  9. Crypter ce dossier partagé : Si vous souhaitez chiffrer le dossier partagé, procédez comme suit :
    1. Spécifiez et confirmez une nouvelle clé de chiffrement. La clé de chiffrement ne peut pas contenir des virgules (,) ou le signe égal (=).
    2. Si nécessaire, cochez la case Ajouter une clé de chiffrement au gestionnaire de clés pour monter le dossier chiffré automatiquement quand le système démarre. Si cette option n'est pas désactivée, le dossier crypté partagé sera déconnecté la prochaine fois que Synology Synology NAS redémarre. Pour afficher ou accéder à un dossier partagé déconnecté, vous devrez monter le dossier manuellement en important ou en entrant la clé de chiffrement.
  10. Cliquez sur OK pour terminer.

    11. Remarque :

    • Certaines fonctions et options de dossiers partagés sont prises en charge uniquement par VDSM et les modèles dotés des architectures de paquets suivantes : Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview et Grantley. Pour savoir quelle architecture de paquet votre modèle prend en charge, reportez-vous à cet article.

Règles de nommage du dossier partagé :

  • Les noms de dossiers partagés sont sensibles à la casse et peuvent être composés de 1 à 32 caractères. Les noms ne peuvent pas inclure les caractères suivants : ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Les noms suivants sont réservés pour le système : ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Le premier caractère du nom d'un dossier partagé ne peut être ni le signe moins ni un espace tout comme le dernier caractère qui ne peut également pas être un espace.
  • La description du dossier partagé est sensible à la casse et est limitée à 64 caractères Unicode affichables.

Dossiers partagés chiffrés :

Le chiffrage AES 256 bits protège les données en bloquant toute tentative d'accès non autorisée. Quand un dossier partagé est chiffré, ceux qui ne possèdent pas la clé de chiffrement ne pourront pas accéder ou modifier les données cryptées même s'ils retirent les disques durs de votre Synology NAS et les montent sur un autre appareil.

Remarque :

  • Le nom d'un fichier ou d'un dossier dans un dossier partagé chiffré ne peut pas comporter plus de 143 caractères anglais ou 47 caractères asiatiques (CJK) ; si vous accédez à vos dossiers partagés à l'aide de Mac OS, le nom d'un fichier ou d'un dossier dans un dossier partagé chiffré ne peut pas comporter plus de 130 caractères anglais ou 43 caractères asiatiques (CJK).
  • Il n'est pas possible d'accéder aux dossiers partagés cryptés via NFS.
  • Si vous restaurez les mots de passe par défaut à l'aide du bouton reset sur le Synology NAS, les dossiers partagés cryptés seront démontés, et l'option de montage automatique des dossiers partagés cryptés sera désactivée.
  • Les dossiers partagés chiffrés ne sont pris en charge qu'en mode à privilèges élevés (uniquement applicable à Dockerx64). Pour utiliser cette fonction, accédez au paquet Docker sur votre DSM hôte, sélectionnez le Docker DSM pour lequel vous souhaitez activer des dossiers partagés chiffrés, cliquez sur Modifier et cochez la case en regard de Exécuter le conteneur à l'aide de privilèges élevés.
