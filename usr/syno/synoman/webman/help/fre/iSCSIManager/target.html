Cible

Selon le modèle, le Synology NAS en charge la création plusieurs cibles sur lesquelles il est possible de mapper des LUN.

Remarque : Le nombre de cibles prises en charge varie en fonction des modèles de Synology NAS.

Pour créer une cible :

  1. Cliquez sur le bouton Créer.
  2. Saisissez les informations de la cible, notamment Nom, IQN et indiquez s'il convient d'activer Authentification CHAP.
  3. Cliquez sur Suivant.
  4. Configurer les propriétés d'iSCSI LUN. Sélectionnez Créer un nouveau iSCSI LUN ou Mapper des LUN existants.
  5. Cliquez sur Suivant.
  6. Cliquez sur Retour pour modifier les paramètres ou sur Appliquer pour les confirmer.

Terminologie

  • IQN : IQN (iSCSI Qualified Name) est un ensemble de noms uniques pour une cible. Il ne devrait pas provoquer de conflits avec d'autres IQN cible sur d'autres serveurs. L'IQN doit comporter moins de 128 caractères et peut inclure des lettres (sensible à la casse), des numéros, des double-points, des tirets et des points. Le format de nomination suggéré est iqn.[date].[domaine]:[identifiant périphérique], par exemple : iqn.2000-04.com.synology:storage.diskarray.1.
  • Authentification CHAP mutuel : Dans CHAP mutuel, le client et le serveur s'authentifieront l'un l'autre. Tout mot de passe CHAP utilisé pour l'authentification de client ne doit pas être configuré pour l'authentification d'un serveur, et tout mot de passe CHAP utilisé pour l'authentification de serveur ne doit pas être configuré pour l'authentification d'un client.

    • Remarque : le processus de découverte sur l'initiateur iSCSI peut échouer si l'authentification CHAP de votre NAS est activée. Finissez d'abord la découverte, puis activer l'authentification CHAP avant de passer à la connexion à une cible. Vous pouvez également vous connecter à une cible avec les IP et IQN.

Pour supprimer une cible :

Vous pouvez supprimer une cible et tous les LUN mappés. Lors de la suppression d'un iSCSI LUN, toutes les données qui s'y trouvent sont supprimées.

  1. Sélectionnez la cible que vous souhaitez supprimer.
  2. Cliquez sur Supprimer.
  3. Si vous voulez supprimer des iSCSI LUN mappés, sélectionnez Supprimer également les LUN mappés et sélectionnez ceux que vous voulez supprimer.
  4. Cliquez sur OK.

Pour modifier une cible :

  1. Sélectionnez la cible que vous souhaitez modifier.
  2. Sélectionnez Modifier dans le menu déroulant Action.
  3. Modifiez les paramètres à votre convenance. Cliquez sur OK pour enregistrer les modifications.

Terminologie

  • Somme de contrôle CRC : Activez la somme de contrôle CRC pour vérifier les erreurs qui se sont produites pendant la transmission de données.
  • Sessions multiples : Autorisez plusieurs sessions à partir de plusieurs initiateurs à connecter à la même cible et en même temps. Pour protéger vos fichiers contre toute altération ou perte de données, nous vous conseillons fortement d'utiliser un système de fichiers partagés ou un système à clusters tel que Virtual Machine File System (VMFS) de VMware et Oracle Cluster File System (OCFS).
  • Masquage : Le masquage permet à une cible d'être accédé de manière exclusive par des IQN d'initiateurs iSCSI spécifiés.

Pour activer ou désactiver une cible :

  1. Sélectionnez la cible que vous souhaitez activer ou désactiver.
  2. Sélectionnez Activer ou Désactiver depuis le menu déroulant Action.
