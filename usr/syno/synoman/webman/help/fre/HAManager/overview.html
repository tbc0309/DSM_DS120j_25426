Vue d'ensemble

Cette page affiche des informations de base concernant le cluster High Availability et les serveurs membres. En outre, cette page fournit des outils pour créer, supprimer ou effectuer l'entretien sur les clusters High Availability et les serveurs membres.

État du cluster High Availability

Ce champ affiche l'état global de votre cluster High Availability et vous alerte en cas d'erreurs éventuelles. Voici une liste de conditions possibles affichées dans ce champ :

État Description
Autonome Les serveurs ne sont pas combinés dans un cluster High Availability.
Sain La condition générale de votre cluster High Availability est bonne. La réplication des données du serveur actif vers le serveur passif fonctionne normalement.
Préparation Les serveurs ont été combinés avec succès dans un cluster High Availability, mais la réplication initiale des données n'est pas encore terminée. Le basculement ne peut pas être exécuté à ce moment. Pour plus de détails, veuillez consulter Basculement.
Chargement Le système charge l'état du cluster. Le basculement ne peut pas être effectué tant que le chargement de l'état n'est pas fini.
Avertissement L'état du cluster HA est anormal, mais les services sont toujours en fonction.
Danger Le serveur actif et le serveur passif sont tous les deux défectueux. Le cluster High Availability ne peut pas fonctionner normalement. Veuillez vérifier l'état de tous les espaces de stockage (p.ex. RAID Groups, groupes de disques, volumes et iSCSI LUN).

Avertissement/Erreur Description

Espace de stockage planté Un espace de stockage (par ex. RAID Group, groupes de disques, volumes, iSCSI LUN et cache SSD) sur le serveur actif ou le serveur passif est en panne. Veuillez vérifier la page État du stockage.

Les données ne peuvent pas être répliquées sur le serveur passif Cette erreur peut se produire en raison d'une panne de volume de stockage ou de disque. Dans ce cas, le basculement ne sera pas disponible. Veuillez vérifier la page État du stockage.

Informations du cluster High Availability

Ce champ est situé en dessous de l'état du cluster High Availability et affiche des informations de base sur le cluster. Si un cluster High Availability n'existe pas, ce champ fournira uniquement une explication de la fonction des clusters High Availability.

Ce champ comprend les informations suivantes :

  • Nom du serveur cluster :Le nom de serveur utilisé par le cluster High Availability.
  • Adresse IP du cluster : L'adresse IP utilisée par le cluster High Availability. Quel que soit le serveur qui exécute les services, les utilisateurs peuvent se connecter au cluster High Availability en utilisant cette adresse IP.
  • Temps de construction : Temps écoulé depuis que le cluster High Availability a été créé.

Boutons des opérations du cluster High Availability

Les trois boutons suivants sont situés sous les informations du cluster High Availability :

  1. Créer le cluster HA : En l'absence de cluster High Availability, vous pouvez cliquer sur ce bouton pour combiner deux serveurs et créer un cluster High Availability. Pour plus d'informations sur la création d'un cluster High Availability, veuillez consulter Assistant Synology High Availability.
  2. Gestion : Ce bouton est uniquement disponible lorsqu'un cluster High Availability existe déjà. Comprend les options suivantes :
    • Assistant de gestion : Lance un assistant qui contient des instructions concernant diverses procédures de gestion.
    • Basculement : Basculez les services depuis le serveur actif vers le serveur passif. Pendant ce processus, le serveur actif cesse de fournir les services et devient serveur passif. Ensuite, le serveur passif devient le serveur actif et commence à fournir tous les services du cluster High Availability. Cette option n'est pas disponible lorsque l'état du serveur passif et erreur ou si aucun serveur passif n'a été sélectionné.
    • Mettre à jour DSM : Ouvre la page Mise à jour DSM dans le Panneau de configuration. Ici, les deux serveurs peuvent être mis à jour.
    • Arrêter le cluster HA : Arrête à la fois les serveurs actif et passif.
  3. Désactiver le bip : Cette option n'est disponible que lorsqu'une erreur survient sur le serveur actif ou passif. Interrompt le son du bip de l'un des serveurs lorsqu'une erreur survient. Les serveurs émettent encore un bip si d'autres erreurs se produisent par la suite.

Assistant de gestion

L'assistant de gestion vous guide tout au long des processus suivants de gestion.

  • Gérer le cluster :
    • Modifier le serveur actif en serveurs passifs : Basculez les services depuis le serveur actif vers le serveur passif. Pendant ce processus, le serveur actif cesse de fournir les services et devient serveur passif. Ensuite, le serveur passif devient le serveur actif et commence à fournir tous les services du cluster High Availability. Cette option n'est pas disponible lorsque l'état du serveur passif et erreur ou si aucun serveur passif n'a été sélectionné.
    • Mise à jour simultanée de DSM sur les serveurs actif et passif : Ouvre la page Mise à jour DSM dans le Panneau de configuration. Ici, les deux serveurs peuvent être mis à jour.
    • Supprimer le cluster HA : Annule la liaison du cluster. Les deux serveurs fonctionnent indépendamment après le redémarrage.
    • Ajout d'un nouveau serveur passif : Ajoute un serveur passif dans le cluster. Cette option n'est disponible que lorsqu'il n'y a pas de serveur passif dans le cluster High Availability.
    • Comment délier le serveur passif du cluster High Availability : Déliez le serveur passif du cluster. Quand un serveur est ajouté au cluster High Availability à nouveau après avoir été délié, une réplication complète des données sera nécessaire. Par conséquent, il est recommandé d'utiliser cette option que si vous souhaitez remplacer le serveur passif existant par un nouveau.
    • Déplacez les deux serveurs à un autre emplacement : Sélectionnez cette option si vous devez déplacer le cluster. Les deux serveurs s'arrêtent après l'opération.
  • Gérer le serveur actif :
    • Arrêter le serveur actif.
    • Redémarrer le serveur actif.
    • Remplacer/mettre à jour des composants en panne sur le serveur actif : Cette option arrête le serveur actif.
  • Gérer le serveur passif :
    • Arrêtez le serveur passif.
    • Redémarrez le serveur passif.
    • Remplacer/mettre à jour des composants en panne sur le serveur passif : Cette option arrête le serveur passif.

Remarque :

  • Une fois un cluster High Availability supprimé, la réplication des données sera nécessaire lorsque vous souhaitez en créer un nouveau.
  • Si vous arrêtez ou déliez le serveur passif, il ne sera plus possible de basculer si une erreur se produit sur le serveur actif.

État des serveurs et de la réplication des données

Cette section contient des images qui indiquent l'état du serveur actif et du serveur passif, ainsi que le flux de réplication des données entre les serveurs.

L'image de la réplication des données indique les conditions suivantes possibles :

  • La réplication des données fonctionne normalement : Indiqué par les données qui circulent du serveur actif vers le serveur passif.
  • Pas de réplication des données : Indiqué par l'absence de l'image de flux des données.

Informations du serveur

Cette section est située sous les images des serveurs. Les informations concernant le serveur actif et le serveur passif sont affichées dans cette section. Si aucun serveur passif n'a été sélectionné, cette section indiquera N/A.

  • Nom du serveur : Nom du serveur Synology.
  • Nom de modèle : Modèle serveur Synology.
  • Numéro de série : Numéro de série du serveur Synology.
  • État du ventilateur : État des ventilateurs de refroidissement d'un serveur. Si un ventilateur dysfonctionne, un avertissement apparaîtra. Dans ce cas, veuillez contacter votre distributeur local pour obtenir de l'aide.
  • Température : Température du serveur Synology.
  • État de l'alimentation : Indique si les alimentations sur un serveur fonctionnent toutes normalement ou non. Si le modèle de serveur est équipé d'alimentations redondantes, un avertissement apparaîtra si l'une d'elle dysfonctionne.
