Arrêt et réinitialisation

Les instructions ci-dessous vous guideront dans la procédure d'arrêt, de suppression ou de réinitialisation d'un cluster high-availability.

Pour arrêter un cluster high-availability en utilisant Synology High Availability :

Lorsque vous souhaitez arrêter un cluster high-availability pour transporter les serveurs à un nouvel emplacement physique ou effectuer un entretien, mais que vous ne voulez pas supprimer les configurations du cluster high-availability, veuillez suivre ces instructions.

  1. Connectez-vous à DSM.
  2. Ouvrez Synology High Availability.
  3. Cliquez sur le bouton marqué Gestion.
  4. Sélectionnez Arrêter le cluster HA.
  5. Un message de confirmation apparaît. Cliquez sur Oui pour confirmer.

Pour arrêter manuellement un cluster high-availability :

Le cluster high-availability peut également être arrêté manuellement en utilisant les boutons d'alimentation situés sur chaque serveur.

  1. Appuyez le bouton d'alimentation sur un serveur (actif ou passif) et maintenez-le enfoncé jusqu'à ce qu'il émette un bip.
  2. Une fois qu'un serveur est complètement éteint, attendez au moins 30 secondes (le serveur qui est toujours en ligne nécessite du temps pour enregistrer les paramètres).
  3. Appuyez sur le bouton d'alimentation sur l'autre serveur et maintenez-le enfoncé jusqu'à ce qu'il émette un bip.
  4. Attendez que le serveur soit complètement éteint.

Allumer les serveurs :

Quand vous voulez allumer les serveurs à nouveau, vous devez allumer simultanément le serveur actif et le serveur passif (ou à cinq minutes d'intervalle maximum). Si vous n'allumez que l'un des serveurs, il essaie d'allumer l'autre serveur en diffusant un message Wake-on-LAN.

Arrêt anormal :

Quand un cluster high-availability est créé, l'option de redémarrage après une coupure de courant du Panneau de configuration est automatiquement activée. Si les serveurs s'arrêtent de manière anormale suite à une panne, ils s'allument automatiquement une fois le courant rétabli.

Pour supprimer un cluster high-availability :

Quand vous souhaitez délier les deux serveurs et supprimer un cluster high-availability afin de lier un serveur différent, veuillez suivre ces instructions.

  1. Allez à la page Aperçuu.
  2. Cliquez sur le bouton Gestion.
  3. Sélectionnez Supprimer le cluster HA.

Après confirmation, le système supprime le cluster high-availability. Le champ d'état affiche la progression du processus de suppression. Lorsque le cluster high-availability a été supprimé avec succès, vous serez dirigé vers l'adresse IP d'origine du serveur actif. L'état des deux serveurs deviendra Autonome.

Remarque :

  • Pendant la procédure de suppression d'un cluster high-availability, il peut y avoir quelques opérations d'écriture de données incomplètes. Pour assurer l'exhaustivité des données, veuillez effectuer la réplication de données basée sur le serveur actif, car le serveur passif manque peut-être quelques mises à jour de données.

Pour réinitialiser un cluster high-availability défectueux :

Si vous ne parvenez pas à accéder à DSM ou au Synology High Availability Manager, du fait d'une erreur de dédoublement du cerveau ou d'autres défaillances, vous pouvez essayer de forcer le cluster high-availability pour le délier en appuyant sur le bouton Réinitialiser situé sur vos serveurs.

  1. Appuyez sur le bouton Reset (habituellement situé sur la face arrière d'un serveur) et maintenez-le enfoncé jusqu’à ce que le serveur émette un bip.
  2. Le système sera supprimé du cluster high-availability et l'état de ce serveur passera à Autonome.
  3. Appuyer sur le bouton Réinitialiser restaurera également certains paramètres par défaut, comme l'adresse IP, le DNS et le mot de passe admin, en plus de ce qui suit :
    • Le mot de passe admin sera réinitialisé à vide.
    • Le compte guest sera désactivé.
    • DHCP sera activé sur IP, DNS, Passerelle et autres interfaces réseau.
    • Les Jumbo Frames seront désactivées, et MTU sera défini à 1500.
    • Le port de gestion sera réinitialisé à 5000/5001.
    • PPPoE sera réinitialisé à la valeur par défaut.
    • Le blocage automatique d'IP sera désactivé (l'adresse IP bloquée ne sera pas effacée).
    • Link Aggregation sera désactivé.
    • Les règles de pare-feu seront désactivées (les règles ne seront pas effacées).

Remarque :

  • Le fait d'appuyer sur le bouton Reset d'un serveur supprime uniquement ce serveur du cluster high-availability. Si vous souhaitez supprimer les deux serveurs du cluster high-availability, appuyez sur le bouton Reset des deux serveurs.
