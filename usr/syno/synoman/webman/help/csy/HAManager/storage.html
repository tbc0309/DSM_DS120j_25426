Stav úložiště

Tato stránka obsahuje podrobnosti o stavu svazků úložiště a jednotek iSCSI LUN (na úrovni bloků) na aktivním a pasivním serveru clusteru high-availability.

Jelikož musí být kapacita a konfigurace odpovídajících svazků a jednotek iSCSI LUN (na úrovni bloků) na aktivním i pasivním serveru identická, zobrazují se svazky a jednotky iSCSI LUN (na úrovni bloků) společně jako jedna jednotka. Pokud si tedy chcete zobrazit stav svazků a jednotek iSCSI LUN (na úrovni bloků) aktivního i pasivního serveru, musíte přejít na stránku Stav úložiště [při přístupu do clusteru high-availability si lze v programu Správce úložiště zobrazit pouze svazky a jednotky iSCSI LUN (na úrovni bloků) na aktivním serveru].

Tato stránka zobrazuje následující informace:

  • Název: Zobrazuje názvy svazků a jednotek iSCSI LUN (na úrovni bloků).
  • Využito: Zobrazuje procentuální hodnotu využité kapacity svazků a jednotek iSCSI LUN (na úrovni bloků).
  • Využito/celková velikost: Zobrazuje celkovou a využitou kapacitu svazků a jednotek iSCSI LUN (na úrovni bloků).
  • Stav: Zobrazuje stav svazků, jednotek iSCSI LUN (na úrovni bloků) a mezipamětí SSD.
  • Popis: Zobrazuje popis chyb a další informace týkající se svazku, jednotky iSCSI LUN (na úrovni bloků) a mezipaměti SSD.

Postup opravy vadného svazku nebo jednotky iSCSI LUN (na úrovni bloků):

  • Pokud dojde ke zhroucení pevného disku na jednom ze serverů, můžete přejít na stránku Stav disku, na které zjistíte, u kterého disku došlo ke zhroucení a je tedy nutné jej vyměnit. V takovém případě proveďte následující:
    1. Vyměňte zhroucený pevný disk (postup instalace pevného disku naleznete v Rychlé instalační příručce pro váš produkt Synology).
    2. Přejděte na stránku Stav úložiště a klikněte na položku Opravit.
  • Pokud je pasivní server offline, bude stav svazků a jednotek iSCSI LUN (na úrovni bloků) oznamovat výskyt chyby a na stránce Stav disku se nezobrazí žádný z disků na pasivním serveru. V takovém případě se můžete pokusit vyřešit tento problém následujícím způsobem:
    1. Ověřte, zda je pasivní server zapnutý.
    2. Zkontrolujte všechny napájecí zdroje.
    3. Zkontrolujte všechna připojení.
  • Pokud na jednom ze serverů dojde k chybě systému souborů ve svazku nebo na jednotce iSCSI LUN (na úrovni bloků), postupujte následovně:
    • Přejděte na stránku Stav úložiště.
    • Klikněte na položku Opravit.

Poznámka: Maximální počet svazků a jednotek iSCSI LUN (na úrovni bloků) v clusteru high-availability je 64.

Server kvora

Servery kvora pomáhají snižovat možnost výskytu chyb split brain. V nastaveních split brain můžete přiřadit další server, který bude sloužit jako server kvora. Tento server kvora se musí neustále připojovat jak k aktivnímu, tak k pasivnímu serveru. Vhodnou volbou pro server kvora je například server brány nebo server DNS, protože takový server se může připojit jak k aktivnímu, tak k pasivnímu serveru. Pamatujte si, že server kvora slouží pouze k poskytování služeb Ping a není tedy nutné instalovat žádné aplikace. Níže naleznete situace, které mohou nastat, když je povolen server kvora:

  • Pokud se pasivní server nemůže připojit jak k aktivnímu serveru, tak k serveru kvora, neprovede se přepnutí služeb za provozu za účelem prevence chyb split brain.
  • Pokud se aktivní server nemůže připojit k serveru kvora, ale pasivní server se k němu připojit může, provede se přepnutí za účelem zajištění lepší dostupnosti.
