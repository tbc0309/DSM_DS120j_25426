Vypnutí a reset

Níže uvedené pokyny obsahují popis vypnutí, odebrání nebo resetu clusteru high-availability.

Vypnutí clusteru high-availability pomocí nástroje Synology High Availability:

Chcete-li vypnout cluster high-availability kvůli přemístění serverů nové fyzické místo nebo provedení údržby, ale nechcete odebrat konfigurace clusteru high-availability, postupujte podle následujících pokynů.

  1. Přihlaste se do systému DSM.
  2. Otevřete nástroj Synology High Availability.
  3. Klikněte na tlačítko s označením Správa.
  4. Vyberte možnost Vypnout cluster HA.
  5. Zobrazí se zpráva s potvrzením. Potvrďte kliknutím na možnost Ano.

Ruční vypnutí clusteru high-availability:

Cluster high-availability lze vypnout také ručně pomocí tlačítek napájení nacházejících se na obou serverech.

  1. Na jednom serveru (aktivním nebo pasivním) stiskněte a podržte tlačítko napájení, dokud nezazní zvukový signál.
  2. Až se jeden server úplně vypne, počkejte alespoň 30 sekund (server, který je stále online, potřebuje čas na uložení nastavení).
  3. Na druhém serveru stiskněte a podržte tlačítko napájení, dokud nezazní zvukový signál.
  4. Počkejte, až se server zcela vypne.

Zapnutí serverů:

Pokud chcete servery opět zapnout, je potřeba zapnout aktivní i pasivní server současně (případně s rozestupem do pěti minut). Pokud zapnete pouze jeden ze serverů, pokusí se zapnout druhý server odesláním zprávy Probuzení přes LAN.

Nenormální vypnutí:

Při vytvoření clusteru high-availability je možnost restartování po selhání napájení v části Ovládací panel automaticky povolená. Pokud dojde k vypnutí serverů z důvodu poruchy napájení, servery se po obnovení napájení automaticky zapnou.

Odebrání clusteru high-availability:

Chcete-li odpojit oba servery a odebrat cluster high-availability, aby bylo možné připojit jiný server, postupujte podle následujících pokynů.

  1. Přejděte na kartu Přehled.
  2. Klikněte na tlačítko Správa.
  3. Vyberte možnost Odebrat cluster HA.

Po potvrzení systém cluster high-availability odebere. V poli stavu se zobrazuje průběh procesu odebrání. Po úspěšném odebrání clusteru high-availability budete přesměrováni na původní IP adresu aktivního serveru. Stav obou serverů se změní na Samostatný.

Poznámka:

  • V průběhu procesu odebírání clusteru high-availability může dojít k nedokončeným operacím zápisu dat. Chcete-li zajistit úplnost dat, proveďte replikaci dat založenou na aktivním serveru, protože pasivní server nemusí obsahovat některé aktualizace dat.

Obnovení nefunkčního clusteru high-availability:

V případě, že není možný přístup do systému DSM nebo do clusteru Synology High Availability z důvodu chyby split brain nebo jiných závad, můžete se pokusit o vynucení odpojení clusteru high-availability stisknutím tlačítka Obnovit nacházejícího se na serverech.

  1. Stiskněte a podržte tlačítko Obnovit (obvykle se nachází na zadním panelu serveru), dokud nezazní zvukový signál serveru.
  2. Tento systém bude odebrán z clusteru high-availability a stav serveru se změní na Samostatný.
  3. Stisknutím tlačítka Obnovit rovněž obnovíte na výchozí hodnoty některá nastavení, například IP adresu, DNS a heslo uživatele admin. Dále se provedou následující akce:
    • Heslo správce bude obnoveno jako prázdné.
    • Účet typu guest se zakáže.
    • U síťových rozhraní IP, DNS, brány a dalších se aktivuje protokol DHCP.
    • Rámce JumboFrame budou zakázány a hodnota MTU se nastaví na 1500.
    • Port pro správu bude obnoven jako 5000/5001.
    • Protokol PPPoE se vyresetuje na výchozí hodnoty.
    • Automatické blokování IP bude zakázáno (blokovaná IP adresa se nevymaže).
    • Funkce Link Aggregation bude zakázána.
    • Pravidla brány firewall budou zakázána (pravidla se nevymažou).

Poznámka:

  • Stisknutím tlačítka Obnovit na serveru odeberete z clusteru high-availability pouze tento server. Chcete-li odebrat z clusteru high-availability oba servery, stiskněte tlačítko Obnovit na obou serverech.
