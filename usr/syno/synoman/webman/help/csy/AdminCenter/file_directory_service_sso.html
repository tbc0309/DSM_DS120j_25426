Klient SSO

Klient SSO domény/LDAP

Pokud se zařízení Synology NAS připojilo ke stejné doméně nebo službě LDAP jako server SSO, můžete nastavit zařízení Synology NAS jako klienta SSO. Po přihlášení k serveru SSO pomocí svých přihlašovacích údajů mohou uživatelé využívat služby poskytované zařízením Synology NAS.

Než začnete, ověřte si, jestli se zařízení Synology NAS připojilo do stejné domény nebo služby LDAP jako server SSO.

Nastavení zařízení Synology NAS jako klienta SSO:

  1. Přejděte do nabídky Ovládací panel > Doména/LDAP > Klient SSO.
  2. Zaškrtněte políčko Povolit službu SSO.
  3. Do polí pod zaškrtávacím políčkem zadejte adresu URL serveru SSO a ID aplikace.

    Poznámka:

    ID aplikace získáte jedním z těchto dvou způsobů:

    • Metoda č. 1:
      1. Přihlaste se do aplikace DSM spuštěné jako server SSO.
      2. Přidejte svoje zařízení Synology NAS, které bude sloužit jako klientská aplikace, do seznamu aplikací serveru SSO Server. Podrobné pokyny najdete v článku nápovědy pro SSO Server > Seznam aplikací.
      3. Zkopírujte a vložte ID aplikace přidělené serverem SSO Server po přidání vašeho zařízení Synology NAS do serveru SSO.
    • Metoda č. 2:
      1. Po zadání adresy URL serveru SSO klikněte na položku Rychlá registrace nacházející se vedle pole ID aplikace.
      2. V poli pro registraci zadejte do polí Správce serveru a Heslo přihlašovací údaje účtu správce serveru SSO.
      3. Do pole Název aplikace zadejte název své klientské aplikace.
      4. Zkontrolujte, zda možnost Přesměrovat URI odkazuje přesně na stránku přihlášení vašeho zařízení Synology NAS.
      5. Klikněte na možnost OK, vaše zařízení Synology NAS bude přidáno do serveru SSO Server a ID aplikace se automaticky vyplní.

    ______

    * Je potřeba, abyste při konfiguraci všech klientů SSO zadali totožnou adresu URL serveru SSO (název hostitele nebo IP).

  4. Nastavení uložíte kliknutím na možnost Použít.

Poznámka:

  • Pokud server SSO používá protokol HTTPS, může být přihlášení SSO zablokováno prohlížeči z důvodu absence důvěryhodného certifikátu.
  • Pokud se k vašemu serveru SSO přistupuje prostřednictvím protokolu HTTP a pokud vaše zařízení Synology NAS (nastavené jako klient SSO) používá připojení HTTPS, musí uživatelé nakonfigurovat nastavení svých prohlížečů tak, aby mohli používat nezabezpečené připojení.
  • IP adresa serveru SSO ve formátu IPv6 nebude podporována, pokud jste v systému DSM v části Ovládací panel > Zabezpečení > Zabezpečení zaškrtli políčko Vylepšit zabezpečení pomocí záhlaví HTTP Content Security Policy (CSP).

Klient SSO Azure AD

Pokud se vaše zařízení Synology NAS připojilo do domény Azure AD pomocí služby site-to-site VPN nebo do domény Windows synchronizované s doménou Azure AD, je možné zařízení Synology NAS nastavit jako klienta Azure SSO. Po přihlášení k serveru Azure SSO pomocí svých přihlašovacích údajů mohou uživatelé využívat služby poskytované zařízením Synology NAS.

Než začnete, přesvědčte se, jestli se zařízení Synology NAS připojilo do domény Azure AD pomocí služby VPN nebo domény Windows, která je již synchronizovaná s doménou Azure AD.

Nastavení zařízení Synology NAS jako klienta Azure SSO:

  1. Přejděte do nabídky Ovládací panel > Doména/LDAP > Klient SSO.
  2. Zaškrtněte políčko Povolit službu Azure SSO.
  3. Do příslušných polí zadejte hodnoty ID klienta, Tajný klíč klienta, ID nájemce a Přesměrovat URI.
  4. Nastavení uložíte kliknutím na možnost Použít.

Poznámka:

  • Podrobný postup získání hodnot ID/tajný klíč klienta a ID nájemce se nachází v následujících částech.
  • Z bezpečnostních důvodů se uživatelé musí do systému DSM se službou Azure SSO přihlašovat prostřednictvím protokolu HTTPS.

Získání ID klienta a tajného klíče klienta pro aplikaci:

  1. Pomocí účtu správce se přihlaste ke klasickému portálu Azure.
  2. V části ACTIVE DIRECTORY vyberte adresář, který obsahuje vaše uživatelské účty.
  3. Přejděte na kartu APPLICATIONS (Aplikace) a kliknutím na položku ADD (Přidat) dole přidejte novou aplikaci.
  4. Klikněte na možnost Add an application my organization is developing (Přidat aplikaci vyvíjenou mojí organizací).
  5. Do pole Name (Název) zadejte název své aplikace a zaškrtněte políčko WEB APPLICATION AND/OR WEB API (Webová aplikace nebo webové rozhraní API).
  6. Vyplňte pole SIGN-ON URL (Přihlašovací adresa URL). Například https://vaše_doména:port/webman/login.cgi
  7. Vyplňte pole APP ID URI (URI ID aplikace). Například https://vaše_doména:port/
  8. Po dokončení nastavení vyberte svoji aplikaci. Zobrazí se CLIENT ID (ID klienta).
  9. Přidejte klíč a klikněte na možnost SAVE (Uložit). Následně se jako prostý text zobrazí CLIENT SECRET (Tajný klíč klienta).

Získání ID nájemce pro aplikaci:

  1. Opakujte výše uvedené kroky 1 až 3.
  2. Na kartě APPLICATIONS (Aplikace) vyberte aplikaci a dole klikněte na možnost VIEW ENDPOINTS (Zobrazit koncové body).
  3. Ve všech textových polích se zobrazí sekvence ID. Zkopírujte a vložte ID do pole Tenant ID (ID nájemce).

Poznámka:

  • Uživatelské rozhraní služby Microsoft Azure™ se může změnit bez předchozího upozornění.
