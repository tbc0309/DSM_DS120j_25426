Zabezpečení

V nastavení Ovládací panel > Zabezpečení > Zabezpečení je možné pro relace procházení systému DSM změnit následující bezpečnostní nastavení:

  • Časovač odhlášení (minuty): Pokud budou uživatelé po dobu zadanou v této části nečinní, budou od systému DSM automaticky odhlášeni. Zadejte libovolnou hodnotu od 1 do 65535.
  • Vylepšit kompatibilitu prohlížeče přeskočením kontroly IP: Pokud k zařízení Synology NAS přistupujete přes server HTTP proxy a dochází k náhodným odhlášením, je možné povolit tuto možnost a přeskočit kontrolu IP adresy.
  • Zvýšit ochranu před webovými útoky CSRF: Tato možnost posiluje ochranu systému proti útokům typu cross-site scripting. Tato možnost začne platit při příštím přihlášení do systému DSM.
  • Vylepšit zabezpečení pomocí záhlaví HTTP Content Security Policy (CSP): Tato možnost zvyšuje zabezpečení systému proti útokům typu cross-site scripting (XSS) povolením dat pouze z důvěryhodných zdrojů a omezením spouštění inline skriptů.
  • Nepovolit systému DSM sloučení s iFrame: Povolením této možnosti zakážete jiným webům sloučit systém DSM s jinými webovými stránkami pomocí prvku iFrame, takže se zabrání některým typům útoků ze škodlivých webů. Konkrétním webům je možné sloučení systému DSM s prvkem iFrame povolit kliknutím na tlačítko Povolené webové stránky.
  • Vymazat neuložené relace přihlášení uživatele při restartu systému: Tato možnost předchází neočekávaným chybám systému v případě, že uživatelé zůstanou přihlášeni a provádějí v systému operace v době, kdy se systém připravuje. Je-li tato možnost povolená, budou se všichni uživatelé muset po restartování systému znovu přihlásit.
  • Při změně aktuální adresy IP zobrazit upozornění na ploše systému DSM: Při změně IP adresy aktuálně připojeného uživatele odeslat upozornění na plochu tohoto uživatele.
  • Důvěryhodné servery proxy: Systém DSM použije informace o vzdálené adrese IP předané z důvěryhodného serveru proxy. Důvěryhodné servery proxy je možné změnit.
