DDNS

DDNS (Dynamic Domain Name Service) vereinfacht die Verbindung der Synology NAS über das Internet durch Zuweisung eines Hostnamens zur IP-Adresse. DDNS erlaubt Ihnen z. B. den Zugriff auf die Synology NAS unter Verwendung eines Domainnamens (z. B. www.john.synology.me), ohne dass Sie sich eine IP-Adresse (z. B. 172.16.254.1) merken müssen.

DDNS einrichten

Sie können der IP-Adresse der Synology NAS einen vorhandenen Hostnamen zuweisen oder einen von Synology oder verschiedenen anderen DDNS-Anbietern bereitgestellten neuen Namen beantragen. Weitere Informationen über die Registrierung von Hostnamen erhalten Sie vom jeweiligen Anbieter.

DDNS-Hostnamen einrichten:

  1. Klicken Sie auf die Schaltfläche Hinzufügen.
  2. Ein Dialogfeld wird geöffnet, in dem Sie die folgenden Einstellungen ändern:
    • Serviceanbieter: Wählen Sie einen Serviceanbieter aus. Um einen kostenlosen Hostnamen von Synology zu beantragen, wählen Sie Synology aus dem Dropdown-Menü aus.
    • Hostname: Geben Sie einen registrierten DDNS Hostnamen ein, zum Beispiel john.synology.me.
    • Benutzername/E-Mail: Geben Sie Benutzernamen/E-Mail für Ihren DDNS-Anbieter ein.
    • Kennwort/Schlüssel: Geben Sie Kennwort/Schlüssel für Ihren DDNS-Anbieter ein.
    • Externe Adresse: Geben Sie die externe IP-Adresse der Synology NAS ein, die den Hostnamen verwenden wird.
    • Heartbeat: Aktivieren Sie diese Option, um Alarme in Bezug auf den Status des zugewiesenen Hostnamens zu erhalten (nur für von Synology bereitgestellte Hostnamen).
  3. Klicken Sie auf Verbindung testen, um zu prüfen, ob die Einstellungen korrekt sind.
  4. Klicken Sie zum Speichern und Fertigstellen auf OK.
  5. Wenn Sie Synology als Serviceanbieter ausgewählt haben, wird eine Nachricht angezeigt, wenn kein passendes SSL-Zertifikat für Ihren DDNS-Hostnamen vorhanden ist und es wird empfohlen, dass Sie ein Zertifikat von Let’s Encrypt für den Hostnamen signieren und als Standardzertifikat festlegen. Klicken Sie auf Ja.

    Anmerkung: Wenn es bereits ein passendes SSL-Zertifikat für Ihren DDNS-Hostnamen gibt, werden Sie gefragt, ob Sie dieses als Standardzertifikat festlegen möchten. Diese Nachricht wird jedes Mal angezeigt, wenn der DDNS-Hostname bearbeitet wird. Weitere Informationen zu SSL-Zertifikaten finden Sie hier.

Nach dem Abschluss der Einrichtung können Sie durch Eingeben des DDNS-Hostnamens in einen Webbrowser über das Internet auf die Synology NAS zugreifen.

Anforderungen und Einschränkungen

  • Die Server des DDNS-Serviceanbieters funktionieren.
  • Die Synology NAS kann sich mit dem Internet verbinden.
  • DDNS funktioniert nur in einer Netzwerkumgebung, in der kein Proxyserver benötigt wird.
  • Sie können nur einen Hostnamen für jeden DDNS-Anbieter eingeben.

Verwaltung von Hostnamen

Verwenden Sie die folgenden Schaltflächen, um bestehende Hostnamenseinträge zu verwalten:

  • Löschen: Entfernt den ausgewählten Hostnameneintrag aus der Liste.
  • Bearbeiten: Ändert Einstellungen des ausgewählten Hostnameneintrags.
  • Jetzt aktualisieren: Aktualisiert den Status aller Hostnameneinträge, für die der DDNS-Dienst aktiviert ist.

Benutzerdefinierten DDNS-Anbieter einrichten

Sie können einen benutzerdefinierten DDNS-Anbieter hinzufügen, wenn Ihr Anbieter nicht bereits in der Liste der Serviceanbieter angezeigt wird.

Benutzerdefinierten DDNS-Anbieter einrichten:

  1. Klicken Sie auf Anpassen.
  2. Ein Dialogfeld wird geöffnet, in das Sie die nachstehenden Einstellungen eingeben:
    • Serviceanbieter: Geben Sie den Namen Ihres benutzerdefinierten Serviceanbieters ein.
    • Abfrage-URL: Geben Sie die URL Ihres benutzerdefinierten Serviceanbieters ein.
  3. Klicken Sie auf Speichern, um den benutzerdefinierten Serviceanbieter zur Liste der DDNS-Serviceanbieter hinzuzufügen.

Anforderungen und Einschränkungen

  • Die Abfrage-URL muss dem von Ihrem Serviceanbieter angegebenen Protokoll entsprechen. Diese URL wird verwendet, um aktuelle IP-Informationen zum DDNS-Server Ihres Anbieters zu senden.
  • Eine Abfrage-URL enthält grundsätzlich die vier folgenden Parameter. Ersetzen Sie bei der Eingabe der Abfrage-URL die folgenden Parameter durch die entsprechende Variable.
    • Benutzername: __USERNAME__
    • Kennwort: __PASSWORD__
    • Hostname: __HOSTNAME__
    • Externe Adresse: __MYIP__
  • Die Abfrage-URL darf keine Leerzeichen oder Semikolons enthalten.
