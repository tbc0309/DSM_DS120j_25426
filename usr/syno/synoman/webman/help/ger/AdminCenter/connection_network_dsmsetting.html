DSM-Einstellungen

Unter Systemsteuerung > Netzwerk > DSM-Einstellungen können die von den Clients verwendeten Portnummern für den Zugriff auf Synology NAS geändert werden.

So ändern Sie die Standardportnummer:

Sowohl HTTP- wie auch HTTPS-Verbindungen sind in DSM immer aktiviert. Der Standard-HTTP-Port ist 5000 und der Standard-HTTPS-Port ist 5001.

  1. Geben Sie die Portnummer im Feld HTTP oder HTTPS ein.
  2. Klicken Sie auf Übernehmen.

HTTP-Verbindungen zu HTTPS umleiten:

Sie können Clients, die versuchen, über den HTTP-Port auf Ihre Synology NAS zuzugreifen, zum HTTPS-Port umleiten.

  1. Wählen Sie HTTP-Verbindungen automatisch zu HTTPS umleiten (Web Station und Photo Station ausgeschlossen) aus.
  2. Klicken Sie auf Übernehmen.

Ports, die für die HTTPS-Verbindung verwendet werden:

  • DSM: Die Standardportnummer ist 5001, damit Sie über die folgende Adresse sicher auf den DSM zugreifen können: https://server-hostname:5001/
  • Web Station und Photo Station: Die Standardportnummer ist 443, damit Sie über die folgende Adresse sicher auf Web Station und Photo Station zugreifen können: https://server-hostname/

HTTP/2 aktivieren:

Sie können die HTTPS-Verbindungen beschleunigen, indem Sie HTTP/2 aktivieren.

  1. HTTP/2 aktivieren wählen.
  2. Klicken Sie auf Übernehmen.

Anpassen der maximal zulässigen gleichzeitigen HTTP-Verbindungen:

Sie können die Option Anzahl max. gleichzeitiger HTTP-Verbindungen anpassen aktivieren, um die HTTP-Verbindungen Ihrer Synology NAS zu erhöhen.

Anmerkung:

  • Nur anwendbar auf folgende Plattformen: Bromolow, Cedarview, Avoton, Grantley, Broadwell, Dockerx64, Braswell, Broadwellnk, Apollolake. In diesem Artikel erfahren Sie, ob Ihr Modell diese Funktion unterstützt.
  • Je höher die Anzahl der maximal zulässigen gleichzeitigen HTTP-Verbindungen ist, desto mehr Systemspeicher wird benötigt.

Zur Anpassung der „Server“-Überschrift in den HTTP-Antworten:

  1. „Server“-Überschrift in den HTTP-Antworten aktivieren wählen.
  2. Geben Sie eine Überschrift für Ihren Webserver an (z. B. nginx).
  3. Klicken Sie auf Übernehmen.

Sie können die „Server“-Überschrift deaktivieren, indem Sie das Kästchen deaktivieren.

Domain

Unter Domain können Sie einen benutzerdefinierten Domainnamen für den Zugriff auf Synology NAS angeben.

Benutzerdefinierte Domain aktivieren:

  1. Markieren Sie Benutzerdefinierte Domain aktivieren.
  2. Geben Sie einen Domainnamen an, der für Synology NAS angewendet werden soll.
  3. Sie können auch HSTS aktivieren markieren, um auf dieser Domain HTTPS-verschlüsselte Verbindungen zu erzwingen.
  4. Klicken Sie auf Übernehmen.

Anmerkung:

  • Jede Domain kann nur für jeweils eine bestimmte Anwendung auf Ihrer Synology NAS verwendet werden.
  • Sofern der Domainname nicht länger als 15 Zeichen ist und den NetBIOS-Namenskonventionen entspricht, wird das System den Domainnamen (z. B. MyDSM) automatisch registrieren und übertragen. Windows-Benutzer können mit dieser URL auf den DSM-Desktop im selben lokalen Netzwerk zugreifen: http://MyDSM oder https://MyDSM
  • Der Zugriff auf Anwendungen über URLs, die Domainnamen verwenden, erfolgt über den Standard-HTTP-Port (80) oder den Standard-HTTPS-Port (443).
