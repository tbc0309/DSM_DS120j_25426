Überblick

Diese Seite zeigt allgemeine Informationen über den High Availability-Cluster und Mitglieder-Server an. Zusätzlich bietet diese Seite Werkzeuge zum Erstellen, Entfernen oder zum Ausführen von Wartungsarbeiten an High Availability-Clustern und Mitglieder-Servern.

Status des High Availability-Clusters

Dieses Feld zeigt den Gesamtstatus des High Availability-Clusters an und benachrichtigt Sie bei Fehlern. Es folgt eine Tabelle mit Bedingungen, die in diesem Feld angezeigt werden können:

Status Beschreibung
Einzeln Die Server sind nicht in einem High Availability-Cluster kombiniert.
In Ordnung Der Gesamtzustand des High Availability-Clusters ist gut. Die Replikation von Daten vom aktiven Server auf den passiven Server funktioniert normal.
Wird vorbereitet Die Server wurden erfolgreich zu einem High Availability-Cluster kombiniert, aber die erstmalige Datenreplikation wurde noch nicht abgeschlossen. Die Übergabe kann zu diesem Zeitpunkt nicht durchgeführt werden. Weitere Details finden Sie unter Übergabe.
Laden Das System lädt den Cluster-Status. Übergabe kann erst ausgeführt werden, nachdem der Status geladen wurde.
Warnhinweis Der Status des HA-Clusters ist abnormal, aber die Dienste funktionieren noch.
Gefahr Sowohl der aktive als auch der passive Server sind defekt. Der High Availability-Cluster kann nicht normal funktionieren. Prüfen Sie den Status aller Speicherplätze (z. B. RAID Groups, Diskgruppen, Volumes und iSCSI LUNs).

Warnmeldungen

Die folgende Tabelle enthält Warnmeldungen, die angezeigt werden könnten, wenn ein Fehler im System auftritt. Wenn mehr als eine Warnmeldung angezeigt wird, können Sie mit den Vorwärts-/Rückwärts-Schaltflächen zwischen ihnen umschalten.

Warnung/Fehler Beschreibung
Überwachter Dienst fehlerhaft Ein Fehler ist bei einem oder mehreren überwachten Diensten aufgetreten.
Speicherplatz zeigt Leistungseinbußen Ein Speicherplatz (z. B. RAID Group, Diskgruppe, Volume, iSCSI LUN oder SSD-Cache) auf dem aktiven oder passiven Server zeigt Leistungseinbußen. Überprüfen Sie die Seite Speicher-Status.
Speicherplatz abgestürzt Ein Speicherplatz (z. B. RAID Group, Diskgruppe, Volume, iSCSI LUN oder SSD-Cache) auf dem aktiven oder passiven Server ist abgestürzt. Überprüfen Sie die Seite Speicher-Status.
Erkennung des passiven Servers fehlgeschlagen Dieser Fehler kann auftreten, wenn der passive Server offline ist oder der aktive und der passive Server nicht über die Heartbeat-Verbindung verbunden werden können. In diesem Fall ist die Übergabe nicht verfügbar.
Daten können nicht auf den passiven Server repliziert werden Dieser Fehler kann wegen eines Fehlers des Speicher-Volumes oder der Festplatte auftreten. In diesem Fall ist die Übergabe nicht verfügbar. Überprüfen Sie die Seite Speicher-Status.
Passiver Server ist nicht ausgewählt Wenn der passive Server vom High Availability-Cluster getrennt ist, wird dieser Status angezeigt. In diesem Fall ist die Übergabe nicht verfügbar. Wenn Sie den passiven Server wieder einbinden möchten, klicken Sie auf das Dropdown-Menü Beibehalten und auf Bind. Wenn Sie den passiven Server wieder einbinden, ist eine vollständige Datenreplikation erforderlich.

Informationen des High Availability-Clusters

Dieses Feld befindet sich unter dem High Availability-Cluster-Status und zeigt grundlegende Informationen über den Cluster an. Wenn ein High Availability-Cluster nicht existiert, zeigt dieses Feld nur die Beschreibung der Funktion des High Availability-Clusters an.

Dieses Feld enthält die folgenden Informationen:

  • Cluster-Servername: Der vom High Availability-Cluster verwendete Servername.
  • Cluster IP-Adresse: Die vom High Availability-Cluster verwendete IP-Adresse. Unabhängig davon, auf welchem Server Dienste ausgeführt werden, können sich Benutzer mit dieser IP-Adresse mit dem High Availability-Cluster verbinden.
  • Erstellungszeit: Das Erstellungsdatum des High Availability-Clusters.

Bedienschaltflächen des High Availability-Clusters

Die folgenden drei Schaltflächen befinden sich unter den High Aailability-Cluster-Informationen:

  1. HA-Cluster erstellen: Wenn kein High Availability-Cluster existiert, können Sie auf diese Schaltfläche klicken, um zwei Server zu kombinieren und einen High Availability-Cluster zu erstellen. Weitere Informationen über die Erstellung eines High Availability-Clusters finden Sie im Synology High Availability-Assistent.
  2. Verwaltung: Diese Schaltfläche ist nur verfügbar, wenn ein High Availability-Cluster existiert. Sie enthält die folgenden Optionen:
    • Verwaltungsassistent: Startet einen Assistenten, der Anweisungen für die verschiedenen Verwaltungsverfahren beinhaltet.
    • Übergabe: Initiiert eine Übergabe vom aktiven auf den passiven Server. Während des Vorgangs stoppt der aktive Server die Bereitstellung von Diensten und wird zum neuen passiven Server. Der alte passive Server wird in dem Fall zum neuen aktiven Server und beginnt mit der Bereitstellung von Diensten des High Availability-Clusters. Diese Option ist nicht verfügbar, wenn der passive Server defekt oder nicht ausgewählt ist.
    • DSM aktualisieren: Öffnet die Seite DSM aktualisieren der Systemsteuerung. Von hier aus können beide Server aktualisiert werden.
    • HA-Cluster herunterfahren: Fährt den aktiven und passiven Server herunter.
  3. Signalton deaktivieren: Diese Option ist nur verfügbar, wenn ein Fehler auf dem aktiven oder passiven Server auftritt. Deaktiviert den Signalton des aktiven/passiven Servers, wenn ein Fehler auftritt. Die Server geben trotzdem einen Signalton ab, wenn später ein anderer Fehler auftritt.

Verwaltungsassistent

Der Verwaltungsassistent führt Sie durch die folgenden Verwaltungsvorgänge:

  • Cluster verwalten:
    • Aktiven Server zum passiven Server machen: Initiiert eine Übergabe vom aktiven auf den passiven Server. Während des Vorgangs stoppt der aktive Server die Bereitstellung von Diensten und wird zum neuen passiven Server. Der alte passive Server wird in dem Fall zum neuen aktiven Server und beginnt mit der Bereitstellung von Diensten des High Availability-Clusters. Diese Option ist nicht verfügbar, wenn der passive Server defekt oder nicht ausgewählt ist.
    • DSM auf den aktiven und den passiven Servern aktualisieren. Öffnet die Seite DSM aktualisieren der Systemsteuerung. Von hier aus können beide Server aktualisiert werden.
    • HA-Cluster entfernen: Entfernt den Cluster. Die beiden Server funktionieren nach dem Neustart unabhängig voneinander.
    • Neuen passiven Server hinzufügen: Fügt einen passiven Server zu einem Cluster hinzu. Diese Option ist nur dann verfügbar, wenn im High Availability-Cluster kein passiver Server vorhanden ist.
    • Bindung des passiven Servers aus dem High Availability-Cluster entfernen: Entfernt den passiven Server aus dem Cluster. Wird der Server nach dem Trennen wieder zum High Availability-Cluster hinzugefügt, ist eine vollständige Datenreplikation erforderlich. Wir empfehlen daher diese Option nur zu verwenden, wenn Sie den existierenden passiven Server durch einen neuen ersetzen möchten.
    • Beide Server an einen anderen Ort verschieben: Wählen Sie diese Option, wenn Sie den Cluster verschieben müssen. Beide Server werden nach dem Vorgang heruntergefahren.
  • Aktiven Server verwalten:
    • Aktiven Server herunterfahren.
    • Aktiven Server neu starten.
    • Ausgefallene Komponenten auf dem aktiven Server aktualisieren/ersetzen. Diese Option fährt den aktiven Server herunter.
  • Passiven Server verwalten:
    • Passiven Server herunterfahren.
    • Passiven Server neu starten.
    • Ausgefallene Komponenten auf dem passiven Server aktualisieren/ersetzen. Diese Option fährt den passiven Server herunter.

Anmerkung:

  • Wenn ein High Availability-Cluster entfernt wird, ist eine vollständige Datenreplikation erforderlich, wenn Sie einen neuen erstellen möchten.
  • Wenn Sie den passiven Server herunterfahren oder trennen, können Dienste bei Auftreten von Fehlern auf dem aktiven Server nicht übergeben werden.

Server- und Datenreplikationsstatus

Dieser Abschnitt enthält Bilder, die den Status des aktiveren und passiven Servers anzeigen sowie die Datenreplikation zwischen den beiden Servern.

Das Bild der Datenreplikation zeigt die folgenden möglichen Situationen an:

  • Datenreplikation funktioniert normal: Dies wird durch die Darstellung des Datenflusses vom aktiven Server zum passiven Server angezeigt.
  • Keine Datenreplikation: Dies wird durch das Fehlen des Datenflussbilds angezeigt.

Serverinformationen

Dieser Bereich befindet sich unter den Server-Images. Informationen zum aktiven und zum passiven Server werden in diesem Bereich angezeigt. Wenn kein passiver Server ausgewählt wurde, zeigt dieser Abschnitt N/A an.

  • Servername: Synology-Servername.
  • Modellname: Synology-Servermodell.
  • Seriennummer: Seriennummer des Synology-Servers.
  • Lüfterstatus: Status der Lüfter eines Servers. Wenn ein Lüfter ausfällt, wird eine Warnung angezeigt. Wenden Sie sich in diesem Fall an Ihren Händler vor Ort, um Hilfe zu erhalten.
  • Temperatur: Temperatur des Synology-Servers.
  • Stromstatus: Zeigt an, ob die Netzteile an einem Server normal funktionieren oder nicht. Wenn das Synology-Modell mit redundanten Netzteilen ausgestattet ist, wird eine Warnung angezeigt, wenn eines von ihnen ausfällt.
