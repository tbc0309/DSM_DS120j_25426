Speicherpool reparieren

Speicherpool-Fehler können verschiedene Ursachen haben, etwa abgestürzte Laufwerke. In bestimmten Fällen können Sie die Fehler mit den folgenden Anweisungen identifizieren und beheben.

Die in diesem Artikel genannte Funktion Reparieren ist nur verfügbar für Speicherpools mit RAID-Typen, die Datenredundanz unterstützen (d. h. SHR mit mindestens 2 Laufwerken, RAID 1, RAID 5, RAID 6, RAID F1 und RAID 10). Beachten Sie bitte, dass mit dieser Funktion Speicherpools jedoch keine fehlerhaften Laufwerke repariert werden können.

Anmerkung:

  • Wir empfehlen, Laufwerke aus der Synology-Kompatibilitätsliste zu verwenden, um Probleme zu vermeiden. Die Verwendung nicht kompatibler Laufwerke kann die Systemleistung beeinträchtigen oder sogar zu Datenverlust führen.
  • SHR (Synology Hybrid RAID) ist nur bei bestimmten Modellen verfügbar. Weitere Informationen zu SHR finden Sie hier.

Einen Speicherpool mit defekten Systempartitionen reparieren

Wenn Systempartitionen defekt sind, wird eine Meldung angezeigt, dass auf die Systempartitionen nicht zugegriffen werden kann. Gehen Sie wie folgt vor, um das Problem zu beheben:

  1. Starten Sie den Speicher-Manager.
  2. Gehen Sie zur Seite Überblick und klicken Sie auf den Link Reparieren. Das System sollte mit der Reparatur der Systempartitionen beginnen.

Einen fehlerhaften Speicherpool reparieren

Ein Speicherpool wird fehlerhaft, wenn nicht auf die Daten auf einem Laufwerk zugegriffen werden kann, aber noch kein Datenverlust aufgetreten ist. Dies geschieht nur bei Speicherpools mit RAID-Typen mit Datenredundanz (d. h. SHR mit mindestens 2 Laufwerken, RAID 1, RAID 5, RAID 6, RAID F1 und RAID 10). Bevor Sie einen fehlerhaften Speicherpool reparieren, ersetzen Sie bitte zuerst die defekten Laufwerke durch einwandfreie und entfernen Sie sie dann, oder tauschen Sie sie einfach gegen einwandfreie Laufwerke aus.

Voraussetzungen:

Ersetzen Sie die defekten Laufwerke bitte durch einwandfreie, die folgende Kriterien erfüllen:

  • Für Speicherpools mit RAID 1, RAID 5, RAID 6, RAID F1 und RAID 10: Das Ersatzlaufwerk muss mindestens so groß sein wie das kleinste Laufwerk im Speicherpool.
  • Für SHR-1-Speicherpools: Das Ersatzlaufwerk muss eines der folgenden Kriterien erfüllen.
    • Das Ersatzlaufwerk muss mindestens so groß sein wie das defekte Laufwerk.
    • Das Ersatzlaufwerk muss mindestens so groß sein wie das zweitgrößte Laufwerk im Speicherpool.
      Beispiele:
      • Beispiel 1: SHR-1 umfasst Laufwerke mit 3 TB, 2 TB, 2 TB und 1 TB. Wenn das 3-TB-Laufwerk defekt ist, kann es durch ein Laufwerk repariert werden, dessen Kapazität gleich oder größer 2 TB ist.
      • Beispiel 2: SHR-1 umfasst Laufwerke mit 4TB, 4TB, 2 TB und 2TB. Wenn das 4-TB-Laufwerk defekt ist, kann es durch ein Laufwerk repariert werden, dessen Kapazität gleich oder größer 4TB ist.
      • Beispiel 3: SHR-1 umfasst Laufwerke mit 4TB, 4TB, 2 TB und 2TB. Wenn das 2-TB-Laufwerk defekt ist, kann es durch ein Laufwerk repariert werden, dessen Kapazität gleich oder größer 2 TB ist.
  • Für SHR-2-Speicherpools: Das Ersatzlaufwerk muss eines der folgenden Kriterien erfüllen.
    • Das Ersatzlaufwerk muss mindestens so groß sein wie das defekte Laufwerk.
    • Das Ersatzlaufwerk muss mindestens so groß sein wie das drittgrößte Laufwerk im Speicherpool.

Anmerkung:

  • Wenn die verfügbaren Laufwerke als Hot-Spare-Laufwerke zum Schutz eines oder mehrerer Speicherpools zugewiesen wurden, reparieren die Hot-Spare-Laufwerke den fehlerhaften Speicherpool automatisch. Weitere Informationen dazu finden Sie unter Hot Spare.

Fehlerhaften Speicherpool reparieren:

Gehen Sie wie folgt vor, um einen fehlerhaften Speicherpool zu reparieren:

  1. Gehen Sie zu Speicher-Manager > Speicherpool und suchen Sie den Speicherpool mit dem Status Fehlerhaft.
  2. Zeigen Sie die Informationen zum Speicherpool an und kontrollieren Sie den Laufwerkstatus.
  3. Gehen Sie danach zur Seite HDD/SSD, um sich zu vergewissern, welches Laufwerk defekt ist.
  4. Entfernen Sie das defekte Laufwerk aus Ihrem Synology NAS und ersetzen Sie es durch ein neues. Wenn Ihr Synology NAS „Hot-Swapping“ (Austausch während des Betriebs) nicht unterstützt, müssen Sie das Gerät vor dem Aus- und Einbau von Laufwerken herunterfahren.
    Anmerkung: Alle auf dem Ersatzlaufwerk vorhandenen Daten werden gelöscht.
  5. Nachdem das Ersatzlaufwerk installiert wurde, gehen Sie wieder zur Seite Speicherpool und wählen Sie den fehlerhaften Speicherpool aus.
  6. Wählen Sie im Dropdown-Menü Aktion Reparieren aus.
  7. Wählen Sie das Ersatzlaufwerk aus, das zum Speicherpool hinzugefügt werden soll, und folgen Sie den Anweisungen des Assistenten.

Anmerkung:

  • Wenn die Reparatur eines Speicherpools lange dauert, kann es an den hier aufgelisteten Gründen liegen.
