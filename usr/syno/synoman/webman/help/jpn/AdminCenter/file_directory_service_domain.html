ドメイン

Synology NAS を Windows ドメインに加えることができます。ドメインに加えると、わざわざ別のユーザー名とパスワードの組み合わせを覚えなくても、ドメイン ユーザーは各自のドメイン アカウントとパスワードを使って Synology NAS にサインインし、ファイルにアクセスしたり、DSM アプリケーションを使用したりすることができます。

ドメインに加入する：

  1. [コントロール パネル] > [ドメイン/LDAP] > [ドメイン] を選択します。
  2. [ドメインに参加] ボックスにチェックマークを付けます。
  3. 相当する欄にドメイン名を入力し、DNS Server を指定します。
  4. 必要であれば、[詳細ドメインオプション] ボックスにチェックマークを付けます。（詳細は、次の説明をお読みください）。
  5. [適用]をクリックします。
  6. 開いたウィンドウで、ドメインの管理者用ユーザー名とパスワードを入力します。
  7. [OK] をクリックします。

DNS Server

Synology NAS でドメイン検索をできるようにするため、DNS Server を指定するよう推奨しています。ただし、ドメインが DNS Server に登録されていない場合は、ドメイン加入処理ができない場合があります。

ドメインサーバーの種類

Synology NAS がドメインに加入すると、この欄にはドメイン タイプが表示されます。ドメインタイプは「AD ドメイン」になります。

管理モード

このオプションは、ドメイン ユーザーとグループの権限を管理する方法を決定します。

  • 信頼できるドメイン：結合したドメインのユーザーとグループと信頼できるドメインを管理できます。ユーザーとグループのリストをドメイン別にフィルタできます。
  • OU のシングルドメイン：このモードでは、結合したドメインのユーザーとグループのみ表示されます。このモードでは、組織単位 (OU) 別にユーザーとグループの一覧をフィルタできます。

詳細ドメイン オプション

ほとんどの場合、詳細ドメインオプションの設定は必要ありません。詳細ドメインオプションは、特定のドメイン環境でのみ必要です。

  • DC IP/FQDN：DC（ドメインコントローラー）IP アドレスまたは FQDN を指定すると、Synology NAS がその情報を使用して通信しようとします。複数の IP アドレスまたは FQDN が存在する場合は、IP の間にコンマを入力します。必要に応じて最後の DC IP アドレスまたは FQDN の後にアスタリスク(*)を追加すると、指定したすべての IP アドレスで失敗したときに Synology NAS が別の DC との通信を試みます。このとき、最後の IP アドレスまたは FQDN とアスタリスクの間には必ずコンマを追加してください。
  • ドメイン NetBIOS 名：ドメインの NetBIOS 名を指定します。
  • Domain FQDN (DNS 名)：ドメインの FQDN (DNS名)を指定します。
  • DNS インターフェイスの登録：ドメインを加えると、ここで指定した NIC (Network Interface Card) は DNS Server に登録されます。Synology NAS のホスト名にアンダースコア（_）が含まれていると、DNS にはアンダースコア（_）は使用できないため、登録できません。
  • ユーザー/グループ リストを更新：Synology NAS がドメイン ユーザー／グループ リストを自動更新する間隔を指定します。これは、毎日、毎週、毎月のいずれかで設定できます。また、[ドメイン ユーザー] タブを選択し、[ドメインデータを更新] をクリックすると、ドメイン ユーザー／グループ リストを手動で更新できます。自動更新はシステムのハイバネーションに影響しますのでご注意ください。

読み取り専用ドメイン コントローラに加入する

Synology NAS を読み取り専用ドメイン コントローラ (RODC) に加入するには、DC IP/FQDN 欄で読み書き可能ドメイン コントローラ (RWDC) の「コンピュータのフルネーム」(FQDN) を指定することができます。フルコンピュータ名は次のフォーマットに従う必要があります。コンピューター名.ドメイン、例：「ad01.synology.com」。

注：

  • ドメイン ユーザー名に % や $ が含まれていると、home フォルダにアクセスできない場合があります。その場合は、ドメイン管理者に新しいユーザー名を要請してください。
  • ドメイングループ「Domain Admins」と「Enterprise Admins」は、自動的に「administrators」という名前のローカルグループに追加されます。要するに、これらのグループのドメイン ユーザーは、CIFS/FTP/AFP/WebDAV、その他の DSM アプリケーションなどのファイル サービスへのアクセスをはじめ、Synology NAS についての管理権を有することになります。
  • また、Synology NAS の共有フォルダへアクセスする権限をドメインユーザーに設定することもできます。詳しくは、こちらを参照してください。

ドメイン オプション

このセクションには、次のようなドメインの追加オプションがあります。

  • DiskStation はドメイン ユーザーがログインすると、毎回 NTP サーバーと同期を取る：このオプションを有効にすると、Synology NAS と NTP サーバー間を強制的に同期させます。
  • NT4 対応モードでユーザー/グループ リストを取得：NT4 RPC モードでシステムにユーザー／グループ リストを取得させます。このオプションは、デフォルトの設定であるドメイン ユーザー／グループ リストを取得できないときに有効にしてください。
  • サーバーの署名を有効にする：クライアントでクライアントの署名を有効にしてある場合は、サーバーの署名を Synology NAS で有効にして、ユーザーがドメインサービスを正常に使用できるようにする必要があります。
  • ネスト構造のグループレベルを列挙する：ネスト構造の数と番号を付けられるドメイン グループのメンバーを指定します。
  • ドメイン管理者：管理者権限を与える先のユーザーを最大で10グループまで指定できます。管理者権限を持つユーザーには、Synology NAS およびそれに保存されたファイルを管理する権限があります。
  • 統合 Windows 認証を有効にする：このオプションを有効化すると、ドメイン アカウントからコンピュータにすでにサインインしているユーザーが、ユーザー名とパスワードの入力なしでインターネット ブラウザ経由で DSM にサインインできるようになります。

統合 Windows 認証の使用

  • クライアント コンピュータは Windows のオペレーティング システムを実行している必要があります。また、バージョンは Windows 7 以上でなければなりません。
  • クライアント コンピュータは Synology NAS として同じドメインに参加する必要があります。
  • Windows の [コントロール パネル] > [インターネット オプション] > [詳細設定] の順に進み、[セキュリティ] セクションの下の [統合 Windows 認証を有効にする] のチェックボックスにチェックが入っていることを確かめます。
  • 統合 Windows 認証は、すべてのブラウザで動作します。Firefox については、さらに数ステップがセットアップに必要です。
    • Firefox ブラウザを開き、URL バーに about:config を入力します。
    • network.automatic-ntlm-auth.trusted-uris を検索します。
    • [値] 欄をダブルクリックし、ドメイン の IP アドレスを入力します。
  • Synology NAS を Synology Directory Server で作成されたドメインに参加させる場合は、統合 Windows 認証を使用するために追加の構成が必要です。
    1. DSM で [コントロール パネル] > [ファイルサービス] > [SMB/AFP/NFS] の順に選択します。
    2. [SMB] セクション下で、[詳細設定] をクリックします。
    3. [最小 SMB プロトコル] ドロップダウン メニューから [SMB1] を選択し、[保存] をクリックします。

注：

  • 統合 Windows 認証によってサインインすることは DSM 6.2.2 以降でのみ利用可能です。

ドメイン ステータス チェック

[ドメイン ステータス チェック] をクリックすると、お使いの DSM および加わったドメイン サーバー間の接続ステータスをチェックできます。トラブルシューティングについての詳細は、次のチェック項目を参照してください。

  • Synology NAS およびドメイン コントローラ間で適切な接続が行われているかを確認する：Synology NAS は、ドメイン コントローラとの通信に失敗しました。
    • ポート 389、139、445 がドメイン コントローラで適切に動作していることを確認してください。
    • NTDS サービスがドメイン コントローラで適切に動作していることを確認してください。
  • 競合したホスト名がサブネットに存在しているかを確認する：Synology NAS には、サブネットに既に存在するホスト名があります。
    • お使いの Synology NAS が複数のネットワーク インターフェイスを持つサブネットに接続されている場合は、DSM が適切に動作します。
    • お使いの Synology NAS が、単一のネットワーク インターフェイスでサブネットに接続されている場合、Synology NAS のホスト名を変更し、再びそのドメインに参加してください。
  • Synology NAS および Kerberos サーバー間で適切な接続が行われているか確認する：Synology NAS は、Kerberos サーバーとの通信に失敗しました。
    • 適切な Kerberos DNS SRV 記録や関連した A 記録が DSN Server に存在していることを確認してください。
    • ポート 88 が、KDC サーバーで利用できることを確認します。
  • Synology NAS および LDAP Server 間で適切な接続が行われているか確認するSynology NAS は、LDAP Server との通信に失敗しました。
    • 適切な LDAP DNS SRV 記録や関連した A 記録が DSN Server に存在していることを確認してください。
    • ポート 389 が、LDAP Server で適切に動作することを確認します。
  • Synology NAS およびドメイン コントローラー間で接続が適切に動作するために、MTU が適切に構成されているかを確認する：1 つ以上のスイッチがパケットの無効な MTU サイズにより、Synology NAS およびドメイン コントローラ間で転送されるパケットが低下しています。
    • Synology NAS および ドメイン コントローラ間のスイッチのインターフェイスの設定を確認してください。
  • Synology NAS およびドメイン間で時間が同期されているかを確認する：Synology NAS の時間は、ドメイン サーバーと同期していません。
    • Synology NAS で NTP サービスをセットアップすることで、ドメイン サーバーと時間を同期することや、ドメイン オプションで [ドメインユーザーがログインしたときに、Synology NAS は NTP サーバー時間と同期する] を有効化することができます。
