全般

[コントロール パネル] > [ネットワーク] > [全般] を選択すると、デフォルトのゲートウェイ、DNS server、プロキシおよび IPv6 トンネリングなどの一般的なネットワーク オプションを設定できます。

Synology NAS デバイスに別名を付ける

Synology NAS デバイスに別名を付けるには、新しいサーバー名を入力した後で [適用] をクリックします。

サーバー名は大文字小文字を区別します。1 ～ 15 文字で指定し、文字、数字、アンダースコアー、マイナス記号などが使用可能です。ただし名前の先頭は文字で開始しなければなりません。

デフォルトのゲートウェイを指定する

デフォルトのゲートウェイの IP アドレスを入力し、[適用] をクリックするだけで設定が保存されます。

DNS Server の IP アドレスを指定する

DNS server の IP アドレスは手動で設定できます。その場合、優先 DNS server の IP アドレスを入力してください。優先 DNS server に不具合が生じたときのバックアップ用として、代替 DNS server の IP アドレスも入力できます。DNS server の IP アドレスを設定できたら、[適用] をクリックして設定を保存してください。

詳細設定：

  • ターゲットの IP アドレスが着信インターフェイスで設計したローカル アドレスとして認識されると、ARP リクエストに返信しますネットワーク プロトコルにおいて、ARP は同じ IP アドレスからのリクエストには応答しません。ARP リクエストに応答し、ARP テーブルを完成させるには、このオプションを有効にしてください。
  • 複数のゲートウェイを有効にするインターフェイスで複数のゲートウェイをサポートさせるには、このオプションを有効にしてください。
  • IP 競合の検出を有効にする：ネットワークに同じ IP アドレスがあるかどうかを自動的に検出するには、このオプションを有効にします。

注：

すべてのインターフェースを自動的に構成した場合、または PPPoE 接続を使用している場合は、DNS server とデフォルトゲートウェイの欄は自動的に設定されますので変更することはできません。

プロキシ サーバー経由で接続

プロキシ サーバー経由で Synology NAS を接続するには、[プロキシ サーバー経由で接続] にチェックマークを付けて、[アドレス] と [ポート] を入力した後、[適用] をクリックします。

注：

  • [詳細設定] をクリックして HTTP プロキシ、HTTPS プロキシ、および認証情報を設定します。
  • デフォルトにより、HTTPS プロキシ設定は HTTP プロキシと同じです。
  • [プロキシ サーバー経由で接続] にチェックマークが付けてあると、他の一部のパッケージと共に [DSM の更新]、[パッケージ センター] も常にプロキシを経由してインターネットに接続します。
  • [ローカル アドレスはプロキシ サーバーを通らない] にチェックマークを付けた場合は、Synology NAS が使用する DNS サーバーがローカル ネットワーク内のデバイスのクエリーを解決できるか確認してください。

IPv6 トンネリングを有効にする

IPv6 が完全に IPv4 にとって代わるまで、IPv6 トンネルが IPv4 サービスにアクセスするためには IPv6 アドレスでホストを有効にする必要があります。.このテクニックは、既存の IPv4 ネットワーク内で IPv6 トラフィックを送信する方法を可能にします。
IPv6 の設定の仕方がわからない、または接続に問題が発生した場合は、インターネットのサービスプロバイダにご相談ください。

IPv6 トンネルを有効にする：

  1. [トンネルを有効にする] にチェックマークを付けて、トンネル サポートを有効にします。
  2. [サーバー アドレス] にホスト名またはトンネル サービス プロバイダーの IP アドレスを入力します(例：anonymous.freenet6.net)。
  3. 必要に応じて、[匿名で接続] または [既存のアカウントで接続] を選択します。

注：

外部 IPv6 アドレスは、サービス プロバイダーによって自動的に更新されます。ただし、Synology NAS デバイスは古い IPv6 アドレスをすぐには削除しません。
