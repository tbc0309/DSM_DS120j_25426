ターゲット

モデルによっては、Synology NAS は LUN をマップできる複数のターゲットの作成をサポートします。

注： サポートされるターゲットの数は、Synology NAS のモデルの数によって異なります。

ターゲットの作成方法：

  1. [作成] ボタンをクリックします。
  2. 名前、IQN などを含むターゲットの情報と、CHAP 認証を有効にするかどうかを入力します。
  3. [次へ] をクリックします。
  4. iSCSI LUN マッピングを設定します。 [新しい iSCSI LUN の作成] または [既存の iSCSI LUN をマップ] を選択します。
  5. [次へ] をクリックします。
  6. 編集を続行する場合は [戻る]、設定を確定する場合は [適用] をクリックします。

用語集

  • IQN: IQN (iSCSI Qualified Name) は、ターゲットに対する固有の名前です。 別のサーバーのターゲット IQN と衝突しない名前にします。 IQN は、最大 128 文字で、文字（大文字小文字を区別する）、数字、コロン、ダッシュ、ドットが使用できます。 推奨される名前の形式は、iqn.[日付].[ドメイン]:[デバイス識別子] です。 例： iqn.2000-04.com.synology:storage.diskarray.1
  • 相互 CHAP 認証： 相互 CHAP では、クライアントとサーバーが互いに認証し合います。 クライアントの認証に使用する CHAP パスワードは、あらゆるサーバーを認証するために構成されなければなりません。またサーバーの認証に使用するあらゆる CHAP パスワードは、クライアントを認証するために構成されなければなりません。

    • 注：NAS の CHAP 認証が有効化されている場合、iSCSI 開始プログラムの検索プロセスは失敗することがあります。 検索を最初に完了して、その後 CHAP 認証を有効化してからターゲットに接続してください。 あるいは、IP と IQN でターゲットに接続することができます。

ターゲットの削除方法：

ターゲットおよびマップされた LUN を削除することができます。 iSCSI LUN を削除すると、iSCSI LUN 上のデータもすべて削除されます。

  1. 削除するターゲットを選択します。
  2. [削除] をクリックします。
  3. マップした LUN を削除する場合は、[マップした LUN も削除する] を選択して、削除する対象を選択します。
  4. [OK] をクリックします。

ターゲットの編集方法：

  1. 編集するターゲットを選択します。
  2. [操作]ドロップダウンメニューから[設定]を選択します。
  3. 希望する内容に設定を編集します。 [OK] をクリックして、変更内容を保存します。

用語集

  • CRCチェックサム： CRC チェックサムを有効にすると、データ転送中にエラーが発生したかどうか確認できます。
  • マルチセッション： 1つ以上の開始プログラムによる複数のセッションから同時に同じターゲットへの接続を可能にします。 ファイルの破損や予期しないデータ損失から守るためには、VMware Virtual Machine File System (VMFS) や Oracle Cluster File System (OCFS) などの共有ファイルシステムまたはクラスタシステムを採用することを強く推奨します。
  • マスキング： マスキングすると、ターゲットへ特定の iSCSI 開始プログラムの IQN から排他的にアクセスできるようになります。

ターゲットの有効化または無効化の方法：

  1. 有効/無効にするターゲットを選択します。
  2. [操作] ドロップダウンメニューから [有効化] または [無効化] を選択します。
