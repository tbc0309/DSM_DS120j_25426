성능

성능 탭을 클릭하면 리소스 사용률 데이터가 표시됩니다. 마우스를 올려놓으면 상세한 날짜, 시간 및 차트 정보가 표시됩니다.

개요 탭에서 CPU, GPU, 시스템 메모리, GPU 메모리, 네트워크, 디스크, 볼륨 및 iSCSI 차트가 표시됩니다.

CPU

사용자, 시스템 및 I/O 대기의 전반적인 CPU 사용량을 확인할 수 있습니다. 리소스 모니터를 처음 실행하면 시스템이 리소스 데이터를 수집하고 동시에 UI 페이지를 로드해야 하므로, CPU 부하가 높을 수 있습니다.

  • 사용자는 사용자 응용 프로그램을 실행하기 위해 사용된 CPU를 백분율로 표시합니다.
  • 시스템은 핵심 시스템과 하드웨어/소프트웨어 IRQ에 사용된 CPU를 백분율로 표시합니다.
  • I/O 대기는 I/O 대기 시 사용된 CPU를 백분율로 표시합니다.
  • 이용은 사용자와 시스템 CPU 사용량의 합계입니다.
  • 1, 5 및 15분 내에 실행-대기열 길이(예: 현재 실행 중인 프로세스 수와 실행 대기 중인 수의 합)를 보여주는 부하 평균이 표시됩니다.

용어 설명

  • I/O 대기: I/O 대기는 CPU가 유휴 상태이고 다른 프로그램에서 사용되지 않는 것을 나타냅니다.

    • I/O 대기는 정상적인 동작입니다. CPU 속도는 I/O보다 빠르므로, CPU는 일반적으로 디스크 또는 네트워크의 전송이 완료될 때까지 기다립니다. 하지만 I/O 대기율이 너무 높으면 대부분의 CPU 시간이 I/O 대기에 소모되어 활용률이 저하될 수 있습니다. 더욱 효율적인 디스크 그룹 구성(예: SHR 또는 RAID 5) 또는 네트워크 구성(예: 본딩)을 사용하여 보다 나은 시스템 성능을 구현할 수 있습니다.

GPU

전반적인 GPU 사용률이 표시됩니다. 드롭다운 메뉴에서 GPU 또는 GPU 메모리로 전환할 수 있습니다.

  • 이용은 사용자와 시스템 GPU 사용량의 합계입니다.
  • GPU 메모리에서 사용됨과 여유 공간에 의한 메모리 구성을 확인할 수 있습니다.

참고:

  • GPU를 지원하는 모델에서만 사용할 수 있습니다.

시스템 메모리

물리적 메모리의 전반적인 메모리 사용량과 스왑 공간이 표시됩니다. 하드 디스크에 액세스하지 않고도 데이터를 빠르게 가져올 수 있도록 시스템이 자주 액세스하는 데이터를 캐시에 저장하므로, 메모리 사용량은 높은 상태로 유지됩니다. 전체 메모리가 부족하면 캐시 메모리가 해제됩니다.

스왑 공간 사용량이 많다는 점은 시스템 메모리가 부족하다는 것을 나타내며 시스템 성능도 영향을 받습니다. 맨 위에 있는 드롭다운 메뉴에서 스왑을 선택하여 스왑 인과 스왑 아웃 속도를 확인할 수 있습니다.

네트워크

네트워크 흐름 차트는 전송 또는 수신된 데이터를 초당 KB로 표시합니다. PPPoE가 연결되면 차트에 전송 속도도 표시됩니다.

디스크

차트 맨 위에 있는 드롭다운 메뉴를 클릭하여 전체 디스크 사용률, 전송 속도 또는 IOPS를 확인할 수 있습니다. 아래에 있는 모두 보기를 클릭하고 데이터를 차트에 추가하는 확인란을 선택하여 보다 많은 디스크를 선택할 수 있습니다.

볼륨

이 페이지는 볼륨의 전반적인 전송 상태를 표시합니다. 차트 위에 있는 드롭다운 메뉴를 클릭하여 전송 사용률, 전송 속도 또는 IOPS를 확인합니다. 모두 보기 버튼을 클릭하여 개별 볼륨을 선택하고 차트에 추가합니다.

iSCSI

이 페이지는 현재 실행 중인 iSCSI LUN의 전반적인 상태를 표시합니다. 차트 위에 있는 드롭다운 메뉴를 클릭하여 LUN을 선택하고 해당 상태를 확인합니다. 기본적으로 처리량, IOPS 및 대기 시간 등 세 가지 차트가 표시됩니다. 차트 선택 버튼을 클릭하여 확인할 다른 차트를 선택하면 보다 자세한 내용을 확인할 수 있습니다.
