설정

설정을 클릭하고 다음 옵션을 설정합니다.

  • 드롭다운 메뉴에서 실시간 경과를 분 단위로 선택합니다.
  • 이전 데이터를 보려면 사용 기록 활성화를 선택합니다. 성능 차트 확인 시 오른쪽 위 구석에 있는 드롭다운 메뉴에서 실시간, 1일, 1주, 1개월 또는 1년으로 전환할 수 있습니다.
  • 성능 알람 규칙을 설정하려면 성능 알람 활성화를 선택합니다.

적용을 클릭하여 설정을 저장합니다.

성능 알람 규칙

성능 알람 규칙 생성하기:

  1. 설정 > 설정으로 이동하고 성능 알람 활성화를 선택했는지 확인합니다.
  2. 설정 > 성능 알람 규칙으로 이동합니다.
  3. 생성 버튼을 클릭합니다.
  4. 다음 규칙 설정을 사용할 수 있습니다.
    • 리소스: 규칙을 적용할 리소스를 선택합니다.
    • 유형: 생성할 규칙의 유형을 선택합니다.
    • 수준: 알람 수준을 선택합니다.
    • 보다 큼: 초과할 경우, 시스템이 사용자에게 알릴 규칙 유형의 최대 백분율을 설정합니다.
  5. 이 규칙을 즉시 활성화하려면 활성화를 선택합니다.
  6. 확인을 클릭하여 규칙을 추가합니다.

성능 알람 메커니즘:

성능 알람은 잘못된 알람을 최소화하고 단기간 내에 시스템 리소스가 과도하게 소비되는 것을 방지하기 위해 다음 메커니즘을 통해 작동합니다.

  1. 성능 알람 서비스는 5분마다 모든 규칙을 확인합니다. 모니터링된 모든 값이 사용자가 설정한 임계값 이하이면 어떠한 동작도 트리거되지 않습니다.
  2. 모니터링된 값이 임계값을 초과하면 알람은 즉시 트리거되지 않습니다. 대신 시스템은 30초 간격으로 규칙을 5회 이상 확인합니다.
  3. 간격을 변경한 후 모니터링된 값이 임계값을 5회 연속으로 초과하면 알람이 트리거됩니다. 하지만 모니터링된 값이 5회 확인 중에 한 번이라도 임계값 이하이면 간격은 다시 5분으로 변경됩니다.

따라서 단일 이벤트 한 개의 경우, 알람은 최초 이벤트 감지 후 늦어도 450초에 트리거됩니다.

알람 수준:

성능 알람은 알람 수준에 따라 여러 가지 동작을 트리거합니다.

  • 경고: 시스템 로그에 경고 메시지를 생성하지만 어떠한 시스템 알림도 트리거하지 않습니다.
  • 중대: 시스템 로그에 메시지를 생성할 뿐 아니라 시스템 알림도 트리거합니다.

성능 알람 탭 또는 DSM의 로그 센터에서 성능 알람 로그를 확인할 수 있습니다.
