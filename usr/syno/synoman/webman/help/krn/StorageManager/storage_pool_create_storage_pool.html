새 스토리지 풀 만들기

중요:

  • 시작하기 전에 스토리지 풀을 만들 드라이브에 중요 데이터가 없는지 확인하십시오. 생성 프로세스 중에 모든 기존 데이터가 삭제됩니다.
  • HDD/SSD 페이지로 이동하고 각 드라이브 상태가 초기화됨 또는 초기화되지 않음인지 확인합니다.
  • 모든 문제를 방지하기 위해 Synology 제품 호환 목록에 있는 드라이브를 사용하는 것이 좋습니다. 호환되지 않는 드라이브를 사용하면 시스템 성능이 저하되거나 Synology NAS에서 데이터가 손실될 수 있습니다.

스토리지 풀 만들기:

  1. 저장소 관리자를 엽니다.
  2. 스토리지 풀로 이동하고 생성 버튼을 클릭합니다.
  3. 스토리지 풀 유형 선택하기:
    • 보다 우수한 성능: 이 스토리지 풀 유형은 단일 볼륨만 지원하지만 보다 우수한 성능을 제공합니다.
    • 보다 높은 유연성: 이 스토리지 풀 유형은 여러 볼륨, Synology Hybrid RAID(SHR) 및 RAID Group 기능을 지원합니다.
  4. 스토리지 풀 속성 구성:
    • 스토리지 풀 설명: 이 스토리지 풀을 설명합니다.
    • RAID 유형: RAID 유형에 따라 데이터 보호 수준이 다릅니다. 자세한 정보는 일반을 참조하십시오.
    • RAID당 최소 드라이브 개수: 선택한 RAID 유형에 필요한 최소 드라이브 개수입니다.
    • RAID당 최대 드라이브 개수: RAID당 드라이브 개수가 많을수록 RAID 실패 가능성이 높아집니다. 데이터 보안과 스토리지 풀 용량 간의 균형이 손상되지 않도록 단일 RAID의 드라이브 개수 제한을 조정하거나 RAID Group을 사용하는 것이 좋습니다.
    • RAID Group 기능: 선택한 RAID 유형이 RAID Group 기능을 지원하는 지 여부입니다. 단일 RAID에서는 드라이브 개수가 제한되지만 RAID Group 기능을 사용하면 여러 RAID로 저장소 블록을 구성할 수 있으므로, 용량과 드라이브 중복이 향상됩니다. RAID Group은 RAID 5, RAID 6 및 RAID F1에서 지원됩니다.
  5. 마법사의 나머지 단계를 따라 스토리지 풀 생성을 완료합니다.

참고:

  • 특정 모델에서만 SHR(Synology Hybrid RAID)을 사용할 수 있습니다. 자세한 내용은 Synology 웹사이트를 참조하십시오.
  • 특정 모델에서만 RAID당 최대 드라이브 개수와 RAID Group 기능을 사용할 수 있습니다. 자세한 내용은 Synology 웹사이트를 참조하십시오.
  • 선택한 RAID 유형이 RAID Group 기능을 지원하는 경우에만 RAID당 최대 드라이브 개수를 변경할 수 있습니다.
  • 스토리지 풀의 RAID 유형이 RAID 5, RAID 6, RAID F1, SHR(드라이브 세 개 이상) 또는 SHR-2인 경우, 시스템은 스토리지 풀에 RAID 다시 동기화를 수행하여 액세스 속도와 데이터 오류 복원을 향상시킵니다.
