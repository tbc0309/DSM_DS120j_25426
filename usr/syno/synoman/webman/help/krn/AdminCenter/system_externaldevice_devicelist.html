외부 드라이브

외부 드라이브를 시스템에 연결하면 usbshare[number](USB 드라이브, 해당되는 경우) 또는 satashare[number](eSATA 드라이브, 해당되는 경우)라는 시스템이 만든 공유 폴더를 통해 드라이브 용량을 공유할 수 있습니다. 공유 폴더는 외부 드라이브를 시스템에서 꺼내면 자동으로 제거됩니다.

Synology NAS은(는) 다음 형식을 인식합니다. Btrfs, ext3, ext4, FAT, FAT32, exFAT, HFS, HFS Plus 및 NTFS. 인식되지 않은 외부 드라이브는 시스템에서 사용하기 전에 먼저 포맷되어야 합니다.

참고:

  1. 일부 모델은 읽기 전용으로 HFS/HFS Plus를 지원합니다.
  2. 저널은 HFS/HFS Plus에서 지원되지 않습니다.
  3. Synology NAS에서 exFAT를 지원할 수 있도록 패키지 센터에서 exFAT Access를 설치해야 합니다. exFAT 형식을 지원하는 Synology 제품 모델 목록은 이 페이지를 참조하십시오.
  4. 연결을 끊기 전에 외부 드라이브를 꺼냈는지 확인합니다.
  5. 특정 모델에서만 Btrfs를 사용할 수 있습니다. 사용 중인 모델에서 Btrfs를 지원하는지 확인하려면 이 문서를 참조하십시오.

디스크 이미지에 액세스하기:

Synology NAS는 File Station을 사용하여 외부 장치의 디스크 이미지 콘텐츠에 액세스하도록 가상 드라이브를 탑재할 수 있습니다. 가상 드라이브를 탑재하는 방법은 이 문서를 참조하십시오.

참고:

  • 지원되는 디스크 이미지 형식: ISO 9960 및 UDF 파일 시스템을 사용하는 ".iso" 파일입니다.

드라이브 포맷하기:

포맷을 클릭하고 마법사를 따릅니다. 외부 드라이브를 Synology NAS에서 ext4 및 FAT32로 포맷할 수 있습니다.

참고:

  1. 외부 드라이브를 포맷하기 전에 데이터를 백업해야 합니다. 이 절차 중에 모든 데이터가 삭제되며 다시 복원될 수 없습니다.
  2. 외부 드라이브를 ext3 또는 ext4 형식으로 포맷하려면 드라이브 용량이 1GB 이상이어야 합니다.
  3. 드라이브가 ext3 또는 ext4 형식으로 포맷되면 Synology NAS에서만 인식됩니다. PC 및 Mac에서 외부 드라이브의 데이터를 직접 읽으려면 FAT32 형식을 선택합니다.

S.M.A.R.T. 테스트 수행하기:

드라이브 S.M.A.R.T. 테스트가 하드 드라이브 상태를 검사한 후 보고하여 곧 발생할 수 있는 드라이브 오류에 대한 조기 경고를 보냅니다. 오류가 발견되면 드라이브를 바꾸는 것이 좋습니다.
S.M.A.R.T. 테스트를 수행하려면 S.M.A.R.T. 테스트 버튼을 클릭하고 마법사를 따르면 됩니다.
테스트 결과를 확인하려면 S.M.A.R.T. 정보를 클릭합니다.

참고:

  • 테스트 시간은 시스템 환경에 따라 달라질 수 있습니다.

제한 사항

다음 상황에서는 드라이브를 포맷할 수 없습니다.

  • 외부 드라이브가 초기화 중인 경우(즉, 외부 장치가 아직 연결되지 않은 경우).
  • 외부 드라이브 용량이 1GB 이하인 경우(ext3 또는 ext4 형식).
  • 외부 드라이브가 포맷 중인 경우.
  • 시스템이 파일을 외부 드라이브에 백업 중인 경우.
  • 외부 드라이브가 손상된 경우.

문제 해결

  1. 외부 드라이브 포맷이 실패했습니다. 그 이유는 무엇입니까?
    • 외부 드라이브가 올바르게 연결되어 있는지 확인합니다.
    • 외부 드라이브에 쓰기 금지가 설정되어 있는지 확인합니다. 그럴 경우 포맷하기 전에 쓰기 금지를 해제합니다.
  2. 외부 드라이브의 파일 시스템이 지원되지 않음으로 표시되거나 시스템이 외부 드라이브의 usbshare[number] 또는 satashare[number] 공유 폴더를 만들지 않습니다. 어떻게 해야 합니까?
    • 현재 시스템에서는 Btrfs, ext3, ext4, FAT, FAT32, HFS, HFS Plus 및 NTFS 형식을 인식할 수 있습니다. 위의 형식으로 포맷되지 않은 외부 드라이브는 인식되지 않으며 따라서 상태가 지원되지 않음으로 표시됩니다.
  3. usbshare[number] 또는 satashare[number] 공유 폴더에 액세스할 수 없습니다. 어떻게 해야 합니까?
    • 이 문제는 외부 드라이브의 파일 시스템 문제로 인해 발생할 수 있습니다. 이를 해결하려면 외부 드라이브를 PC에 연결하고 파일 시스템 검사를 수행하십시오.
  4. ext3 또는 ext4 형식의 외부 드라이브를 PC 및 Mac에 연결하여 직접 액세스할 수 있습니까?
    • 아니요. ext3 또는 ext4는 Synology NAS에서만 인식됩니다. PC 및 Mac에서 외부 드라이브의 데이터를 직접 읽으려면 FAT32 형식을 선택합니다.
  5. 외부 드라이브에 "lost+found"라는 폴더가 있는 이유는 무엇입니까?
    • 이 폴더는 ext3/ext4 파일 시스템을 사용할 때 자동으로 생성되는 시스템 기본 폴더입니다. 외부 드라이브에 이 폴더가 표시되는 것은 정상입니다. 관리자가 이 폴더에 있는 콘텐츠에 액세스하여 관리할 수 있는 권한이 있더라도 폴더를 변경하지 않는 것이 좋습니다.

참고:

  • 특정 모델에서만 Btrfs를 사용할 수 있습니다. 사용 중인 모델에서 Btrfs를 지원하는지 확인하려면 이 문서를 참조하십시오.

모든 외부 장치에 대한 기본 권한 설정하기:

설정 > 기본 권한을 클릭하고 외부 장치 및 해당 콘텐츠에 액세스, 보기 또는 수정할 수 있는 사용자나 그룹을 지정하여 기본 권한을 설정할 수 있습니다. 외부 장치를 Synology NAS에 연결하면 사용자가 지정한 권한이 자동으로 이 외부 장치에 할당됩니다.

권한 창에서 다음을 수행하여 기본 권한을 설정할 수 있습니다.

  1. 드롭다운 메뉴에서 다음 중 하나를 선택합니다.
    • 시스템 내부 사용자: 익명 FTP/WebDAV 사용자와 같은 기본 시스템 사용자 권한을 할당합니다.
    • 로컬 사용자: 로컬 사용자 권한을 할당합니다(guest 포함).
    • 로컬 그룹: 로컬 그룹 권한을 할당합니다.
  2. 각 사용자 또는 그룹에 대한 외부 장치의 액세스 권한을 사용자 지정하려면 확인란을 선택 또는 선택 취소합니다.
    • 읽기/쓰기: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 액세스하여 변경할 수 있습니다.
    • 읽기 전용: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 액세스할 수 있습니다.
    • 액세스 금지: 사용자 또는 그룹이 외부 장치에 있는 파일과 폴더에 액세스할 수 없습니다.
