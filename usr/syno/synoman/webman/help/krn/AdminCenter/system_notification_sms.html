SMS

SMS 알림을 활성화하여 Synology NAS의 상태가 변경되거나 연결 손실, 팬 동작 변경, 정전, 저장소 볼륨 공간 부족 또는 백업 및 복원 실패 등과 같은 오류가 시스템에 발생한 경우에 SMS를 통해 알림 메시지를 발송할 수 있습니다.

참고:

  • EDS14와 같이 팬이 없는 모델에서는 팬 동작 변경 이벤트를 사용할 수 없습니다. 자세한 내용은 Synology 웹사이트를 참조하십시오.

SMS 알림 활성화하기:

SMS 알림 활성화를 선택하고 다음 설정을 완료합니다.

  1. SMS 서비스 공급자 드롭다운 목록에서 SMS 공급자를 선택합니다. 기본 서비스 공급자는 "clickatell", "clickatell-2017" 및 "SendinBlue-v3"입니다.
  2. SMS 서비스 공급자 추가를 클릭하여 새 SMS 서비스 공급자 구성을 생성하거나 삭제를 클릭하여 선택한 SMS 공급자 구성을 삭제합니다.
  3. SMS 서비스 공급자가 인증을 위해 사용자 이름과 패스워드를 사용하는 경우, 매개 변수의 해당 범주를 선택하십시오. 사용자 이름 필드에 SMS 서비스 계정 사용자 이름을 입력합니다. 패스워드 및 패스워드 확인 필드에 SMS 서비스 계정 패스워드를 입력하고 확인합니다.
  4. SMS 서비스 공급자가 인증을 위해 API 키를 사용하는 경우, 해당 매개 변수의 API 키 범주를 선택하십시오. API 키 필드에 SMS 서비스 공급자가 제공한 API 키를 입력합니다.
  5. SMS 서비스 공급자가 보낸 사람 이름을 수정할 수 있는 매개 변수를 제공하는 경우, 해당 매개 변수의 보낸 사람 범주를 선택하십시오. 보낸 사람 필드에 보낸 사람 이름을 입력합니다. 이름은 SMS 서비스 공급자의 규칙을 준수해야 합니다.
  6. 기본 전화 번호 또는 보조 전화 번호를 입력합니다 두 번호를 모두 입력하면 SMS 알림 메시지가 두 번호 모두에 전송됩니다.
  7. 너무 많은 메시지 수신을 원치 않으면 SMS 간격 제한 활성화를 선택하고 지정한 간격마다 메시지가 1개씩 전송되도록 알림 메시지 수를 제한합니다.
  8. 테스트 SMS 메시지 전송을 클릭하여 지정한 전화 번호로 테스트 SMS 메시지를 발송합니다. 메시지가 수신되지 않으면 구성이 잘못된 것입니다.
  9. 적용을 클릭하여 구성을 저장하거나 재설정을 클릭하여 설정을 취소합니다.

참고:

  • SMS 알림 언어는 알림 언어에서 설정됩니다. SMS 알림 언어를 변경하려면 메인 메뉴 > 제어판 > 지역 옵션으로 이동하고 언어 탭을 클릭한 후 알림 언어 섹션의
    드롭다운 메뉴에서 원하는 언어를 선택합니다.
  • 기본 SMS 서비스 공급자인 "clickatell" 및 "clickatell-2017"을 제외한 모든 SMS 서비스 공급자는 영어로 알림 메시지를 발송합니다.

제한 사항:

  • 사용자 이름: 대소문자를 구분하며 1~32 자의 Unicode 문자여야 합니다.
  • 패스워드: 대소문자를 구분하며 문자, 숫자, 기호 및 공백을 포함하여 0~128 자의 표시 가능한 문자여야 합니다.
  • API 키: 대소문자를 구분하며 1~256 자의 Unicode 문자여야 합니다.
  • 보낸 사람: 대소문자를 구분하며 1~64 자의 Unicode 문자여야 합니다.
  • SMS 간격 제한은 1~10000 분 사이여야 합니다.

SMS 공급자 구성 생성하기:

  1. SMS 서비스 공급자 추가를 클릭합니다.
  2. 공급자 이름 필드에 SMS 서비스 공급자 이름을 입력합니다.
  3. SMS URL 필드에서 SMS URL을 입력합니다. SMS 서비스 공급자로부터 SMS URL 을 제공 받을 수 있습니다.
  4. HTTP 메서드에서 GET 또는 POST를 선택합니다. SMS 서비스 공급자의 API 문서에서 이 정보를 제공 받을 수 있습니다.
  5. 다음을 클릭합니다.
  6. HTTP 요청 머리글 편집에서 SMS 서비스 공급자의 API 문서에 기재된 매개 변수를 추가해야 합니다.
  7. 다음을 클릭합니다.
  8. HTTP 메서드에서 POST를 선택한 경우, HTTP 요청 본문 편집에서 SMS 서비스 공급자의 API 문서에 기재된 매개 변수를 추가해야 합니다.
  9. 다음을 클릭합니다.
  10. 시스템은 여러 매개 변수에서 SMS URL, HTTP 요청 머리글 및 HTTP 요청 본문을 분석합니다. 해당 범주를 선택하여 각 매개 변수를 정의합니다. 기타를 제외한 각 매개 변수 범주는 한 번만 사용할 수 있습니다.

제한 사항:

  1. 공급자 이름: 대소문자를 구분하며 1~32 자의 Unicode 문자여야 합니다.
  2. SMS URL: 대소문자를 구분하며 1~500 자의 Unicode 문자여야 합니다.이 SMS URL은 설정 용도로만 사용됩니다. 설정을 완료한 후에도 SMS 메시지가 발송되지 않습니다. SMS 서비스 공급자가 이 URL 형식을 제공합니다.예로 Clickatell을 사용하는 경우, 사용할 수 있는 URL은 https://api.clickatell.com/http/sendmsg?user=TestUser&password=TestPassword&api_id=3148203&to=886123456789&text=Hello+world입니다.
  3. 매개 변수: 대소문자를 구분하며 1~32 자의 Unicode 문자여야 합니다.
  4. 값: 대소문자를 구분하며 1~256 자의 Unicode 문자여야 합니다.
  5. 매개 변수를 최대 10개까지 추가할 수 있습니다.
  6. "메시지 내용" 매개 변수에서 "Hello"와 "world" 사이의 공백은 서비스 공급자가 정의한 특정 문자로 대체된다는 점에 유의하십시오.
    예로 Clickatell을 사용하는 경우, "메시지 내용" 매개 변수 값은 "Hello+world"이고 문자 "+"는 Clickatell에서 특수 문자로 사용됩니다.
