도메인

Synology NAS은(는) Windows 도메인에 가입할 수 있습니다. 도메인에 가입하면 도메인 사용자는 도메인 계정과 패스워드를 사용하여 Synology NAS에 로그인할 수 있습니다. 이를 통해 또 다른 사용자 이름과 패스워드를 기억할 필요 없이 파일에 액세스하고 DSM 응용 프로그램을 사용할 수 있습니다.

도메인에 가입하기:

  1. 제어판 > 도메인/LDAP > 도메인으로 이동합니다.
  2. 도메인에 가입 확인란을 선택합니다.
  3. 도메인 이름을 입력하고 적절한 필드에 DNS 서버를 지정합니다.
  4. 필요한 경우, 고급 도메인 옵션 확인란을 선택합니다. (자세한 내용은 아래를 참조하십시오.)
  5. 적용을 클릭합니다.
  6. 팝업 창에 도메인 관리자의 사용자 이름과 패스워드를 입력합니다.
  7. 확인을 클릭합니다.

DNS Server

도메인의 Synology NAS을(를) 검색할 수 있도록 DNS 서버를 지정하는 것이 좋습니다. 하지만 도메인이 DNS 서버에 등록되어 있지 않으면 도메인 가입 절차가 실패할 수 있습니다.

도메인 서버 유형

이 필드는 Synology NAS이(가) 도메인에 가입 시 도메인 유형을 표시합니다. 도메인 유형은 "AD 도메인"입니다.

관리 모드

이 옵션은 도메인 사용자 및 그룹 권한 관리 방법을 지정합니다.

  • 신뢰할 수 있는 도메인: 가입된 도메인 및 신뢰할 수 있는 도메인의 사용자와 그룹을 관리할 수 있습니다. 사용자와 그룹의 목록을 도메인 기준으로 필터링할 수 있습니다.
  • OU가 있는 단일 도메인: 이 모드에서는 가입된 도메인의 사용자 및 그룹만 표시됩니다. 이 모드를 사용하면 사용자와 그룹의 목록을 조직 구성 단위(OU) 기준으로 필터링할 수 있습니다.

고급 도메인 옵션

대부분의 경우, 고급 도메인 옵션을 입력할 필요가 없습니다. 고급 도메인 옵션은 특정 도메인 환경에서만 필요합니다.

  • DC IP/FQDN: DC(도메인 컨트롤러) IP 주소 또는 FQDN을 지정하면 Synology NAS이(가) 통신을 시도합니다. IP 주소 또는 FQDN이 두 개 이상 있으면 이들 각 사이에 쉼표(,)를 삽입합니다. 원하는 경우, 마지막 DC IP 주소 또는 FQDN 다음에 별표(*)를 추가합니다. 그러면 지정된 모든 DC가 실패할 경우 Synology NAS이(가) 다른 DC와 통신을 시도합니다. 별표와 마지막 IP 주소 또는 FQDN 사이에 쉼표를 추가해야 합니다.
  • 도메인 NetBIOS 이름: 도메인의 NetBIOS 이름을 지정합니다.
  • 도메인 FQDN(DNS 이름): 도메인의 FQDN(DNS 이름)을 지정합니다.
  • DNS 인터페이스 등록: 도메인에 가입한 경우, 여기에서 지정된 NIC(네트워크 인터페이스 카드)가 DNS 서버로 등록됩니다. Synology NAS 호스트 이름에 밑줄(_)이 있는 경우 DNS에 밑줄(_)을 사용할 수 없으므로 등록이 실패합니다.
  • 사용자/그룹 목록 업데이트: Synology NAS에서 얼마나 자주 도메인 사용자/그룹 목록을 자동으로 업데이트할 지를 지정합니다. 업데이트를 수행할 시간을 매일, 매주 또는 매월로 사용자 지정할 수 있습니다. 또한 도메인 사용자 탭으로 이동하고 도메인 데이터 업데이트를 클릭하여 도메인 사용자/그룹 목록을 수동으로 업데이트할 수 있습니다. 자동 업데이트는 시스템 최대 절전 모드에 영향을 준다는 점에 유의하십시오.

읽기 전용 도메인 컨트롤러에 가입

Synology NAS을(를) 읽기 전용 도메인 컨트롤러(RODC)에 가입하려면 읽기-쓰기 도메인 컨트롤러(RWDC)의 "전체 컴퓨터 이름"(FQDN)을 DC IP/FQDN 필드에 지정해야 합니다. 전체 컴퓨터 이름은 다음 형식을 따라야 합니다. ComputerName.Domain(예: "ad01.synology.com")

참고:

  • 도메인 사용자 이름에 % 또는 $가 있으면 홈 폴더에 액세스하지 못할 수 없습니다. 도메인 관리자에게 새 사용자 이름을 문의하십시오.
  • Domain Admins 및 Enterprise Admins 도메인 그룹이 로컬 administrators 그룹에 자동으로 추가됩니다. 따라서, 이들 그룹의 도메인 사용자는 CIFS/FTP/AFP/WebDAV 및 기타 DSM 응용 프로그램과 같은 파일 서비스에 액세스를 비롯한 Synology NAS에 대한 관리 권한을 갖습니다.
  • 또한 Synology NAS에 있는 공유 폴더에 대한 도메인 사용자의 액세스 권한을 구성할 수 있습니다. 자세한 내용은 여기를 참조하십시오.

도메인 옵션

이 섹션에서는 아래와 같은 추가 도메인 옵션을 포함합니다.

  • 도메인 사용자가 로그인할 때마다 DiskStation이 NTP 서버와 동기화합니다: 이 옵션을 활성화하면 Synology NAS 및 NTP 서버 간의 시간이 강제로 동기화됩니다.
  • NT4 호환 모드가 있는 사용자/그룹 목록 가져오기: 이 옵션을 활성화하면 시스템에서 NT4 RPC 모드를 사용하는 사용자/그룹 목록을 가져올 수 있습니다. 기본 설정을 사용하여 특정 도메인 사용자/그룹 목록을 가져올 수 없는 경우에 이 옵션을 활성화할 수 있습니다.
  • 서버 등록 활성화: 클라이언트가 클라이언트 등록을 활성화한 경우 Synology NAS에서도 서버 등록이 활성화되어야 사용자가 정상적으로 도메인 서비스를 사용할 수 있습니다.
  • 중첩된 그룹 수준 열거: 열거할 수 있는 중첩된 도메인 그룹의 구성원 수를 지정합니다.
  • 도메인 관리자: 관리자 권한을 부여할 사용자 그룹을 최대 10개까지 지정합니다. 관리자 권한을 가진 사용자는 Synology NAS 및 저장된 파일에 대한 전체 제어 권한을 가집니다.
  • 통합된 Windows 인증 활성화: 이 옵션을 활성화하면 이미 자신의 도메인 계정을 통해 자신의 컴퓨터에 로그인한 사용자는 사용자 이름과 패스워드를 입력하지 않고 인터넷 브라우저를 통해 DSM에 로그인할 수 있습니다.

통합된 Windows 인증 사용

  • 클라이언트 컴퓨터는 Windows 운영 체제를 실행하고 있어야 하고 버전은 Windows 7 이상이어야 합니다.
  • 클라이언트 컴퓨터는 Synology NAS과(와) 같은 도메인에 가입되어야 합니다.
  • Windows의 제어판 > 인터넷 옵션 > 고급으로 이동하고 보안 섹션에서 통합된 Windows 인증 활성화 확인란이 선택되어 있는지 확인합니다.
  • 통합된 Windows 인증은 모든 브라우저에서 작동합니다. Firefox의 경우, 설정하려면 추가 단계를 진행해야 합니다.
    • Firefox 브라우저를 열고 URL 표시줄에 about:config를 입력합니다.
    • network.automatic-ntlm-auth.trusted-uris를 검색합니다.
    • value 필드를 더블 클릭하고 도메인의 IP 주소를 입력합니다.
  • Synology NAS을(를) Synology Directory Server에서 만든 도메인에 가입할 경우 통합 Windows 인증을 사용하려면 추가 구성이 필요합니다.
    1. DSM 제어판 > 파일 서비스 > SMB/AFP/NFS로 이동합니다.
    2. SMB 섹션에서 고급 설정을 클릭합니다.
    3. 최소 SMB 프로토콜 드롭다운 메뉴에서 SMB1을 선택하고 저장을 클릭합니다.

참고:

  • DSM 6.2.2 이상에서만 통합된 Windows 인증을 통한 로그인을 사용할 수 있습니다.

도메인 상태 확인

도메인 상태 확인을 클릭하면 가입한 도메인 서버와 DSM 간의 연결 상태를 확인할 수 있습니다. 문제 해결에 대한 자세한 내용은 다음 검사 항목을 참조하십시오.

  • Synology NAS와 도메인 컨트롤러 간의 연결이 제대로 작동하는지확인: Synology NAS가 도메인 컨트롤러와의 통신에실패했습니다.
    • 포트 389, 139 및 445가 도메인 컨트롤러에서 제대로 작동하는지확인합니다.
    • NTDS 서비스가 도메인 컨트롤러에서 올바르게 작동하는지확인합니다.
  • 서브넷에 충돌하는 호스트 이름이 있는지 확인: 이미 서브넷에 있는 호스트 이름이 Synology NAS에 있습니다.
    • Synology NAS가 여러 네트워크 인터페이스를 통해 서브넷에 연결되면 DSM이 올바르게 작동합니다.
    • Synology NAS가 단일 네트워크 인터페이스를 통해 서브넷에 연결된 경우 Synology NAS의 호스트 이름을 변경하고 도메인에 다시 가입하십시오.
  • Synology NAS와 Kerberos 서버 간의 연결이 제대로 작동하는지 확인: Synology NAS에서 Kerberos 서버와의 통신에 실패했습니다.
    • 올바른 Kerberos DNS SRV 레코드와 관련 A 레코드가 DNS 서버에 있는지 확인합니다.
    • KDC 서버에서 포트 88을 사용할 수 있는지 확인합니다.
  • Synology NAS와 LDAP 서버 간의 연결이 제대로 작동하는지 확인: Synology NAS가 LDAP 서버와의 통신에 실패했습니다.
    • 올바른 LDAP DNS SRV 레코드와 관련 A 레코드가 DNS 서버에 있는지 확인합니다.
    • LDAP 서버에서 포트 389가 올바르게 작동하는지 확인합니다.
  • Synology NAS 및 도메인 컨트롤러 사이에 연결이 제대로 작동할 수 있도록 MTU가 올바르게 구성되었는지 확인: 패킷의 잘못된 MTU 크기로 인해 스위치 한 개 이상이 Synology NAS와도메인 컨트롤러 간에전송된 패킷을 삭제했습니다.
    • Synology NAS와 도메인 컨트롤러 간의 스위치 인터페이스 설정을확인하십시오.
  • Synology NAS와 도메인 간에 시간이 동기화되는지 확인: Synology NAS의 시간은 도메인 서버와 동기화되지 않습니다.
    • 시간이 도메인 서버와 동기화되도록 Synology NAS에서 NTP 서비스를 설정하거나 도메인 옵션에서 도메인 사용자가 로그인할 때마다 Synology NAS가 NTP 서버와 동기화합니다 옵션을 활성화할 수 있습니다.
