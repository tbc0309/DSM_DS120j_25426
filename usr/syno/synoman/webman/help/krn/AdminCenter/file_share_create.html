공유 폴더 생성

공유 폴더는 Synology NAS에 파일과 폴더를 저장하고 관리할 수 있는 기본 디렉토리입니다. 파일을 저장하기 전에 DSM에 공유 폴더를 최소 1개 이상 생성해야 합니다.

공유 폴더 생성하기:

  1. 제어판 > 공유 폴더로 이동하고 생성을 클릭합니다.
  2. 공유 폴더 이름과 설명을 지정합니다.
  3. 공유 폴더를 생성할 저장소를 선택합니다(저장소가 하나만 있는 경우 이 단계를 건너뜀).
  4. 필요한 경우 다음 옵션을 수정합니다.
    • 이 공유 폴더를 "내 네트워크 환경"에서 숨기기: Windows 파일 탐색기에서 "네트워크" 아래에 해당 공유 폴더가 표시되지 않도록 합니다. 이 옵션은 공유 폴더의 액세스 권한에 영향을 미치지 않습니다.

      참고:

      • "내 네트워크 환경"에서 공유 폴더를 숨겨도 액세스 권한은 영향을 받지 않습니다. 숨겨진 공유 폴더의 액세스 권한은 변하지 않습니다. 공유 폴더에 대한 적절한 액세스 권한을 가진 사용자는 "\\서버 이름\공유 폴더 이름"을 입력하여 계속 공유 폴더에 액세스할 수 있습니다.
    • 권한 없는 사용자의 하위 폴더 및 파일 숨기기: 권한 없는 사용자의 하위 폴더 및 파일 숨기기를 활성화하면 읽기 권한이 없는 사용자가 Windows 파일 공유 프로토콜을 통해 공유 폴더에 액세스하면 공유 폴더 안에 있는 하위 폴더나 파일이 보이지 않게 됩니다. 이 기능은 사용자가 Synology 제품에 연결할 때 자신이 열 수 없는 많은 파일과 하위 폴더를 확인하는 번거로움을 덜어줍니다.

      참고:

      • 이 기능은 Windows 파일 공유 프로토콜을 통해서만 지원됩니다.
      • File Station에서 마우스 오른쪽 버튼을 클릭하여 각 폴더와 파일의 권한 설정을 수정할 수 있습니다. 드롭다운 메뉴에서 속성을 선택하고 권한 탭을 클릭합니다.
      • 권한 없는 사용자의 하위 폴더 및 파일 숨기기를 활성화하면 권한이 없는 사용자의 공유 폴더 자체를 숨기지는 못합니다. 이 공유 폴더 안에 있는 하위 폴더와 파일만 숨겨집니다.
  5. 휴지통 활성화: 공유 폴더에서 파일을 삭제하면 파일은 #recycle 폴더로 이동합니다. 휴지통에 대한 액세스를 administrators 그룹에 속한 사용자로 제한할 수 있습니다.
  6. 공유 폴더 할당량 활성화: 각 공유 폴더의 최대 용량을 지정하려면 공유 폴더 할당량 활성화 확인란을 선택하고 다음 필드에 최대 용량(예: 10 GB)을 입력합니다.

    참고:

    • 저장소가 Btrfs 파일 시스템을 사용하는 경우에만 공유 폴더 할당량 활성화 옵션을 공유 폴더에 사용할 수 있습니다.
    • 모든 사용자의 home 디렉토리를 포함한 저장소 사용량은 “homes”의 할당량을 초과할 수 없습니다.
  7. 고급 데이터 무결성을 위한 데이터 체크섬 활성화: CRC32 체크섬 및 쓰기 시 복사 전략으로 공유 폴더를 보호하기 위해 공유 폴더를 생성하는 동안 고급 데이터 무결성 데이터 체크섬을 활성화할 수 있습니다. CRC32 체크섬은 데이터가 원래 쓰여질 때와 동일하게 남아 있는지 확인하는 데 적용되며, 시스템은 자동으로 RAID 중복을 사용하여 손상된 데이터를 수정합니다. 쓰기 시 복사를 통해 비정상적인 시스템 종료 시 데이터 일관성을 향상시킬 수 있습니다.

    참고:

    • 이 옵션은 Btrfs 파일 시스템을 사용하는 볼륨에만 사용할 수 있습니다.
    • 공유 폴더를 생성하는 동안 각 공유 폴더에 대해 이 옵션을 활성화/비활성화할 수 있습니다. 공유 폴더가 생성되면 이 옵션을 수정할 수 없습니다.
    • 이 옵션을 활성화하면 시스템은 전체 Data Scrubbing을 수행할 수 있습니다. Data Scrubbing에 대해 자세히 알아보기
    • 공유 폴더가 다음 서비스를 사용하는 경우 서비스 품질을 보장하기 위해 데이터 체크섬을 비활성화하는 것이 좋습니다.
      • 데이터베이스 또는 가상 컴퓨터 호스팅
      • Surveillance Station의 비디오 녹화물 저장
      • 작은 임의 쓰기 패턴이 필요한 서비스 실행
  8. 파일 압축 활성화: 생성된 공유 폴더의 데이터는 저장소 공간을 줄이기 위해 Btrfs 파일 시스템을 통해 자동으로 압축합니다. 사용하기 위해 검색하면 데이터 압축이 자동으로 풀립니다.

    참고:

    • Btrfs 볼륨에서 공유 폴더를 생성하거나 고급 데이터 무결성 보호가 활성화된 경우에만 이 옵션을 사용할 수 있습니다.
    • 공유 폴더가 생성되면 이 옵션을 수정할 수 없습니다.
    • 압축 비율은 데이터 유형별로 다릅니다. 텍스트 파일의 경우 압축 비율이 가장 높은 반면 멀티미디어 파일(예: 이미지, 음악 또는 비디오)은 출력을 위해 이미 압축되어 있으므로 크게 압축되지 않을 수 있습니다.
    • 공유 폴더 암호화가 활성화된 경우에는 이 옵션을 활성화할 수 없습니다.
  9. 이 공유 폴더 암호화: 공유 폴더를 암호화하려면 다음을 수행합니다.
    1. 새 암호화 키를 지정하고 확인합니다. 암호화 키에는 쉼표(,) 또는 등호(=)가 포함될 수 없습니다.
    2. 필요하면 키 관리자에 암호화 키 추가 확인란을 선택하여 시스템 시작 시 암호화된 폴더를 자동으로 마운트합니다. 이 옵션을 비활성화하면 다음 번에 Synology NAS 다시 시작 시 암호화된 공유 폴더가 분리됩니다. 분리된 공유 폴더를 보거나 액세스하려면 암호화 키를 가져오거나 입력하여 폴더를 수동으로 마운트해야 합니다.
  10. 확인을 클릭하여 완료합니다.

    11. 참고:

    • VDSM과 다음 패키지 아키텍처가 있는 모델만 공유 폴더의 특정 기능과 옵션을 지원합니다. Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview 및 Grantley. 사용하고 있는 모델이 지원하는 패키지 아키텍처를 확인하려면 이 문서를 참조하십시오.

공유 폴더 명명 규칙:

  • 공유 폴더 이름은 대소문자를 구분하지 않으며 1~32 자여야 합니다. 이름에 다음 문자가 포함될 수 없습니다. ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • 다음 이름은 시스템 용도로 예약되어 있습니다. ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • 공유 폴더 이름의 첫 번째 문자는 마이너스(-) 기호 또는 공백일 수 없으며 마지막 문자는 공백일 수 없습니다.
  • 공유 폴더 설명은 대소문자를 구분하며 최대 64자의 표시 가능한 Unicode 문자로 제한됩니다.

공유 폴더 암호화:

AES 256비트 암호화는 승인되지 않은 모든 액세스 시도를 차단하여 데이터를 보호합니다. 공유 폴더가 암호화된 경우, 암호화 키가 없는 사용자는 Synology NAS에서 하드 드라이브를 제거하고 다른 장치에 마운트한 경우에도 암호화된 데이터에 액세스하거나 수정할 수 없습니다.

참고:

  • 암호화된 공유 폴더 내의 파일 또는 폴더 이름은 영문자 143자 또는 아시아(CJK) 문자 47자를 초과할 수 없습니다. Mac OS를 사용하여 공유 폴더에 액세스하는 경우, 암호화된 공유 폴더 내의 파일 또는 폴더 이름은 영문자 130자 또는 아시아(CJK) 문자 43자를 초과할 수 없습니다.
  • NFS 를 통해 암호화된 공유 폴더에 액세스할 수 없습니다.
  • Synology NAS에 있는 재설정 버튼을 사용하여 패스워드를 기본값으로 복원하면 암호화된 공유 폴더가 분리되고 암호화된 공유 폴더를 자동으로 마운트하는 옵션이 비활성화됩니다.
  • 암호화된 공유 폴더는 높은 권한 모드에서만 지원됩니다(Dockerx64에만 적용). 이 기능을 사용하려면 호스트 DSM의 Docker 패키지로 이동하고 암호화된 공유 폴더를 활성화할 Docker DSM을 선택합니다. 그런 다음 편집을 클릭하고 높은 권한을 사용하여 컨테이너 실행 옆의 확인란을 선택합니다.
