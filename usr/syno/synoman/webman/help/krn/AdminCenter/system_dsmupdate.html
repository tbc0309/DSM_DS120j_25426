DSM 업데이트

Synology는 주기적으로 DSM 무료 업데이트를 제공합니다. 업데이트에는 새로운 기능, 기능 개선 및 성능 강화 등이 포함될 수 있습니다. DSM 업데이트는 부 또는 주 업데이트 형식일 수 있습니다. 업데이트가 부 업데이트인 경우, 시스템은 상태 옆에 메시지를 표시합니다. 주 DSM 버전 출시가 있으면 상태 아래 섹션에 이 출시 버전에 대한 정보와 주 업데이트 버전의 다운로드 링크가 표시됩니다. 일반적으로 부 업데이트에는 심각한 버그 수정 및 시스템 안정성을 위한 보안 패치가 포함되며, 주 업데이트에는 주로 심각한 버그 수정 및 보안 패치 이외에 완전히 새로운 기능과 성능 향상이 포함됩니다.

참고:

  • DSM 업그레이드는 Synology NAS에 있는 데이터에 영향을 주지 않습니다. 그렇지만 데이터 보안을 위해 시작하기 전에 데이터를 백업하는 것이 좋습니다.
  • 새 DSM 업데이트 버전으로 업그레이드한 후에는 이전 DSM 업데이트 버전으로 다운그레이드할 수 없습니다.
  • Synology NAS는 자동으로 검사 목록을 실행하여 새 버전이 서비스 기반 시스템 및 시스템 구성과 관련되어 있는지 확인합니다. 따라서 현재 DSM 구성에 따라 사용 가능한 업데이트와 최신 버전이 다를 수 있습니다.

수동으로 DSM 업데이트 수행

Synology의 다운로드 센터에서 .pat 업데이트 파일을 다운로드하여 수동으로 DSM 업데이트를 수행할 수 있습니다.

수동으로 DSM 업데이트 수행하기:

  1. 다운로드 센터를 방문하여 새 DSM 업데이트를 로컬 컴퓨터에 다운로드합니다. .pat 파일 압축을 풀지 않아도 됩니다.
  2. 수동 DSM 업데이트를 클릭하여 수동 DSM 업데이트 페이지로 이동합니다.
  3. 찾아보기를 클릭하여 업데이트 파일을 선택합니다.
  4. 적용을 클릭하여 DSM을 업데이트합니다. 업데이트가 완료되면 시스템이 자동으로 다시 부팅됩니다.

문제 해결:

DSM을 업데이트할 수 없으면 다음 제안 사항으로 문제를 해결할 수 있습니다.

  • 업그레이드 절차가 사용자에 의해 중단되었거나 정전과 같은 예상하지 못한 사고로 중단되었는지 확인합니다.
  • DSM 버전을 확인합니다. 최신 DSM 버전으로만 시스템을 업데이트할 수 있습니다.
  • 업데이트 파일이 손상되었을 수 있으므로, DSM 업데이트를 다시 다운로드합니다.

문제가 지속되면 Synology의 기술 지원팀에 연락하십시오.

자동 DSM 업데이트 및 설치 기본 설정 지정

업데이트 설치 설정을 구성하고 스케줄을 확인하려면 업데이트 설정을 클릭합니다.

사용 가능한 새 DSM 업데이트가 있으면 시스템이 알려주기:

새 업데이트 설치 여부를 결정할 수 있도록 알려주기 옵션을 사용하면 시스템은 사용 가능한 새 버전이 있는 경우에만 이를 사용자에게 알립니다. 알림 수신 후 업데이트 다운로드 여부를 선택할 수 있습니다.

DSM 업데이트를 다운로드하여 설치할 수 있으면 바탕 화면 알림이 표시됩니다. DSM 자동 업데이트를 중지하려면 바탕 화면 알림에서 이를 취소할 수 있습니다. 또한 전원 예약에서 종료를 활성화한 경우, 예약 종료 전에 DSM 업데이트가 설치됩니다.

새 DSM 업데이트를 자동으로 설치하기:

자동으로 새 업데이트 설치 옵션을 사용하면 시스템은 사용 가능한 새 업데이트가 있는 경우에 자동으로 새 DSM 업데이트를 설치할 수 있습니다. 이는 상태에 표시된 부 업데이트에만 적용되며 주 업데이트에는 적용되지 않습니다.

확인 스케줄 설정하기:

확인 스케줄을 사용하면 시스템이 사용 가능한 업데이트를 확인해야 하는 시기를 정의할 수 있습니다. 업데이트를 확인할 요일과 특정일의 시간을 선택합니다. 예: 화요일과 금요일 11:00

주 업데이트 버전

사용 가능한 주 DSM 버전이 있으면 DSM 업데이트 섹션 하단에 이 정보가 표시됩니다. 다운로드를 클릭하여 주 업데이트 버전을 다운로드할 수 있습니다. 다운로드가 완료되면 지금 업데이트를 클릭하여 프로세스를 완료합니다.

문제 해결:

새 DSM 확인 또는 다운로드 중에 오류 메시지(빨간색 텍스트)가 표시되는 경우, 다음을 확인하십시오.

  • Synology NAS이(가) 인터넷에 연결되어 있는지 확인합니다.
  • Synology의 다운로드 서버에 현재 사용량이 많을 수 있습니다. 나중에 다운로드를 다시 시도하면 문제가 해결될 수 있습니다.

문제가 지속되면 Synology의 기술 지원팀에 연락하십시오.

확장 유닛 펌웨어 업데이트

SAS 확장 유닛을 사용하는 경우, 가끔 펌웨어를 업데이트해야 할 수도 있습니다. 시스템은 자동으로 펌웨어 업데이트를 확인하고 펌웨어를 즉시 업데이트해야 합니다, 모든 확장 유닛 펌웨어가 최신 버전입니다 또는 연결된 확장 유닛이 없습니다 등의 상태 중에서 하나를 표시합니다.

  • 펌웨어를 즉시 업데이트해야 합니다: 이 상태가 표시되면 가능한 빨리 확장 유닛 펌웨어를 업데이트하는 것이 좋습니다. 업데이트가 완료될 때까지 확장 유닛을 사용할 수 없습니다. 또한 이 시간 동안 확장 유닛에 새 볼륨을 만들 수 없습니다.
  • 모든 확장 유닛 펌웨어가 최신 버전입니다: 이 상태는 시스템이 업데이트를 완료했고 안정적인 환경에서 작동 중임을 나타냅니다.
  • 연결된 확장 유닛이 없습니다: 이 상태는 Synology NAS에 연결된 확장 유닛이 없음을 나타냅니다. 따라서 펌웨어를 업데이트할 필요가 없습니다.

중요: 일반적으로 확장 유닛 펌웨어 업그레이드는 Synology NAS에 있는 데이터에 영향을 주지 않습니다. 그렇지만 데이터 보안을 위해 시작하기 전에 데이터를 백업하는 것이 좋습니다.

참고:

  • 확장 유닛 펌웨어 업데이트는 특정 SAS 모델에서만 필요합니다.
  • DSM 6.2.2 이상에서만 확장 유닛 펌웨어 업데이트를 사용할 수 있습니다.

확장 유닛 펌웨어 업데이트 수행하기:

확장 유닛 펌웨어 업데이트는 몇 분만에 끝나며 시스템을 다시 시작할 필요가 없습니다. 업데이트 동안 실행 중인 다른 서비스는 중단되지 않습니다.

  1. 지금 업데이트를 클릭하여 업데이트를 시작합니다.
  2. 경고 대화상자가 나타납니다. 다음 옵션 중 하나를 선택합니다.
    • 계속 업데이트하려면 확인을 클릭합니다. 진행 상태 막대가 나타납니다.
    • 나중에 업데이트하려면 취소를 클릭합니다.
  3. 업데이트 성공 여부를 확인할 수 있도록 시스템은 확장 유닛 펌웨어 업데이트 완료 또는 확장 유닛 펌웨어 업데이트 실패와 같은 바탕 화면 알림 중 하나를 표시합니다.
  4. 다음 상태 메시지 중 하나가 상태에 표시됩니다.
    • 모든 확장 유닛 펌웨어가 최신 버전입니다: 이 메시지는 업데이트가 성공적으로 완료되면 표시됩니다. 시스템이 안정적인 환경에서 작동 중이며, 확장 유닛을 사용할 수 있습니다.
    • 업데이트 실패: 확장 유닛 펌웨어 업데이트가 실패하면 지원팀에 문의를 클릭하여 문제를 해결합니다.

참고:

  • 시스템이 펌웨어 업데이트 필요성을 감지하면 시스템 안정성을 보장하기 위해 확장 유닛 서비스를 사용할 수 없습니다. 펌웨어 업데이트가 완료되면 서비스를 다시 사용할 수 있습니다.
  • 확장 유닛 펌웨어 업데이트 실패 시 드라이브 또는 확장 유닛을 다시 시작하거나 제거하려 하지 마십시오. 이 문제를 해결하려면 이 문서를 참조하거나 Synology 기술 지원팀에 연락하여 추가 도움을 받으십시오.
