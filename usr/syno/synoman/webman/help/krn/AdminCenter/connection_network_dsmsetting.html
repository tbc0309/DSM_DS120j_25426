DSM 설정

제어판 > 네트워크 > DSM 설정에서 클라이언트가 Synology NAS에 액세스하는 데 사용하는 포트 번호를 변경할 수 있습니다.

기본 포트 번호 변경하기:

HTTP 연결 및 HTTPS 연결은 둘다 DSM에서 항상 활성화됩니다. 기본 HTTP 포트는 5000이고, 기본 HTTPS 포트는 5001입니다.

  1. HTTP 또는 HTTPS 필드에 포트 번호를 입력합니다.
  2. 적용을 클릭합니다.

HTTP에서 HTTPS 연결로 리디렉션하기:

클라이언트가 HTTP 포트를 통해 Synology NAS에 액세스하려고 할 때 클라이언트를 HTTPS 포트로 리디렉션할 수 있습니다.

  1. HTTP 연결에서 HTTPS로 자동으로 리디렉션(Web Station 및 Photo Station 제외)을 선택합니다.
  2. 적용을 클릭합니다.

HTTPS 연결에 사용되는 포트:

  • DSM: 기본 포트 번호가 5001이므로 다음 주소를 입력하여 안전하게 DSM에 액세스할 수 있습니다:
  • Web Station 및 Photo Station: 기본 포트 번호가 443이므로 다음 주소를 입력하여 안전하게 Web Station 및 Photo Station에 액세스할 수 있습니다:

HTTP/2 활성화하기:

HTTP/2를 활성화하여 HTTPS 연결 속도를 높일 수 있습니다.

  1. HTTP/2 활성화를 선택합니다.
  2. 적용을 클릭합니다.

최대 동시 HTTP 연결 사용자 지정하기:

최대 동시 HTTP 연결 사용자 지정을 활성화하여 Synology NAS에서 HTTP 연결을 늘릴 수 있습니다.

참고:

  • Bromolow, Cedarview, Avoton, Grantley, Broadwell, Dockerx64, Braswell, Broadwellnk, Apollolake 플랫폼에만 적용할 수 있습니다. 사용 모델이 Btrfs를 지원하는지 확인하려면 이 항목을 참조하십시오.
  • 최대 동시 HTTP 연결 수가 많을수록 시스템 메모리를 더 많이 필요로 합니다.

HTTP 응답에서 "Server" 헤더 사용자 지정하기:

  1. HTTP 응답에서 "Server" 헤더 활성화를 누릅니다.
  2. 웹 서버의 헤더를 지정합니다(예: nginx).
  3. 적용을 클릭합니다.

확인란의 선택을 해제하여 "Server" 헤더를 비활성화할 수 있습니다.

도메인

도메인 아래에서, Synology NAS에 액세스하기 위한 사용자 지정된 도메인 이름을 지정할 수 있습니다.

사용자 지정된 도메인 활성화하기:

  1. 사용자 지정된 도메인 활성화를 선택합니다.
  2. Synology NAS에 대해 적용되는 도메인 이름을 지정합니다.
  3. 또한 HSTS 활성화를 선택하여 이 도메인에서 HTTPS 암호화된 연결을 강제 적용할 수 있습니다.
  4. 적용을 클릭합니다.

참고:

  • 각 도메인 이름은 Synology NAS에서 별개의 한 응용 프로그램에 대해서만 사용될 수 있습니다.
  • 도메인 이름의 길이가 15자를 초과하지 않고 NetBIOS 명명 규칙을 준수하면, 시스템이 자동으로 도메인 이름(예: MyDSM)을 등록하고 브로드캐스트합니다. Windows 사용자는 다음과 유사한 URL을 사용하여 동일한 로컬 네트워크에서 DSM 데스크톱에 액세스할 수 있습니다: http://MyDSM 또는 https://MyDSM
  • 도메인 이름을 사용하는 URL을 통해 응용 프로그램에 액세스할 때, 표준 HTTP 포트(80) 또는 표준 HTTPS 포트(443)를 통과합니다.
