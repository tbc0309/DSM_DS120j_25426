SMB

SMB(서버 메시지 블록) 프로토콜을 사용하면 SMB/CIFS를 지원하는 Windows, Mac, Linux 클라이언트가 Synology NAS에 저장된 데이터에 액세스할 수 있습니다. Windows 탐색기를 통해 Synology NAS에 있는 파일을 검색하거나 공유 폴더를 네트워크 드라이브에 탑재하려면 제어판 > 파일 서비스 > SMB/AFP/NFS에서 이 옵션을 활성화합니다.

SMB 프로토콜 활성화:

  1. SMB 서비스 활성화를 선택합니다.
  2. 적용을 클릭합니다.

참고:

  • SMB 프로토콜을 활성화한 후 SMB를 통해 인덱싱된 폴더를 Mac 컴퓨터에 탑재한 후 Finder를 사용하여 탑재된 폴더 내에서 파일 콘텐츠를 검색할 수 있습니다(NVR216 또는 VS360HD에는 적용되지 않음).

작업 그룹

이 옵션을 사용하여 네트워크 관리 목적으로 시스템을 특정 작업 그룹의 구성원으로 지정할 수 있습니다.

참고:

  • 작업 그룹 이름에는 1~15자가 포함될 수 있습니다. 이름에 다음 문자를 포함할 수 없습니다. [ ] ; : "< >* + = \ / | ? ,
  • OS X 10.9를 실행하는 Mac을 사용 중이며 Synology NAS이(가) Windows 도메인에 가입되어 있는 경우, SMB 프로토콜을 통해 Synology NAS에 저장된 파일에 액세스하기 위해 Synology NAS 로컬 사용자 이름 앞에 localhost\를 추가하여 로그인할 수 있습니다.

이전 버전에 대한 액세스 허용 안 함

Windows 탐색기에서 사용자가 공유 폴더 내 이전 버전 탭을 클릭하여 폴더 또는 파일의 스냅샷에 액세스할 수 있으므로, 관리자 도움 없이 데이터를 복원 또는 복사할 수 있습니다. 이전 버전에 대한 액세스 허용 안 함을 선택하면 "이전 버전" 페이지에 어떠한 정보도 표시되지 않습니다.

참고:

  • 이 기능은 VDSM과 Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview 및 Grantley 아키텍처가 있는 모델에서만 지원됩니다. 사용하고 있는 모델이 지원하는 패키지 아키텍처를 확인하려면 이 문서를 참조하십시오.
  • 이전 버전에 해당하는 스냅샷 촬영 당시에 사용자 계정이 생성되었고 해당 스냅샷이 촬영된 공유 폴더에 대한 읽기 권한이 사용자에게 부여된 경우에만 사용자에게 이전 버전에 대한 액세스가 허용됩니다.

전송 로깅 활성화

전송 로그 활성화 확인란을 선택하여 CIFS/SMB 프로토콜을 통해 수행된 파일 작업을 로깅할 수 있습니다. 기본적으로 전송 로그는 파일 또는 폴더 삭제만 모니터링합니다. 하지만 로그 설정으로 이동하여 만들기, 이동, 이름 바꾸기, 읽기, 쓰기 등 시스템의 다른 파일 작업 이벤트를 선택해 로깅할 수도 있습니다. 파일 작업 이벤트를 많이 선택할수록 시스템 성능이 더욱 영향을 받을 수 있다는 점에 유의하십시오.

고급 설정

고급 설정 버튼을 클릭하여 아래 옵션을 볼 수 있습니다.

  • WINS 서버: Microsoft Windows 인터넷 이름 서비스(WINS)는 NetBIOS 이름 - IP 주소 매핑 서비스입니다. 이 시스템이 WINS 서버에 등록되도록 구성된 경우, Windows 사용자는 TCP/IP 네트워크에서 보다 쉽게 시스템을 찾을 수 있습니다.
  • 최대 SMB 프로토콜: 네트워크 구성에 따라 Synology NAS에서 지원되는 최신 SMB 프로토콜을 설정할 수 있습니다.
    • SMB1: SMB1(또는 CIFS라고 함)은 Windows NT 이상에서 지원됩니다.
    • SMB2: SMB2는 Windows Vista 이상에서 지원되며 SMB의 향상된 버전입니다. SMB2는 네트워크 패킷 수를 줄이고 성능을 강화하기 위해 여러 SMB 작업을 단일 요청에 통합하는 기능을 추가합니다.
    • SMB2 및 Large MTU: 최대 전송 단위(MTU)는 통신 프로토콜에서 전달될 수 있는 가장 큰 데이터 단위를 의미합니다. 1기가비트 또는 10기가비트 이더넷과 같은 더욱 빠른 네트워크 인터페이스를 활용하기 위해 Large MTU는 최대 전송 단위를 1MB로 늘립니다. Large MTU를 활성화하면 처리해야 할 패킷 수가 줄어들어 대형 파일 전송 속도와 효율성이 향상됩니다.
    • SMB3: SMB3는 Windows 8 및 Windows Server 2012 이상에서 지원되며, SMB2의 향상된 버전입니다. SMB3는 AES 기반 파일 암호화 전송을 지원하므로, 피어 투 피어 파일 전송 보안이 강화됩니다.

    참고:

    • SMB2는 Windows Vista, Windows Server 2008 R2, Windows 7 이상에서 지원됩니다.
    • Large MTU는 Windows Server 2008 R2, Windows 7 이상에서 지원됩니다.
    • Windows 컴퓨터에서 Large MTU를 효과적으로 만들려면 컴퓨터의 레지스트리 설정을 수정해야 합니다. 자세한 내용은 Microsoft 기술 지원(이 문서 등)을 참조하십시오.
  • 최소 SMB 프로토콜: 네트워크 구성에 따라 Synology NAS에서 지원되는 초기 SMB 프로토콜을 설정할 수 있습니다.
  • 전송 암호화 모드: SMB3가 활성화되어 있으면 SMB 프로토콜은 전송 암호화를 추가하여 파일 전송 보안을 강화합니다.
    • 비활성화: 전송 암호화가 적용되지 않습니다.
    • 자동: 전송 암호화는 이 기능을 지원하는 클라이언트에만 적용됩니다.
    • 강제: 전송 암호화가 항상 적용됩니다. 따라서 전송 암호화를 지원하지 않는 클라이언트는 SMB 프로토콜을 사용할 수 없습니다.

    • 참고:

    • 전송 암호화는 Windows 8, Windows 2012 이상에서 지원됩니다.
    • 전송 암호화가 활성화되면 SMB 프로토콜에 대한 익명 로그온이 지원되지 않습니다.
    • 전송 암호화가 활성화되면 응용 프로그램 시간 제한을 방지하기 위해 Opportunistic Locking 활성화를 선택한 상태로 유지하십시오.
  • Opportunistic Locking 활성화: 이 옵션을 선택하여 CIFS의 "Opportunistic Locking"을 활성화하면 클라이언트의 파일 콘텐츠 캐시 성능이 향상되고 낮은 읽기/쓰기 오버헤드가 방지됩니다. 이 옵션을 선택 취소하면 데이터베이스 응용 프로그램이 네트워크를 통해 Synology NAS에 저장된 데이터베이스 파일에 더욱 원활하게 액세스할 수 있습니다.
    • SMB2 임대 활성화: 이 옵션을 활성화하면 더 많은 SMB2 관련 설정을 사용하여 Opportunistic Locking이 제공하는 클라이언트 캐싱 메커니즘을 미세 조정할 수 있습니다.
  • Local Master Browser 활성화: 동일한 서브넷 내에 여러 Windows 컴퓨터가 있으면 협상하여 이들 중 한 컴퓨터를 "Local Master Browser"로 선택합니다. Local Master Browser는 서브넷 내의 다른 컴퓨터 및 해당 공유 리소스의 목록을 유지하고, 이러한 목록을 다른 컴퓨터와 공유합니다. 이 옵션은 Synology NAS이(가) Local Master Browser 역할로 간주되도록 허용합니다.

    • 참고:

    • 이 기능을 활성화하면 하드 디스크 최대 절전 기능이 비활성화되고 패스워드 없이 게스트 계정이 활성화됩니다.
  • DirSort VFS 모듈 활성화: 폴더 이름을 정렬하여 알파벳순으로 나열합니다. 이 옵션을 활성화하면 시스템 성능이 영향을 받습니다.
  • 거부 파일: 지정한 기준에 따라 파일을 숨깁니다. 기준에는 와일드카드(*)가 포함될 수 있으며, 항목이 여러 개이면 슬래시(/)로 항목이 구분됩니다. 예: /abc*/*.txt/*.conf/. 이 옵션을 활성화하면 시스템 성능이 영향을 받습니다.
  • 공유 폴더 내의 기호 링크 허용: 이 옵션을 선택하면 기호 링크가 동일한 공유 폴더 내의 파일이나 폴더를 가리킬 수 있습니다. DSM 터미널을 사용하여 기호 링크를 만들 수 있습니다.
    • 공유 폴더 전반에 기호 링크 허용: 이 옵션을 선택하면 기호 링크가 다른 공유 폴더에 있는 파일 또는 폴더를 가리킬 수 있습니다. DSM 터미널을 사용하여 기호 링크를 만들 수 있습니다. 그러나 권한이 없는 사용자가 기호 링크의 대상 공유 폴더에 액세스할 수도 있습니다.
  • Mac 특수 문자 변환을 위해 VFS 모듈 활성화: 이 옵션을 선택하면 파일 생성 중에 파일 이름의 SMB 예약 문자(예: < > " * ?)를 기본 라틴 문자로 변환할 수 있습니다. 이러한 파일은 AFP를 통해 인식될 수 있습니다.
  • MSDFS VFS 모듈 활성화: 이 옵션을 선택하면 기호 링크가 다른 SMB 서버에 위치한 파일이나 폴더를 가리킬 수 있습니다. Windows XP와 같은 Windows 구 버전을 사용하는 경우, 이 옵션을 활성화하기 전에 클라이언트와 Synology NAS이(가) Windows 도메인에 가입되어야 합니다. MSDFS 링크 생성에 대한 자세한 내용은 여기를 참조하십시오. 이 옵션을 비활성화한 후에 클라이언트 장치의 캐시를 삭제해야 합니다(클라이언트 장치를 다시 시작하는 것도 좋음). 그렇지 않으면 정상적인 성능과 동작이 영향을 받을 수 있습니다.
  • 동일한 IP 주소로부터의 다중 연결 비활성화: 이 옵션을 활성화하면 Synology NAS은(는) SMB 프로토콜을 사용한 클라이언트에서 IP 주소마다 하나의 연결만 허용합니다. IP 주소가 동일한 다중 클라이언트가 SMB 프로토콜을 사용하는 경우, Synology NAS은(는) 최신 연결을 유지하고 기존 연결을 차단합니다. NAT 환경에서 클라이언트는 단일 IP 주소를 사용할 수 있습니다. 이 경우, 이 옵션은 정상 성능에 영향을 미칠 수 있습니다.
  • 디버그 로그 수집: 이 옵션을 활성화하면 디버깅을 위해 상세한 로그가 저장됩니다. 이 옵션을 활성화하면 시스템 성능이 영향을 받습니다.
  • 기본 UNIX 권한 적용: 이 옵션을 활성화하면 파일과 폴더 업로드 또는 생성 시 기본 UNIX 권한이 적용됩니다. UNIX 권한은 파일에 대해 744, 폴더에 대해 755입니다. 이 옵션을 비활성화하면 UNIX 권한은 파일과 폴더에 대해 777입니다.

    • 참고:

    • Windows ACL 지원 공유 폴더("photo"를 제외한 모든 공유 및 외부 드라이브의 공유)의 경우, Linux 또는 FTP 클라이언트에서 chmod 명령을 실행하여 폴더와 파일 권한 유형을 Windows ACL에서 UNIX로 변경하십시오(EDS14에 적용되지 않음).
    • 이 옵션을 활성화하면 여러 프로토콜에서 권한이 일치하지 않는 문제가 발생할 수 있습니다. 일관성을 유지하려면 이 옵션을 비활성화 상태로 유지하는 것이 좋습니다.
  • 파일 생성 시 디스크 공간 예약 안 함: 이 옵션을 활성화하면 Synology NAS에 파일 생성 시 디스크 공간이 예약되지 않습니다.
  • 와일드카드 검색 캐시 활성화: 클라이언트가 반복된 와일드카드 검색 요청을 전송할 경우, 검색 성능을 향상시키려면 이 옵션을 활성화합니다.
  • SMB 지속 가능 핸들 활성화(프로토콜 간 파일 잠금 비활성화): 이 옵션을 활성화하면 연결이 일시적으로 끊긴 후 사용자가 손실된 SMB2 연결을 이 Synology NAS(으)로 복원할 수 있습니다. 열린 파일은 네트워크가 끊긴 동안에도 사용할 수 있습니다. 그러나, 프로토콜 간 파일 잠금이 비활성화되므로 사용자는 SMB2가 아닌 프로토콜(예: AFP)을 통해 잠긴 파일에 액세스하거나 수정할 수 있습니다.
  • SMB 캐시 지우기: 문제 해결을 위해 이 버튼을 클릭하여 아래의 SMB 캐시 파일을 삭제합니다.
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
