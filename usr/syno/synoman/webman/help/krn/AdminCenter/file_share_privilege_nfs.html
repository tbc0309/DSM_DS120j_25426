NFS 권한 할당

Linux 클라이언트가 공유 폴더에 액세스할 수 있도록 모든 공유 폴더에 NFS 권한을 할당할 수 있습니다.

공유 폴더에 NFS 권한 할당하기:

  1. 공유 폴더 목록에서 편집하려는 공유 폴더를 선택합니다.
  2. 편집 > NFS 권한을 클릭합니다.
  3. 생성을 클릭하여 NFS 규칙을 추가합니다.
  4. 팝업 창이 나타납니다. 아래 옵션을 정의합니다.
  5. 호스트 이름 또는 IP: 공유 폴더에 액세스할 NFS 클라이언트의 IP 주소를 입력합니다. 세 가지 방식으로 호스트를 지정할 수 있습니다:
    • 단일 호스트: 정규화된 도메인 이름이나 IP 주소입니다.
    • 와일드카드: *, *.synology.com
    • IP 네트워크: 203.74.205.32/255.255.252.0, /24
  6. 권한: NFS 클라이언트에 대한 읽기/쓰기 권한을 선택합니다.
  7. 보안: 구현할 보안 특성을 지정합니다.
    • AUTH_SYS: NFS 클라이언트의 UID(사용자 ID) 및 GID(그룹 ID)를 사용하여 액세스 권한을 확인합니다.
    • Kerberos 인증: NFS 클라이언트가 공유 폴더에 연결되면 Kerberos 인증을 수행합니다. 클라이언트는 Kerberos 인증을 통과한 경우에만 공유 폴더에 액세스할 수 있습니다.
    • Kerberos 무결성: Kerberos 인증을 수행하고 데이터 전송 중 패킷 무결성을 보장합니다.
    • Kerberos 개인 정보 보호: Kerberos 인증을 수행하고 데이터 전송 중에 NFS 패킷을 암호화하여 악의적인 당사자가 NFS 트래픽을 변경하는 것을 방지합니다.
  8. Squash: 이 필드를 사용하여 NFS 클라이언트의 사용자 액세스 권한을 제어할 수 있습니다. 다음 중 하나를 선택하십시오:
    • 매핑 없음: root 사용자를 포함하여 NFS 클라이언트의 모든 사용자가 원래의 액세스 권한을 유지할 수 있습니다.
    • admin에 root 매핑: 시스템에서 admin 사용자 액세스 권한과 동등한 액세스 권한이 NFS 클라이언트의 root 사용자에게 할당됩니다.
    • guest에 root 매핑: 시스템에서 guest 액세스 권한과 동등한 액세스 권한이 NFS 클라이언트의 root 사용자에게 할당됩니다.
    • admin에 모든 사용자 매핑: 시스템에서 admin 사용자 액세스 권한과 동등한 액세스 권한이 NFS 클라이언트의 모든 사용자에게 할당됩니다.
    • guest에 모든 사용자 매핑: 시스템에서 guest 액세스 권한과 동등한 액세스 권한이 NFS 클라이언트의 모든 사용자에게 할당됩니다.
  9. 비동기 활성화: 이 옵션을 선택하면 파일 변경이 완료되기 전에 Synology NAS이(가) NFS 클라이언트의 요청에 응답할 수 있어, 성능이 향상됩니다.
  10. 권한이 없는 포트로부터 연결 허용(1024보다 높은 포트): 이 옵션을 선택하면 Synology NAS에 연결 시 NFS 클라이언트가 권한이 없는 포트(예: 1024보다 큰 포트)를 사용할 수 있습니다.
  11. 사용자에게 마운팅된 하위 폴더 액세스 허용： 이 옵션을 선택하면 NFS 클라이언트가 마운팅된 하위 폴더에 액세스할 수 있습니다.
  12. 확인을 클릭하여 종료합니다.
  13. 확인을 클릭하여 NFS 권한을 적용합니다.

참고:

  • 서버 이름 형식이 *.domain인 경우에는 Synology NAS에서 해당 IP 주소를 검색하여 *.domain 이름을 찾을 수 있도록 NFS 클라이언트의 IP 주소에 해당 DNS PTR 레코드가 있어야 합니다.

보안 특성:

특정 사용자 계정을 사용하여 NFS를 통해 공유 폴더에 액세스하는 경우:

  • AUTH_SYS 보안 특성이 구현된 경우: 클라이언트에는 NFS 클라이언트와 Synology NAS에서 정확하게 동일한 숫자의 UID(사용자 ID) 및 GID(그룹 ID)가 있어야 합니다. 그렇지 않으면 공유 폴더에 액세스 시 클라이언트에 기타 권한이 할당됩니다. 권한 충돌을 방지하기 위해 Squash에서 Admin에 모든 사용자 매핑을 선택하거나 공유 폴더에 "Everyone" 권한을 부여할 수 있습니다.
  • Kerberos(krb5, krb5i, krb5p) 보안 특성이 구현된 경우: Win/Mac/NFS > NFS 서비스 > Kerberos 설정 > ID 매핑으로 이동하여 NFS 클라이언트를 특정 사용자에 매핑하거나, 해당 사용자 계정을 사용하여 Windows /LDAP 도메인에 가입해야 합니다. 그렇지 않으면 공유 폴더에 액세스 시 클라이언트에 guest 권한이 할당됩니다.
  • 공유 폴더가 생성된 외부 장치의 파일 시스템이 NTFS 또는 FAT인 경우, admin에 모든 사용자 매핑 옵션이 강제 적용됩니다.

Kerberos 보안 특성을 사용하여 Synology NAS에 연결하려면 Win/Mac/NFS > NFS 서비스 > Kerberos 설정으로 이동하여 Kerberos 인증을 구성해야 합니다.
