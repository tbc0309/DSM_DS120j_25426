대상

모델에 따라 Synology NAS에서 LUN을 매핑할 수 있는 대상을 여러 개 생성할 수 있습니다.

참고: 지원되는 대상 개수는 Synology NAS 모델에 따라 다릅니다.

대상 생성하기:

  1. 생성 버튼을 클릭합니다.
  2. 이름, IQN, CHAP 인증 활성화 여부를 포함한 대상 정보를 입력합니다.
  3. 다음을 클릭합니다.
  4. iSCSI LUN 매핑을 설정합니다. 새 iSCSI LUN 생성 또는 기존 LUN 매핑을 선택합니다.
  5. 다음을 클릭합니다.
  6. 뒤로를 클릭하여 설정을 수정하거나 적용을 클릭하여 설정을 확인합니다.

용어

  • IQN: IQN(iSCSI Qualified Name)은 대상의 고유한 이름 세트로, 다른 서버에 있는 대상 IQN 과 충돌해서는 안됩니다. IQN 은 128 자 미만이어야 하며 영문자(대소문자 구문), 숫자, 콜론, 대시 및 점을 사용할 수 있습니다. 제안된 이름 지정 형식은 iqn.[날짜].[도메인]:[장치 ID]입니다. 예: iqn.2000-04.com.synology:storage.diskarray.1.
  • 상호 CHAP 인증: 상호 CHAP에서 클라이언트와 서버가 서로 인증합니다. 클라이언트 인증용으로 사용된 CHAP 패스워드를 서버 인증용으로 구성하면 안됩니다. 또한 서버 인증용으로 사용된 CHAP 패스워드를 클라이언트 인증용으로 구성하면 안됩니다.

    • 참고: 사용 중인 NAS에서 CHAP 인증이 활성화되면 iSCSI 초기자에서의 검색 프로세스가 실패할 수 있습니다. 먼저 검색을 완료하고 CHAP 인증을 활성화한 후에 대상에 연결하십시오. 또는 IP 및 IQN을 사용하여 대상에 연결할 수 있습니다.

대상 제거하기:

대상 및 매핑된 모든 LUN을 제거할 수 있습니다. iSCSI LUN을 제거하면 iSCSI LUN에 있는 모든 데이터가 삭제됩니다.

  1. 제거할 대상을 선택합니다.
  2. 제거를 클릭합니다.
  3. 매핑된 LUN을 제거하려면 매핑된 LUN도 삭제를 선택하고 제거할 매핑된 LUN을 선택합니다.
  4. 확인을 클릭합니다.

대상 편집하기:

  1. 편집할 대상을 선택합니다.
  2. 동작 드롭다운 메뉴에서 편집을 선택합니다.
  3. 원하는 대로 설정을 편집합니다. 확인을 클릭하여 변경 사항을 저장합니다.

용어

  • CRC 체크섬: CRC 체크섬을 활성화하여 데이터 전송 중에 오류가 발생했는지 확인할 수 있습니다.
  • 다중 세션: 초기자 하나 이상에서 다중 세션을 동일한 대상에 동시에 연결할 수 있습니다. 파일 손상 또는 예기치 않은 데이터 손실을 방지하기 위해 VMFS(VMware Virtual Machine File System) 및 OCFS(Oracle Cluster File System)와 같은 공유 파일 시스템이나 클러스터 시스템을 사용하는 것이 좋습니다.
  • 마스킹: 마스킹을 사용하면 지정된 iSCSI 초기자 IQN이 대상에 독점적으로 액세스할 수 있습니다.

대상 활성화 또는 비활성화하기:

  1. 활성화 또는 비활성화할 대상을 선택합니다.
  2. 동작 드롭다운 메뉴에서 활성화 또는 비활성화를 선택합니다.
