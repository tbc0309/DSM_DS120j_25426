iSNS

iSNS(Internet Storage Name Service)는 중앙 집중 관리 메커니즘을 제공합니다. iSNS 서버를 사용하여 모든 iSCSI Target을 등록할 수 있습니다.

iSNS 서버를 사용하여 iSCSI Target 등록하기:

  1. iSNS를 클릭하고 iSNS 활성화를 선택합니다.
  2. iSNS 서버의 IP 주소를 입력합니다. 확인을 클릭하여 설정을 적용합니다.

I/O 대기열 크기

기본 설정은 16이지만 필요에 따라 모든 대상의 I/O 대기열 크기를 수정할 수 있습니다.

저용량 쓰기

사용 가능한 볼륨 공간이 1GB 미만이면 파일 시스템 충돌을 방지하기 위해 시스템에서 볼륨의 LUN을 보호하므로, 사용자는 LUN에 데이터를 쓸 수 없습니다. 볼륨 여유 공간이 부족할 때 저용량 쓰기를 사용하면 사용자는 LUN에 데이터를 계속 쓸 수 있습니다. 하지만 공간 부족 문제가 해결되면 이 옵션을 비활성화하는 것이 좋습니다.

참고:

  • 이 기능을 활성화하더라도 LUN에 데이터를 계속 기록하지 마십시오.

공간 재사용 모드

공간 재사용에는 다음과 같은 세 가지 모드가 있습니다. 속도를 조정하면 LUN 성능의 공간 재사용 영향을 제어할 수 있습니다.

  • I/O 일관성 우선 순위 지정: I/O 일관성 우선 순위를 지정하면 시스템이 레거시 고급 LUN이 사용한 공간을 재사용하고 성능 영향이 최소화됩니다. 연속으로 I/O가 수신되면 속도가 느려집니다. 연속으로 공간 재사용 명령을 수신하면 다른 공간 재사용 명령이 없을 때까지 재사용 프로세스가 지연됩니다.
  • 균형: 기본 재사용 모드입니다. 이 모드는 I/O 속도와 재사용 속도 간의 균형을 유지합니다.
  • 재사용 우선 순위 지정: 재사용 우선 순위 지정은 수신 I/O가 재사용 속도에 영향을 거의 미치지 않는다는 의미입니다. I/O 일관성이 저하되지 않고 최고 속도로 재사용이 수행됩니다.
