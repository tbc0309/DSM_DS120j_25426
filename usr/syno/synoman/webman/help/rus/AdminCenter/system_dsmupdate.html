Обновление

Обновление DSM

Периодически компания Synology выпускает бесплатные обновления для DSM. Обновления могут включать в себя новые функции, усовершенствования функций и решения для повышения производительности. Обновления DSM могут быть основными или промежуточными. Если обновление является промежуточным, рядом с полем Состояние отображается соответствующее сообщение. При наличии основных обновлений DSM в разделе Состояние будет отображаться информация об этой версии, а также ссылка для скачивания основного обновления. Обычно промежуточные обновления включают в себя исправления критических ошибок и исправления системы безопасности для обеспечения стабильности системы, в то время как основные обновления, как правило, включают в себя новые функции и обеспечивают повышение производительности, а также содержат исправления критических ошибок и исправления системы безопасности.

Примечание.

  • Обновление DSM не влияет на данные в Synology NAS. Однако чтобы обеспечить безопасность данных, настоятельно рекомендуется создать резервные копии данных перед началом процесса обновления.
  • После обновления до новой версии DSM невозможно перейти на предыдущую версию DSM.
  • Synology NAS автоматически запускает контрольный список, чтобы определить, соответствует ли новая версия вашей системе в зависимости от служб и конфигурации системы. В этой связи доступные обновления и последние версии могут различаться в зависимости от текущих конфигураций DSM.

Обновление DSM вручную

Можно скачать файл обновления .pat в Центре загрузок Synology и выполнить обновление DSM вручную.

Обновление DSM вручную

  1. Посетите Центр загрузок, чтобы скачать обновление DSM на локальный компьютер. Файл .pat распаковывать не нужно.
  2. Нажмите Обновление DSM вручную для перехода на страницу обновления DSM вручную.
  3. Нажмите Обзор, чтобы выбрать файл обновления.
  4. Нажмите Применить, чтобы начать обновление DSM. Система перезагрузится автоматически по завершении обновления.

Устранение неполадок

Если не удается выполнить обновление DSM, следующие предложения могут помочь в решении данной проблемы:

  • Проверьте, не был ли процесс обновления прерван пользователями или в результате непредвиденных событий, например, отключения питания.
  • Проверьте версию DSM. Обновление возможно только при наличии более поздней версии DSM.
  • Еще раз скачайте обновление DSM, поскольку файл обновления мог быть поврежден.

Если проблема все еще не устранена, обратитесь в службу технической поддержки Synology.

Настройка автоматической проверки обновлений DSM и параметров установки

Чтобы настроить параметры установки обновлений и изменить расписание, нажмите Настройки обновления.

Чтобы система отображала уведомления о наличии нового обновления DSM, выполните следующие действия.

Параметр Уведомить и разрешить мне принять решение об установке нового обновления позволяет системе получать уведомления только при наличии новой версии. После получения уведомления можно выбрать, требуется ли скачать обновление.

Когда обновление DSM будет скачано и готово к установке, на рабочем столе появится соответствующее уведомление. Чтобы отключить автоматическое обновление DSM, отмените его в уведомлении на рабочем столе. Обновления DSM будут установлены перед запланированным выключением, если вы включили параметр Завершение работы в разделе График включения и отключения питания.

Чтобы новые обновления DSM устанавливались автоматически, выполните следующие действия.

Параметр Автоматическая установка обновлений позволяет системе автоматически устанавливать новые обновления DSM, если они обнаружены в результате проверки. Этот параметр применим только к промежуточным обновлениям, отображаемым в разделе Состояние, но не к основным обновлениям.

Чтобы настроить расписание проверки, выполните следующие действия.

При помощи параметра Проверка расписания можно указать, когда система должна проверять наличие доступных обновлений. Выберите дни недели и время суток для проверки обновлений. Например, по вторникам и по пятницам в 11:00.

Основное обновление

Если доступна основная версия DSM, эта информация будет отображаться в нижней части раздела "Обновление DSM". Чтобы скачать основную версию, нажмите Скачать. После скачивания нажмите Обновить сейчас, чтобы завершить процесс.

Устранение неполадок

При отображении сообщений об ошибке (красный текст) во время проверки или скачивания новой версии DSM проверьте следующее.

  • Убедитесь, что Synology NAS подключен к Интернету.
  • Сервер скачивания Synology в данный момент может быть перегружен. Попробуйте снова выполнить скачивание позже.

Если проблема все еще не устранена, обратитесь в службу технической поддержки Synology.

Обновление микропрограммы модуля расширения

При наличии модулей расширения SAS периодически может потребоваться обновление микропрограммы. Система автоматически проверит наличие обновлений микропрограммы и отобразит одно из следующих состояний: Необходимо немедленно обновить микропрограмму, Микропрограмма всех модулей расширения обновлена или Нет подключенных модулей расширения

  • Необходимо немедленно обновить микропрограмму. При отображении этого состояния настоятельно рекомендуется как можно скорее обновить микропрограмму модуля расширения. Модули расширения будут недоступны до завершения обновления. В это время также невозможно создать новые тома на модулях расширения.
  • Микропрограмма всех модулей расширения обновлена. Это указывает на то, что обновление завершено и система работает в стабильной среде.
  • Нет подключенных модулей расширения. Это значит, что к Synology NAS не подключены модули расширения. Поэтому нет необходимости в обновлении микропрограммы.

Внимание! Обычно обновление микропрограммы модулей расширения не влияет на данные Synology NAS. Однако чтобы обеспечить безопасность данных, настоятельно рекомендуется создать резервные копии данных перед началом процесса обновления.

Примечание.

  • Обновление микропрограммы модулей расширения требуется только для определенных моделей SAS.
  • Обновление микропрограммы модулей расширения доступно только в DSM 6.2.2 или более поздней версии.

Чтобы обновить микропрограмму модуля расширения, выполните следующие действия.

Обновление микропрограммы модуля расширения выполняется за несколько минут и не требует перезагрузки системы. Работа остальных запущенных служб не будет прервана во время обновления.

  1. Нажмите Обновить сейчас, чтобы начать обновление.
  2. Отобразится диалоговое окно с предупреждением. Выберите один из следующих параметров:
    • Чтобы продолжить обновление, нажмите OK. Появится индикатор выполнения.
    • Чтобы выполнить обновление позже, нажмите Отмена.
  3. Система отобразит одно из следующих уведомлений о состоянии обновления: Обновление микропрограммы модуля расширения завершено или Сбой обновления микропрограммы модуля расширения.
  4. В разделе Состояние будет отображаться одно из следующих сообщений о состоянии:
    • Микропрограмма всех модулей расширения обновлена. Это сообщение отображается после успешного завершения обновления. Система работает в стабильной среде, модули расширения доступны.
    • Сбой обновления. Если не удалось обновить микропрограмму модуля расширения, нажмите Обратиться в службу поддержки для устранения проблемы.

Примечание.

  • Для обеспечения стабильности системы службы модуля расширения недоступны, если система обнаруживает, что требуется обновление микропрограммы. Службы будут снова доступны после завершения обновления микропрограммы.
  • Если не удалось обновить микропрограмму модуля расширения, не пытайтесь перезапустить либо удалить диски или модули расширения. Чтобы устранить эту проблему, ознакомьтесь с этой статьей и обратитесь за помощью в службу технической поддержки Synology.
