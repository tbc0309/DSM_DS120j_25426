Общие

Общие параметры сети, такие как шлюз по умолчанию, DNS-сервер, прокси-сервер и туннелирование IPv6, доступны в разделе Панель управления > Сеть > Общие.

Переименование устройства Synology NAS

Для того чтобы переименовать устройство Synology NAS, введите новое имя сервера и нажмите Применить.

Имя сервера вводится без учета регистра и может содержать от 1 до 15 символов, в числе которых могут быть буквы, цифры, символы подчеркивания и знаки минус. Первым символом имени должна быть буква.

Указание шлюза по умолчанию

Просто введите IP-адрес шлюза по умолчанию и нажмите Применить, чтобы сохранить настройку.

Указание IP-адреса DNS Server

Можно настроить IP-адрес DNS-сервера вручную. В этом случае необходимо ввести IP-адрес основного DNS-сервера. Также можно ввести альтернативный IP-адрес DNS-сервера, обеспечивающий резервное копирование при возникновении проблем с основным DNS Server. После настройки IP-адресов DNS-сервера нажмите Применить, чтобы сохранить настройки.

Дополнительные настройки.

  • Ответьте на запрос ARP, если целевой IP-адрес совпадает с локальным адресом, настроенным на входящем интерфейсе. В сетевом протоколе ARP не отвечает на запрос с идентичного IP-адреса. Включите этот параметр, чтобы ответить на запрос ARP и убедиться, что таблица ARP заполнена.
  • Включите несколько шлюзов. Включите этот параметр для поддержки нескольких шлюзов в интерфейсе.
  • Включить обнаружение конфликтов IP-адресов. Включите этот параметр, чтобы автоматически определять наличие одинаковых IP-адресов в сети.

Примечание.

Если все интерфейсы настраиваются автоматически или если используется подключение PPPoE, поля «DNS-сервер» и «Шлюз по умолчанию» будут заполнены автоматически и не могут быть изменены.

Подключение с помощью прокси-сервера

Для подключения Synology NAS с помощью прокси-сервера установите флажок Подключиться с помощью прокси-сервера, введите Адрес и Порт, а затем нажмите Применить, чтобы сохранить настройки.

Примечание.

  • Нажмите Дополнительные настройки, чтобы настроить прокси HTTP, HTTPS и информацию об аутентификации.
  • По умолчанию параметры прокси-сервера HTTPS совпадают с параметрами прокси-сервера HTTP.
  • При активации параметра Подключиться с помощью прокси-сервера функции «Обновление DSM» и «Центр пакетов» вместе с другими пакетами будут подключены к Интернету через прокси.
  • Если установлен флажок Обойти прокси-сервер для локального адреса, убедитесь, что используемый Synology NAS DNS-сервер может разрешать запросы для устройств в локальной сети.

Включение туннелирования по протоколу IPv6

Пока IPv6 полностью не заменит IPv4, туннелирование IPv6 необходимо для того, чтобы хосты с адресами IPv6 могли использовать службы IPv4. Этот метод позволяет пропускать трафик IPv6 по существующим сетям IPv4.
Если вы не знаете, как настраивать IPv6 или устранять неисправности подключения, обратитесь к провайдеру услуг Интернет.

Включение туннелирования по протоколу IPv6:

  1. Установите флажок Включить Tunnel, чтобы включить поддержку туннелирования.
  2. В поле Адрес сервера введите имя хоста или IP-адрес провайдера услуг туннелирования, например: anonymous.freenet6.net.
  3. Выберите Подключаться анонимно или Подключиться с помощью существующей учетной записи.

Примечание.

Внешний адрес IPv6 может обновляться провайдером услуг автоматически. Однако Synology NAS не сразу выполнит удаление старого адреса IPv6.
