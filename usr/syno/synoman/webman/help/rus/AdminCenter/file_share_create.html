Создание общей папки

Общие папки являются базовыми каталогами, где можно хранить и управлять файлами и папками на устройстве Synology NAS. Перед сохранением файлов необходимо создать как минимум одну общую папку в DSM.

Создание общей папки

  1. Выберите Панель управления > Общая папка и нажмите Создать.
  2. Укажите имя и описание для общей папки.
  3. Выберите хранилище, в котором будет создана общая папка (пропустите этот шаг, если существует только одно хранилище).
  4. При необходимости измените следующие параметры.
    • Скрыть эту общую папку в меню «Сетевое окружение». Общая папка не будет отображаться в меню «Сеть» в Проводнике Windows. Данный параметр не влияет на права доступа к общей папке.

      Примечание

      • Скрытие общей папки в меню «Сетевое окружение» не влияет на права доступа к ней. Право доступа к скрытой общей папке останется неизменным. Пользователи, имеющие необходимые права доступа к общей папке, смогут получать доступ к этой папке, указывая путь «\\имя сервера\имя общей папки».
    • Скрыть подпапки и файлы от пользователей без прав доступа. При активации пункта Скрыть подпапки и файлы от пользователей без прав доступа пользователь, выполняющий доступ к общей папке по протоколу обмена файлами Windows, не сможет просматривать подпапки или файлы в общей папке, если у него нет прав на чтение. Благодаря этому пользователи, подключающиеся к продуктам Synology, не будут видеть большое количество недоступных для них файлов и папок.

      Примечание

      • Данная функция поддерживается только в протоколе обмена файлами Windows.
      • Вы можете настроить параметры прав для каждой папки и файла в File Station, нажав его правой кнопкой мыши. Выберите Свойства в раскрывающемся меню и нажмите вкладку Разрешение.
      • Сама общая папка не будет скрыта от пользователей без прав доступа при включении параметра Скрыть подпапки и файлы от пользователей без прав доступа. Будут скрыты только подпапки и файлы в данной общей папке.
  5. Включить корзину. При удалении файлов из общей папки они будут перемещены в папку #recycle. Доступ к корзине можно ограничить списком пользователей, принадлежащих группе administrators.
  6. Включить квоту общей папки. Чтобы указать максимальную емкость каждой общей папки, установите флажок в поле Включить квоту общей папки и введите максимальную емкость (например, 10 ГБ) в соседнем поле.

    Примечание

    • Параметр Включить квоту общей папки будет доступен для общих папок, только если хранилище использует файловую систему Btrfs.
    • Использование объединенного хранилища домашних каталогов всех пользователей не должно превышать квоту папок homes.
  7. Включите контрольную сумму данных для дополнительной целостности данных. Чтобы защитить общую папку с помощью контрольной суммы CRC32 и метода копирования при записи, при создании общей папки можно включить контрольную сумму данных для дополнительной защиты целостности данных. С помощью метода контрольной суммы CRC32 можно проверить идентичность данных исходным данным, и система будет автоматически использовать избыточное пространство RAID для исправления поврежденных данных. Копирование при записи обеспечивает согласованность данных в случае внезапного сбоя системы.

    Примечание

    • Данной функцией можно воспользоваться только в томах с файловой системой Btrfs.
    • Можно включить/отключить данную функцию при создании каждой общей папки. После создания общей папки изменить значение данного параметра нельзя.
    • После включения этого параметра система сможет выполнять полномасштабное исправление ошибок данных. Подробнее об исправлении ошибок данных
    • Для хорошей работы не рекомендуется включать контрольную сумму данных, если общая папка будет использоваться для следующих целей:
      • Размещение баз данных или виртуальных машин на хост-сервере
      • Хранение видеозаписей Surveillance Station
      • Запуск служб с небольшой и случайной схемой записи
  8. Включить сжатие файлов. Данные в созданной общей папке будут автоматически сжиматься файловой системой Btrfs в целях экономии места хранилища. При извлечении данных для использования они автоматически разархивируются.

    Примечание

    • Этот параметр доступен только при создании общей папки в томе Btrfs или когда включен параметр Усиленная защита целостности данных.
    • После создания общей папки изменить значение данного параметра нельзя.
    • Степень сжатия зависит от типа данных. Текстовые файлы сжимаются сильнее всего, тогда как мультимедийные (например, изображения, музыка или видео) могут сжиматься меньше, поскольку они уже сжаты для вывода.
    • Если включено шифрование общих папок, включить этот параметр нельзя.
  9. Зашифровать общую папку. Чтобы зашифровать общую папку, выполните следующие действия.
    1. Укажите и подтвердите новый ключ шифрования. Ключ шифрования не может содержать запятые (,) и знаки равенства (=).
    2. При необходимости установите флажок Добавить ключ шифрования в Диспетчер ключей, чтобы автоматически подключать зашифрованную папку при запуске системы. Если данный параметр отключен, зашифрованная общая папка будет отключена при следующем запуске Synology NAS. Для просмотра или доступа к удаленной общей папке необходимо подключить ее вручную, выполнив импорт или указав ключ шифрования.
  10. Нажмите OK для завершения.

    11. Примечание

    • Определенные функции и параметры для общей папки поддерживаются только VDSM и моделями со следующими архитектурами пакетов: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview, и Grantley. Поддерживаемую вашей моделью архитектуру пакета см. в данной статье.

Правила именования общих папок.

  • Имена общих папок вводятся с учетом регистра и могут иметь длину от 1 до 32 символов. Имена не могут содержать следующие символы: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Следующие имена зарезервированы для системного использования: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Первый символ в названии общей папки не может быть знаком минуса или пробелом. Последний символ также не может быть пробелом.
  • Общая папка description чувствительна к регистру и может содержать до 64 отображаемых символов в кодировке Unicode.

Шифрование общих папок

256-битное шифрование AES блокирует все попытки несанкционированного доступа к данным. После шифрования общей папки без ключа шифрования другие пользователи не смогут использовать зашифрованные данные даже в случае удаления жестких дисков из устройства Synology NAS и их установки на собственное устройство.

Примечание

  • Имя файла или папки в зашифрованной общей папке может содержать не более 143 букв латинского алфавита или не более 47 символов азиатского алфавита (CJK); при входе в общую папку с помощью ОС Mac имя файла или папки в зашифрованной общей папке может содержать не более 130 букв латинского алфавита или не более 43 символов азиатского алфавита (CJK).
  • К зашифрованным общим папкам нельзя получить доступ по протоколу NFS.
  • При восстановлении паролей по умолчанию с помощью кнопки сброса, расположенной на устройстве Synology NAS, зашифрованные общие папки будут отключены, а параметр автоматического подключения зашифрованных общих папок будет деактивирован.
  • Зашифрованные общие папки поддерживаются только в режиме доступа высокого уровня (применимо только к Dockerx64). Чтобы использовать данную функцию, откройте пакет Docker в DSM хост-сервера, выберите Docker DSM, для которого необходимо включить зашифрованные общие папки, выберите Редактировать и установите флажок Использовать контейнер с правами доступа высокого уровня.
