DDNS

DDNS (динамическая служба доменных имен) позволяет упростить подключение к Synology NAS по Интернету с помощью сопоставления имени хоста с IP-адресом. Например, DDNS предоставляет доступ к Synology NAS с помощью имени домена (например, www.john.synology.me) без необходимости запоминания IP-адреса (например, 172.16.254.1).

Настройка DDNS

Вы можете указать существующее имя хоста IP-адреса устройства Synology NAS, либо зарегистрировать новое имя, предоставленное Synology или другими провайдерами DDNS. Для получения дополнительной информации о регистрации имени хоста обратитесь к соответствующему провайдеру.

Настройка имени хоста DDNS:

  1. Нажмите кнопку Добавить.
  2. Отобразится диалоговое окно с предложением изменить следующие настройки:
    • Поставщик услуг: выберите поставщика услуг. Чтобы зарегистрировать бесплатное имя хоста, предоставленное компанией Synology, выберите Synology в раскрывающемся меню.
    • Имя хоста: введите зарегистрированное имя хоста DDNS, например john.synology.me.
    • Имя пользователя/электронная почта: введите имя пользователя или адрес электронной почты провайдера DDNS.
    • Пароль/ключ: введите пароль или ключ провайдера DDNS.
    • Внешний адрес: введите внешний IP-адрес устройства Synology NAS, для которого будет использоваться имя хоста.
    • Heartbeat: включите данный параметр для получения предупреждений о состоянии сопоставленного имени хоста (только для имен хостов, предоставленных компанией Synology).
  3. Нажмите Тестирование подключения для проверки правильности настроек.
  4. Нажмите OK, чтобы сохранить и завершить.
  5. При выборе Synology в качестве поставщика услуг появится окно сообщения, если для имени хоста DDNS не существует соответствующего сертификата SSL, с рекомендацией подписать сертификат Let's Encrypt для имени хоста и установить его в качестве сертификата по умолчанию. Нажмите Да.

    Примечание. Если сертификат SSL уже сопоставлен с именем хоста DDNS, появится окно сообщения с запросом о необходимости установить этот сертификат в качестве сертификата по умолчанию. Это сообщение появляется при каждом изменении имени хоста DDNS. Дополнительные сведения о сертификате SSL см. здесь.

После завершения настройки вы сможете выполнить доступ к Synology NAS по Интернету, введя имя хоста DDNS в веб-браузере.

Требования и ограничения

  • Серверы поставщика услуг DDNS работают нормально.
  • Synology NAS может подключиться к Интернету.
  • DDNS будет работать только в сетевой среде, в которой не требуется прокси-сервер.
  • Вы можете ввести только одно имя хоста для каждого провайдера DDNS.

Управление именами хостов

Используйте следующие кнопки для управления существующими именами хостов:

  • Удалить: удаление выбранного имени хоста из списка.
  • Редактировать. изменение параметров выбранного имени хоста.
  • Обновить сейчас: обновление состояния всех имен хостов, для которых включена служба DDNS.

Настройка пользовательского поставщика DDNS

Если ваш провайдер не появился в списке поставщиков услуг, вы можете добавить пользовательского поставщика DDNS.

Настройка пользовательского поставщика DDNS:

  1. Нажмите Настроить.
  2. Отобразится диалоговое окно с предложением ввести следующие настройки:
    • Поставщик услуг: Введите название пользовательского поставщика услуг.
    • URL-адрес запроса: Введите URL-адрес пользовательского поставщика услуг.
  3. Нажмите Сохранить, чтобы добавить пользовательского поставщика услуг в список поставщиков DDNS.

Требования и ограничения

  • URL-адрес запроса должен относиться к протоколу, указанному вашим поставщиком услуг. Данный URL-адрес используется для обновления информации об IP-адресе на DDNS-сервере вашего поставщика.
  • Как правило, URL-адрес запроса включает в себя четыре нижеприведенных параметра. При вводе URL-адреса запроса замените следующие параметры переменными величинами.
    • Имя пользователя: __USERNAME__
    • Пароль: __PASSWORD__
    • Имя хоста: __HOSTNAME__
    • Внешний адрес: __MYIP__
  • В URL-адресе запроса не должно быть пробелов или точек с запятой.
