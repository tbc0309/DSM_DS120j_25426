SMB

Протокол SMB (Server Message Block) позволяет клиентам Windows, Mac и Linux с поддержкой SMB/CIFS получать доступ к данным, сохраненным на Synology NAS. Чтобы просмотреть файлы на Synology NAS с помощью Проводника Windows или подключить общие папки как сетевые диски, включите данный параметр в пункте Панель управления > Файловые службы > SMB/AFP/NFS.

Включение протокола SMB

  1. Установите флажок Включить службу SMB.
  2. Нажмите Применить.

Примечание.

  • После включения протокола SMB можно подключать полностью проиндексированные папки к компьютеру Mac по протоколу SMB и использовать Finder для поиска содержимого файлов в этих папках (не применимо к NVR216 или VS360HD)

Рабочая группа

Данный параметр позволяет использовать систему как часть определенной рабочей группы для сетевого управления.

Примечание.

  • Имя рабочей группы может содержать от 1 до 15 символов. Имя не может содержать следующие символы: [ ] ; : »< >* + = \ / | ? ,
  • При использовании Mac с OS X 10.9 и устройства Synology NAS, подключенного к домену Windows, можно выполнить вход, добавив localhost\ перед локальным именем пользователя устройства Synology NAS. После этого вы сможете получать доступ к файлам на Synology NAS по протоколу SMB.

Запрет доступа к предыдущим версиям

В Проводнике Windows пользователи могут получить доступ к моментальным снимкам папок или файлов, открыв вкладку Предыдущие версии в общей папке и разрешив восстановление и копирование данных без помощи администратора. Если выбрать Запрет доступа к предыдущим версиям, на странице «Предыдущие версии» не будет отображаться никакой информации.

Примечание.

  • Данная функция поддерживается только VDSM и моделями со следующими архитектурами пакетов: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview, и Grantley. Поддерживаемую вашей моделью архитектуру пакета см. в данной статье.
  • Пользователи могут получить доступ к предыдущей версии только в том случае, если во время создания моментального снимка, соответствующего предыдущей версии, учетные записи этих пользователей были созданы, и им были назначены права на чтение общей папки, в которой был сделан моментальный снимок.

Включение журнала передачи

Установите флажок Включить журнал передачи, чтобы включить запись журнала файловых операций, выполняемых по протоколу CIFS/SMB. По умолчанию журнал передачи отслеживает только удаление файла или папки; однако можно перейти в раздел Настройки журналов, чтобы выбрать другие операции с файлами — создание, перемещение, переименование, чтение, запись. Обратите внимание, что чем больше операций с файлами выбрано, тем больше влияние на производительность системы.

Дополнительные настройки

Нажмите кнопку Дополнительные настройки для просмотра указанных ниже параметров.

  • WINS-сервер. Microsoft Windows Internet Name Service (WINS) — это услуга сопоставления имени NetBIOS с IP-адресом. Пользователям Windows будет проще найти систему в сетях TCP/IP, если для системы настроена регистрация с WINS-сервером.
  • Максимальный протокол SMB. В зависимости от конфигурации вашей сети установите протокол SMB последней версии, поддерживаемый Synology NAS.
    • SMB1. SMB1 (также называется CIFS) поддерживается, начиная с версии Windows NT.
    • SMB2. SMB2 поддерживается, начиная с версии Windows Vista, и является расширенной версией SMB. SMB2 добавляет возможность объединять различные действия SMB в единый запрос для сокращения числа сетевых пакетов и повышения производительности.
    • SMB2 и Large MTU. Максимальным размером передаваемых данных (MTU) называется самый крупный блок данных, который можно передать по протоколу связи. Для использования преимуществ более быстрых сетевых интерфейсов, например 1- или 10-гигабитного Ethernet, Large MTU увеличивает максимальный размер передаваемых данных до 1 мегабайта (МБ). Включение Large MTU повышает скорость и эффективность передачи больших файлов, сокращая количество обрабатываемых пакетов.
    • SMB3. SMB3 поддерживается, начиная с Windows 8 и Windows Server 2012. Это улучшенная версия SMB2. SMB3 позволяет передавать файлы, зашифрованные по алгоритму AES, что повышает безопасность однорангового обмена файлами.

    Примечание.

    • SMB2 поддерживается на Windows Vista, Windows Server 2008 R2, Windows 7 и более поздних версиях.
    • Large MTU поддерживается на Windows Server 2008 R2, Windows 7 и более поздних версиях.
    • Чтобы обеспечить эффективную работу Large MTU с компьютером Windows, необходимо изменить настройки реестра компьютера. Дополнительную информацию можно найти на веб-сайте технической поддержки Microsoft (например, здесь).
  • Минимальный протокол SMB. В зависимости от конфигурации вашей сети установите протокол SMB самой ранней версии, поддерживаемый Synology NAS.
  • Режим шифрования передачи. При включении SMB3 протокол SMB будет применять дополнительное шифрование передачи для повышения безопасности передачи файлов.
    • Отключить. Шифрование передачи не будет использоваться.
    • Авто. Шифрование передачи будет использоваться только для клиентов, поддерживающих данную функцию.
    • Принудительно. Шифрование передачи будет использоваться всегда. Это приведет к тому, что клиенты, не поддерживающие функцию шифрования передачи, не смогут пользоваться протоколом SMB.

    • Примечание.

    • Шифрование передачи поддерживается в Windows 8, Windows Server 2012 и более поздних версиях.
    • Когда функция шифрования передачи включена, анонимный вход для протокола SMB не поддерживается.
    • При включенной функции шифрования передачи рекомендуется установить флажок Включить уступающую блокировку, чтобы предотвратить случаи ожидания приложения.
  • Включить уступающую блокировку. Выберите этот параметр для включения «уступающей блокировки» CIFS, чтобы клиент мог лучше выполнять кэширование содержимого файлов и избежать незначительных проблем с чтением-записью. При отмене выбора этого параметра приложения базы данных будут более эффективно осуществлять доступ к файлам базы данных, которые сохраняются на Synology NAS по сети.
    • Включить аренду SMB2. Включение данного параметра позволяет настроить дополнительные параметры SMB2 для точной регулировки механизма кэширования клиента посредством уступающей блокировки.
  • Включить Local Master Browser. Если в одной подсети расположено несколько компьютеров Windows, среди них будет выбран один компьютер в качестве Local Master Browser. Local Master Browser содержит список других компьютеров в подсети и их общих ресурсов и обменивается данными списками с другими компьютерами. Данный параметр позволяет Synology NAS выполнять роль Local Master Browser.

    • Примечание.

    • Включение данной функции приведет к отключению режима сна жестких дисков и включит гостевую учетную запись без пароля.
  • Включить модуль DirSort VFS. Сортировка папок в алфавитном порядке. Включение данного параметра влияет на производительность системы.
  • Запрещенные файлы. Скрытие файлов по определенным критериям. Критерии могут включать символы (*); несколько критериев необходимо разделять косой чертой (/). Например: /abc*/*.txt/*.conf/. Включение данного параметра влияет на производительность системы.
  • Разрешить символьные ссылки в общих папках. Включение данного параметра разрешает использование символьных ссылок для обозначения файлов или папок, расположенных в одной и той же общей папке. Символьные ссылки можно создать с помощью терминала DSM.
    • Разрешить символьные ссылки между различными общими папками. Включение данного параметра разрешает использование символьных ссылок для обозначения файлов или папок, расположенных в одной и той же общей папке. Символьные ссылки можно создать с помощью терминала DSM. Однако символьные ссылки позволяют получать доступ к целевым общим папкам без соответствующих разрешений.
  • Включить модуль VFS для преобразования специальных символов Mac. Выбор данного параметра обеспечивает преобразование зарезервированных SMB символов (например, < > " * ?) в именах файлов в символы основной латиницы во время создания файлов. Такие файлы будут распознаваться через AFP.
  • Включить модуль MSDFS VFS. Позволяет символьным ссылкам указывать на файлы и папки на других серверах SMB. В ранних версиях Windows, например Windows XP, перед включением данного параметра клиенту и Synology NAS необходимо присоединиться к домену Windows. Дополнительную информацию о создании ссылок MSDFS см. здесь. Обратите внимание, что кэш клиентского устройства должен быть очищен после отключения данного параметра (рекомендуется также перезагрузить клиентское устройство), в противном случае это может оказать влияние на производительность.
  • Отключить несколько подключений с одного IP-адреса. При включении данного параметра Synology NAS будет поддерживать только одно подключение клиента на IP-адрес с помощью протокола SMB. Если несколько клиентов с одним IP-адресом используют протокол SMB, Synology NAS будет использовать только последнее подключение и отключит предыдущие. В среде NAT клиент может использовать один IP-адрес, в этом случае включение данного параметра может повлиять на производительность.
  • Сбор журналов отладки. При включении данного параметра в целях отладки будет выполняться сохранение подробных журналов. Включение данного параметра влияет на производительность системы.
  • Применение разрешений UNIX по умолчанию. Включение данного параметра позволяет применять разрешения UNIX по умолчанию при загрузке или создании папок и файлов. Для файлов используется разрешение UNIX 744, а для папок — 755. При отключении данного параметра для файлов и папок используется разрешение UNIX 777.

    • Примечание.

    • Для всех общих папок с включенными разрешениями Windows ACL (кроме папки «photo» и папок на внешних дисках) необходимо выполнить команду chmod на клиенте Linux или FTP. В результате типы разрешений для папок и файлов изменятся с Windows ACL на UNIX (не применимо к EDS14).
    • Включение данного параметра может привести к несогласованности разрешений между различными протоколами. Во избежание несогласованности рекомендуется не включать данный параметр.
  • Не резервировать дисковое пространство при создании файлов. Включение данного параметра позволяет не резервировать дисковое пространство при создании файлов на Synology NAS.
  • Включить кэш поиска специальных символов. Включите этот параметр для улучшения эффективности поиска в случае отправки клиентами повторных запросов на поиск специальных символов.
  • Включить надежные обработчики SMB (Блокировка файла кросс-протокола будет отключена). При включении этого параметра пользователи смогут восстановить потерянное подключение по протоколу SMB2 к Synology NAS после временного отключения; во время отключения от сети открытые файлы останутся доступными. При этом межпротокольная блокировка файла будет отключена, чтобы пользователи могли получить доступ к заблокированному файлу или изменить его посредством других протоколов помимо SMB2 (например, AFP).
  • Очистить кэш SMB. Нажмите эту кнопку, чтобы удалить следующие файлы кэша SMB для устранения неисправностей.
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
