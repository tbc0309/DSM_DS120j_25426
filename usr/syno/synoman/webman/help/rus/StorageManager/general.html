Общие

Диспетчер хранения позволяет пользователям управлять всем объемом хранения в продуктах Synology. Чтобы упростить управление хранилищем и создать гибкие объемы хранения с защитой данных, Synology DSM поддерживает несколько типов пулов ресурсов хранения.

Пул ресурсов хранения

Synology NAS позволяет объединять несколько дисков в единый блок хранения, так называемый «пул ресурсов хранения». Synology NAS предоставляет следующие типы пулов ресурсов хранения.

  • Пул ресурсов хранения для повышения производительности.
    • Выделяет всю доступную емкость для одного тома.
    • Обеспечивает лучшую производительность, но меньшую гибкость управления хранилищем.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.
    • Поддерживает не более одного массива RAID.
  • Пул ресурсов хранения для большей гибкости.
    • Позволяет создавать несколько томов в одном пуле ресурсов хранения.
    • Обеспечивает большую гибкость в управлении хранилищем.
    • Позволяет создавать несколько iSCSI LUN (обычные файлы) на томах.
    • Позволяет объединять несколько массивов RAID в один пул ресурсов хранения (в конфигурациях RAID 5, RAID 6 или RAID F1).

Тип RAID

При создании пула ресурсов хранения сначала выберите тип RAID. Различные типы RAID обеспечивают разные уровни защиты данных, возможности хранилища и производительность. В настоящий момент Synology NAS поддерживает RAID следующих типов.

  • Базовый. Для создания пула ресурсов хранения используется один диск. Пулы ресурсов хранения Basic не обеспечивают избыточность данных.
  • JBOD*. В одном пуле ресурсов хранения объединяются не менее двух дисков. Пулы ресурсов хранения JBOD не обеспечивают избыточность данных. Доступная емкость пула ресурсов хранения JBOD равна общей емкости всех дисков, входящих в этот пул. JBOD поддерживает объединение дисков разной емкости.
  • RAID 0*. В одном пуле ресурсов хранения объединяются не менее двух дисков. RAID 0 не обеспечивает избыточности данных. Если один из дисков отказывает, данные в пуле ресурсов хранения будут утеряны. Доступная емкость пула ресурсов хранения RAID 0 равняется общей емкости всех составляющих дисков. RAID 0 поддерживает объединение дисков разной емкости.
  • RAID 1*. В одном пуле ресурсов хранения объединяются 2–4 диска. Система записывает идентичные данные на каждый из дисков. RAID 1 обеспечивает защиту данных, пока хотя бы один диск нормально работает. Доступная емкость пула ресурсов хранения RAID 1 равняется емкости наименьшего из составляющих дисков.
  • RAID 5*. В одном пуле ресурсов хранения объединяются как минимум три диска. Этот пул разделяет данные и информацию о четности между всеми дисками. RAID 5 обеспечивает избыточность данных. При сбое одного из дисков систему можно перестроить с помощью информации о четности, хранящейся на других дисках. Эта конфигурация поддерживает объединение нескольких массивов RAID при создании пула ресурсов хранения для нескольких томов. Доступная емкость пула ресурсов хранения RAID 5 = общая емкость объединенных массивов RAID. Емкость массива RAID = (количество дисков – 1) x (емкость наименьшего диска).
  • RAID 6*. В одном пуле ресурсов хранения объединяются как минимум 4 диска. Этот пул разделяет данные и информацию о четности между всеми дисками. RAID 6 обеспечивает избыточность данных. При сбое двух дисков систему можно перестроить с помощью информации о четности, хранящейся на других дисках. Эта конфигурация поддерживает объединение нескольких массивов RAID при создании пула ресурсов хранения для нескольких томов. Доступная емкость пула ресурсов хранения RAID 6 = общая емкость объединенных массивов RAID. Емкость массива RAID = (количество дисков - 2) x (емкость наименьшего диска).
  • RAID 10*. В одном пуле ресурсов хранения объединяются не менее четырех дисков. Количество дисков должно быть четным. RAID 10 имеет производительность RAID 0 и уровень защиты данных RAID 1. Доступная емкость пула ресурсов хранения RAID 10 = (емкость наименьшего диска) X (количество дисков / 2)
  • RAID F1*. Данный тип RAID предназначен для твердотельных дисков (SSD). Этот пул ресурсов хранения объединяет как минимум три твердотельных диска и разделяет данные и информацию о четности между всеми твердотельными дисками. При этом во избежание одновременного сбоя нескольких твердотельных дисков на один конкретный твердотельный диск записывается большее число битов четности. RAID F1 обеспечивает избыточность данных. При сбое одного из твердотельных дисков систему можно перестроить с помощью информации о четности, хранящейся на других твердотельных дисках. Эта конфигурация поддерживает объединение нескольких массивов RAID при создании пула ресурсов хранения для нескольких томов. Доступная емкость пула ресурсов хранения RAID F1 = общая емкость объединенных массивов RAID. Емкость массива RAID = (количество дисков – 1) x (емкость наименьшего диска).

Примечание.

  • Типы RAID, помеченные символом *, доступны только в системах NAS с достаточным количеством отсеков для дисков. Чтобы узнать, какие типы RAID поддерживаются в вашей модели, см. технические характеристики модели на сайте www.synology.com.
