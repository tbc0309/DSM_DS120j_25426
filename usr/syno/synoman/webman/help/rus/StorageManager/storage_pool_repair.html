Восстановление пула ресурсов хранения

Ошибки в пулах ресурсов хранения возникают по разным причинам, например из-за неисправности дисков. В некоторых случаях для диагностики ошибки и устранения неисправности необходимо выполнить описанные ниже действия.

Функция восстановления, упомянутая в этой статье, доступна только для пулов ресурсов хранения с типами RAID с поддержкой избыточности данных (например, SHR, содержащий не менее 2 дисков, RAID 1, RAID 5, RAID 6, RAID F1 и RAID 10). Обратите внимание, что эта функция может восстановить пул ресурсов хранения, но не может восстановить неисправные диски.

Примечание.

  • Рекомендуется использовать диски, указанные в Списке совместимых продуктов Synology, чтобы избежать ошибок. Использование несовместимых дисков может повлиять на производительность системы и даже привести к потере данных.
  • SHR (Synology Hybrid RAID) поддерживается только в некоторых моделях. Дополнительная информация о SHR представлена здесь.

Восстановление пула ресурсов хранения с неисправными системными разделами

При отказе системных разделов появляется сообщение с информацией о том, что к системным разделам нельзя получить доступ. Выполните указанные ниже действия для устранения проблемы.

  1. Запустите Диспетчер хранения.
  2. Перейдите на страницу Обзор и нажмите ссылку Восстановить. Система начнет восстановление системных разделов.

Восстановление поврежденного пула ресурсов хранения

Пул ресурсов хранения переходит в состояние Поврежденный, если он не может получить доступ к данным на диске, но данные еще не потеряны. Это происходит только в пулах ресурсов хранения с типами RAID, которые обеспечивают избыточность данных (например, SHR, содержащий не менее 2 дисков, RAID 1, RAID 5, RAID 6, RAID F1 и RAID 10). Прежде чем приступить к восстановлению поврежденного пула ресурсов хранения, восстановите неисправные диски с помощью исправных и извлеките неисправные диски (или просто замените их исправными дисками).

Требования к дискам

Замените неисправные диски исправными, соответствующими следующим критериям.

  • Для пулов ресурсов хранения RAID 1, RAID 5, RAID 6, RAID F1 и RAID 10. Размер заменяющего диска должен быть равен размеру наименьшего диска в пуле ресурсов хранения или превышать его.
  • Для пулов ресурсов хранения SHR-1. Заменяющий диск должен соответствовать одному из следующих критериев.
    • Размер заменяющего диска должен быть не меньше размера неисправного диска.
    • При исключении самого большого диска в пуле ресурсов хранения размер заменяющего диска должен быть не меньше размера самого большого из оставшихся дисков.
      Примеры:
      • Пример 1. SHR-1 состоит из четырех дисков емкостью 3 ТБ, 2 ТБ, 2 ТБ и 1 ТБ. При неисправности диска объемом 3 ТБ его можно заменить диском, емкость которого равна или превышает 2 ТБ.
      • Пример 2. SHR-1 состоит из четырех дисков емкостью 4 ТБ, 4 ТБ, 2 ТБ и 2 ТБ. При неисправности диска объемом 4 ТБ его можно заменить диском, емкость которого равна или превышает 4 ТБ.
      • Пример 3. SHR-1 состоит из четырех дисков емкостью 4 ТБ, 4 ТБ, 2 ТБ и 2 ТБ. При неисправности диска размером 2 ТБ его можно заменить диском, объем которого равен или превышает 2 ТБ.
  • Для пулов ресурсов хранения SHR-2. Заменяющий диск должен соответствовать одному из следующих критериев.
    • Размер заменяющего диска должен быть не меньше размера неисправного диска.
    • При исключении двух самых больших дисков в пуле ресурсов хранения размер заменяющего диска должен быть не меньше размера самого большого из оставшихся дисков.

Примечание.

  • Если доступные диски были назначены в качестве дисков Hot Spare для защиты одного или нескольких пулов ресурсов хранения, диски Hot Spare автоматически восстанавливают поврежденные пулы ресурсов хранения. Для получения дополнительной информации см. Hot Spare.

Восстановление поврежденного пула ресурсов хранения

Для восстановления поврежденного пула ресурсов хранения выполните следующие действия.

  1. Перейдите в Диспетчер хранения > Пул ресурсов хранения и найдите пул ресурсов хранения в состоянии Проблемный.
  2. Просмотрите сведения о пуле ресурсов хранения и проверьте состояние диска.
  3. Затем, чтобы посмотреть, на каком диске произошел сбой, перейдите на страницу HDD/SSD.
  4. Извлеките неисправный диск из Synology NAS и замените его новым. Если Synology NAS не поддерживает «горячую замену», отключите его, прежде чем извлекать и устанавливать диски.
    Примечание. Все существующие данные на заменяющем диске будут удалены.
  5. После установки заменяющего диска вернитесь на страницу Пул ресурсов хранения и выберите поврежденный пул ресурсов хранения.
  6. В раскрывающемся меню Действие нажмите Восстановить.
  7. Выберите заменяющий диск, который необходимо добавить в пул ресурсов хранения, и следуйте инструкциям мастера для завершения.

Примечание.

  • Если восстановление пула ресурсов хранения занимает много времени, это может быть связано с указанными здесь причинами.
