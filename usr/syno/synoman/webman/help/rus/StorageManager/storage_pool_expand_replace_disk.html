Замена дисков для расширения емкости системы хранения

При использовании определенных типов RAID емкость пула ресурсов хранения можно расширить, заменив старые диски новыми дисками большего размера. В этой статье приводится важная информация о процессе расширения, а также пример замены существующих дисков для увеличения объема хранения.

Примечание.

  • Процедура, описанная в этой статье, применима только к следующим типам RAID: SHR, RAID 1, RAID 5, RAID 6, RAID 10, RAID F1.
  • SHR (Synology Hybrid RAID) поддерживается только в некоторых моделях. Для получения дополнительных сведений посетите веб-сайт Synology.
  • RAID F1 поддерживается только в некоторых моделях. Для получения дополнительных сведений посетите веб-сайт Synology.
  • Если пул ресурсов хранения поддерживает исправление ошибок данных, рекомендуется выполнить эту процедуру, чтобы убедиться в исправности тома и пула ресурсов хранения до замены дисков.
  • Рекомендуется использовать диски, указанные в Списке совместимых продуктов Synology, чтобы избежать ошибок. Использование несовместимых дисков может снизить производительность системы или даже привести к потере данных на устройстве Synology NAS.

При замене нескольких дисков необходимо заменять диски по одному. После замены одного диска необходимо восстановить пул ресурсов хранения перед тем, как заменять следующий диск. Для указанных ниже типов RAID емкость пула системы хранения будет расширена после замены всех дисков: RAID 1, RAID 5, RAID 6, RAID 10, RAID F1.

Перед началом работы

Перед заменой диска обратите внимание на следующие примечания.

  • Резервное копирование данных. Несмотря на то, что в процессе расширения существующие данные не удаляются, мы настоятельно рекомендуем произвести резервное копирование данных перед тем, как перейти к данной процедуре.
  • Состояние пула ресурсов хранения Перед началом работы убедитесь, что состояние пула ресурсов хранения является Исправным. В противном случае сначала необходимо его/ее исправить.
  • Будут ли данные доступны при расширении? При расширении пулов ресурсов хранения данные в системе будут доступны, но при первом запуске процедуры система может временно приостановить соединения.
  • Использование дисков емкостью 3 ТБ и более. Для расширения пула ресурсов хранения с помощью дисков емкостью 3 ТБ или более убедитесь, что пул ресурсов хранения создан на базе ОС DSM 3.0 или более поздней версии.

SHR (Synology Hybrid RAID)

Поскольку пулы ресурсов хранения SHR оптимизируют объем хранилища в соответствии с размером и количеством установленных дисков, выполните действия ниже для правильного расширения объема хранилища.

  • Если емкость всех существующих дисков одинакова, следует заменить не менее двух дисков на диски большего размера для расширения объема хранилища. Например, если пул ресурсов хранения SHR состоит из трех дисков емкостью 2 ТБ, то для расширения пула ресурсов хранения необходимо заменить не менее двух дисков.
  • Если емкость существующих дисков различна, то новые диски должны быть не меньше самого большого из существующих дисков. Помимо этого, для оптимизации использования хранилища сначала нужно заменить диски самого малого объема. Например, если пул ресурсов хранения SHR состоит из трех дисков емкостью 4, 3 и 2 ТБ, то новые заменяющие диски должны иметь емкость не менее 4 ТБ. Также необходимо сначала заменить диски объемом 3 и 2 ТБ.

RAID 5 / RAID 6 / RAID 10 / RAID F1

При замене дисков в пулах ресурсов хранения RAID 5 / RAID 6 / RAID 10 / RAID F1 сначала необходимо заменить диск самой меньшей емкости для максимального использования диска. Каждый тип RAID имеет следующую емкость хранения:

  • RAID 5: (кол-во дисков – 1) x (размер наименьшего диска)
  • RAID 6: (кол-во дисков – 2) x (размер наименьшего диска)
  • RAID 10: (кол-во дисков / 2) x (размер наименьшего диска)
  • RAID F1: (кол-во дисков – 1) x (размер наименьшего диска)

Например, если пул ресурсов хранения RAID 5 состоит из трех дисков емкостью 4, 3 и 2 ТБ, то доступная емкость пула ресурсов хранения будет равна 4 ТБ. При замене диска емкостью 4 или 3 ТБ емкость пула ресурсов хранения не изменится. Чтобы увеличить емкость пула ресурсов хранения, сначала необходимо заменить диск емкостью 2 ТБ.

Замена диска для расширения емкости системы хранения

Ниже приведен пример замены дисков для пула ресурсов хранения SHR.

Замена дисков

  1. Запустите Диспетчер хранения.
  2. Выберите Пул ресурсов хранения, чтобы узнать, какой из существующих жестких дисков имеет наименьшую емкость.
  3. Выключите Synology NAS. (Если Synology NAS поддерживает горячую замену, пропустите этот шаг.)
  4. Извлеките самый малый диск и установите новый жесткий диск большего размера.
    Примечание. Во избежание получения травм и повреждения сервера Synology NAS следуйте инструкции по установке дисков в Руководстве по установке оборудования вашего устройства Synology.
  5. Включите Synology NAS.
  6. Перейдите в Диспетчер хранения еще раз.
  7. Откройте HDD/SSD, чтобы проверить, распознается ли новый диск.
  8. Пул ресурсов хранения должен находиться в состоянии Поврежден. Выберите пул ресурсов хранения и в раскрывающемся меню Действие нажмите Восстановить.
  9. Выберите замещающий жесткий диск для добавления в пул ресурсов хранения. Выполните инструкции мастера для завершения операции.
  10. Повторяйте описанную выше процедуру до тех пор, пока все необходимые диски не будут заменены дисками большего размера.
