Резервное копирование Synology NAS

Выход из строя жестких дисков, природные катастрофы и атаки программ-вымогателей представляют потенциальную угрозу потери самых важных данных. Чтобы избежать потери данных в подобных ситуациях, широко используется стратегия защиты данных «3-2-1», которая обеспечивает сохранение нескольких версий данных и дополнительную защиту в случае, если текущая версия будет потеряна или повреждена программой-вымогателем.

Стратегия защиты данных «3-2-1» означает создание не менее трех резервных копий данных (одна копия хранится в локальной системе, две остальные копии хранятся на двух удаленных устройствах). Для индивидуальных пользователей: эту стратегию можно использовать для обеспечения защиты корпоративного уровня (для сохранения ценных воспоминаний, например фотографий, семейных видеозаписей и личных альбомов) — либо можно создать две копии данных для обеспечения их доступности. Для бизнес-пользователей: рекомендуется применить стратегию «3-2-1», чтобы обеспечить безопасность всех основных корпоративных данных и важнейших информационных ресурсов.

Эта статья содержит пошаговое руководство для нескольких бесплатных инструментов, доступных в Центре пакетов в Synology DSM, которые позволяют разработать собственное решение защиты данных «3-2-1» (в том числе Hyper Backup, Snapshot Replication, USB Copy и Cloud Sync).

1. Резервное копирование в той же системе Synology NAS (для создания локальной копии с несколькими версиями)

Локальная копия с несколькими версиями позволяет быстро восстанавливать данные в случае их повреждения или случайного удаления, поскольку данные можно извлечь прямо из системы Synology NAS. Однако даже если принять эту меру защиты, сбой в системе RAID или выход из строя жесткого диска может привести к необратимой потере данных. Поэтому настоятельно рекомендуется создать локальную копию с несколькими версиями, чтобы быстро восстанавливать данные, а также выполнить резервное копирование на удаленные устройства для дополнительной защиты данных.

Приведенные далее в этом разделе инструкции содержат короткое описание пакетов Hyper Backup и Snapshot Replication. Можно выбрать наиболее подходящий пакет в соответствии с вашими предпочтениями.

Hyper Backup позволяет выполнять резервное копирование приложений и системных конфигураций, а также восстанавливать их в случае необходимости для обеспечения согласованности данных. Кроме того, этот пакет также позволяет выполнять инкрементное резервное копирование на уровне блоков. Это означает, что копируются только файлы, которые изменены по сравнению с первоначальными версиями, а переименованные или дублирующиеся файлы удаляются с помощью функции дедупликации файлов для экономии дискового пространства.

Если ваша модель системы NAS поддерживает файловую систему Btrfs1, то можно создать локальную копию с помощью пакета Snapshot Replication. Операция создания снимка выполняется всего за несколько секунд и не влияет на производительность системы. По сравнению с Hyper Backup, Snapshot Replication выполняет резервное копирование и восстановление быстрее; кроме того, это решение можно применить к общим папкам и iSCSI LUN.

2. Резервное копирование в другую систему

Чтобы защититься от потери данных, вызванной сбоем в системе RAID или повреждением аппаратного обеспечения, можно создать внешнюю копию данных на самых разных устройствах. В следующих разделах содержится описание некоторых инструментов, с помощью которых можно выполнить подобные задачи резервного копирования.

2.1 На внешние устройства (USB)

Резервное копирование на USB-накопитель — это простой и удобный способ благодаря широкому распространению подобных устройств. Этот способ резервного копирования доступен для различных файловых систем и типов внешних устройств (например, можно выполнить резервное копирование важных данных на RDX).

Hyper Backup позволяет выполнять резервное копирование приложений и системных конфигураций, а также восстанавливать их точные копии в случае необходимости. Кроме того, этот пакет также позволяет выполнять инкрементное резервное копирование на уровне блоков. Это означает, что копируются только файлы, которые изменены по сравнению с первоначальными версиями, а переименованные или дублирующиеся файлы удаляются с помощью функции дедупликации файлов для экономии дискового пространства.

В отличие от Hyper Backup, USB Copy не поддерживает резервное копирование приложений и системных конфигураций, а отсутствие функции дедупликации файлов может снизить эффективность использования хранилища. Однако данный пакет может запустить резервное копирование данных при подключении внешнего устройства, а по завершении процедуры копирования может извлечь это устройство. Кроме того, формат скопированных с помощью USB Copy данных доступен для чтения, а формат данных Hyper Backup не доступен для чтения.

  • Hyper Backup
  • USB Copy

2.2 В другую систему Synology NAS

При резервном копировании в другую систему Synology NAS данные передаются по протоколу Synology NAS.

С помощью представленного выше пакета Hyper Backup, который содержит встроенную технологию для эффективного использования хранилища с поддержкой резервного копирования приложений и системных конфигураций, можно с легкостью скопировать данные на удаленное устройство NAS. Для этого требуется установить Hyper Backup Vault на целевой сервер (этот пакет позволяет просмотреть принятые задачи резервного копирования и журналы для каждой задачи на принимающем устройстве). Hyper Backup поддерживает файловые системы EXT4 и Btrfs, однако формат копируемых данных недоступен для чтения.

Snapshot Replication обеспечивает удобную репликацию снимков в несколько удаленных систем Synology NAS. Выполняется репликация только измененных блоков данных для сокращения используемой пропускной способности и экономии времени. Для дополнительной гибкости управления одну репликацию можно применить к трем удаленным серверам. Следует иметь в виду, что пакет Snapshot Replication доступен только для моделей с поддержкой файловой системы Btrfs1.

  • Hyper Backup
  • Snapshot Replication

2.3 На файловые серверы

Этот способ резервного копирования поддерживает протоколы rsync, HTTP, WebDAV, OpenStack и S3. Он требует больше дискового пространства на стороне клиента для сохранения кэша (по сравнению с пространством для резервного копирования в другую систему Synology NAS).

Hyper Backup позволяет копировать данные на файловые сервера по протоколам rsync и HTTP (WebDAV и OpenStack). Благодаря поддержке нескольких версий этот пакет обеспечивает дополнительную защиту данных, а дедупликация версий сокращает емкость, необходимую для хранения нескольких версий. Кроме того, поддержка резервного копирования приложений и системных конфигураций также позволяет с легкостью восстанавливать изначальные настройки.

2.4 В облачные сервисы

Резервное копирование данных в облачные сервисы — наиболее гибкий способ внешнего резервного копирования, особенно с точки зрения экономии пространства и требований к первоначальному приобретению и обслуживанию оборудования для хранения данных. Организации могут увеличивать и уменьшать облачное пространство в соответствии с требованиями — без дополнительных инвестиций в развертывание ИТ-систем и настройку оборудования.

Hyper Backup позволяет копировать данные в облачный сервис Synology — Synology C2, интегрированное решение, которое обеспечивает доступность данных в облаке для пользователей Synology NAS. Благодаря возможности выбора различных тарифных планов вы можете получать доступ к файлам и восстанавливать их на любом браузере, а используемая технология шифрования обеспечивает дополнительную безопасность данных.

Помимо облачного сервиса Synology можно также выполнить резервное копирование данных в другие общедоступные облачные службы. Hyper Backup не только позволяет создавать резервные копии с несколькими версиями в общедоступных облачных службах, но также обеспечивает согласованность данных резервной копии благодаря копированию приложений и системных конфигураций.

Cloud Sync также можно использовать в качестве решения для синхронизации в реальном времени данных на Synology NAS и в публичных облачных сервисах (если функция односторонней загрузки включена). Однако этот пакет сохраняет только текущую версию всех файлов на Synology NAS (при этом если облачный сервис поддерживает управление версиями, возможность резервного копирования нескольких версий сохраняется). Кроме того, синхронизированные в каталоге назначения файлы сохраняют свой исходный формат.

3. Сравнение решений резервного копирования, представленных в этой статье

Чтобы предотвратить потерю данных из-за выхода из строя жесткого диска, природных катастроф и атак программ-вымогателей, настоятельно рекомендуется использовать решения внешнего резервного копирования. Среди таких решений оптимальным является копирование в облачные сервисы (для пользователей, не имеющих специального оборудования). Облачные сервисы позволяют сэкономить на аппаратном обеспечении, обслуживании и электричество, а также сократить амортизационные расходы. Помимо этого, пользователи могут выбрать наиболее оптимальное решение в соответствии со своими потребностями, поскольку некоторые поставщики облачных сервисов предоставляют гибкие тарифные планы. Однако если у вас уже есть необходимые устройства (например, USB-накопители), другие системы Synology NAS или файловые серверы, то можно воспользоваться существующими системами хранения для резервного копирования данных с NAS (при этом рекомендуется расположить целевое устройство на определенном расстоянии от исходной системы NAS, чтобы защитить данные от потери).

Внешнее резервное копирование данных, несомненно, повышает уровень их защиты, однако рекомендуется также иметь локальную копию файлов для быстрого восстановления. В связи с этим рекомендуется воспользоваться функциями Hyper Backup или Snapshot Replication для создания копии данных NAS в этой же системе Synology NAS или на других устройствах, расположенных вблизи от исходной системы для эффективного восстановления.

Благодаря реализации стратегии «3-2-1» и созданию как минимум двух копий файлов вы можете быть уверены, что ваши данные защищены и всегда доступны.
