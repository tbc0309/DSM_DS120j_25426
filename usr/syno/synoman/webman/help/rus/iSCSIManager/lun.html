LUN

Прежде чем создать LUN, сначала в Диспетчере хранения необходимо создать том.

Типы LUN с различными функциями отображаются разными цветами.

  • LUN с функцией Thin Provisioning, созданные в томе ext4.
  • LUN с функцией Thin Provisioning, созданные в томе Btrfs.
  • LUN с функцией Thick Provisioning.

Важно!

  • Только LUN, созданные в томе Btrfs в DSM 6.2 и более поздних версиях, поддерживают новые расширенные функции и моментальные снимки. LUN в томе ext4 поддерживают только прежнюю версию расширенных функций.

Создание LUN

  1. Нажмите Создать.
  2. Установите свойства этого LUN.
    • Имя LUN
    • Местоположение: Местоположение не может быть изменено после создания LUN, однако пользователь может скопировать этот LUN на другой том.
    • Емкость. Размер LUN должен составлять не менее 1 ГБ. Для LUN с прежней версией расширенных функций размер должен составлять не менее 10 ГБ.
    • Выделение пространства. Функция Thick Provisioning обеспечивает более высокую стабильность и производительность, а функция Thin Provisioning поддерживает выделение ресурсов по требованию и все расширенные функции, но может привести к сбою файловой системы LUN, когда емкость тома заканчивается.
    • Расширенные функции. LUN с функцией Thick Provisioning поддерживают только функции Hardware-Assisted Zeroing и Hardware-Assisted Locking, а LUN на томах ext4 поддерживают только прежнюю версию расширенных функций.
  3. Следуйте инструкциям мастера по выполнению процесса.

Примечание.

  • Невозможно будет создать новые LUN, если в томе остается меньше 1 ГБ свободного места.
  • Можно включить расширенные функции для поддержки соответствующих функций VMware VAAI.
  • Можно включить Hardware-Assisted Data Transfer, чтобы LUN могли одновременно поддерживать Windows Offload Data Transfer (ODX).
  • Включение прежней версии расширенных функций может повлиять на производительность операций ввода-вывода.
  • Количество поддерживаемых LUN зависит от моделей Synology NAS. Подробные характеристики продукта представлены на сайте www.synology.com.

Терминология:

  • Hardware-Assisted Zeroing. Помогает хостам VMware ESXi инициализировать виртуальные диски с большей эффективностью при создании виртуальных дисков.
  • Hardware-Assisted Locking. Блокирует только требуемые компоненты LUN, когда хост VMware ESXi получает доступ к данным на нем. Это позволяет нескольким хостам VMware ESXi одновременно получать доступ к одним и тем же LUN без снижения производительности.
  • Hardware-Assisted Data Transfer. Передача данных быстро осуществляться на Synology NAS без использования ресурсов хостов при копировании данных на хостах VMware ESXi и Microsoft Hyper-V.
  • Моментальный снимок. Функции файловой системы Btrfs позволяют использовать моментальные снимки и репликацию.
  • Реорганизация пространства. Поддержка команды реорганизации пространства инициатора. При удалении или передаче данных освобождается пространство, которое они используют. Однако это может повлиять на производительность операций ввода-вывода.

Удаление LUN

При удалении LUN все данные на LUN будут удалены. Подключения между LUN и любыми сопоставленными Target также будут утеряны.

  1. Выберите LUN, который хотите удалить.
  2. Нажмите кнопку Удалить и следуйте инструкциям мастера.

Редактирование LUN

  1. Выберите LUN, который хотите отредактировать.
  2. В раскрывающемся меню Действие выберите Редактировать.
  3. Выполните редактирование необходимых свойств.
  4. Нажмите OK, чтобы сохранить настройки.

Примечание.

  • Избежать потери данных при редактировании емкости LUN можно, увеличив эту емкость.
  • Если состояние LUN меняется на Недоступно, значит, при доступе к LUN произошли ошибки, связанные с данными. Во избежание серьезной потери данных в системе этот LUN будет временно недоступен. В этом случае необходимо как можно скорее связаться со специалистами службы поддержки Synology.

Если Synology NAS используется в качестве внутреннего хранилища для OpenStack Cinder, для создания томов Cinder будут использоваться LUN с командами ускорения хранения. Операции с LUN/Target, которые используются томами Cinder, и управление ими на Synology NAS будут ограничены.

  • Cinder LUN нельзя редактировать.
  • Невозможно изменить такие сведения, как имя, IQN, параметры сопоставления и маски Cinder Target.
  • Сопоставить Cinder LUN и Cinder Target с общими LUN и Target также невозможно.
  • Невозможно создавать снимки Cinder LUN в интерфейсе пользователя DSM, а также изменять, удалять и восстанавливать снимки Cinder LUN.
  • LUN, клонированные на основе LUN и моментальных снимков Cinder, будут общими LUN.

Клонирование LUN

  1. Выберите LUN, который требуется клонировать.
  2. В раскрывающемся меню Действие выберите Клонировать.
  3. Измените имя клонированного LUN, затем выберите целевой том.
  4. Нажмите OK для начала клонирования.

Примечание.

  • LUN нельзя клонировать, если оставшийся объем тома составляет менее 1 ГБ.

Дефрагментация LUN.

Дефрагментация может восстановить производительность LUN, ухудшившуюся из-за фрагментации данных или создания снимков. Во время дефрагментации производительность ввода-вывода LUN может снизиться, но вы можете в любой момент остановить процесс и возобновить его позднее.

  1. Выберите LUN, для которого требуется выполнить дефрагментацию.
  2. Нажмите Дефрагментация > Запуск.

Остановка дефрагментации LUN

  1. Выберите LUN, для которого выполняется дефрагментация.
  2. Нажмите Дефрагментация > Остановить.

Примечание.

  • Дефрагментация поддерживается только для LUN на томе Btrfs с Thin Provisioning.
  • Во время выполнения дефрагментации вы не можете редактировать/клонировать LUN или создавать снимки, однако можете удалить LUN.
  • Дефрагментацию можно остановить в любое время. Система сохранит данные о ходе выполнения дефрагментации и продолжит ее выполнение с момента последней остановки.
  • Существующие снимки LUN потребуют дополнительное пространство тома после дефрагментации LUN.

Дополнительные сведения

  • LUN с включенной функцией создания моментальных снимков или прежней версией расширенных функций можно быстро клонировать на одном и том же томе.
  • LUN с новыми расширенными функциями можно клонировать только на тома Btrfs.
  • При клонировании LUN с прежней версией расширенных функций на том Btrfs выполняется их преобразование в LUN с новыми расширенными функциями LUN. При клонировании на другой том ext4 прежняя версия расширенных функций LUN будет отключена.
  • Свойства LUN не изменяются после клонирования (кроме LUN с прежней версией расширенных функций).
  • Во время клонирования LUN без поддержки моментальных снимков или прежней версии расширенных функций рекомендуется отключить эти LUN, чтобы предотвратить ошибки клонирования или несогласованность данных.

Преобразование LUN с прежней версией расширенных функций

  1. Выберите LUN для преобразования.
  2. В раскрывающемся меню Действие выберите Преобразовать.
  3. Настройте параметры.
    • Имя LUN
    • Местоположение: Местоположение не может быть изменено после создания LUN, однако пользователь может преобразовать этот LUN на другой том Btrfs.
  4. Установите флажок, если вы хотите удалить исходный LUN и сопоставить Target с новым LUN после преобразования.
  5. Нажмите Применить для подтверждения настроек.

Примечание.

  • Только LUN с прежней версией расширенных функций могут быть преобразованы в LUN с новыми расширенными функциями.
  • LUN можно преобразовать только на томах Btrfs с достаточным объемом свободного пространства.
  • Для обеспечения согласованности данных привязка LUN к Target будет удалена во время преобразования.
  • В преобразованном новом расширенном LUN не будет моментальных снимков.

Терминология:

  • Thin Provisioning. Thin Provisioning — это способ оптимизации места хранения с помощью динамического распределения места или распределения по требованию.
  • VMware VAAI. VAAI (VMware vStorage APIs for Array Integration) позволяет перенести выполнение стандартных операций с виртуальных массивов хранения на устройство iSCSI в среде VMware vSphere и оптимизировать производительность системы хранения.
  • Windows ODX. Offload Data Transfer (ODX) — это новая технология Microsoft для передачи данных в Windows Server 2012 и 8. ODX повышает скорость передачи за счет перераспределения части нагрузки на массив хранения.
    Если источник и место назначения данных расположены в LUN на одном томе, клонирование будет производиться при помощи команд ускорения хранения, что позволяет одновременно сэкономить 99,9% пространства для хранения и повысить скорость клонирования.
  • Сопоставление Target. Сопоставление LUN с одной или несколькими целями Target.
