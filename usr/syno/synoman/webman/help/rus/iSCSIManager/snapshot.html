Snapshot

Порядок создания снимков LUN:

Снимок LUN представляет собой копию LUN на определенный момент времени, доступную только для чтения.

  1. Выберите LUN, снимки которого требуется сделать.
  2. Нажмите Моментальный снимок и выберите Сделать моментальный снимок.
  3. Введите описание моментального снимка.
  4. Выберите один из следующих типов моментальных снимков в соответствии со своими потребностями:
    • С учетом используемого приложения. Данный тип снимков доступен, только если на сервере Windows или VMware vCenter Server установлен Synology Storage Console или Synology Snapshot Manager для обмена данными с DSM. Когда моментальные снимки с учетом используемого приложения запущены на DSM, на приложения отправится уведомление и все данные из памяти будут перенаправлены на LUN, на котором выполнен мгновенный снимок, для обеспечения единообразия данных.
    • С учетом сбоя. Если ни на одном клиенте не установлен Storage Console или Snapshot Manager, вы можете создавать снимки LUN только с учетом сбоя. При их запуске на приложения не будет отправлено уведомление по перенаправлению данных из памяти на LUN, поэтому согласованность данных не гарантируется.
  5. Нажмите OK.

Примечание.

  • Рекомендуется установить Synology Storage Console или Snapshot Manager, чтобы избежать возможной несогласованности данных.

Терминология:

  • Synology Storage Console Synology Storage Console — это плагин для управления хранилищем для VMware vSphere Client или сервера Windows, который упрощает мониторинг и управление несколькими системами хранения Synology. Кроме того, он позволяет создавать снимки с учетом используемого приложения непосредственно из DSM.
    • Для сред VMware плагин совместим только с устройством VMware vCenter Server 6.5 U2 или более поздних версий.
    • Вы можете скачать плагин в Центре загрузок и получить дополнительную информацию в разделах Synology Storage Console for VMware и Synology Storage Console for Windows.
  • Synology Snapshot Manager Synology Snapshot Manager — это подключаемый к программному обеспечению плагин, который позволяет создавать снимки с учетом используемого приложения непосредственно в DSM.
    • Для сред VMware плагин совместим только с VMware vCenter Server 6.0 или предыдущих версий.

Удаление фотоснимка:

  1. Выберите LUN, снимки которого сделаны.
  2. Нажмите Снимок > Список снимков.
  3. Выберите снимок, который необходимо удалить, и нажмите кнопку Удалить.

Примечание.

  • Реорганизация пространства требует времени для обработки после удаления снимков.

Порядок восстановления LUN

Выберите снимок на определенный момент времени для восстановления данных на LUN.

  1. Выберите LUN, который требуется восстановить.
  2. Нажмите Снимок и выберите Список снимков.
  3. Выберите моментальный снимок для восстановления и нажмите кнопку Восстановить.

Клонирование LUN из снимков

Создайте копию снимка LUN или LUN с возможностью записи.

  • В Список снимков выберите снимок LUN и нажмите Клонировать.

Примечание.

  • Функции LUN по созданию снимка, восстановлению и клонированию доступны только на LUN с командами ускорения хранения.
  • Если после создания моментального снимка или клона состояние изменится на Ненормально/Повреждение, удалите данный моментальный снимок или клон и создайте другой.
  • Все изменения LUN после создания моментального снимка будут утеряны при восстановлении LUN с помощью данного моментального снимка.
  • Восстановление и клонирование LUN, а также создание снимков и клонирование снимков LUN и другие функции недоступны, если в томе меньше 1 ГБ свободного места.

Политика хранения

Это оптимальный способ определения максимального количества версий снимков, позволяющий экономить пространство в томе. Однако при этом может потребоваться увеличить период хранения снимков. В отношении политики хранения необходимо обратить внимание на следующее.

  • В компании Synology принята политика хранения данных GFS (Grandfather-Father-Son). Можно настроить хранение максимального количества версий снимков в течение следующих диапазонов времени соответственно: каждый час, ежедневно, еженедельно, ежемесячно и ежегодно.
  • Один снимок может одновременно соответствовать нескольким правилам хранения. Например, один моментальный снимок может соответствовать правилам ежедневного, еженедельного и ежемесячного хранения.
  • Самые ранние снимки сохраняются, если в течение определенного периода будет создано несколько снимков. Например, если задано правило хранения для 10 еженедельных снимков и расписание создания снимков настроено на создание 1 снимка в день, то Snapshot Replication будет сохранять самый ранний снимок каждую неделю в течение последних 10 недель.
  • Если снимки не были созданы в течение определенного периода времени, система сохранит последние снимки. Например, если правило хранения настроено для создания 15 ежедневных снимков, а за последние 30 дней не было создано ни одного снимка, система сохранит снимки, созданные 30–45 дней назад.
  • По умолчанию сохраняются все версии снимков, созданные в течение часа с момента создания последнего снимка. Это позволяет упростить поиск и восстановление данных из последних версий снимков. Также можно заблокировать снимок, чтобы предотвратить его автоматическое удаление согласно правилам хранения.

Пример 1.
Если требуется хранить ежедневные снимки за последние 3 месяца, еженедельные снимки за последние 4–6 месяцев и ежемесячные снимки за последние 7–9 месяцев, необходимо ввести 0 для ежечасных снимков, 90 для ежедневных снимков, 24 для еженедельных снимков, 9 для ежемесячных снимков и 0 для ежегодных снимков.

Пример 2.
Если требуется хранить ежечасные снимки за последние 2 дня, ежедневные снимки за последнюю неделю, еженедельные снимки за последний месяц и ежемесячные снимки за последний год, необходимо ввести 48 для ежечасных снимков, 7 для ежедневных снимков, 4 для еженедельных снимков, 12 для ежемесячных снимков и 0 для ежегодных снимков.
