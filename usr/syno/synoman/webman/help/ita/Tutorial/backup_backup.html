Eseguire il backup di Synology NAS

La perdita di dati imprevista, dovuta a guasti al disco rigido, danni naturali e attacchi ransomware comporta potenziali minacce ai dati più importanti. Per evitare la perdita di dati causata da queste situazioni, sarà adottata ampiamente la strategia di protezione dati 3-2-1 per mantenere più versioni dei dati di backup, fornendo quindi una maggiore protezione in caso di crash della versione corrente o di attacchi ransomware.

Una strategia di protezione dati 3-2-1 comporta la disponibilità di almeno tre copie di dati, di cui una viene archiviata a livello locale, mentre le altre due sono archiviate in due dispositivi differenti. Gli utenti privati possono adottare questa strategia per ottenere una protezione dati di livello aziendale per importanti ricordi, come foto, video di famiglia e album personalizzati, oppure scegliere di mantenere almeno due copie di dati per assicurarsene la disponibilità. Come per gli utenti business, la strategia 3-2-1 è consigliata per garantire che tutti i dati business core e le informazioni critiche siano sempre protetti.

In questo articolo sono descritti numerosi strumenti gratuiti disponibili nel Centro pacchetti DSM di Synology, in grado di creare la propria soluzione di protezione dati 3-2-1, fra cui Hyper Backup, Snapshot Replication, USB Copy e Cloud Sync.

1. Eseguire il backup nello stesso Synology NAS per ottenere una copia locale multi-versione

Una copia locale multiversione offre il vantaggio di un ripristino veloce in caso di danneggiamento dei dati o di eliminazione accidentale dei file. Infatti la copia dati può essere recuperata direttamente da Synology NAS. Tuttavia, anche nel caso sia adottata questa misura, il crash del RAID o guasti al disco rigido potrebbero comunque causare disastrose perdite di dati. Pertanto, è altamente consigliato creare una copia locale multiversione per garantire un ripristino efficiente e, nel contempo, il backup su una destinazione differente per ottenere un'ulteriore protezione dati.

Il contenuto di questa sezione, riportato di seguito, fornisce una breve presentazione di Hyper Backup e Snapshot Replication. Il pacchetto può essere scelto in base alle funzioni e alle esigenze.

Hyper Backup consente di eseguire il backup delle applicazioni e delle configurazioni di sistema, nonché di ripristinarle quando necessario per garantire la consistenza dei dati. Inoltre, questo pacchetto consente anche il backup incrementale di livello blocco, vale a dire che solo i file modificati rispetto alle versioni iniziali saranno sottoposti a backup, mentre i file semplicemente rinominati o duplicati saranno eliminati tramite deduplicazione file trasversale per evitare inutili consumi dello spazio di archiviazione.

Se il modello NAS in uso supporta il file system Btrfs1, è possibile creare una copia locale mediante Snapshot Replication. Eseguito in pochissimi secondi, il processo snapshot causa solo impatti minimi sulle prestazioni di sistema. Rispetto a Hyper Backup, Snapshot Replication è più veloce in termini di tempi di ripristino/velocità di backup e, inoltre, può essere applicato a cartelle condivise e iSCSI LUN.

2. Backup su una destinazione diversa

Per proteggersi dalla perdita dati causata da crash RAID o danni hardware, è possibile scegliere di creare una copia offsite in varie destinazioni. Nelle sezioni seguenti è fornita una panoramica di alcuni strumenti da usare per completare le attività di backup.

2.1 Su dispositivi esterni (USB)

Il backup su USB è semplice e conveniente, grazie all'ampio uso di questo dispositivo. Questo tipo di backup supporta vari file system e dispositivi esterni, come il backup di informazioni critiche su RDX.

Hyper Backup consente di eseguire il backup delle applicazioni e delle configurazioni di sistema, nonché di ripristinare gli stessi identici dati quando necessario. Inoltre, questo pacchetto consente anche il backup incrementale di livello blocco, vale a dire che solo i file modificati rispetto alle versioni iniziali saranno sottoposti a backup, mentre i file semplicemente rinominati o duplicati saranno eliminati tramite deduplicazione file trasversale per evitare inutili consumi dello spazio di archiviazione.

Rispetto a Hyper Backup, USB Copy non supporta il backup di applicazioni o configurazioni di sistema e la mancanza della deduplicazione file trasversale potrebbe ridurre l'efficienza di archiviazione, ma questo pacchette può iniziare il backup dati non appena un dispositivo esterno viene collegato ed espelle il dispositivo al termine della procedura di backup. Inoltre, il formato dei dati sottoposti a backup tramite copia USB è leggibile, mentre la controparte in Hyper Backup non lo è.

2.2 Su un altro Synology NAS

Se il backup è eseguito su un altro Synology NAS, i dati vengono trasferiti tramite protocollo NAS di Synology.

Con il summenzionato Hyper Backup, dotato di tecnologia integrata per migliorare l'efficienza di archiviazione, nonché il supporto backup di configurazioni e applicazioni, sarà possibile eseguire facilmente il backup di dati su un dispositivo NAS remoto installando Hyper Backup Vault nella destinazione, un pacchetto che consente di visualizzare le attività di backup ricevute e i log relativi alle attività nel dispositivo ricevente. Hyper Backup supporta entrambi i file system EXT4 e Btrfs, ma il formato dei dati di backup non è leggibile.

Snapshot Replication fornisce la possibilità di replicare le snapshot su più Synology NAS remoti, replicando soltanto i blocchi dati modificati per ridurre la larghezza di banda e i tempi. Per la flessibilità di gestione, una replica può essere estesa fino a tre siti remoti. Inoltre, Snapshot Replication è applicabile solo a modelli che supportano il file system Btrfs1.

2.3 Su server file

Questo tipo di backup supporta la risincronizzazione, HTTP, WebDAV, OpenStack, oltre ai protocolli S3 e consuma più spazio di archiviazione su lato client per la cache rispetto al volume occupato quando si esegue il backup su un altro Synology NAS.

Hyper Backup consente di eseguire il backup dei dati su server file tramite i protocolli rsync e HTTP (WebDAV e OpenStack), mentre il suo supporto multiversione consente di migliorare la protezione dei dati e la deduplicazione a versione trasversale riduce il volume richiesto per archiviare più versioni. Inoltre, il supporto il backup di configurazioni di applicazione e di sistema consente anche di ripristinare facilmente le impostazioni originali.

2.4 Su servizi cloud

Il backup dati su servizi cloud rappresenta la scelta offsite più flessibile, specialmente per l'occupazione dello spazio, oltre che per l'acquisto iniziale e la manutenzione dei dispositivi di archiviazione. Le aziende potranno effettuare ampliamenti o riduzioni, in base ai cambiamenti, senza ulteriori investimenti in distribuzione IT o adeguamenti hardware.

Hyper Backup consente di eseguire l backup di dati su servizio cloud Synology Synology C2, una soluzione ottimale in grado di garantire la disponibilità dei dati nel cloud per gli utenti Synology NAS. Grazie ai diversi piani di acquisto fra cui scegliere, l'utente potrà accedere e ripristinare file in qualsiasi browser, sfruttando la sicurezza dati garantita dalla tecnologia di crittografia.

Oltre al servizio cloud di Synology, potrebbe anche essere necessario seguire il backup dei dati in altri cloud pubblici. Hyper Backup non solo consente di creare il backup multiversione su cloud pubblici, ma anche di garantire la consistenza dei dati di backup, oltre al backup di applicazioni e configurazioni di sistema.

Cloud Sync può essere impiegato anche come soluzione in tempo reale per sincronizzare i dati Synology NAS su cloud pubblici se la funzione di caricamento a una via è abilitata. Tuttavia, questo pacchetto mantiene solo la versione corrente di tutti i file in Synology NAS, ma se il servizio cloud utilizzano supporta il versioning, sarà ancora possibile creare attività di backup multiversione. Inoltre, i file sincronizzati nella destinazione restano dello stesso formato dei dati di origine.

3. Confrontare le soluzioni di backup presentate in questo articolo

Per evitare perdite di dati causate da guasti a dischi rigidi, danni naturali e attacchi ransomware, sono consigliate le soluzioni di backup offsite su cloud. Fra queste, il backup su cloud è la scelta ideale per gli utenti senza alcuna apparecchiatura dal momento che i servizi cloud consentono di ottimizzare i costi di hardware, manutenzione ed elettricità, nonché le spese di ammortamento. Inoltre, gli utenti possono anche scegliere soluzioni più specifiche alle proprie esigenze, ad esempio scegliendo provider di servizi cloud che offrono piani di acquisto flessibili. Tuttavia, se si dispone già di dispositivi come unità USB, altri Synology NAS o server file, sarà comunque possibile sfruttare lo spazio per eseguire il backup dei dati NAS, ma occorre posizionare il dispositivo di destinazione lontano dal NAS di origine, come protezione da potenziali perdite di dati.

Le copie offsite dei dati aumentano il livello di protezione, ma è importante anche disporre di una copia locare per un ripristino veloce. Pertanto, è consigliabile impiegare le funzioni di Hyper Backup o Snapshot Replication per creare una copia dei dati NAS nello stesso Synology NAS o in altri dispositivi, da posizionare vicino ai dati di origine per ottenere migliori prestazioni di ripristino.

Dopo avere approfondito la strategia 3-2-1 e avere creato almeno due copie di dati, questi saranno subito disponibili e protetti.
