Creazione di una cartella condivisa

Le cartelle condivise sono directory di base dove è possibile salvare e gestire i file e cartelle sulla Synology NAS. Prima di salvare qualsiasi file, è necessario creare almeno una cartella condivisa in DSM.

Per creare una cartella condivisa:

  1. Andare su Pannello di controllo > Cartella condivisa e fare clic su Crea.
  2. Specificare un nome e una descrizione per la cartella condivisa.
  3. Selezionare un volume in cui creare la cartella condivisa (saltare questo passaggio solo se esiste un solo volume).
  4. Modificare le seguenti opzioni, se necessario:
    • Nascondere questa cartella condivisa in "Risorse di rete": Evitare che la cartella condivisa appaia in “Rete” in Windows File Explorer. Questa opzione non influisce sui privilegi di accesso della cartella condivisa.

      Nota:

      • Nascondere una cartella condivisa in "La Mia Rete" non influisce sui diritti di accesso ad essa. Il diritto di accesso di una cartella condivisa nascosta resta invariato. Gli utenti che dispongono dei diritti di accesso alla cartella condivisa possono ancora accedervi immettendo "\nome server\nome cartella condivisa".
    • Nascondi sottocartelle e file da utenti senza autorizzazioni: Quando viene abilitato Nascondi sottocartelle e file da utenti senza autorizzazioni, se un utente che non dispone dei privilegi di lettura tenta di accedere a una cartella condivisa tramite il protocollo Condivisione file di Windows, l'utente non sarà in grado di visualizzare le cartelle o i file nella cartella condivisa. In questo modo si evita di confondere gli utenti quando si connettono ai prodotti Synology e non possono accedere a file e sottocartelle.

      Nota:

      • questa funzione è supportata solo tramite il protocollo di condivisione file di Windows.
      • È possibile modificare le impostazioni dei privilegi di ciascuna cartella e file in File Station facendo clic con il tasto destro sull'elemento. Scegliere Proprietà dal menu a discesa e fare clic sulla scheda Autorizzazione.
      • La cartella condivisa stessa non sarà nascosta dagli utenti senza autorizzazione abilitando Nascondi file e sottocartelle da utenti senza autorizzazioni. Solo sottocartelle e file in questa cartella condivisa saranno nascosti.
  5. Abilitazione del cestino: Quando i file nella cartella condivisa sono eliminati, vengono spostati in una cartella chiamata #recycle. L'accesso al cestino può essere limitato agli utenti appartenenti al gruppo administrators.
  6. Abilita quota cartella condivisa: per specificare la capacità massima di ogni cartella condivisa, spuntare la casella Abilita quota cartella condivisa e inserire la capacità massima (es. 10 GB) nel campo seguente.

    Nota:

    • solo quando l'archivio utilizza il file system Btrfs, l'opzione Abilita quota cartella condivisa saranno disponibili per le cartelle condivise.
    • L'utilizzo dell'archiviazione combinata delle directory home di tutti gli utenti non supera la quota di “homes”.
  7. Abilitare la checksum dati per l'integrità dati avanzata: per proteggere una cartella condivisa con le strategie Somma di controllo CRC32 e copy-on-write, è possibile abilitare la somma di controllo dati per l'integrità dei dati avanzata durante la creazione della cartella condivisa. La somma di controllo CRC32 è applicata al controllo se i dati restano identici a quando sono stati scritti inizialmente e il sistema utilizzerà automaticamente la ridondanza RAID per correggere i dati danneggiati. Copy-on-write consente di migliorare la coerenza dei dati in caso di arresto anomalo del sistema.

    Nota:

    • quest'opzione è disponibile soltanto per i volumi che utilizzano il file system Btrfs.
    • È possibile abilitare/disabilitare quest'opzione per ogni cartella condivisa durante la relativa creazione. Quest'opzione non può essere modificata dopo la creazione della cartella condivisa.
    • L'abilitazione di quest'opzione consente al sistema di eseguire una pulizia dati completa. Altre informazioni sulla pulizia dei dati
    • Per garantire la qualità del servizio, si consiglia di non abilitare la checksum dati quando la cartella condivisa sarà usata per i seguenti servizi:
      • Database host o macchine virtuali
      • Archiviazione di registrazioni video di Surveillance Station
      • Esecuzione dei servizi che richiedono un modello di scrittura piccolo e random
  8. Abilita compressione file: i dati nella cartella condivisa creata saranno compressi automaticamente dal file system Btrfs per salvare lo spazio di archiviazione. Quando sono recuperati per l'uso, i dati vengono decompressi automaticamente.

    Nota:

    • quest'opzione è disponibile solo quando si crea una cartella condivisa in un volume Btrfs o quando l'opzione Protezione integrità dati avanzata è abilitata.
    • Quest'opzione non può essere modificata dopo la creazione della cartella condivisa.
    • La velocità di compressione varia con i tipi di dati: I file di testo presentano la velocità di compressione più elevata, mentre i file multimediali (es. immagini, musica o video) potrebbero non essere correttamente compressi in quanto sono già compressi per l'output.
    • Quest'opzione non può essere abilitata se la crittografia cartella condivisa è abilitata.
  9. Crittografa questa cartella condivisa: per codificare la cartella condivisa, procedere come segue:
    1. specificare e confermare una nuova chiave di crittografia. La chiave di crittografia non può includere virgole (,) o segni uguale (=).
    2. Se necessario, selezionare la casella Aggiungi chiave di crittografia a Gestione chiavi per montare la cartella crittografata automaticamente quando il sistema si avvia. Se l'opzione è disabilitata, la cartella crittografata condivisa sarà smontata al successivo avvio di Synology NAS. Per vedere o accedere a una cartella condivisa non montata, sarà necessario montare manualmente la cartella importando o inserendo la chiave di codifica.
  10. Fare clic su OK per terminare.

    11. Nota:

    • alcune funzioni e opzioni di una cartella condivisa sono supportate solo da VDSM e da modelli con le seguenti architetture di pacchetto: Apollo Lake, Avoton, Braswell, Broadwell, Bromolow, Cedarview e Grantley. Per vedere quale architettura di modelli è supportata, consultare questo questo articolo.

Regole per i nomi della cartella condivisa:

  • I nomi della cartella condivisa non fanno la distinzione tra lettere minuscole e maiuscole e possono comprendere tra 1-32 caratteri. I nomi non possono contenere i seguenti caratteri: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • I seguenti nomi sono riservati per il sistema: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Il primo carattere di una cartella condivisa non può essere un segno meno o uno spazio; l'ultimo carattere non può essere uno spazio.
  • La descrizione della cartella condivisa è limitata a maiuscole e minuscole, visualizzabili fino a 64 caratteri Unicode.

Crittazione cartelle condivise:

la crittografia AES 256-bit protegge i dati bloccando tutti i tentativi di accesso non autorizzati. Dopo la crittografia, senza l'apposita chiave di crittografia, gli utenti non potranno accedere alla cartella condivisa o modificarne i dati crittografati, anche rimuovendo i dischi rigidi da Synology NAS e montandoli su un altro dispositivo.

Nota:

  • il nome di un file o di una cartella in una cartella condivisa crittata non deve superare 143 caratteri inglesi o 47 caratteri asiatici (CJK); se si accede alle cartelle condivise con Mac OS, il nome di un file o di una cartella in una cartella condivisa crittata non deve superare 130 caratteri inglesi o 43 caratteri asiatici (CJK).
  • È impossibile accedere alle cartelle condivise crittografate tramite NFS.
  • Se si ripristinano le password predefinite usando il pulsante di ripristino posto su Synology NAS, le cartelle condivise crittografate verranno smontate, e l'opzione di montaggio automatico delle cartelle condivise crittografate verrà disabilitata.
  • Le cartelle condivise crittografate sono supportate solo nella modalità privilegi elevati (non applicabile a Dockerx64). Per usare questa funzione, andare nel pacchetto Docker del DSM host, selezionare il Docker DSM per cui abilitare le cartelle condivise crittografate, fare clic su Modifica e spuntare le caselle accanto Esegui contenitore utilizzando i privilegi elevati.
