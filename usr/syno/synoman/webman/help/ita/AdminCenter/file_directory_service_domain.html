Dominio

È possibile aggiungere Synology NAS a un dominio Windows. Una volta aggiunto il dominio, gli utenti di dominio possono accedere alla Synology NAS con i propri account e password di dominio, consentendo di accedere ai file e usare applicazioni DSM senza la necessità di ricordare un altro set di nomi utente e password.

Per unirsi a un dominio:

  1. Andare su Pannello di controllo > Domino/LDAP > Dominio.
  2. Spuntare la casella Accedi al dominio.
  3. Inserire il nome dominio e specificare il DNS Server nelle caselle appropriate.
  4. Se necessario, spuntare la casella Opzioni dominio avanzate. Per i dettagli, vedere di seguito.
  5. Fare clic su Applica.
  6. Nella finestra popup, inserire il nome utente e la password dell'amministratore sul server di dominio.
  7. Fare clic su OK.

DNS Server

Si consiglia di specificare un DNS server per consentire a Synology NAS di cercare il dominio. Tuttavia, se il dominio non è registrato nel DNS server, la procedura di accesso al dominio potrebbe avere esito negativo.

Tipo di server dominio

Questo campo mostra il tipo di dominio quando Synology NAS si collega a un dominio. Il tipo di dominio sarà "Dominio AD".

Modalità gestione

questa opzione determinerà come gestire i privilegi degli utenti e gruppi dominio.

  • Dominio fidato: è possibile gestire utenti e gruppi in un dominio congiunto nonché domini fidati. È possibile filtrare l'elenco di utenti e gruppi in base ai domini.
  • Dominio singolo con OU: solo utenti e gruppi nel dominio congiunto visualizzeranno in questa modalità. Questa modalità consente di filtrare l'elenco di utenti e gruppi in base alle unità organizzative (OU).

Opzioni dominio avanzate

Nella maggior parte dei casi, non è necessario riempire nessuna delle opzioni avanzate del dominio. Le opzioni avanzate del dominio sono necessarie solo per determinati ambienti di dominio.

  • DC IP/FQDN: specificare un indirizzo IP DC (controller di dominio) o FQDN, con cui Synology NAS cercherà di comunicare. In presenza di più di un indirizzo IP o FQDN, inserire una virgola (,) tra ciascun indirizzo. Aggiungere un asterisco (*) dopo l'ultimo indirizzo IP DC o FQDN; Synology NAS proverà a comunicare con l'altro DC se non ha avuto successo con tutti gli altri specificati. Sarà necessario aggiungere una virgola tra l'asterisco e l'ultimo indirizzo IP o FQDN.
  • Nome del dominio NetBIOS: indicare il nome del dominio NetBIOS.
  • Dominio FQDN (nome DNS): indicare l'FQDN (nome DNS) del dominio.
  • Registrare l'interfaccia DNS: quando si accede a un dominio, la NIC (scheda di interfaccia di rete) specificata qui sarà registrata con il server DNS. Se il nome host di Synology NAS include un sottolineato (_), la registrazione fallisce in quanto i sottolineati (_) non possono essere usati con DNS.
  • Aggiorna elenco utenti/gruppo: specificare con quale frequenza Synology NAS aggiorna automaticamente l'elenco utenti/gruppi di dominio. È possibile personalizzare il periodo di tempo per eseguire aggiornamenti quotidiani, settimanali o mensili. Inoltre, gli elenchi utente/gruppo dominio possono essere aggiornati manualmente andando alla scheda Utenti di dominio e facendo clic su Aggiorna dati di dominio. Gli aggiornamenti automatici influiranno sulla sospensione del sistema.

Accesso al controller dominio di sola lettura

Per collegare Synology NAS a un controller dominio di sola lettura (RODC), è necessario specificare il "nome completo del computer" (FQDN) del proprio controller di sola lettura (RWDC) nel campo DC IP/FQDN. Il nome del computer intero deve essere nel seguente formato: NomeComputer.Dominio, ad es. "ad01.synology.com".

Nota:

  • se il nome utente dominio include % o $, l'accesso alla cartella home potrebbe risultare impossibile. Chiedere al proprio amministratore del dominio di fornire un nuovo nome utente.
  • I gruppi di domini Domain Admins e Enterprise Admins verranno aggiunti automaticamente al gruppo locale administrators. Quindi, gli utenti del dominio in questi gruppi dispongono dei privilegi di amministrazione su Synology NAS, incluso l'accesso ai servizi di file come CIFS/FTP/AFP/WebDAV e altre applicazioni DSM.
  • È anche possibile configurare i privilegi di accesso degli utenti del dominio per le cartelle condivise su Synology NAS. Fare clic qui per maggiori informazioni.

Opzioni dominio

Questa sezione include le opzioni di dominio aggiuntive seguenti.

  • DiskStation eseguirà la sincronizzazione con un server NTP ogni volta che un utente dominio esegue l'accesso: Se si abilita questa opzione si forza la sincronizzazione dell'orario tra Synology NAS e il server NTP.
  • Ottieni elenchi utenti/gruppo con modalità compatibile con NT4: abilitando questa opzione si consente al sistema di ottenere elenchi utente/gruppo usando la modalità NT4 RPC. Questa opzione può essere abilitata se alcuni elenchi utente/gruppo non possono essere ottenuti usando le impostazioni predefinite.
  • Abilita accesso server: se il client ha abilitato l'accesso al client, è necessario abilitare anche l'accesso al server su Synology NAS. In questo modo gli utenti potranno utilizzare normalmente i servizi di dominio.
  • Enumera livelli gruppo compatto: specificare il numero di membri del gruppo dominio compatto che possono essere enumerati.
  • Amministratori di dominio: specificare fino a 10 gruppi di utenti a cui fornire le autorizzazioni di amministratori. Qualsiasi utente con permessi amministratore ha il controllo completo di Synology NAS e dei file archiviati.
  • Abilita autenticazione Windows integrata: l'abilitazione di questa opzione consente agli utenti, che hanno già eseguito il login nei rispettivi computer tramite gli account di dominio, di accedere a DSM tramite il browser Internet senza inserire nome utente e password.

Utilizzo dell'autenticazione Windows integrata

  • Il computer client deve eseguire un sistema operativo Windows con versione Windows 7 o superiore.
  • Il computer client deve essere associato allo stesso dominio di Synology NAS.
  • In Windows, andare su Pannello di controllo > Opzioni Internet > Avanzate e verificare che la casella di controllo Abilita autenticazione Windows integrata nella sezione Sicurezza sia spuntata.
  • L'autenticazione Windows integrata funziona con tutti i browser. Per Firefox, la configurazione richiede qualche passaggio in più.
    • Aprire un browser Firefox, immettere about:config nella barra degli URL.
    • Ricercare network.automatic-ntlm-auth.trusted-uris.
    • Fare doppio clic sul campo valore e inserire l'indirizzo IP del dominio.
  • Se Synology NAS viene associato a un dominio creato da Synology Directory Server, è richiesta un'ulteriore configurazione per utilizzare Windows Authentication integrato:
    1. Andare su DSM Pannello di controllo > Servizi File > SMB/AFP/NFS.
    2. Nella sezione SMB, fare clic su Impostazioni avanzate.
    3. Selezionare SMB1 dal menu a discesa Protocollo SMB minimo e fare clic su Salva.

Nota:

  • l'accesso tramite l'autenticazione Windows integrata è disponibile solo su DSM 6.2.2 o superiore.

Controllo stato dominio

È possibile fare clic su Controllo stato dominio per controllare lo stato della connessione tra DSM e il server dominio a cui è associato. Per maggiori informazioni sulla risoluzione dei problemi, consultare gli elementi seguenti:

  • Verificare se la connessione funziona correttamente tra Synology NAS e il controller di dominio: Synology NAS non è stato in grado di comunicare con il controller di dominio.
    • Verificare che le porte 389, 139 e 445 funzionino correttamente nel controller di dominio.
    • Verificare che il servizio NTDS funzioni correttamente nel controller di dominio.
  • Controllare se nella subnet esistono conflitti tra nomi host: Synology NAS ha un nome host già presente nella subnet.
    • Se Synology NAS è connesso a una subnet con più interfacce di rete, DSM funzionerà correttamente.
    • Se Synology NAS è connesso alla subnet con una singola interfaccia di rete, modificare il nome host di Synology NAS, e associarlo di nuovo al dominio.
  • Verificare se la connessione funziona correttamente tra Synology NAS e il server Kerberos: Synology NAS non è stato in grado di comunicare con il server Kerberos.
    • Verificare il corretto record Kerberos DNS SRV mentre il record A correlato esiste nel server DNS.
    • Verificare che la porta 88 sia disponibile nel server KDC.
  • Verificare se la connessione funziona correttamente tra Synology NAS e il server LDAP: Synology NAS non è stato in grado di comunicare con il server LDAP.
    • Verificare il corretto record LDAP DNS SRV mentre il record A correlato esiste nei server DNS.
    • Verificare che la porta 389 funzioni correttamente nel server LDAP.
  • Controllare che MTU sia configurato correttamente per verificare che la connessione funzioni correttamente tra Synology NAS e il controller di dominio: Uno o più switch hanno rilasciato i pacchetti trasferiti tra Synology NAS e il controller di dominio a causa delle dimensioni MTU non valide dei pacchetti.
    • Verificare le impostazioni di interfaccia degli switch tra Synology NAS e il controller di dominio.
  • Controllare se la connessione tra Synology NAS e il dominio funziona correttamente: Il tempo indicato in Synology NAS è sincronizzato con il server di dominio.
    • È possibile configurare il servizio NTP su Synology NAS per sincronizzare l'orario con il server di dominio o per abilitare l'opzione Synology NAS eseguirà la sincronizzazione con un server NTP ogni volta che un utente di dominio esegue l'accesso in Opzioni di dominio.
