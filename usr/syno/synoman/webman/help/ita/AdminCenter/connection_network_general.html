Generale

Opzioni di rete generiche come gateway predefinito, server DNS, proxy e tunneling IPv6 sono disponibili su Pannello di controllo > Rete > Generale.

Rinominare dispositivo Synology NAS

Per rinominare il dispositivo Synology NAS, inserire il nuovo nome del server e fare clic su Applica.

Il nome del server non fa distinzione tra maiuscole e minuscole e può contenere da 1 a 15 caratteri, che possono essere lettere, numeri, segni di sottolineatura o segni meno. Il primo carattere del nome deve essere una lettera.

Specificare gateway predefinito

Basta inserire l'indirizzo IP del gateway predefinito e fare clic su Applica per salvare le impostazioni.

Specificare indirizzo IP DNS Server

È possibile scegliere di configurare manualmente l'indirizzo IP del server DNS. In questo caso, è necessario inserire l'indirizzo IP del server DNS preferito. È possibile inserire anche un indirizzo IP di un server DNS alternativo da utilizzare per il backup in caso di problemi con il server DNS preferito. Una volta configurato l'indirizzo IP di DNS Server, fare clic su Applica per salvare le impostazioni.

Impostazioni avanzate:

  • Rispondere alla richiesta ARP se l'indirizzo IP di destinazione è identico all'indirizzo locale configurato sull'interfaccia in arrivo: nel protocollo di rete, ARP non risponde alla richiesta dell'indirizzo IP identico. Abilitare quest'opzione per rispondere alla richiesta ARP e verificare che la tabella ARP sia completa.
  • Abilita più gateway: abilitare quest'opzione per supportare più di un gateway nell'interfaccia.
  • Abilita rilevamento conflitto IP: abilitare quest'opzione per rilevare automaticamente se è presente lo stesso indirizzo IP nella rete.

Nota:

i campi Server DNS e gateway predefinito verranno impostati automaticamente e non possono essere modificati nel caso vengano configurate automaticamente tutte le interfacce di rete o se si sta utilizzando la connessione PPPoE.

Collegamento tramite Server Proxy

Per consentire al dispositivo Synology NAS di collegarsi tramite un server proxy, spuntare Connetti tramite server proxy, immettere l'Indirizzo e la Porta, e quindi fare clic su Applica.

Nota:

  • fare clic su Impostazioni avanzate per configurare il proxy HTTP, proxy HTTPS e le informazioni di autenticazione.
  • Come impostazione predefinita, le impostazioni proxy HTTPS sono le stesse del proxy HTTP.
  • Se Connetti tramite server proxy è spuntato, Aggiornamento DSM e Centro pacchetti insieme ad altri pacchetti si connetteranno a Internet tramite un proxy.
  • Se è spuntato Bypassa server proxy per indirizzi locali, assicurarsi che il server DNS usato da Synology NAS può risolvere le query per i dispositivi nella rete locale.

Abilitare Tunneling IPv6

Fino a quando IPv6 subentra completamente a IPv4, il tunneling IPv6 sarà necessario per abilitare gli host con gli indirizzi IPv6 per raggiungere i dispositivi IPv4. Questa tecnica fornisce un modo per portare il traffico IPv6 nelle reti IPv4 esistenti.
Se non si è sicuri di come configurare IPv6 o si riscontrano problemi di connessione, contattare il proprio provider di servizi Internet.

Per abilitare Tunnel IPv6:

  1. Spuntare la casella Abilita tunnel per abilitare il supporto di tunnel.
  2. In Indirizzo server, inserire il nome host o l'indirizzo IP del provider del servizio tunnel, ad esempio: anonymous.freenet6.net.
  3. A seconda delle preferenze, selezionare Connessione anonima o Connettere a un account esistente.

Nota:

il proprio indirizzo IPv6 esterno potrebbe essere aggiornato automaticamente dal provider del servizio. Tuttavia, Synology NAS non rimuoverà immediatamente il vecchio indirizzo IPv6.
