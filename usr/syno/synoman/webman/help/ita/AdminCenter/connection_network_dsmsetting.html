Impostazioni DSM

Su Pannello di controllo > Rete > Impostazioni DSM, è possibile modificare il numero di porta che i client utilizzano per accedere a Synology NAS.

Per cambiare il numero di porta predefinito:

Entrambe le connessioni HTTP e HTTPS sono sempre abilitate in DSM. La porta HTTP predefinita è 5000 mentre la porta HTTPS predefinita è 5001.

  1. Immettere il numero porta nel campo HTTP o HTTPS.
  2. Fare clic su Applica.

Per reindirizzare le connessioni da HTTP a HTTPS:

È possibile reindirizzare i client alla porta HTTPS quando tentano di accedere a Synology NAS tramite la porta HTTP.

  1. Selezionare Reindirizza automaticamente le connessioni HTTP in HTTPS (escluso Web Station e Photo Station).
  2. Fare clic su Applica.

Porte utilizzate per la connessione HTTPS:

  • DSM: il numero di porta predefinito è 5001, quindi è possibile accedere a DSM inserendo il seguente indirizzo: https://server-hostname:5001/
  • Web Station e Photo Station: il numero di porta predefinito è 443, quindi è possibile accedere in modo sicuro a Web Station e Photo Station inserendo il seguente indirizzo: https://server-hostname/

Per abilitare HTTP/2:

È possibile aumentare la velocità delle connessioni HTTPS abilitando HTTP/2.

  1. Spuntare Abilita HTTP/2.
  2. Fare clic su Applica.

Per personalizzare il numero massimo di connessioni HTTP contemporanee:

Abilitare Personalizza numero massimo di connessioni HTTP contemporanee per aumentare le connessioni HTTP su Synology NAS.

Nota:

  • applicabile solo alle piatteforme Bromolow, Cedarview, Avoton, Grantley, Broadwell, Dockerx64, Broadwellnk, Apollolake e Braswell. Per verificare se il proprio modello supporta questa funzione, consultare questo articolo.
  • maggiore è il numero massimo di connessioni HTTP contemporanee, maggiore sarà la memoria di sistema richiesta.

Per personalizzare l'intestazione "Server" nelle risposte HTTP:

  1. Spuntare Abilitare l'intestazione "Server" nelle risposte HTTP.
  2. Specificare un'intestazione per il server Web (es. nginx).
  3. Fare clic su Applica.

L'intestazione "Server" può essere disattivata deselezionando la casella di controllo.

Dominio

In Dominio, è possibile specificare un nome dominio personalizzato per accedere a Synology NAS.

Per abilitare il dominio personalizzato:

  1. Spuntare Abilita dominio personalizzato.
  2. Specificare un nome dominio applicato per Synology NAS.
  3. È possibile anche spuntare Abilita HSTS per forzare le connessioni crittate HTTPS in questo dominio.
  4. Fare clic su Applica.

Nota:

  • ogni nome dominio può essere usato solo per una distinta applicazione in Synology NAS.
  • Se il nome dominio non supera 15 caratteri in lunghezza ed è conforme alle convenzioni di denominazione NetBIOS, il sistema registrerà e trasmetterà il nome dominio (es. MyDSM) automaticamente. Gli utenti Windows possono accedere al desktop DSM nella stessa LAN con un URL simile: http://MyDSM o https://MyDSM
  • Quando si accede alle applicazioni tramite gli URL che utilizzano nomi dominio, verrà attraversata la porta HTTP standard (80) o la porta HTTPS standard (443).
