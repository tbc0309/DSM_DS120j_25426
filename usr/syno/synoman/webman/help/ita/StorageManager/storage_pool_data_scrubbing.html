Pulitura dati

Esegui pulitura dati

Pulitura dati è una funzione di manutenzione che consente di ispezionare gli storage pool. I due seguenti tipi di Pulitura dati saranno eseguiti in sequenza:

  • Pulitura del file system: questo funzione controllerà i volumi nel file system Btrfs. Se sono rilevati dati inconsistenti con la checksum, il sistema tenterà di usare un backup per riparare i dati e il percorso file sarà registrato in Log Center.
  • Pulitura RAID: questa funzione può essere utilizzata con gli storage pool che implementano SHR (con tre o meno unità), RAID 5 o RAID 6. Consente di riparare tutti le inconsistenze di dati rilevate.

Si consiglia di eseguire regolarmente la pulitura dei dati in modo da garantire la consistenza dei dati ed evitare di perdere i dati in caso di guasto dell'unità.

Nota:

  • Pulitura dati e le funzioni correlate descritte in questo articolo sono disponibili solo su specifici modelli.
  • La Pulitura dati non può essere eseguita quando lo stato dello storage pool è Crash o Danneggiato.
  • Se la somma di controllo dati per l'integrità dati avanzata non è abilitata per una cartella condivisa, la pulizia file system non sarà in grado di controllare e riparare i dati nella cartella per mancanza di somme di controllo.

Impostare la programmazione della Pulitura dati

È possibile impostare una programmazione di Pulitura dati. Il sistema eseguirà la Pulitura dati in base alle impostazioni.

Per impostare una programmazione di Pulitura dati:

  1. Aprire Storage Manager.
  2. Andare su Storage Pool > Pulitura dati.
  3. Fare clic su Imposta programmazione.
  4. Spuntare Abilita programmazione Pulitura dati.
  5. Specificare Frequenza, Data di inizio e Sessione tempo.
  6. Andare nella scheda Seleziona destinazione.
  7. Selezionare lo storage pool su cui eseguire la Pulitura dati.
  8. Fare clic su OK.

Nota:

  • per essere aggiunto a una programmazione, lo stato dello storage pool deve essere Normale e supportare la Pulitura dati.
  • Se il sistema è aggiornato dalla versione 6.1, gli storage pool con capacità di eseguire la Pulitura dati saranno aggiunti per impostazione predefinita alla pianificazione.
  • Se la pianificazione è stata impostata nella versione 6.1, verrà abilitata automaticamente in base alle precedenti impostazioni.

Per aggiungere uno storage pool a una programmazione o per rimuovere uno storage pool da una programmazione:

  1. Fare clic su Imposta pianificazione
  2. Andare nella scheda Seleziona destinazione.
  3. Selezionare o deselezionare la casella di controllo di uno storage pool per stabilire se aggiungerlo o rimuoverlo da una pianificazione.
  4. Fare clic su OK.

Nota:

  • uno storage pool non può essere aggiunto o rimosso da una programmazione quando la Pulitura dati è in esecuzione manuale su questo storage pool.
  • Per rimuovere uno storage pool in fase di Pulitura dati pianificata, occorre annullare l'avanzamento di questo storage pool.

Per modificare le impostazioni di programmazione:

  1. Fare clic su Imposta pianificazione.
  2. A seconda delle esigenze, modificare frequenza, data di inizio e sessione ora.
  3. Fare clic su OK.

Nota:

  • la modifica della Data di inizio comporterà la reimpostazione dell'avanzamento della programmazione e la Pulitura dati sarà riavviata in base alla nuova data di inizio.

Per sospendere una programmazione:

  1. Selezionare uno storage pool in fase di programmazione della Pulitura dati.
  2. Fare clic su Sospendi programmazione dal menu a discesa Azione.

Nota:

  • per riprendere la programmazione della Pulitura dati, fare clic su Riprendi programmazione dal menu a discesa Azione. In caso contrario, la programmazione sarà sospesa.

Eseguire la Pulitura dati manualmente:

Se il sistema rileva delle anomalie nello spazio di archiviazione, provare a riparare i dati eseguendo la Pulitura dati manualmente.

Per eseguire manualmente la Pulitura dati:

  1. Aprire Storage Manager.
  2. Andare su Storage Pool > Pulitura dati.
  3. Selezionare lo storage pool su cui eseguire la Pulitura dati.
  4. Fare clic su Esecuzione manuale dal menu a discesa Azione.

Nota:

  • se il sistema viene arrestato prima del completamento della pulitura dati, alcuni progressi potrebbero andare persi e sarà necessario riavviare manualmente la pulitura dati.
  • Se l'esecuzione manuale di Pulitura dati si sovrappone alla Pulitura dati programmata, questa fase di programmazione non sarà completata fino al termine della Pulitura dati manuale.
  • L'esecuzione della Pulitura dati potrebbe influire sulle prestazioni di sistema; quindi, si consiglia di non eseguire la Pulitura dati su più storage pool, tranne se necessario.

Per sospendere una programmazione su uno storage pool soggetto a esecuzione manuale della Pulitura dati:

  1. Selezionare uno storage pool da sospendere.
  2. Fare clic su Sospendi dal menu a discesa Azione.
