Generale

Gestione archiviazione consente agli utenti di gestire tutti gli spazi di archiviazione su prodotti Synology. Per semplificare la gestione dell'archiviazione e creare spazi di archiviazione più flessibili con protezione dei dati, Synology DSM supporta più tipi di storage pool.

Storage Pool

Con Synology NAS, è possibile unire più unità in un'unica unità di archiviazione denominata storage pool. Synology NAS fornisce i seguenti tipi di storage pool:

  • Storage pool per prestazioni migliori:
    • alloca tutta la capacità disponibile su un solo volume.
    • Offre prestazioni migliori con minore flessibilità nella gestione dell'archiviazione.
    • Consente la creazione di più iSCSI LUN (File normali) sui volumi.
    • Supporta un RAID Array al massimo.
  • Storage pool per maggiore flessibilità:
    • supporta la creazione di più volumi su uno storage pool.
    • Offre una maggiore flessibilità nella gestione dell'archiviazione.
    • Consente la creazione di più iSCSI LUN (File normali) sui volumi.
    • Supporta la combinazione di più array RAID in uno storage pool (se configurato come RAID 5, RAID 6 o RAID F1).

Tipo di RAID

Durante la creazione di uno storage pool è necessario prima scegliere il tipo di RAID. Diversi tipi di RAID offrono diversi livelli di protezione dei dati, funzioni di archiviazione e prestazioni. Synology NAS supporta correntemente i seguenti tipi di RAID:

  • Basic: utilizza una unità per creare uno storage pool. Gli storage pool di base non offrono la ridondanza dei dati.
  • JBOD*: combina almeno due unità per creare uno storage pool. Gli storage pool JBOD non offrono la ridondanza dei dati. La capacità disponibile di uno storage pool JBOD equivale alla capacità di tutte le unità presenti nello storage pool. JBOD supporta l'uso di unità di varie dimensioni.
  • RAID 0*: combina almeno due unità per creare uno storage pool. RAID 0 non offre la ridondanza dei dati. In caso di guasto di una unità membro, i dati nello storage pool andranno persi. La capacità disponibile di uno storage pool RAID 0 equivale alla capacità totale di tutte le unità membri. RAID 0 supporta l'uso di unità di varie dimensioni.
  • RAID 1*: combina 2-4 unità per creare uno storage pool. Il sistema scriverà dati identici per ogni unità. RAID 1 offre la protezione dei dati fino a quando una unità funziona in modo corretto. La capacità disponibile di uno storage pool RAID 1 equivale alla dimensione dell'unità più piccola.
  • RAID 5*: combina almeno tre unità per creare uno storage pool ed esegue lo striping dei dati e delle informazioni di parità attraverso le unità. RAID 5 offre la ridondanza dei dati. In caso di guasto di una unità membro, il sistema può essere ricreato utilizzando le informazioni di parità su altre unità membri. Supporta l'unione di più array RAID durante la creazione di uno storage pool per volumi multipli. La capacità disponibile di uno storage pool RAID 5 = capacità totale delle array RAID combinate. Capacità array RAID = (numero di unità - 1) x (capacità dell'unità più piccola)
  • RAID 6*: combina almeno quattro unità per creare uno storage pool ed esegue lo striping dei dati e delle informazioni di parità attraverso le unità. RAID 6 offre la ridondanza dei dati. Anche in caso di guasto di due unità, il sistema può essere ricreato utilizzando le informazioni di parità sulle altre unità membri. Supporta l'unione di più array RAID durante la creazione di uno storage pool per volumi multipli. La capacità disponibile di uno storage pool RAID 6 = capacità totale delle array RAID combinate. Capacità array RAID = (numero di unità - 2) x (capacità dell'unità più piccola)
  • RAID 10*: combina almeno quattro unità per creare uno storage pool. Il numero delle unità deve essere pari. RAID 10 fornisce le prestazioni di RAID 0 oltre allo stesso livello di protezione dati di RAID 1. La capacità disponibile dello storage pool RAID 10 = (capacità dell'unità più piccola) X (numero di unità - 2)
  • RAID F1*: questo tipo di RAID è progettato per le unità allo stato solido (SSD). Combina almeno tre SSD per creare uno storage pool ed esegue lo striping dei dati e delle informazioni di parità attraverso tutte le SSD membri, scrivendo nel contempo più bit di parità in una specifica SSD per evitare che più SSD abbiano esito negativo contemporaneamente. RAID F1 offre la ridondanza dei dati. In caso di guasto di una SSD membro, il sistema può essere ricreato utilizzando le informazioni di parità su altre SSD membri. Supporta l'unione di più array RAID durante la creazione di uno storage pool per volumi multipli. La capacità disponibile di uno storage pool RAID F1 = capacità totale di array RAID combinati. Capacità array RAID = (numero di unità - 1) x (capacità dell'unità più piccola)

Nota:

  • i tipi RAID con * sono disponibili solo sui modelli di NAS con abbastanza vani per i vani unità. Per informazioni su quali tipi di RAID sono supportati dal modello in uso, andare su www.synology.com per verificare le specifiche del modello.
