Volume

I volumi sono gli spazi di archiviazione di base nei quali l'utente può creare cartelle condivise, iSCSI LUN (File normali), salvare dati o installare i pacchetti. Prima di creare qualsiasi volume, verificare di aver creato in precedenza un Raid Group. Per ulteriori informazioni relative ai Gruppi RAID, fare riferimento a Raid Group.

Per creare un volume:

  1. Verificare che Synology NAS disponga di un Raid Group esistente con spazio allocabile.
  2. Fare clic sul pulsante Crea.
  3. Scegliere un Raid Group nel quale creare il volume.
  4. Modificare le impostazioni desiderate.
  5. Impostare le dimensioni di allocazione per il volume. (L'impostazione delle dimensioni di allocazione non è disponibile con i volumi creati su un Raid Group per Volumi singoli o iSCSI LUN)
  6. Fare clic sul pulsante Applica per salvare le impostazioni.

Nota:

  • le dimensioni massime di volume allocabili sono 200 TB.
  • Le dimensioni minime allocate a un volume BTRFS devono essere di almeno 10 GB.
  • I volumi btrfs manterranno il 4% di spazio per i metadati.

Per rimuovere un volume:

  1. Selezionare il volume da rimuovere.
  2. Fare clic sul pulsante Rimuovi e seguire la procedura guidata per completare il processo.

Nota:

  • la rimozione del volume comporterà la cancellazione di tutti i dati esistenti nel volume.

Per forzare il montaggio di un volume:

Se la cache su un volume è mancante, il sistema non monterà il volume al riavvio. Se la cache non è rilevabile, e si vuole accedere al volume, è possibile fare clic su Forza montaggio.

Nota:

  • questa funzione è disponibile solo su modelli specifici.

Per riparare un volume:

Se lo stato del volume è Danneggiato, passare alla scheda Raid Group e cercare il Raid Group appropriato. Seguire le istruzioni consigliate per la riparazione.

Per espandere le dimensioni di un volume

  • Per i volumi creati su un Raid Group per volumi o iSCSI LUN (Livello blocco) multipli.
    1. Selezionare il volume da espandere.
    2. Fare clic su Modifica.
    3. Immettere le nuove dimensioni e fare clic su OK per espandere.
  • Per i volumi creati su un Raid Group per un volume/iSCSI LUN singolo (Livello blocco), consultare la sezione Per espandere le dimensioni di un Raid Group di Raid Group, le dimensioni del volume saranno applicate dopo avere ampliato il Raid Group.

Nota:

Se i volumi sono usati come target replicati di Snapshot Replication, le prestazioni di replica possono essere migliorate regolando le impostazioni di Frequenza di registrazione ora di accesso file nella finestra Modifica dopo la creazione dei volumi:

  • Giornalmente: l'ora di accesso al file viene aggiornata ogni giorno oppure quando il file viene modificato, proprio come nella maggior parte dei sistemi operativi. Le funzionalità di alcune applicazioni si basano sulla precisione dell'ora di accesso al file. Selezionando Giornaliero si influirà sulle prestazioni di replica; tuttavia, quest'opzione è consigliata se si utilizza una delle seguenti funzioni o applicazioni frequentemente:
    • Ora di accesso meno recente in Storage Analyzer
    • SSD Cache Advisor in Storage Manager
    • Qualsiasi applicazione di terzi basata su Registrazione ora di accesso file.
  • Mensile: l'ora di accesso al file viene aggiornata mensilmente oppure quando il file viene modificato. Quest'opzione comporta un piccolo miglioramento nelle prestazioni di replica, mentre le applicazioni che si basano su Registrazione ora di accesso file saranno in grado di funzionare, anche se con minore precisione.
  • Mai: consigliato per ottenere le migliori prestazioni di replica e non sono necessarie applicazioni che si basano su Registrazione ora di accesso file nell'uso quotidiano.

SSD TRIM

Se un volume consiste interamente di SSD (unità a stato solido), si raccomanda di abilitare SSD TRIM. Questa funzione migliora le prestazioni di lettura e scrittura dei volumi creati sulle SSD, accrescendo l'efficienza nonché estendendo la durata delle SSD.

Per abilitare SSD TRIM:

  1. Aprire Storage Manager e andare su Volume.
  2. Selezionare un volume che consiste unicamente di SSD, quindi fare clic sul pulsante Configura.
  3. Nella finestra visualizzata, andare sulla scheda SSD TRIM e fare clic sulla casella Abilita TRIM.
  4. Fare clic sul pulsante Imposta programmazione per scegliere quando il sistema eseguirà SSD TRIM.
  5. Fare clic su OK.

Nota:

  • SSD TRIM è disponibile solo su alcuni modelli di Synology NAS e tipi di RAID.
  • alcuni modelli SSD non sono in grado di eseguire SSD TRIM su volumi RAID 5,, RAID 6 e RAID F1. Consultare l'elenco compatibilità sul sito Web ufficiale di Synology.
  • Gli SSD TRIM non possono essere abilitati nelle seguenti situazioni:
    • Il volume è montato con cache SSD.
    • Il pacchetto Time Backup è installato.
  • SSD TRIM non può essere abilitato su iSCSI LUN (livello blocco).
  • Le prestazioni del sistema saranno influenzate quando si esegue SSD TRIM. Il tempo necessario a finire l'esecuzione TRIM varia a seconda delle dimensioni del volume e del modello di SSD.
  • Quando TRIM è abilitato su un volume, esso sarà anche abilitato su altri volumi creati sullo stesso Gruppo unità o RAID Group.

Pulizia del file system*

La pulizia del file system è una funzione di manutenzione dei dati che ispeziona i volumi. In caso di rilevamento di dati inconsistente con il checksum, il percorso del file verrà registrato in Log Center e sarà opportuno utilizzare un backup per proteggere i dati.

Si consiglia di eseguire regolarmente la pulizia del file system per garantire la consistenza dei dati.

Per eseguire la pulizia del file system:

  1. Aprire Storage Manager e andare su Volume.
  2. Selezionare il volume in cui eseguire la pulizia del file system.
  3. Fare clic su Gestisci.
  4. Selezionare Esecuzione della pulitura del file system.

Nota:

  • la deframmentazione del file system può essere eseguita solo su volumi nel file system Btrfs.
  • La pulizia del file system può essere eseguita solamente quando lo stato del volume è Normale.
  • Non spegnere il sistema prima del termine della pulizia del file system. Se il sistema viene spento prima del termine, sarà necessario riavviare manualmente dall'inizio la pulizia del file system.

Deframmentazione del file system*

Se i dati vengono modificati di continuo, la quantità di dati frammentati aumenterà nel file system, causando un rallentamento delle prestazioni di accesso dei file nelle unità. La deframmentazione del file system può migliorare tali prestazioni.

Per eseguire la deframmentazione del file system:

  1. Aprire Storage Manager e andare su Volume.
  2. Selezionare il volume in cui eseguire la deframmentazione del file system.
  3. Fare clic su Gestisci.
  4. Selezionare Deframmentazione del file system.

Nota:

  • La deframmentazione del file system può essere eseguita solo su volumi nel file system Btrfs.
  • La deframmentazione del file system può essere eseguita solamente quando lo stato del volume è Normale.
  • Non spegnere il sistema prima del termine della deframmentazione del file system. Se il sistema viene spento prima del termine, sarà necessario riavviare manualmente dall'inizio la deframmentazione del file system.
  • L'uso del volume potrebbe aumentare se il volume contiene istantanee di cartelle condivise.
