域/LDAP 用户或组

如果 Synology NAS 作为 Windows 域或 LDAP 客户端加入目录服务，您可以设置和修改 Synology NAS 共享文件夹和 SRM 应用程序的域/LDAP 用户或组的访问权限，并启用所有域/LDAP 用户的本地服务。

域/LDAP 用户

根据您要加入的目录服务类型，单击域用户或 LDAP 用户选项卡，设置共享文件夹和应用程序的权限，以及各用户的磁盘容量配额。
注：磁盘容量配额设置不适用于Docker DSM。

权限设置的规则包括：

  • 如果权限发生冲突，则权限的优先顺序为：NA > RW > RO。
  • 权限预览: 显示用户对共享文件夹真正权限, 此权限由用户本身权限及所属群组权限共同决定。
  • 群组权限: 用户所属群组对共享文件夹的权限。

用户主目录

所有的域/LDAP 用户与本机用户一样，都可以通过 CIFS、AFP、FTP 或 File Station 访问自己的 home 文件夹。

拥有管理权限的用户可以访问 homes 共享文件夹中的所有个人文件夹。所有用户的本地文件夹都位于“@DH-domain name”（适用于域用户）或“@LH-FQDN name”（适用于 LDAP 用户）。而域用户本地文件夹的名称是域用户帐户加唯一的数字。

    注：
  1. 若要删除 homes 共享文件夹，您必须先禁用用户主目录服务。
  2. 启用域/LDAP 用户主目录服务也将启用本地用户的本地服务（如果未启用）。

域/LDAP 组

根据您所在的目录服务类型，单击域组或 LDAP 组选项卡可配置共享文件夹域/LDAP 组的访问权限。权限优先级为：NA > RW > RO
