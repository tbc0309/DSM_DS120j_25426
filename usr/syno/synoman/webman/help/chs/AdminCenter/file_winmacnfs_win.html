SMB

SMB (Server Message Block) 协议可让支持 SMB/CIFS 的 Windows、Mac 和 Linux 客户端访问存储在 Synology NAS 中的数据。如果您要通过 Windows 资源管理器浏览 Synology NAS 上的文件或装载共享文件夹作为网络驱动器，请进入 控制面板 > 文件服务 > SMB/AFP/NFS启用此选项。

若要启用 SMB 协议：

  1. 勾选启用 SMB 服务。
  2. 单击应用。

注：

  • 启用 SMB 协议后，您可通过 SMB 将全面建索文件夹装载到 Mac 计算机，并使用 Finder 搜索已装载文件夹中的文件内容（不适用于 NVR216 或 VS360HD）。

工作群组

此选项可让您将系统指定为特定工作群组成员以进行网络管理。

注：

  • 工作群组的名称可包含 1 至 15 个字符。名称不得包含以下字符：[ ] ; : "< >* + = \ / | ? ,
  • 如果您 Mac 的系统版本为 OS X 10.9，且 Synology NAS 已加入 Windows 域，则您可在 Synology NAS 本地用户名前添加“localhost\”进行登录，通过 SMB 协议访问存储在 Synology NAS 上的文件。

不可访问之前版本

在 Windows 资源管理器中，用户可在共享文件夹中单击以前版本选项卡来访问文件夹或文件的快照，允许还原或复制数据而无需管理员的帮助。如果您选择不可访问以前版本，则“以前版本”页面将不会显示任何信息。

注：

  • 仅具有以下套件架构的 VDSM 和型号支持此功能：Apollo Lake、Avoton、Braswell、Broadwell、Bromolow、Cedarview 和 Grantley。若要查看您的型号支持哪种套件架构，请参阅本文。
  • 用户仅在以之前版本拍摄快照、其用户帐户已创建，并已授予拍摄相应快照所在共享文件夹的读取权限后，才可访问之前的版本。

启用传输日志记录

勾选启用传输日志复选框可对通过 CIFS/SMB 协议执行的文件操作启用日志记录。默认情况下，传输日志仅监控文件或文件夹的删除；但是，也可以进入日志设置以选择其他文件操作事件（创建、移动、重命名、读取、写入）以供系统进行日志记录。请注意，选择的文件操作事件越多，便越可能会影响系统性能。

高级设置

单击高级设置按钮可查看以下选项。

  • WINS 服务器：Microsoft Windows Internet 名称服务 (WINS) 是一个 NetBIOS 名称对 IP 地址的映射服务。如果系统配置使用 WINS 服务器来进行注册，Windows 的用户可以更容易地将系统设置在 TCP/IP 网络上。
  • 最大 SMB 协议：根据网络配置，设置 Synology NAS 支持的最新 SMB 协议。
    • SMB1：SMB1（也称为 CIFS）自 Windows NT 发布以来得到支持。
    • SMB2：SMB2 自从 Windows Vista 发布以来得到支持，且为 SMB 的增强版本。SMB2 增加了将多重 SMB 操作功能组合到单个请求的功能，以减少网络数据包的数量并提高性能。
    • SMB2 和 Large MTU：最大传输单元 (MTU) 是指可通过通讯协议的最大数据单元。为利用最快的更快的接口，如 1- 或 10-gigabit 以太网，Large MTU 将最大传输单元提高至 1 megabyte (MB)。启用 Large MTU 可提高大文件传输的速度和效率，同时降低需处理的数据包数量。
    • SMB3：SMB3 自 Windows 8 和 Windows Server 2012 发布以来得到支持。它是 SMB 2 的增强版。SMB3 支持基于 AES 的文件加密传输，从而提高了对等文件传输的安全性。

    注：

    • Windows Vista、Windows Server 2008 R2、Windows 7 和以上的版本支持 SMB2。
    • Windows Server 2008 R2、Windows 7 和以上的版本支持 Large MTU。
    • 您需要修改计算机的注册设置来使 Windows 计算机的 Large MTU 选项生效。详情请参阅 Microsoft 技术支持（如本文章）。
  • 最小 SMB 协议：根据网络配置，设置 Synology NAS 支持的最早 SMB 协议。
  • 传输加密模式：启用 SMB3 后，SMB 协议将增加传输加密以增强文件传输的安全性。
    • 禁用：不应用传输加密。
    • 自动：传输加密将只应用到支持此功能的客户端。
    • 强制：始终应用传输加密。此选项将造成不支持传输加密的客户端无法使用 SMB 协议。

    • 注：

    • Windows 8、Windows Server 2012 和以上的版本支持传输加密。
    • 启用传输加密后，不支持匿名登录 SMB 协议。
    • 启用传输加密后，请保持勾选启用 Opportunistic Locking 以免应用程序超时。
  • 启用 Opportunistic Locking：选择此选项可启用 CIFS 的“Opportunistic Locking”，以使客户端能够更好地缓存文件内容并避免较小的读/写负荷。取消选择此选项后，数据库应用程序将通过网络更好地访问存储在 Synology NAS 的数据库文件。
    • 启用 SMB2 租用：启用此选项可让更多的 SMB2 相关设置来微调 Opportunistic Locking 提供的客户端缓存机制。
  • 启用 Local Master Browser：当多台 Windows 计算机存在于同一子网络时，它们会进行协商并选择一台计算机作为“local master browser”。Local Master Browser 将维护子网中的其它计算机及其它共享资源的列表，并与其它计算机共享这些列表。此选项可让 Synology NAS 作为 Local Master Browser 使用。

    • 注：

    • 启用此选项将禁用硬盘休眠功能，且不需要密码就能激活 guest 帐户。
  • 启用 DirSort VFS 模块：将文件夹按字母顺序排列。启用此选项会影响系统性能。
  • 禁止文件：根据特定条件隐藏文件。条件可包括通配符 (*)，并使用斜线号 (/) 分隔多项条目。例如：/abc*/*.txt/*.conf/。启用此选项会影响系统性能。
  • 允许在共享文件夹中使用符号链接：选择此选项可允许符号链接指向位在同一共享文件夹中的文件或文件夹。符号链接可以使用 DSM 终端机来创建。
    • 允许跨共享文件夹的符号链接：选择此选项可允许符号链接指向位在不同共享文件夹中的文件或文件夹。符号链接可以使用 DSM 终端机来创建。但没有权限的用户可访问符号链接的目的地共享文件夹。
  • 启用 VFS 模块以转换 Mac 特殊字符：选择在文件创建时将文件名中的 SMB 保留字符（如 < > " * ?）转换为基本拉丁字母。此类文件将通过 AFP 可识别。
  • 启用 MSDFS VFS 模块：允许符号链接指向位于其他 SMB 服务器中的文件或文件夹。对于旧版本的 Windows，如 Windows XP，客户端和 Synology NAS 需要加入 Windows 域后才能启用该选项。如需更多有关创建 MSDFS 链接的信息，请参阅此处。请注意，禁用此选项后将会清除客户端设备的缓存（还建议重新启动客户端设备），以避免影响正常性能和行为。
  • 禁用同一 IP 地址的多个连接：启用此选项后，Synology NAS 限制客户端一次只能一个 IP 地址使用 SMB 协议。IP 地址相同的多个客户端使用 SMB 协议，Synology NAS 将保留最近一次连接而中断所有旧连接。在 NAT 环境下，客户端可能只使用一个 IP 地址，在此情况下启用此选项会影响正常的操作性能。
  • 收集调试日志：启用此选项后将保存详细的日志供调试之用。启用此选项会影响系统性能。
  • 应用默认的 UNIX 权限：启用此选项可在上传或创建文件和文件夹时应用默认的 UNIX 权限。文件的 UNIX 权限为 744，而文件夹的 UNIX 权限为 755。禁用此选项后，文件与文件夹的 UNIX 权限均为 777。

    • 注：

    • 对于启用 Windows ACL 的共享文件夹（外部设备上的“照片”和共享文件除外），请在 Linux 或 FTP 客户端上运行 chmod 命令，以将 Windows ACL 上的文件夹和文件权限类型更改到 UNIX。（不适用于 EDS14）。
    • 启动此选项可能会导致不同协议之间出现权限不一致的问题。为避免不一致问题，建议您保持禁用此选项。
  • 请勿在创建文件时保留硬盘空间：启用此选项可在 Synology NAS 上创建文件时禁止保留硬盘空间。
  • 启用通配符搜索缓存：如果客户端发送重复的通配符搜索请求，则启用此选项可提高搜索性能。
  • 启用 SMB 耐久控制（跨协议文件锁定将被禁用）：启用此选项可让用户在暂时断开连接后，把丢失的 SMB2 连接还原到此 Synology NAS；打开的文件将在网络中断时仍保持可用。但跨协议文件锁定将被禁用，因此用户可通过非 SMB2 协议访问或修改锁定的文件（如 AFP）。
  • 清除 SMB 缓存：单击此按钮可删除以下 SMB 缓存文件以排除故障：
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
