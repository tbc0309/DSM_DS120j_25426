iSNS

iSNS（Internet 存储器名称服务）提供集中化管理的机制。您可在 iSNS 服务器上注册所有的 iSCSI Target。

向 iSNS 服务器注册 iSCSI Target：

  1. 单击 iSNS 并勾选启用 iSNS。
  2. 输入 iSNS 服务器的 IP 地址。单击确定来应用设置。

I/O 队列深度

默认设置为 16，但您可以根据需要修改所有目标的 I/O 队列深度。

低容量写入

当存储空间上的可用空间小于 1 GB 时，存储空间上的 LUN 将变为受系统保护，以防止文件系统损毁，且用户无法向其写入数据。低容量写入功能使用户可以在存储空间的可用空间即将用尽时继续向 LUN 写入数据。但是，建议在解决空间不足的问题后禁用此选项。

注：

  • 即使已启用此功能，也请避免继续将数据写入 LUN。

空间回收模式

以下是三种空间回收模式。通过调整速度，可以控制空间回收对 LUN 性能的影响。

  • I/O 一致性优先：I/O 一致性优先可在系统回收旧版高级 LUN 的已用空间时将对性能的影响降至最低。如果存在持续传入 I/O，则速度会降低。在收到连续空间回收命令时，回收过程会延迟，直到没有其他空间回收命令。
  • 平衡：默认回收模式。此模式可保持 I/O 速度与回收速度之间的平衡。
  • 回收优先：回收优先意味着传入 I/O 对回收速度几乎没有影响。回收会全速执行，而不会牺牲 I/O 一致性。
