目标

根据型号，Synology NAS 支持在可映射的 LUN 上创建多个目标。

注：支持的目标数量视 Synology NAS 型号而定。

若要创建目标：

  1. 单击创建按钮。
  2. 输入目标的信息，包括名称、IQN 以及是否要启用 CHAP 加密认证。
  3. 单击下一步。
  4. 设置 iSCSI LUN 映射。选择创建新的 iSCSI LUN 或映射已有的 LUN。
  5. 单击下一步。
  6. 单击返回以修改或单击应用以确认设置。

术语

  • IQN：IQN (iSCSI Qualified Name) 是目标的一组独特名称。这组名称不能和其他服务器上的目标 IQN 相冲突。IQN 必须少于 128 个字符，可包含字母（不分大小写）、数字、冒号、短划线和圆点。我们建议您按照标准格式来命名：iqn.[日期].[域]:[设备标识符]，例如：iqn.2000-04.com.synology:storage.diskarray.1。
  • 相互 CHAP 加密认证：在相互 CHAP 加密认证中，客户端及服务器要互相通过验证才能使用。用于客户端验证的任何 CHAP 密码不可用于配置任何服务器的验证，用于服务器验证的任何 CHAP 密码不可用于配置任何客户端的验证。

    • 注：如果已在 NAS 上启用 CHAP 加密认证，则 iSCSI 启动器上的发现过程可能会失败。请先完成发现，然后启用 CHAP 加密认证，再连接到目标。或者，可以使用 IP 和 IQN 连接到目标。

若要删除目标：

您可以删除一个目标和任何与其映射的 LUN。删除 iSCSI LUN 时，上面所有数据将会被删除。

  1. 选择要删除的目标。
  2. 单击删除。
  3. 如果您要删除映射的 LUN，选择同时删除已映射的 LUN 并选择您想要删除的项目。
  4. 单击确定。

若要编辑目标：

  1. 选择要编辑的目标。
  2. 在操作下拉菜单中选择编辑。
  3. 依您的偏好编辑设置。单击确定来保存更改。

术语

  • CRC 总和检查码：启用 CRC 总和检查码可以检查数据传输时是否有错误发生。
  • 多重链接：多重链接可让多个启动器同时链接至同一个目标。为了确保您的文件不会遭受毁损或无预警数据丢失，我们强烈建议您使用文件共享系统或集群系统，如在 VMware Virtual Machine File System (VMFS) 和 Oracle Cluster File System (OCFS) 下使用此功能。
  • 掩码：掩码可让一个目标专门映射到指定的 iSCSI 启动器的 IQN 上。

若要启用或禁用目标：

  1. 选择您要启用或禁用的目标。
  2. 从操作下拉菜单中选择启用或禁用。
