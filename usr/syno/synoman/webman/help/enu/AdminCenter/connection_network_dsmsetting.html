DSM Settings

At Control Panel > Network > DSM Settings, you can change the port number that clients use to access your Synology NAS.

To change the default port number:

Both HTTP and HTTPS connections are always enabled in DSM. The default HTTP port is 5000 and the default HTTPS port is 5001.

  1. Enter the port number in the HTTP or HTTPS field.
  2. Click Apply.

To redirect HTTP to HTTPS connections:

You can redirect clients to the HTTPS port when they try to access your Synology NAS through the HTTP port.

  1. Select Automatically redirect HTTP connections to HTTPS (Web Station and Photo Station excluded).
  2. Click Apply.

Ports used for HTTPS connection:

  • DSM: The default port number is 5001, so you can securely access DSM by entering the following address: https://server-hostname:5001/
  • Web Station and Photo Station: The default port number is 443, so you can securely access Web Station and Photo Station by entering the following address: https://server-hostname/

To enable HTTP/2:

You can speed up HTTPS connections by enabling HTTP/2.

  1. Tick Enable HTTP/2.
  2. Click Apply.

To customize the maximum of concurrent HTTP connections:

You can enable Customize maximum concurrent HTTP connections to increase the HTTP connections on your Synology NAS.

Note:

  • Only applicable to Bromolow, Cedarview, Avoton, Grantley, Broadwell, Dockerx64, Braswell, Broadwellnk, Apollolake platforms. To see if your model supports this fuction, please refer to this article.
  • The higher the maximum of concurrent HTTP connections, the more system memory it demands.

To customize the "Server" header in the HTTP responses:

  1. Tick Enable the "Server" header in HTTP responses.
  2. Specify a header for your web server (e.g. nginx).
  3. Click Apply.

You can disable the "Server" header by unticking the checkbox.

Domain

Under Domain, you can specify a customized domain name for accessing Synology NAS.

To enable customized domain:

  1. Tick Enable customized domain.
  2. Specify an applied domain name for your Synology NAS.
  3. You can also tick Enable HSTS to force HTTPS encrypted connections on this domain.
  4. Click Apply.

Note:

  • Each domain name can only be used for one distinct application on your Synology NAS.
  • If the domain name does not exceed 15 characters in length and complies with the NetBIOS naming conventions, the system will register and broadcast the domain name (e.g. MyDSM) automatically. Windows users can access the DSM desktop in the same local network with a similar URL: http://MyDSM or https://MyDSM
  • When you access applications via URLs that use domain names, you will go through the standard HTTP port (80) or the standard HTTPS port (443).
