Settings

Click Settings to set up the following options:

  • Choose the Real time duration in minutes from the drop-down menu.
  • Tick Enable usage history if you want to view previous data. You can switch between Real Time, 1 Day, 1 Week, 1 Month and 1 Year from the drop-down menu at the top-right corner when viewing performance charts.
  • Tick Enable performance alarm if you want to set up performance alarm rules.

Click Apply to save the settings.

Performance Alarm Rules

To create a performance alarm rule:

  1. Go to Settings > Settings and make sure you have ticked Enable performance alarm.
  2. Go to Settings > Performance Alarm Rules.
  3. Click the Create button.
  4. The following rule settings are available:
    • Resource: Choose the resource you want to apply the rule to.
    • Type: Choose the type of rule you would like to create.
    • Level: Choose the alarm level.
    • Greater than: Set the maximum percentage of your rule type where the system will notify you if it is exceeded.
  5. Tick Enable if you want to enable this rule immediately.
  6. Click OK to add the rule.

Mechanism for performance alarms:

Performance alarms will operate through the following mechanism to minimize false alarms and to prevent excessive consumption of system resources within a short period of time.

  1. The performance alarm service will check all rules every five minutes. No action will be triggered if all of the monitored values fall below the thresholds that you have set.
  2. When a monitored value exceeds its threshold, an alarm will not be triggered immediately. Instead, the system will check the rule five more times at 30-second intervals.
  3. After the change of the interval length, if the monitored value exceeds the threshold five consecutive times, an alarm will then be triggered; if, however, the monitored value falls below the threshold during any of these five checks, the interval length will be changed back to five minutes.

Therefore, for one single event, an alarm will at latest be triggered 450 seconds after the first detection of the event.

Alarm levels:

Performance alarms will trigger different actions depending on the alarm levels.

  • Warning: Generates a warning message in the system log but will not trigger any system notification.
  • Critical: Not only generates a message in the system log but also triggers system notifications.

Logs of performance alarms can be viewed at the Performance Alarm tab or DSM's Log Center.
